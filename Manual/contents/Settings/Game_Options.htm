Options de jeu

Les options du jeu sont divisées en deux sections : les options principales et les paramètres de la plate-forme. Les options principales commencent par les options générales, où vous pouvez définir certains éléments qui affecteront le jeu, quelle que soit la plate-forme sur laquelle il est conçu.

La fenêtre comporte les sections suivantes :

  • Langage de script par défaut: Choisissez si les événements et les scripts des objets du projet en cours doivent utiliser GML Code ou GML Visual, ou s'il faut toujours demander la création d'un nouveau script.
  • Game Frames Per Second (images par seconde): cette valeur fixe la limite du nombre d'images par seconde que GameMaker doit essayer de maintenir pour vos jeux en cours d'exécution. Cette valeur représente le nombre de fois que la boucle de jeu s'exécutera en une seconde (une seule boucle de jeu est considérée comme une seule "image") et est définie par défaut à 60, bien que 30 soit souvent utilisé aussi. Il existe de nombreux arguments pour déterminer quelle vitesse de salle est idéale, mais la réponse réelle dépend du type de jeu que vous réalisez, de la taille de la salle, du nombre d'instances et de l'intensité du code. Par exemple, si vous avez une grande salle et une vitesse de salle de 30, le jeu peut sembler "haché" car plus la zone que vous regardez est grande, plus il est facile pour l'œil de percevoir des vitesses de salle faibles, mais si le jeu est intensif, régler la vitesse de salle à 120 peut entraîner un "retard" du jeu car les images par seconde diminuent en raison de la quantité de traitement effectuée. Faites donc attention à cette valeur et essayez de toujours l'ajuster aux besoins de votre jeu.
  • Default Draw Colour: La couleur par défaut pour dessiner du texte ou des forums (et d'autres choses) lorsqu'aucune couleur n'a été définie à l'aide du code.
  • Couleur de dessin par défaut en dehors de la région de la salle: La couleur par défaut qui sera utilisée pour effacer l'arrière-plan de toute zone située en dehors de la salle de jeu.
  • Générer des images de projet: Cliquez sur ce bouton pour ouvrir l'outil de génération d'images de projet.
  • Date de début du projet: La date à laquelle le projet a été lancé.
  • Temps d'utilisation du projet: le temps qui s'est écoulé depuis que vous avez lancé le projet.
  • GUID du projet: Il s'agit de l'identifiant unique qui est créé pour chaque projet dans GameMaker. Si vous souhaitez le modifier, vous pouvez cliquer sur le bouton Créer un nouveau GUID pour en générer un nouveau.
  • Générer des mipmaps pour des pages de textures séparées: Cette option permettra à GameMaker de créer des mipmaps pour les sprites qui ont marqué l'option Seperate Texture Page dans les paramètres de texture de l'éditeur Sprite. Activer le mipmapping signifie que la page de texture originale de sprite doit être mise à l'échelle et filtrée en plusieurs résolutions dans le fichier de texture. Cette option est normalement désactivée, mais si vous utilisez des vues pour faire un zoom avant ou arrière sur de grandes zones de la salle de jeu, ou si vous travaillez avec un jeu en 3D, vous pouvez l'activer et utiliser les fonctions de mipmapping pour ajuster la façon dont le jeu traite les différents niveaux de mipmap. Notez que si vous n'indiquez pas à sprites que vous utilisez des pages de texture individuelles, vous pouvez toujours activer le mipmapping pour chaque groupe de textures à partir du gestionnaire de groupes de textures. Cette option est désactivée par défaut.
  • Activer le contrôle de la source: Si vous souhaitez que le projet soit sous SCM (Source Control Management), vous devez cocher cette case. Elle est désactivée par défaut (voir ici pour plus d'informations sur le contrôle de la source).
  • Auteur: Le nom de l'auteur du jeu (ou de l'éditeur ou du studio).
  • Projet modifié en dernier: cette section indique simplement la date et l'heure de la dernière modification du projet (non modifiable).
  • Accepter la licence Spine : Ici, vous pouvez cocher la case pour afficher la licence Spine Runtime et l'accepter. Cela vous permettra d'utiliser Spine sprites dans votre jeu (cette licence sera également affichée la première fois que vous essayez d'ajouter un Spine sprite à votre projet et que vous n'avez pas vérifié la licence ici au préalable).
  • Mode de compatibilité des collisions: Cette option indique à GameMaker d'utiliser l'ancien système de collision utilisé avant les versions 2022 (qui utilisait des valeurs entières au lieu de flottantes). Lisez Mode de compatibilité des collisions pour plus d'informations. Cette option est désactivée par défaut et il est recommandé de la conserver pour les nouveaux projets.
  • Activer le comportement de copie en écriture pour les tableaux: Ceci active le comportement déprécié de copie sur l'écriture de tableau pour le projet actuellement ouvert. Lisez Copie en écriture pour plus d'informations. Cette option est désactivée par défaut et il est recommandé de la conserver pour les nouveaux projets.

La section des options principales contient également un menu pour les informations sur les modèles.

L'explorateur d'actifs vous proposera également d'autres options de jeu en fonction des plates-formes cibles disponibles pour la licence que vous possédez. Il se peut que toutes ces options ne soient pas disponibles ou visibles.

Paramètres de la plate-forme

Les options de jeu suivantes, spécifiques à la plate-forme, existent :

  • Opéra GX
  • Windows
  • macOS
  • Ubuntu
  • Windows UWP
  • HTML5
  • Android
  • iOS
  • tvOS

NOTE Les différentes options de jeux sur console ne sont pas listées ici en raison des diverses restrictions légales maintenues par les différentes sociétés. Si vous avez une licence de console, vous pouvez obtenir ces informations auprès du service d'assistance de YoYo Games, mais vous devez d'abord contacter YoYo Games avec l'adresse e-mail de votre console et demander la permission de consulter les différentes sections.

Certains projets GameMaker peuvent également afficher des options de jeu pour la cible Amazon Fire. Il s'agit d'options anciennes, en lecture seule, car GameMaker prend désormais en charge la plate-forme Fire directement à l'aide des options de jeu Android. Si votre projet affiche toujours Amazon Fire, vous pouvez trouver des informations sur ces options à la page suivante :

  • Amazon Fire (Legacy)
