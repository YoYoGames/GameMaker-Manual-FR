Options de jeu tvOS

Cette section présente les différentes options disponibles qui contrôlent la façon dont vos projets de jeu tvOS seront compilés. Les différentes sections sont les suivantes :

Général

Ici, vous pouvez remplir les informations sur le produit concernant votre jeu, y compris son nom d'affichage, son ID groupé et les informations sur la version . Notez que l'ID de l'offre groupée doit être dans le format URL inverse correct pour que la version finale du jeu fonctionne correctement, par exemple : com.[Company].[GameName]. Vous pouvez également définir l'emplacement de la sortie de l'application sur le Mac de construction, et vous pouvez indiquer l'identifiant d'équipe que vous souhaitez utiliser pour signer l'application finale que GameMaker crée pour vous. Ce paramètre remplacera l'identifiant d'équipe que vous avez fourni dans les préférences tvOS.

Ici, vous avez également la possibilité de générer un Podfile et un Podfile.lock. Ces fichiers sont nécessaires si vous ajoutez CocoaPods à une extension tvOS à l'aide de l'éditeur d'extension. Les fichiers contiennent des macros spécifiques requises par GameMaker pour configurer toute liste de CocaoPods lors de la création de votre jeu. En cliquant sur les boutons, vous créerez les fichiers avec les macros pré-générées, et vous ouvrirez également un explorateur de fichiers à l'emplacement des fichiers, si vous souhaitez les modifier manuellement. Il y a également un bouton qui vous mènera au guide suivant :

  • iOS et tvOS : utilisation des pods Cocoa

Graphiques

Ici, vous pouvez modifier les détails suivants relatifs à la façon dont votre jeu sera affiché, avec les options suivantes disponibles :

  • Interpoler les couleurs entre les pixels: Active l'interpolation linéaire, qui a pour effet de "lisser" les pixels. Pour des graphiques de pixels nets, il est préférable de désactiver cette fonction, mais si vous avez de beaux mélanges alpha et des graphiques aux bords lissés, il est préférable de l'activer. Cette option est désactivée par défaut.
  • Mise à l'échelle: Ici, vous pouvez choisir de mettre à l'échelle le canevas de dessin en conservant le rapport d'aspect de l'écran de l'appareil - en ajoutant un "rembourrage" autour des bords pour s'adapter à l'écran - ou vous pouvez choisir d'étirer le canevas de dessin pour l'adapter à l'écran de l'appareil.

Ensuite, il y a l'option pour définir la taille de la page de texture. La taille par défaut (et la plus compatible) est de 2048x2048, mais vous pouvez choisir entre 256x256 et 8192x8192. Il y a aussi un bouton " Preview" qui génère les pages de texture pour cette plate-forme et ouvre une fenêtre pour que vous puissiez voir à quoi elles ressemblent. Cela peut être très utile si vous souhaitez voir comment les pages de texture sont structurées et éviter d'avoir des pages de texture plus grandes (ou plus petites) que nécessaire. Pour plus d'informations sur les pages de texture, veuillez cliquer ici.

NOTE: Sachez que plus la taille de la page de texture est grande, moins votre jeu sera compatible avec les différents navigateurs et appareils.

Enfin, vous pouvez choisir d'afficher ou non un curseur. Si cette option est cochée, le système d'exploitation affichera un curseur à chaque fois que la souris sera déplacée (à la position de la souris), et le curseur s'estompera et disparaîtra si la souris n'est pas déplacée pendant un certain temps. Si cette case n'est pas cochée, aucun curseur ne sera affiché.

Icônes

Cette section vous permet d'ajouter les différentes icônes requises par les différents appareils et les différentes parties de l'App Store. Ces icônes doivent être créées en tant qu'images 24bit *.png à la taille spécifiée pour chacune d'entre elles.

Il convient de noter que GameMaker dispose d'un outil de génération d'images de projet qui peut être utilisé pour créer automatiquement toutes les images requises pour toutes les différentes plates-formes cibles pour lesquelles votre jeu est compilé. Si vous utilisez cet outil, vous devez revoir les images créées pour vous assurer qu'elles correspondent à vos besoins, en particulier pour cette plate-forme car les icônes requises sont rectangulaires et non carrées. Cet outil génère également des images (expliquées ci-dessous) ainsi que des icônes.

Images

Cette section vous permet d'ajouter des graphiques distincts comme images "Top Shelf" et écrans d'accueil pour chacune des différentes tailles de cible tvOS. Ces images doivent être créées en tant qu'images individuelles 24 bits *.png, sans transparences et à la bonne taille. Vous pouvez également définir une durée minimale d'affichage des écrans d'accueil dans le menu déroulant situé en bas (de 0 à 10 secondes).

Comme mentionné dans la section sur les icônes, vous pouvez utiliser le générateur d'images du projet pour créer ces images.
