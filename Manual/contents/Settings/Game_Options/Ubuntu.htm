Options de jeu d'Ubuntu

Cette section décrit les différentes options disponibles qui contrôlent la façon dont vos projets de jeu Ubuntu (Linux) seront compilés. Les différentes sections sont les suivantes :

Général

Cet onglet vous permet de définir le nom de votre jeu tel qu'il sera affiché lors de son exécution. Vous pouvez également fournir une adresse électronique de contact et une description (courte et longue), ainsi qu'une URL pour la page d'accueil du jeu ou du développeur.

Ici, vous avez également la possibilité d'activer Steam dans votre jeu. Si vous activez cette option, assurez-vous que le SDK Steamworks est installé (voir ici) et que vous avez ajouté l'ID de l'application Steam dans les options générales du jeu.

Enfin, vous avez la possibilité de désactiver le sandbox du système de fichiers. Si vous cochez cette option, le sandboxing de GameMaker sera désactivé pour l'enregistrement et le chargement des fichiers, ce qui permettra à vos jeux d'accéder à access à partir de n'importe quel système exécutant le jeu. Cette option n'est pas cochée par défaut, car nous vous recommandons de maintenir le bac à sable de GameMaker sandbox pour les fichiers sur tous les systèmes et, lorsqu'elle est cochée, certaines zones d'enregistrement/chargement peuvent ne pas être disponibles en fonction des autorisations du système d'exploitation. Pour plus d'informations sur le site sandbox, consultez les pages consacrées au système de fichiers de GameMaker.

Graphiques

Les options graphiques sont celles que vous devez configurer pour déterminer comment votre jeu utilisera la carte graphique de votre machine cible Ubuntu. Les options suivantes sont incluses pour que vous puissiez les modifier :

  • Démarrer en plein écran: Si cette option est cochée, le jeu démarrera en mode plein écran, sinon il démarrera en mode fenêtré. Cette option est désactivée par défaut.
  • Autoriser le passage en plein écran: si cette option est cochée, l'utilisateur peut passer du plein écran au mode fenêtré et inversement en utilisant les raccourcis standard du système d'exploitation Ubuntu. Cette option est désactivée par défaut.
  • Interpoler les couleurs entre les pixels: Active l'interpolation linéaire, qui a pour effet de "lisser" les pixels. Pour des graphiques de pixels nets, il est préférable de désactiver cette fonction, mais si vous avez de beaux mélanges alpha et des graphiques aux bords lissés, il est préférable de l'activer.
  • Afficher le curseur: Lorsque cette option est activée, le curseur par défaut du système d'exploitation sera affiché dans votre jeu. Si cette option est désactivée, lorsque la souris est au-dessus de la fenêtre du jeu, aucun curseur ne sera visible, à moins que vous n'en ayez créé un dans le cadre du projet de jeu (cette option est activée par défaut).
  • Utiliser la synchronisation pour éviter le déchirement: Cette option permet d'activer ou de désactiver v-sync. Notez que si vous avez un jeu avec une vitesse de 120 et que le joueur a un moniteur avec une fréquence de rafraîchissement de 60, l'activation de cette option verrouillera la vitesse du jeu à 60 également. Cette option est désactivée par défaut.
  • Autoriser le redimensionnement de la fenêtre: En cochant cette case, l'utilisateur peut modifier la taille de la fenêtre du jeu (l'option Fenêtre sans bordure doit être désactivée pour que cela fonctionne). Cette option est désactivée par défaut.
  • Mise à l'échelle: Vous pouvez choisir de conserver le rapport hauteur/largeur (une pièce 4:3 sera donc "letter boxée" sur un écran 16:9) ou de mettre l'image à l'échelle (en l'étirant pour l'adapter au plein écran).

    ATTENTION ! La désactivation de la surface d'application désactive toutes les options de mise à l'échelle définies dans les options de jeu de Windows jusqu'à ce qu'elle soit réactivée. Voir La surface d'application pour plus de détails.

Enfin, il est possible de définir la taille de la page de texture. La taille par défaut (et la plus compatible) est de 2048x2048, mais vous pouvez choisir entre 256x256 et 8192x8192. Il y a également un bouton marqué Prévisualisation qui génère les pages de texture pour cette plate-forme et ouvre une fenêtre pour que vous puissiez voir à quoi elles ressemblent. Cela peut être très utile si vous souhaitez voir comment les pages de texture sont structurées et éviter d'avoir des pages de texture plus grandes (ou plus petites) que nécessaire. Pour plus d'informations sur les pages de texture, veuillez cliquer ici.

REMARQUE: plus la taille de la page de texture est grande, moins votre jeu sera compatible avec les PC ayant des spécifications inférieures.

Images

Ici, vous pouvez ajouter un écran d'accueil pour votre jeu, et définir l'icône de jeu qui sera utilisée. L'icône doit être une image de 64x64 pixels au format .png, et l'écran d'accueil doit avoir au moins la même taille que la première pièce de votre jeu (ou son port d'affichage) et être également au format .png. L'écran d'accueil sera affiché pendant le chargement du jeu et peut être activé ou désactivé en cochant l'option Utiliser l'écran d'accueil ( qui est désactivée par défaut).

Il convient de noter que GameMaker dispose d'un outil de génération d'images de projet qui peut être utilisé pour créer automatiquement toutes les images requises pour toutes les différentes plates-formes cibles pour lesquelles votre jeu est compilé. Si vous utilisez cet outil, vous devez réviser les images créées pour vous assurer qu'elles correspondent à vos besoins.
