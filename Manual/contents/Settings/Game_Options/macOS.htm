Options de jeu de macOS

Cette section présente les différentes options disponibles qui contrôlent la façon dont vos projets de jeu macOS seront compilés. Les différentes sections sont les suivantes :

Général

Dans la section Informations sur le produit , la première chose à faire est d'indiquer le nom d'affichage de votre jeu, puis le numéro de version, l'ID de l'application, le fichier de sortie de l'application et les détails du droit d'auteur. Le fichier de sortie de l'application est l'emplacement dans lequel les fichiers d'installation de votre jeu sont placés sur votre machine de développement Mac. Lorsque vous créez votre .app.zip, ce fichier peut aller n'importe où (l'emplacement que vous spécifiez lors du dialogue de sauvegarde n'est pas la seule copie faite de ce .app.zip), mais le .pkg et les autres fichiers sont laissés dans ce dossier, ce qui vous permet de tester les installations locales via le Terminal de manière à simuler l'achat dans le Mac Store, etc...

Vous devez également indiquer ici l'identifiant d'équipe que vous souhaitez utiliser pour signer l'application finale que GameMaker crée pour vous. Ce paramètre remplacera l'identifiant d'équipe que vous avez fourni dans les préférences de macOS. Après l'identifiant de l'équipe, vous avez également la possibilité de fournir l'identifiant de signature, qui est requis par Apple pour toutes les applications hors boutique (comme les jeux Steam, par exemple).

L'option suivante permet d'activer Steam dans votre jeu. Si vous activez cette option, assurez-vous que le SDK Steamworks est installé (voir ici) et que vous avez ajouté l'ID de l'application Steam dans les options générales du jeu.

Enfin, vous avez la possibilité de désactiver le sandbox du système de fichiers. Si vous cochez cette option, le sandboxing de GameMaker sera désactivé pour la sauvegarde et le chargement des fichiers, ce qui permettra à vos jeux d'utiliser access à partir de n'importe quel endroit du système sur lequel le jeu est exécuté. Cette option n'est pas cochée par défaut, car nous vous recommandons de maintenir le bac à sable de GameMaker sandbox pour les fichiers sur tous les systèmes et, lorsqu'elle est cochée, certaines zones de sauvegarde/chargement peuvent ne pas être disponibles en fonction des autorisations du système d'exploitation. Pour plus d'informations sur le site sandbox, consultez les pages consacrées au système de fichiers GameMaker.

Graphiques

Les options graphiques sont celles que vous devez configurer pour déterminer comment votre jeu utilisera la carte graphique de votre Mac cible. Les options suivantes sont incluses pour que vous puissiez les modifier :

  • Autoriser le menu et le dock en plein écran: Lorsqu'elle est cochée, cette option permet d'afficher le menu et le dock du système d'exploitation si le jeu est en mode plein écran. Cette option est désactivée par défaut.
  • Afficher le curseur: Lorsque cette option est cochée, le curseur habituel de macOS sera affiché, et si vous la décochez, aucun curseur ne sera affiché, à moins que vous n'en ayez créé un dans votre code de jeu. Cette option est désactivée par défaut.
  • Démarrer en plein écran: Si cette option est cochée, le jeu démarrera en mode plein écran, sinon il démarrera en mode fenêtré. Cette option est désactivée par défaut.
  • Autoriser le passage au plein écran: si cette option est cochée, l'utilisateur peut passer du plein écran au fenêtré et inversement en utilisant les raccourcis standard de macOS. Cette option est désactivée par défaut.
  • Interpoler les couleurs entre les pixels: Active l'interpolation linéaire, qui a pour effet de "lisser" les pixels. Pour des graphiques en pixels nets, il est préférable de désactiver cette fonction, mais si vous avez de beaux mélanges alpha et des graphiques aux bords lissés, il est préférable de l'activer. Cette option est désactivée par défaut.
  • Utiliser la synchronisation pour éviter le déchirement: Cette option permet d'activer ou de désactiver v-sync. Notez que si vous avez un jeu avec une vitesse de 120 et que le joueur a un moniteur avec une fréquence de rafraîchissement de 60, l'activation de cette option verrouillera la vitesse du jeu à 60 également. Cette option est désactivée par défaut.
  • Autoriser le redimensionnement de la fenêtre: En cochant cette case, l'utilisateur peut modifier la taille de la fenêtre du jeu (l'option Fenêtre sans bordure doit être désactivée pour que cela fonctionne). Cette option est désactivée par défaut.
  • Enable Retina (Activer la rétine): si vous cochez cette case, le fond de l'écran buffer sera à la résolution la plus élevée (réelle) lorsque le jeu est exécuté sur un moniteur compatible avec la rétine, tandis que si vous la décochez, le fond de l'écran buffer sera à la résolution apparente (inférieure). Cela signifie que la fenêtre de jeu initiale aura deux fois la largeur et la hauteur de la pièce/du port de vue de la première pièce. Cependant, cela ne met PAS à l'échelle la surface d'application pour s'adapter, et donc si vous voulez que votre jeu tire le meilleur parti de l'affichage rétina, vous devez également définir la surface d'application pour qu'elle soit de la même taille que la fenêtre (notez que cela doublera les pixels sur la largeur et la hauteur, donc n'activez cette option et ne mettez à l'échelle la surface d'application que si votre jeu bénéficiera de la résolution accrue).

    Cette option est désactivée par défaut, et l'activer ne fera aucune différence lorsque vous exécuterez ou déboguerez le jeu à l'aide de VM, car le site runner est préconstruit dans ce cas. Elle ne fonctionnera que si vous construisez un exécutable via VM, ou si vous utilisez YYC pour tester ou construire le jeu, car cela reconstruira le site runner avec vos options de jeu personnalisées.
  • Mise à l'échelle: Vous pouvez choisir de conserver le rapport hauteur/largeur (une pièce 4:3 sera donc "letter boxée" sur un écran 16:9) ou de mettre l'image à l'échelle (en l'étirant pour qu'elle s'adapte au plein écran).

    ATTENTION ! La désactivation de la surface d'application désactive toutes les options de mise à l'échelle définies dans les options de jeu de Windows jusqu'à ce qu'elle soit réactivée. Voir La surface d'application pour plus de détails.

Enfin, il est possible de définir la taille de la page de texture. La taille par défaut (et la plus compatible) est de 2048x2048, mais vous pouvez choisir entre 256x256 et 8192x8192. Il y a également un bouton marqué Prévisualisation qui génère les pages de texture pour cette plate-forme et ouvre une fenêtre pour que vous puissiez voir à quoi elles ressemblent. Cela peut être très utile si vous souhaitez voir comment les pages de texture sont structurées et éviter d'avoir des pages de texture plus grandes (ou plus petites) que nécessaire. Pour plus d'informations sur les pages de texture, veuillez cliquer ici.

REMARQUE: plus la taille de la page de texture est grande, moins votre jeu sera compatible avec les PC ayant des spécifications inférieures.

Images

La section des images est l'endroit où vous fournissez les images dont votre jeu a besoin. Pour Mac, vous devez fournir un fichier d'icône (qui doit être au format .png et 1024x1024px) et un écran d'accueil (qui peut être au format .png, .bmp, .jpg ou .gif ). L'écran d'accueil s'affichera pendant le chargement du jeu.

Pour les paquets DMG, vous pouvez fournir une image d'arrière-plan de l'installateur qui apparaîtra comme arrière-plan de l'installateur DMG.

Il convient de noter que GameMaker dispose d'un outil de génération d'images de projet qui peut être utilisé pour créer automatiquement toutes les images requises pour toutes les différentes plates-formes cibles pour lesquelles votre jeu est compilé. Si vous utilisez cet outil, vous devez réviser les images créées pour vous assurer qu'elles correspondent à vos besoins.

Emballage

Si vous souhaitez que le jeu terminé soit prêt pour l'App Store, vous devez cocher l'option Build for Mac App Store, mais sachez que cette option ne fonctionnera que si vous êtes un développeur enregistré et que vous disposez des certificats nécessaires. En dessous se trouvent les autorisations de l'application, que vous ne devez vérifier que si elles sont vraies, car Apple peut rejeter votre application si elles sont cochées alors qu'elles ne sont pas nécessaires, ou vice-versa. Ces options permettent simplement à votre jeu d'utiliser les fonctions http_ et url_. Il convient de noter que si vous souhaitez prendre en charge les GamePads dans votre jeu, l'option permettant de créer un paquet prêt pour l'App Store doit être désactivée.

Enfin, vous devez sélectionner sa catégorie d'application (pour plus d'informations, voir ici).

Social

Cette option permet d'utiliser l'extension Apple Sign In sur Mac. Vous pouvez obtenir cette extension sur le Marché GameMaker, et l'extension elle-même contient des instructions complètes d'utilisation. Si vous n'utilisez pas cette extension, vous ne devez pas cocher cette option.
