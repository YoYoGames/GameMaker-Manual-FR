Groupes de textures

Le gestionnaire de groupes de textures est disponible dans le menu Outils sur le site IDE. Vous pouvez y ajouter, supprimer et renommer des groupes de textures.

GameMaker vous permet d'affecter chacune des ressources graphiques (sprites, tilesets, et polices) à différents groupes de textures pour essayer d'optimiser le nombre de changements de textures que le matériel effectue à tout moment dans votre jeu. Pour que cela fonctionne, vous devez définir les groupes de textures nécessaires à partir de cette fenêtre, puis passer par chacun des types de ressources appropriés et sélectionner le groupe auquel vous souhaitez qu'il appartienne dans le menu déroulant.

Vous pouvez également ajouter des ressources à un groupe de textures via le menu du bouton droit de la souris dans le navigateur de ressources. Vous pouvez sélectionner un ou plusieurs sprites, ensembles de tuiles ou polices de caractères, puis utiliser le menu RMB pour ajouter les ressources à un groupe, comme illustré ci-dessous pour les ensembles de tuiles :

il est important de comprendre qu'un groupe de texture unique peut avoir plusieurs pages de texture, et qu'elles ne sont pas équivalentes. Si votre jeu comporte de nombreux graphiques uniques pour chaque niveau, vous pouvez définir un seul groupe de texture pour chacun des niveaux, mais les graphiques peuvent nécessiter deux pages de texture ou plus par groupe. Cela dépend de la capacité de la taille de la page de texture sélectionnée (dans les options de jeu de votre cible) à contenir tous les graphiques d'un groupe de texture.

Par défaut, toutes les images de votre jeu seront ajoutées au groupe de textures "Default", qui existe toujours. Cependant, vous pouvez cliquer sur le bouton "Add New " pour créer un nouveau groupe de textures, et si vous sélectionnez le nom du groupe, vous pouvez le modifier de manière appropriée. Pour commencer, le groupe affichera "ce groupe est vide" pour chaque section du groupe (sprites, tilesets et polices), et vous devrez donc parcourir vos ressources graphiques et les ajouter à un groupe. Vous pouvez ajouter une ressource en utilisant le bouton Ajouter une ressource en bas de page, ou en sélectionnant le groupe de texture dans l'éditeur de ressources pour l'image, le jeu de tuiles ou la police. Les groupes de textures autres que le groupe "Par défaut" peuvent également être supprimés à tout moment, ce qui aura pour effet de déplacer leur contenu dans le groupe "Par défaut" (notez que si vous supprimez un groupe par erreur, vous pouvez utiliser / + "Z" pour annuler l'action).

Une fois que vous avez défini des groupes de textures - et que vous leur avez attribué des ressources graphiques - vous en verrez la liste à gauche de la fenêtre principale de l'éditeur de groupes de textures. En cliquant sur l'une des adresses sprites de cette liste, vous l'afficherez dans la fenêtre de prévisualisation des images à droite. Ceci est particulièrement utile lorsque vous travaillez sur de grands projets et que vous devez être en mesure de voir quelles images ont été assignées à quel groupe. Notez que si les images assignées au groupe ont plusieurs sous-images (c'est-à-dire qu'elles sont animées), seule la première sous-image est affichée dans la fenêtre d'aperçu. Vous pouvez également faire un clic droit sur une ressource pour ouvrir un petit menu vous donnant la possibilité de la déplacer vers un autre groupe de texture ou d'ouvrir l'éditeur de cette ressource.

Il convient de noter que même si les ressources sont indiquées dans la liste comme étant séparées, tous les sprites, les jeux de tuiles et les polices seront ajoutés à la même page (ou aux mêmes pages) de texture, et n'obtiendront pas une page individuelle pour chaque type de ressource. La seule exception à cette règle est lorsqu'un actif sprite a été marqué comme étant pour une "page de texture séparée", dans ce cas il recevra sa propre page de texture individuelle (voir la section sur les " paramètres de texture " dans l'éditeur de sprites pour plus de détails).

Paramètres du groupe

Les paramètres de groupe dans l'éditeur de groupe de texture sont utilisés pour déterminer exactement comment les pages de texture d'un groupe donné doivent être créées, avec les options suivantes disponibles :

  • Allow Scaling (Autoriser la mise à l'échelle) - En désactivant cette option, vous obligerez GameMaker à ne pas mettre à l'échelle le groupe de textures, quelles que soient les circonstances. Cela peut être utile, notamment dans le cas d'applications universelles, car cela vous permet de placer des ressources spécifiques, comme les polices, dans un groupe à part, qui sera utilisé lorsque l'application sera exécutée sur un appareil spécifique. Par exemple, les polices peuvent être très belles sur un écran rétina, mais lorsqu'elles sont mises à l'échelle pour être dessinées sur un écran normal, elles peuvent être terribles ; vous pouvez donc les affecter à un groupe spécial sans mise à l'échelle. Si vous développez pour l'iPhone, vous souhaiterez peut-être cocher cette case, car des applications ont été rejetées parce que les images étaient floues en raison de la mise à l'échelle de la texture. Cette option est activée par défaut.
  • Recadrage automatique - Lorsque vous ajoutez des images dans GameMaker, si elles ont de l'"espace" autour d'elles (c'est-à-dire 100% de transparence), elles sont automatiquement recadrées lorsqu'elles sont ajoutées dans la page de texture et "Recadrage automatique" est marqué. Cependant, si, par exemple, vous utilisez l'image comme une texture non répétitive pour primitives ou quelque chose de similaire, cela signifie que les coordonnées UV pour le mappage de la texture ne seront pas correctement représentées. Dans ce cas, vous pouvez désactiver cette option et le site sprite sera ajouté à la page de texture "tel quel", en conservant les transparences qu'il peut avoir sur les bords. Cette option est activée par défaut.
  • Generate Mipmaps - En activant cette option (elle est désactivée par défaut), vous indiquez à GameMaker que la page de texture originale doit être mise à l'échelle et filtrée en plusieurs résolutions dans le fichier de texture. Cette option est normalement désactivée, mais si vous utilisez des caméras pour zoomer ou dézoomer sur de grandes zones de la salle de jeu, ou si vous travaillez avec un jeu en 3D, vous pouvez l'activer et utiliser les fonctions de mipmap pour ajuster la façon dont le jeu traite les différents niveaux de mipmap. Lorsque vous cochez cette case, la valeur "Border Size" (taille de la bordure) sera fixée à 8 (si elle était fixée à moins de 8 à l'origine). En effet, lorsque vous utilisez des mipmaps, vous avez besoin de bordures plus larges entre les images sur les pages de texture, sinon elles commencent à se mélanger à des niveaux de mipmap plus élevés. La valeur par défaut de 8 permet de réduire l'image à environ un huitième de sa taille d'origine avant que les sous-images ne commencent à se mélanger, mais vous pouvez modifier cette valeur pour qu'elle corresponde à vos besoins. Notez que si vous utilisez l'option "Separate Texture Page" pour sprites, vous n'avez pas besoin de passer en revue chaque texture ici et de la marquer pour générer des mipmaps, mais vous pouvez aller dans les Options générales du jeu et marquer l'option "Generate mipmaps for separate texture pages".
  • Taille de la bordure - La taille de la bordure modifie la quantité de chevauchement (ou "doublage des pixels") autour des bords de toutes les ressources sur les pages générées pour le groupe de textures, de sorte que lorsque les images sont mises à l'échelle, aucun vide (ou couture) n'est visible entre elles. La valeur par défaut est de 2 pixels, mais dans les cas extrêmes où vous effectuez un zoom avant ou arrière important, vous devrez peut-être augmenter cette valeur. Notez que l'éditeur de jeu de tuiles a une option pour définir la bordure de sortie pour les tuiles (et fournit également une explication de la raison pour laquelle cela est nécessaire), et ceci est distinct du réglage de la bordure ici - fondamentalement, les tuiles sont générées avec une bordure de sortie pour chaque tuile sur une texture temporaire, et celle-ci est ensuite ajoutée à la page de texture avec la bordure donnée ici autour du bord extérieur. Notez également que l'activation de Mipmaps définira automatiquement la taille de la bordure par défaut pour la page de texture à 8 pour aider à prévenir le "saignement" entre les textures à des niveaux de mip plus élevés, mais cela peut être changé.
  • Groupe de texture parent - Ici, vous pouvez choisir n'importe quel groupe existant pour qu'il devienne le groupe "parent" du groupe actuellement sélectionné. Les groupes "enfants" héritent de toutes les modifications apportées au groupe de texture parent et servent principalement à la gestion des ressources lorsque vous travaillez avec des jeux multiplateformes. Un groupe de texture avec un parent est toujours un groupe de texture à part entière et génère des pages de texture de la même manière que n'importe quel autre groupe, mais le fait de lui attribuer un parent vous permet d'exclure de grandes parties pour différentes plateformes. Par exemple, vous pouvez ajouter tout un tas de Android joueur sprites à un groupe "joueur", et Android ennemi sprites à un groupe "ennemi", puis leur faire utiliser un groupe parent "android". Si vous excluez ensuite le groupe parent "android" de la construction de Windows ou d'une autre plateforme, les groupes enfants seront également exclus.
  • Format du groupe de textures - Vous pouvez choisir ici le format de votre page de textures, entre BZ2 + QOI, QOI et PNG. Il est recommandé de conserver l'option par défaut, BZ2 + QOI, car elle offre la meilleure compression tout en conservant la qualité.

    Vous pouvez choisir QOI si l'option BZ2 affecte la vitesse sur une plateforme cible (par exemple Android), ou PNG si vous souhaitez manipuler les PNG exportés manuellement ou utiliser l'option PNGCrush.

En dehors de ces paramètres, vous pouvez également choisir d'exporter (ou non) un groupe de texture donné vers une plateforme donnée en cochant ou décochant simplement les cases appropriées dans la section Use Group On... de la fenêtre. Notez cependant que le groupe de textures <Default> est toujours exporté vers toutes les plates-formes, et que vous ne pouvez pas cocher/décocher l'une d'entre elles (seuls les groupes de textures personnalisés le permettent).

Les groupes de textures sont également liés aux paramètres des configurations. Cela signifie que vous pouvez définir des groupes de textures pour chaque configuration individuelle de votre jeu, puis attribuer le site sprite, le jeu de tuiles ou les ressources de polices à ces différents groupes et ils seront "mémorisés" lorsque vous passerez à une autre configuration. Il en va de même pour toutes les options d'exportation que vous avez choisies dans la fenêtre Groupe de textures, de sorte que vous pouvez définir les groupes de textures pour qu'ils soient exportés vers des plates-formes spécifiques sur la base de chaque configuration.

Vous pouvez, par exemple, créer une configuration iPad et créer des pages de texture de 2048x2048px (en configurant cela via les options du jeu). Vous attribuez ensuite toutes les ressources graphiques au groupe de texture approprié et, une fois terminé, vous passez à une autre configuration (pour HTML5, par exemple) et, à partir de la fenêtre Groupe de texture, vous créez de nouveaux groupes de texture de 512x512px, en attribuant les ressources à ces nouveaux groupes. Il existe également un bouton en bas de la page qui vous permet de copier les paramètres du groupe de texture d'une configuration à une autre.

Enfin, les groupes de texture ne concernent que les images bitmap, et les images non bitmap sprites ne seront pas affectées à des groupes de texture. Certaines d'entre elles auront déjà leur propre page de texture unique (animation de squelette sprites), tandis que d'autres n'ont pas besoin de textures car elles sont composées de vecteurs.
