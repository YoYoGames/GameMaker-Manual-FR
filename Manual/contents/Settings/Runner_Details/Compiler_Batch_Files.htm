Fichiers batch du compilateur / Scripts

Le compilateur GameMaker vous permet de placer des fichiers Batch (Windows) ou des scripts Shell (sur macOS et Linux) dans le répertoire de votre projet et dans le répertoire de chaque extension de votre projet.

Placement

Un fichier script peut être placé à la racine du répertoire de votre projet, à côté du fichier .yyp du projet.

Un fichier script peut également être placé dans le répertoire d'une extension, qui peut ressembler à ceci : <répertoire_du_projet>/extensions/<dossier_de_l'extension>/.

Les versions d'extension d'un fichier de script seront exécutées en premier, et sa version racine sera exécutée en dernier. L'ordre dans lequel les extensions sont exécutées ne peut être contrôlé. Le développeur d'une extension doit donc s'assurer que les scripts d'une extension ne dépendent pas des scripts d'une autre extension.

Exécution

Des fichiers de script portant des noms spécifiques, placés dans l'un des répertoires décrits ci-dessus, sont exécutés à différents moments du processus de compilation :

Nom du fichier Batch Windows Nom du script shell macOS/Linux Description
pre_run_step.bat pre_run_step.sh Cette opération est exécutée avant que le déploiement du jeu ne commence.
post_run_step.bat post_run_step.sh Il est exécuté lorsque le jeu est préparé et prêt à être exécuté ; après l'exécution du script, le jeu est lancé.
pre_build_step.bat pre_build_step.sh Cette opération est exécutée avant que le compilateur d'actifs ne soit lancé pour construire le jeu.
post_build_step.bat post_build_step.sh Ceci est exécuté après que le compilateur d'actifs ait été lancé pour construire le jeu.
remote_build_step.bat remote_build_step.sh This is executed on the remote machine, when you are on a Windows machine and compiling remotely for macOS/iOS/tvOS or Ubuntu.
pre_package_step.bat pre_package_step.sh Cette étape est exécutée avant l'étape de conditionnement final, c'est-à-dire lorsque tous les fichiers sont prêts mais que le fichier ZIP final ou le paquet de stockage est sur le point d'être créé.
post_package_step.bat post_package_step.sh Cette opération est exécutée après la dernière étape de l'emballage et la préparation du fichier ZIP final ou du paquet de magasin.
post_textures.bat post_textures.sh Cette étape est exécutée après que les textures de votre jeu ont été générées. Vous pouvez utiliser cette étape pour optimiser manuellement les textures de votre jeu.

Ce script reçoit une variable d'environnement appelée TexturesDir, qui stocke le chemin vers les textures générées.
Platform-Specific Steps
pre_gradle_step.bat pre_gradle_step.sh Android This is executed when the files necessary for the Android tools have been created, but before Gradle is called. You can use this step to access and modify the Android files yourself, before the Android tools compile it into a final executable.

Ces fichiers de script sont pris en charge lors de la construction sur toutes les plateformes cibles.

Désactiver les scripts pour une extension

Si vous utilisez plusieurs extensions qui exécutent des scripts, par exemple Steamworks et GDK, vous voudrez désactiver Steamworks lors de l'exportation vers GDK, et vice-versa.

Pour cela, voir : Comment désactiver les extensions ?
