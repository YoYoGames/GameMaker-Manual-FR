Cloner un référentiel

Lorsque votre projet est stocké dans un dépôt externe, vous devez vous souvenir de pousser vos modifications et de retirer les nouvelles. La raison en est que lorsque vous livrez, vous stockez vos modifications localement, ce qui vous permet de travailler loin d'une connexion Internet et de vous synchroniser lorsque vous êtes prêt, et pour vous synchroniser, vous poussez vos modifications vers le référentiel distant. L'extraction des modifications signifie que vous voulez extraire toutes les modifications que d'autres personnes ont apportées depuis le dépôt distant vers votre dépôt local. Cependant, cela signifie qu'il peut arriver que des personnes modifient les mêmes fichiers, ce qui crée un conflit de fusion. Vous devez savoir comment gérer ces conflits. Nous allons donc en créer délibérément un maintenant et vous montrer comment le gérer.

REMARQUE: Le partage d'un dépôt de projet local (même par le biais d'un service de synchronisation de fichiers comme Dropbox) n'est pas recommandé, car les commits y sont généralement écrits directement, et vous avez donc très peu de protection contre les conflits.

Pour générer notre conflit, nous devons d'abord avoir deux référentiels locaux, un pour le référentiel maître et un pour notre copie. Pour créer cela, nous devons cloner le référentiel, ce qui est fait avec les étapes suivantes :

  • Allez dans Source Control > Clone Repository.
  • Dans le champ URL, tapez le chemin d'accès au projet que nous avons configuré dans le guide sur l'inversion des fichiers ci-dessus (par défaut, ce sera <DRIVE>:\Users\<Username>\Documents\GameMaker\<ProjectName>).
  • Nous devons ensuite définir un chemin vers lequel cloner :
    Lorsque nous cliquons sur le bouton "Ok", nous obtenons l'avertissement suivant concernant la tentative de cloner un dépôt "non nu" :
    Nous pouvons cliquer sur "Ok" ici encore.
  • On nous présente maintenant un explorateur de répertoires pour créer un référentiel proxy "nu" (il s'agit d'un référentiel que vous pouvez stocker en toute sécurité sur Dropbox, par exemple). Naviguez jusqu'à l'endroit où nous voulons créer ce dépôt de proxy et sélectionnez (ou créez) le dossier à utiliser.

Une fois que nous avons fait cela, GameMaker va pousser le projet original vers le proxy, et en tirer un nouveau vers le chemin que nous avons défini dans la fenêtre Clone Repository, puis ouvrir automatiquement un navigateur de fichiers pour que nous puissions ouvrir le projet stocké. Si nous allons maintenant dans Source Control > View History, nous verrons nos commits originaux plus un nouveau commit "Push For Clone".

Vous avez maintenant cloné votre dépôt et pouvez continuer à travailler sur la version locale du projet.
