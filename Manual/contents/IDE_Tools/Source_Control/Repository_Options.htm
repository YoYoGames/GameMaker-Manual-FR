Options du référentiel

Après avoir configuré l'exécutable Git et activé le contrôle de la source pour votre projet, vous devez maintenant lier GameMaker IDE à un dépôt, ce qui peut se faire de deux manières :

  • Créer un référentiel de projet
  • Cloner un référentiel existant

Dans le premier cas, la création d'un référentiel de projet, cela créera un référentiel Git directement à l'endroit où se trouve votre projet. Dans le jargon de Git, il s'agit d'un "dépôt non nu", donc Push/Pull n'a pas de sens dans ce cas, mais vous pouvez toujours livrer et suivre les changements du projet, ainsi que revenir sur les changements et les révisions si nécessaire. Pour le mettre en place, il suffit de cliquer sur Source Control > Create Project Repository dans le menu principal.

En tant que fonction pratique, vous pouvez également cloner un dépôt externe via le site IDE. Là encore, vous aurez besoin du nom d'utilisateur et du mot de passe du dépôt auquel vous allez vous connecter, configurés dans vos préférences d'identité, et il vous suffira ensuite de cliquer sur Source Control > Clone Repository.
