Le Profiler

Le débogueur vous permet de parcourir votre code et de détecter tout problème dans la base de code, mais vous devez parfois vérifier des détails plus subtils afin d'optimiser correctement vos jeux. L'option Profil vous aide à le faire en vous donnant une vue d'ensemble des performances, en indiquant le temps nécessaire pour appeler les fonctions, les scripts et même le temps nécessaire pour qu'un événement spécifique se termine.

Outre la fenêtre principale qui montre les différents événements, scripts et fonctions appelés, vous disposez également des options suivantes pour utiliser le Profiler :

Activer / désactiver

Il s'agit du bouton permettant d'activer ou de désactiver le profileur. Lorsqu'il est activé, le profileur démarre et vous commencerez à voir les événements, les scripts et les fonctions être listés dans la fenêtre principale avec différentes valeurs de données pour chacun d'entre eux. La façon dont ces données sont affichées dépendra de l'affichage du temps et du mode d'affichage (tous deux expliqués plus en détail ci-dessous).


Affichage de l'heure

Cette case à cocher vous permet de basculer entre le temps moyen et le temps absolu pour le profileur. Lorsqu'elle n'est pas cochée, les valeurs affichées dans la fenêtre principale seront le nombre total d'appels faits à la fonction, au script ou à l'événement et le temps total pris (les temps sont en microsecondes) au cours du profil. fonction, script ou événement et le temps total pris (les temps sont en microsecondes) au cours du profil. Toutefois, si vous l'activez, ces colonnes indiquent le temps moyen pris par étape pour chaque fonction, script ou événement, et le nombre moyen de fois qu'il a été appelé.


Mode d'affichage

Ce menu vous permet de choisir le mode d'affichage, qui peut être soit Top Down, soit Bottom Up.

En sélectionnant le mode d'affichage Top Down, le profil sera affiché de haut en bas (plus ou moins de la même manière que la hiérarchie de la pile d'appels), c'est-à-dire qu'il montrera l'événement, puis le script et enfin toutes les fonctions utilisées. Notez que vous pouvez double-cliquer sur n'importe laquelle des entrées pour l'ouvrir dans la fenêtre de débogage de la source si vous en avez une. si vous en avez une. Si vous voyez une icône à côté d'un nom, cela signifie qu'il y a d'autres appels de script ou de fonction dans cette section. Cliquez sur pour développer l'arbre et les afficher.

En sélectionnant le mode d'affichage ascendant, les choses s'affichent de bas en haut, de sorte que vous pouvez voir toutes les fonctions et tous les appels de script individuellement. Dans ce mode d'affichage, si vous cliquez sur le site , la fonction ou le script sera développé pour montrer ce qui l'a appelé.


Voir la cible

La cible de visualisation vous permet de choisir entre l'affichage du projet GML (données sur les événements et le code), les processus du moteur en arrière-plan ou les deux. Si vous choisissez "GML", vous obtiendrez les événements, fonctions et scripts appelés à chaque étape. à chaque étape, tandis que la vue "Moteur" n'affichera que les appels au moteur requis par le projet, ce qui vous permettra de voir les éventuels goulets d'étranglement dans la façon dont votre projet gère les choses. Si vous sélectionnez "Both", alors les deux ensembles de données seront Si vous sélectionnez "Both", les deux ensembles de données seront affichés, mais notez que lorsque vous les combinez avec la vue "Bottom Up", les différents appels de scripts et de fonctions seront imbriqués dans les appels de moteur et vous devrez cliquer sur pour les développer afin de les voir.



Les données du profil sont présentées dans quatre colonnes, et en cliquant sur l'une d'entre elles, les données sont triées selon les critères de la colonne.

  • Nom: Il s'agit du nom de l'événement, du script ou de la fonction appelé(e). Il peut être accompagné d'un pour indiquer d'autres appels à cet événement ou à cette fonction (selon le mode d'affichage que vous avez sélectionné). Vous pouvez double-cliquer sur sur l'un des noms pour ouvrir le bloc de code ou le script dans la fenêtre Source. En cliquant sur le haut de cette colonne, les noms seront classés par ordre alphabétique.
  • Temps: Ceci montrera soit le temps total qu'une fonction (ou un script ou un événement) a pris pendant le profilage, soit le temps moyen qu'il a fallu à chaque étape pour être appelée (selon l'option d'affichage du temps sélectionnée) et il est affiché en microsecondes.
  • Appels: Vous pouvez voir ici soit le nombre total d'appels à l'événement, à la fonction ou au script donné, soit le nombre moyen d'appels à celui-ci en une seule étape (là encore, en fonction de l'option d'affichage du temps sélectionnée).
  • Step%: La valeur indiquée ici est le pourcentage global d'une étape unique que l'appel a pris.
