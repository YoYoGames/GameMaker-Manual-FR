Le débogueur

Le débogueur est un outil puissant pour vérifier votre jeu, et est particulièrement utile pour traquer les erreurs et les bogues dans votre code, ainsi que pour vérifier que les scripts sont exécutés quand ils le doivent, et que les variables et les tableaux contiennent les valeurs attendues, etc. valeurs que vous attendez, etc... Vous pouvez lancer le débogueur en exécutant votre jeu depuis GameMaker IDE à l'aide du bouton Debug en haut du site IDE, ou en allant dans le menu Build et en sélectionnant "Debug ".

REMARQUE: le pare-feu Windows (ou tout autre programme de pare-feu que vous exécutez) peut vous demander de créer une exception de sécurité pour ce module. Vous devez le faire, sinon il ne fonctionnera pas correctement. Ceci est dû à la façon dont le débogueur fonctionne et est essentiel pour les plateformes mobiles debugging.

Vue graphique

La vue Graphique vous montre une représentation graphique de l'utilisation de la mémoire et du taux de rafraîchissement de votre projet au fil du temps :

Le graphique indique l'utilisation actuelle et moyenne de la mémoire ainsi que le nombre de FPS actuel et moyen dans le temps. Vous pouvez effectuer un zoom avant ou arrière en utilisant les touches / ainsi que la molette de la souris , et si vous placez le curseur de la souris sur l'un des éléments du graphique, il vous indiquera la valeur de cet élément au point sélectionné.

La vue Graphique affiche également certains événements de débogage sous forme de points sur la longueur du graphique. Ces événements sont générés automatiquement par GameMaker à certains moments de l'exécution d'un jeu pour montrer des événements importants spécifiques, mais vous pouvez également ajouter vos propres éléments graphiques à l'aide des fonctions show_debug_message() et debug_event(). Les événements système seront marqués d'une couleur différente de celle des événements déclenchés par le code :

Vue des ressources

La vue des ressources vous montre une liste de toutes les ressources utilisées dans le projet qui peuvent contenir du code ou des actions Visual GML. Cette section fonctionne de la même manière que le navigateur d'actifs et vous pouvez développer les éléments pour afficher plus d'informations, par exemple les différents événements pour un objet spécifique. pour afficher plus d'informations, par exemple, les différents événements pour un objet spécifique. Vous pouvez ensuite double-cliquer sur sur l'une des ressources pour ouvrir le code correspondant dans la vue Code. Vous pouvez ensuite double-cliquer sur l'une des ressources pour ouvrir son code dans la fenêtre Code View, où vous pouvez vérifier les valeurs et ajouter ou supprimer des points d'arrêt.

Pile d'appels

Cette fenêtre affiche la pile d'événements en cours, ainsi que le numéro de ligne du code en cours d'exécution. La pile d'appels est essentiellement une structure de données "dernier entré - premier sorti" qui montre l'ordre d'exécution de tout le code du jeu, à la fois depuis GameMaker à partir de GameMaker runner - puisque debugging utilise la machine virtuelle (VM) - et de ce que vous avez écrit pour le jeu. Le débogueur devra avoir été mis en pause pour que des informations soient affichées ici.

Les trois premiers boutons sont utilisés pour contrôler votre jeu, vous permettant de le démarrer, de le mettre en pause, de le redémarrer ou de l'arrêter à tout moment. Lorsque le jeu est en pause (soit en cliquant sur le bouton Pause ou à partir d'un point d'arrêt), vous pouvez alors utiliser les trois boutons d'étape pour parcourir votre code.

Lorsque nous parlons de "pas à pas" dans le code, cela signifie simplement que vous pouvez demander à votre jeu d'exécuter la ligne de code en cours, puis de passer à la suivante, ce qui vous permet d'exécuter l'intégralité de votre jeu une ligne de code à la fois si vous le souhaitez. Normalement, vous utilisez simplement le bouton Step In pour faire cela, mais vous avez également un bouton pour passer au-dessus du code et un autre pour sortir du code. Le passage au code suivant est utile lorsque vous l'ensemble du script ou du bloc de code en une seule étape et s'arrêtera ensuite à la ligne suivante, tandis que l'étape de sortie vous fera sortir d'un script (il s'exécutera, mais en une seule fois) et s'arrêtera ensuite à l'étape suivante lorsque la portée changera. l'étape suivante lorsque la portée change.

La dernière icône pour les mises à jour en temps réel permet à la fenêtre de débogage de mettre à jour ses données en temps réel, de sorte que vous pouvez voir comment les variables changent ou les instances sont créées et détruites au fur et à mesure. Notez que cela peut affecter les performances du jeu et que toutes les informations ne sont pas disponibles en temps réel. Par exemple, la montre Render State (voir la section sur les montres) ne peut pas être visualisée en temps réel.

Vue du code

La vue du code est l'endroit où nous pouvons voir et revoir les différents scripts ou le code d'événement de notre projet pendant son exécution. Par défaut, cette vue est vide, mais vous pouvez double-cliquer sur sur n'importe quelle ressource de la vue des ressources pour l'ouvrir ici. Vous pouvez également ouvrir plusieurs événements ou scripts différents, et ils seront ajoutés à l'espace de travail de la vue Code sous forme d'onglets en haut.

Lorsqu'une fenêtre de code est ouverte dans la vue Code, vous pouvez cliquer avec le bouton gauche de la souris sur dans la gouttière (où se trouvent les numéros de ligne) pour ajouter ou supprimer des points d'arrêt (voir plus bas dans cette page pour plus d'informations sur les points d'arrêt).

Vous pouvez également utiliser le bouton droit de la souris pour ouvrir le menu contextuel, qui vous permet également d'ajouter ou de supprimer des points d'arrêt et de copier le code sélectionné dans le presse-papiers. code sélectionné dans le presse-papiers.

Une autre partie importante de debugging votre jeu est l'utilisation de points d'arrêt. Un point d'arrêt est un endroit du code de votre projet où vous souhaitez qu'il s'arrête pour que vous puissiez voir ce qui se passe. Lorsque le débogueur fonctionne, vous pouvez ajouter ou supprimer des points d'arrêt à tout moment et dans n'importe quel bloc de code ou de script en cliquant à gauche sur dans la gouttière (où se trouvent les numéros de ligne), ou vous pouvez utiliser le bouton droit de la souris pour pour ouvrir le menu contextuel, qui vous permet également d'ajouter ou de supprimer des points d'arrêt :

Les points d'arrêt peuvent également être définis à partir de GameMaker IDE. Lors de l'écriture de votre projet, ils peuvent être ajoutés ou supprimés à tout moment à l'aide de la commande f9 (ou en utilisant le bouton droit de la souris et en sélectionnant "toggle breakpoint"), soit sur une action dans la fenêtre d'objet (pour les utilisateurs de GML Visual), soit sur une ligne dans une fenêtre de l'éditeur de code. Désormais, lorsque vous lancerez votre jeu en mode débogage, il s'arrêtera au(x) point(s) que vous aurez défini(s).

Les points d'arrêt définis à partir de IDE sont enregistrés avec votre projet et persisteront au cours des différentes exécutions jusqu'à ce qu'ils soient supprimés. Toutefois, si vous ajoutez un point d'arrêt à votre code dans le débogueur, il sera perdu après la fermeture du module ou le lancement d'un nouveau test. Notez que tous les points d'arrêt apparaissent dans la fenêtre de sortie où ils peuvent être activés, désactivés ou même supprimés.
