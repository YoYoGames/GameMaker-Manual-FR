Paquets d'actifs locaux

Dans le menu Outils, vous disposez de deux options pour créer et importer des ensembles d'actifs locaux. Un pack d'actifs locaux est un groupe de ressources enregistrées au format de fichier *.yymp qui peut être facilement installé dans des projets existants. Cela signifie que vous pouvez créer votre propre bibliothèque de packages pour les ressources les plus utilisées, comme les bibliothèques de scripts, les intros d'écran d'accueil, les systèmes de menus, etc... sans avoir à utiliser lemarché GameMaker.

Création d'un paquet d'actifs

Pour créer un package d'actifs, vous devez d'abord avoir créé les ressources que vous souhaitez sauvegarder dans un projet, puis dans le menu Outils, sélectionnez l'option Créer un package local. La fenêtre suivante s'ouvre alors :

Dans cette fenêtre, vous pouvez fournir un nom d'éditeur facultatif et un nom d'affichage et un ID de paquet obligatoires. L'ID du paquet sera utilisé pour générer le nom du fichier avec lequel le paquet sera enregistré et sera automatiquement complété par l'extension de fichier *.yymp lors de la création. Vous pouvez également définir le numéro de version du paquet et choisir de le signer ou non (cette option est désactivée par défaut). La signature d'un paquet nécessite la création d'un certificat de signature (vous pouvez le faire à partir des préférences de la place de marché), et signifie que toute ressource qui a été signée ne pourra pas être importée si elle a été altérée après sa création.

Une fois que vous avez rempli les détails du package, vous pouvez sélectionner les ressources que vous souhaitez inclure dans la section "Projet" sur la droite (en utilisant le bouton gauche de la souris ou une combinaison de / et avec le bouton gauche de la souris). Les ressources sélectionnées peuvent être déplacées vers la vue "Asset Package", en utilisant les boutons Add ou Add All et peuvent être retirées à nouveau en utilisant les boutons Remove et Remove All.

Une fois que vous êtes satisfait de la sélection des ressources et que vous avez rempli les détails requis, cliquez sur le bouton Ok pour ouvrir un explorateur de fichiers où vous pouvez choisir l'emplacement où enregistrer le paquet. Une fois sélectionné, le paquet sera créé et une fenêtre de confirmation s'affichera, avec une option pour ouvrir l'emplacement du paquet dans l'explorateur de fichiers de l'OS.

Importation d'un ensemble d'actifs

Il existe deux façons d'importer un Local Asset Package :

  • la première consiste simplement à faire glisser le fichier *.yymp d'une fenêtre d'explorateur vers l'IDE de GameMaker
  • la seconde consiste à sélectionner l'option "Importer un paquet local" dans le menu Outils.

Si vous utilisez le menu Outils, une fenêtre d'exploration s'ouvrira et vous pourrez naviguer jusqu'à l'emplacement du paquet puis le sélectionner pour l'importer. Dans les deux cas, la fenêtre suivante s'ouvrira dans GameMaker:

À gauche se trouve le contenu du paquet et à droite les parties que vous voulez importer. Vous pouvez sélectionner une ou plusieurs ressources du paquet (en utilisant le bouton gauche de la souris ou une combinaison de / et avec le bouton gauche de la souris) et ensuite utiliser le bouton Ajouter pour les ajouter pour l'importation, ou utiliser le bouton Ajouter tout pour tout ajouter pour l'importation.

Lorsque tout est prêt, vous pouvez alors cliquer sur le bouton Importer pour ajouter les ressources sélectionnées à votre projet. Notez que vous avez également la possibilité d'Importer toutes les ressources dans un nouveau projet, ce qui créera un nouveau projet pour vous (et vous demandera de donner un emplacement pour l'enregistrer) et ajoutera ensuite les ressources du paquet à celui-ci.

Veuillez noter que les paquets de ressources créés à l'aide d'une version antérieure à GameMaker.3 devront être convertis car le format de fichier du projet a changé avec la mise à jour 2.3. Cela signifie qu'il peut être préférable d'importer l'ensemble du projet et de tout convertir, puis de l'exporter en tant que nouveau paquet vers un nouvel emplacement si les ressources sont fréquemment importées dans d'autres projets.
