Travailleurs à distance

Lorsque vous compilez votre jeu, vous utilisez généralement le travailleur "Local", c'est-à-dire l'ordinateur sur lequel GameMaker a été installé. Cependant, dans certaines circonstances, vous pouvez avoir besoin d'utiliser un autre ordinateur pour construire vos projets. Par exemple, si vous travaillez sur un Mac et que vous voulez construire un projet Windows, vous pouvez configurer un Remote Worker et demander à un PC Windows d'effectuer la construction. En fait, lorsque vous utilisez un travailleur distant, GameMaker envoie tous les fichiers nécessaires depuis le serveur (local). les fichiers requis de l'ordinateur (local) sur lequel il est exécuté à l'ordinateur du travailleur à distance, puis utilise cet ordinateur pour construire l'exécutable final du projet (en l'enregistrant sur l'ordinateur distant ou en le renvoyant ensuite sur l'ordinateur local). Cependant, avant de pouvoir utiliser cette fonctionnalité, vous devrez avoir installé certains outils sur la machine de construction et effectué certains réglages, que nous allons expliquer ici.

IMPORTANT ! Vous pouvez configurer un travailleur à distance entre un PC local Windows et un PC distant Windows, ou entre un PC Mac local et un PC distant Windows, mais actuellement, vous ne pouvez pas configurer un PC local Windows (ou Mac) et un PC Mac distant, c'est-à-dire que vous ne pouvez pas avoir un travailleur à distance Mac, mais seulement un travailleur à distance Mac. c'est-à-dire que vous ne pouvez pas avoir un travailleur distant Mac, seulement un Windows.

Configuration initiale

Pour configurer un travailleur à distance, vous devez d'abord installer l'application de l'outil de construction Remote Worker sur le PC qui sera utilisé. Vous pouvez trouver cet outil en allant dans le menu Build de GameMaker sur la machine locale et en sélectionnant l'option "Show Remote Worker Installer" :

Alternativement, vous pouvez cliquer sur l'icône "œil" dans le gestionnaire de cibles, et obtenir l'installateur de Remote Worker de cette façon :

L'installateur s'appellera quelque chose comme "GameMakerStudio-Remote-Installer-X.X.X.XXX.exe". et vous devez le copier sur le PC que vous voulez utiliser comme travailleur à distance (il n'est pas nécessaire que GameMaker soit installé sur le travailleur à distance, seulement ce programme).

Une fois que vous l'avez copié sur le PC de base, installez-le en suivant les instructions à l'écran. Lorsque le programme d'installation est terminé, le système d'exploitation peut vous demander d'autoriser access à traverser le pare-feu pour accéder à votre réseau. Vous devez maintenant retourner sur la machine où vous avez installé GameMaker 2 installé et cliquer sur le bouton Target Manager pour ouvrir la fenêtre Target Manager, puis allez dans la section Worker et cliquez sur le bouton Add Worker . La fenêtre suivante s'ouvrira alors fenêtre suivante :

Ici, vous devez donner un nom à votre travailleur à distance (c'est ce qui sera affiché dans la liste des travailleurs dans le Target Manager). Dans la section Nom de l'hôte, ajoutez l'adresse et le port de l'ordinateur de l'employé qui vous ont été montrés et notés précédemment.

Si tout s'est déroulé correctement, vous devriez maintenant recevoir un message indiquant que vous pouvez vous connecter avec succès à la machine du travailleur à distance. Une fois que votre travailleur à distance est installé et connecté à votre ordinateur principal IDE, vous devez configurer les paramètres de construction locaux pour chaque plateforme (voir la section ci-dessous) avant de pouvoir compiler votre projet. Paramètres à distance

En haut des paramètres, vous avez les différents chemins qui seront utilisés par l'application Remote Worker. Avant de faire quoi que ce soit d'autre, il faut les vérifier et définir d'autres chemins si nécessaire (en général, les chemins par défaut devraient convenir). Compiler

Une fois que votre Remote Worker est configuré et que vous avez rempli les paramètres de compilation locale (voir la section ci-dessus), vous pouvez commencer à l'utiliser pour compiler vos jeux. Pour cela, il suffit de sélectionner le Remote Worker dans la liste Target Manager de sélectionner la plate-forme pour laquelle vous souhaitez tester/compiler, ainsi que toute autre option (comme le type de compilation ou la configuration), puis de cliquer sur Play ou Create Executable .

REMARQUE: les plates-formes disponibles dans le Target Manager changent en fonction des capacités de l'ordinateur du travailleur à distance, et toutes les cibles ne sont pas disponibles. Notez également qu'il n'existe pas de cible "Test" pour les travailleurs à distance.

Si vous jouez simplement le projet, la machine Remote Worker le compilera et l'exécutera, mais si vous créez un exécutable, Remote Worker compilera le projet et renverra un fichier *.zip à la machine locale pour qu'elle le sauvegarde. machine locale pour qu'elle l'enregistre.

Il convient de noter que ni le bouton Debug ni le bouton Clean ne seront disponibles lorsque vous utilisez un Remote Worker. Notez également que vous ne pouvez actuellement effectuer qu'une seule construction à distance à la fois.


Une fois que le Remote Worker est installé et fonctionne sur votre machine distante, il apparaît dans la barre des tâches sous forme d'icône et vous pouvez cliquer dessus avec le bouton droit de la souris pour faire apparaître les options suivantes :

  • Ouvrez: Ouvrez l'application Remote Worker dans votre navigateur par défaut.
  • Afficher le journal: Ouvrez le fichier journal du travailleur à distance dans votre navigateur.
  • Paramètres réseau: Ici, vous pouvez filtrer les machines que vous souhaitez autoriser à envoyer des constructions à la machine du télétravailleur. Chaque filtre peut être un filtre de type CIDR ou un nom d'hôte d'une machine à autoriser. Il faut savoir que si vous voulez que plusieurs machines se connectent à un travailleur distant, ou si vous mélangez les types de connexion (par exemple : filaire et wifi), vous devrez peut-être modifier la plage d'adresses IP autorisées ici.
  • Modifier les paramètres: Ouvrez le menu des paramètres du télétravailleur (expliqué dans la section Paramètres, ci-dessus).
  • Quitter: Quittez l'application Remote Worker. Cela signifie que vous ne pourrez plus vous connecter (ni compiler) avec l'application depuis une machine locale jusqu'à ce que vous redémarriez l'application Remote Worker.
