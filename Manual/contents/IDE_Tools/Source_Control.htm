Contrôle des sources

Lagestion du contrôle des sources (GCS) est le nom donné à la méthode permettant de travailler avec des sous-versions ou des sauvegardes de vos projets par le biais d'un référentiel et d'une source locale. Fondamentalement, une solution SCM est un logiciel indépendant qui contrôle tous les aspects de la maintenance, de la modification et de la comparaison des versions de votre projet au fur et à mesure que vous travaillez dessus. Cette solution est particulièrement utile pour les personnes qui travaillent en équipe et doivent pouvoir contrôler qui fait quoi, sans craindre de perdre des données ou d'effectuer des modifications qui devront être annulées ultérieurement, mais les particuliers peuvent également bénéficier de ce système puissant et flexible.

GameMaker utilise Git pour son SCM, qui doit être installé et configuré séparément avant que SCM puisse être utilisé dans GameMaker. Consultez la documentation Git pour des informations plus spécifiques concernant Git.

Vous trouverez ci-dessous des liens vers un petit tutoriel sur la façon de configurer ce plugin et d'utiliser les outils SCM avec un projet.

GameMaker affichera un menu déroulant dans la barre de menu pour le contrôle de la source, qui sera rempli des options suivantes une fois que le contrôle de la source aura été activé pour votre projet (voir la section Configuration du plugin Git ci-dessus pour plus de détails) :

Nous présentons ici chacune des options disponibles (la plupart d'entre elles sont expliquées plus en détail dans les sections du tutoriel ci-dessus) :

  • Créer un dépôt de projet: Cette option vous permet de créer un référentiel local dans le même répertoire que vos fichiers de projet.
  • Cloner le référentiel: Cette option vous permettra de cloner un dépôt d'une source vers une nouvelle destination.
  • Livrer les changements: Avec cette option, vous pouvez mettre en scène les fichiers modifiés dans votre projet et ensuite les livrer au référentiel.
  • Pousser les changements: Après avoir fait un commit ou une fusion, cette option est utilisée pour pousser les changements vers le dépôt maître.
  • Tirer les changements: Avec cette option, vous pouvez mettre à jour le référentiel local en tirant les fichiers modifiés du référentiel maître.
  • Afficher l'historique: Cela ouvrira la fenêtre de l'historique où vous pouvez voir tout l'historique des versions du projet et choisir de revenir en arrière sur des chemins de fichiers spécifiques ou des révisions entières.
  • Afficher les conflits: Cette option ouvre la fenêtre Conflits et répertorie tous les fichiers en conflit qui peuvent exister dans le projet, ce qui vous permet de les traiter soit par le biais de GameMaker IDE, soit à l'aide d'un outil de fusion spécifique.
