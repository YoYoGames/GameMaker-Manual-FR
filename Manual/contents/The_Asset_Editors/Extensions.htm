L'éditeur d'extensions

Un paquet d'extension peut être utilisé pour ajouter un certain nombre de fonctions et de constantes supplémentaires à GameMaker Language (GML) et les transférer facilement entre projets. Les fonctions d'extension peuvent également être liées à des SDK externes ou à du code source écrit dans d'autres langages (comme Java pour Android, Objective-C pour iOS, etc.) qui peuvent être utilisés pour étendre les fonctionnalités de votre jeu (généralement utilisées pour mettre en œuvre des API tierces, comme pour les publicités, les achats in-app, etc.)

Les extensions peuvent être utilisées pour créer une bibliothèque de fonctions que vous utilisez fréquemment dans tous vos projets. Ainsi, au lieu de devoir importer les mêmes scripts dans chaque projet, vous pouvez simplement ajouter une extension qui contient une collection de vos fonctions et constantes préférées. Ces fonctions peuvent également être spécifiques à une certaine plateforme, et vous pouvez facilement choisir les plateformes sur lesquelles une extension peut être utilisée.

Pour un exemple étape par étape de la création d'une extension, voir : Création d'une extension

Pour désactiver les extensions en fonction de la plate-forme cible, voir : Comment désactiver les extensions ?

Importation d'extensions

Normalement, les extensions tierces proviennent du Marché de GameMaker, et celles auxquelles vous avez souscrit sont alors répertoriées dans la fenêtre Bibliothèque du Marché, où vous pouvez les installer. Si vous n'avez pas obtenu votre paquetage d'extension sur le site Marketplace, vous pouvez toujours l'ajouter à un projet en cliquant avec le bouton droit de la souris sur n'importe où dans le navigateur d'actifs et en sélectionnant Ajouter existant, puis en naviguant jusqu'à l'emplacement de l'extension pour la sélectionner.

REMARQUE Si vous avez un paquet .yymps/.yymp/.gmez, vous pouvez le faire glisser dans le IDE pour l'importer, ou aller dans le menu "Outils" et sélectionner "Importer un paquet local".

Plateformes cibles

Une fois que vous avez ajouté une extension à l'Asset Browser, vous devez lui donner un nom et un numéro de version et définir les plates-formes vers lesquelles vous souhaitez l'exporter. En décochant les plates-formes sélectionnées, vous pouvez demander à GameMaker de ne pas exporter l'extension en tant que partie de l'exécutable pour ces plates-formes. Vous devez également vous assurer que les propriétés de l'extension correspondent à la ou aux plates-formes cibles sélectionnées et choisir des cibles d'exportation compatibles pour les fichiers d'extension en fonction des éléments suivants :

  • Toutes les cibles: GML extensions. Ce sont des extensions réalisées à l'aide de GameMaker Language (GML) et sont compatibles avec toutes les plateformes cibles. (Notez que c'est le seul type d'extension que la cible Opera GX supporte actuellement.)
  • Windows (Standard, UWP, Xbox): Vous pouvez utiliser les fichiers Windows DLL (Dynamic Link Libraries). Veuillez noter que si vous avez coché la case "Use x64 Windows Runtime" dans les options de jeu de Windows, toutes les DLL du projet doivent être compatibles avec les 64 bits. Les DLL 32 bits ne fonctionneront pas avec un exécutable 64 bits.
  • macOS: La cible macOS peut utiliser les fichiers standard DyLib pour les extensions.
  • Ubuntu (Linux): Pour la cible Ubuntu vous devez vous assurer que votre extension utilise les fichiers .so.
  • HTML5: Ces plateformes peuvent utiliser les extensions de JavaScript.
  • Android: Cette plate-forme exige que l'extension soit écrite en Java ou en Android Framework SDK.
  • iOS / tvOS: les extensions pour cette plateforme sont écrites en Objective C et nécessitent un fichier *.mm et un fichier *.h, ou si vous utilisez des frameworks, un fichier natif précompilé *.a.
  • PlayStation: Pour PlayStation, vos extensions doivent être des fichiers PRX, chaque cible nécessitant un PRX différent pour se conformer aux spécifications de la cible (PS3, PS4 ou PS5). Si vous travaillez sur une PSVita, l'extension doit être SUPRX à la place.

REMARQUE Certaines extensions doivent respecter des conventions d'appellation spécifiques pour que GameMaker puisse identifier correctement la plate-forme et l'architecture du processeur pour lesquelles elles sont conçues. Veuillez consulter la section Conventions de nommage pour plus d'informations.

Étant donné qu'une extension peut avoir plusieurs fichiers associés, il est possible de créer une seule extension pour plusieurs plates-formes. Dans ce cas, vous devez spécifier toutes les plates-formes compatibles dans l'éditeur d'extension principal (c'est-à-dire cocher la case correspondante), puis dans les propriétés de chaque fichier de l'extension, définir la plate-forme vers laquelle ces fichiers doivent être exportés.

Pour modifier la plate-forme cible d'un fichier d'extension ou d'un espace réservé spécifique, vous devez d'abord ouvrir les propriétés de l'extension (double-cliquez sur l'extension dans l'Asset Browser), puis, une fois dans les propriétés de l'extension, sélectionnez à droite le fichier ressource que vous souhaitez modifier. Décompresser lors de la construction

L'option "Uncompress on build" peut être utilisée pour les fichiers d'archive compressés qui sont destinés à une plate-forme cible spécifique. Si cette option est activée, le contenu de cette archive sera extrait dans le répertoire Included Files du paquet construit, mais seulement après avoir été copié sur la plate-forme cible.

Par exemple, si vous activez cette fonction pour un fichier .zip dans votre extension et que vous lancez ensuite une compilation macOS à partir de Windows, l'archive .zip sera copiée telle quelle de Windows à macOS et sera ensuite décompressée lors de la compilation sur la plate-forme cible (qui, dans ce contexte, est macOS).

Appel des fonctions d'extension

Une fois que vous avez configuré votre extension, vous pouvez utiliser les fonctions et constantes qui lui sont destinées comme vous le feriez avec les fonctions et constantes intégrées utilisées dans le site GameMaker Language. Si votre jeu est multiplateforme (par exemple, il est conçu pour fonctionner sur des cibles Windows et Android ), vous devez utiliser les constantes et fonctions du système d'exploitation pour vérifier la plateforme actuelle avant d'appeler les fonctions spécifiques de l'extension afin d'éviter tout problème, par exemple :

switch(os_type)
{
    case os_android:
        call_android_extension_function(x, y, z);
    break;

    case os_windows:
        call_windows_extension_function(x, y, z);
    break;
}

Notez que pour les différentes plateformes, vos extensions peuvent partager des noms de fonctions. Ainsi, si vous avez une extension avec des fichiers distincts pour (par exemple) faire apparaître un message sur Windows et sur Android, les deux fichiers d'extension peuvent partager les mêmes noms de fonctions pour autant qu'ils soient tous deux configurés pour exporter uniquement vers leurs plateformes respectives.

Extensions officielles de tierces parties

Les anciennes versions de GameMaker intégraient de nombreuses fonctionnalités tierces, mais cela était inefficace et difficile à maintenir, car si une fonctionnalité spécifique était modifiée par une tierce partie, l'ensemble de l'EDI et du Runtime de GameMaker devait être mis à jour. Les versions actuelles ont donc adopté l'utilisation de paquets d'extensions pour ajouter le support de tiers à vos jeux. Cela signifie que vous pouvez ajouter uniquement les fonctions supplémentaires dont vous avez besoin et que vous pouvez même les adapter et les modifier pour répondre à vos besoins spécifiques.

La plupart de ces extensions seront téléchargées et installées pour vous lorsque vous sélectionnerez une option de jeu spécifique (comme les publicités Facebook ou Google Mobile, par exemple), mais vous pouvez trouver de plus amples informations à leur sujet dans la section appropriée pour chaque plateforme sur la base de connaissances GameMaker :

  • Plateformes et configuration deGameMaker

Vous pouvez trouver les extensions elles-mêmes sur la page suivante Marketplace:

  • Official Marketplace Assets

Notez que ces extensions sont accompagnées d'une démo et d'un fichier d'aide (au format *.pdf ) qui expliquent comment les utiliser. Leurs fonctions ne sont donc pas abordées dans ce manuel.
