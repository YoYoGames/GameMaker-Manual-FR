Types de données de plumes

Feather utilise les types de données pour fournir un contrôle syntaxique intelligent lors de l'écriture du code, garantissant que vous n'utilisez pas le mauvais type de données pour une variable ou un paramètre de fonction.

Il vous permet également de spécifier les types de données pour les paramètres et les valeurs de retour de vos propres fonctions de script, en utilisant les commentaires JSDoc.

Les balises JSDoc @param et @return vous permettent de spécifier l'un des types suivants :

NOTE Les types JSDoc sont insensibles à la casse, ce qui signifie que Id.DsList et id.dslist sont identiques.

Cahier des charges

Un spécificateur est ajouté après le type de données de base à l'aide d'un point . pour spécifier le type exact de données dans ce groupe.

Des types tels que Id.DsList, Asset.GMObject et Constant.Color utilisent des spécificateurs.

Types de collections

Les types tels que Array et Id.DsList, qui sont des structures de données contenant plusieurs valeurs, peuvent spécifier un seul type de données pour l'ensemble de leur contenu.

Pour ce faire, on utilise des types de collection, qui sont ajoutés au type à l'aide de crochets <>.

Par exemple, un tableau contenant des chaînes de caractères serait Array<String>, et une liste DS, contenant un tableau, contenant des réels, serait Id.DsList<Array<Real>>.
