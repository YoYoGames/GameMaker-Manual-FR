L'éditeur de script

Cette section traite de l'éditeur de script (également appelé éditeur de texte), dans lequel vous saisissez tout le code dont votre jeu a besoin pour fonctionner. Cet éditeur est utilisé pour créer des actifs de script personnalisés, ainsi que pour coder les événements d'objets, pour ajouter du code de création de salle, et pour bien d'autres choses.

Cette section détaille simplement le fonctionnement de l'éditeur de scripts, et non la manière de créer des ressources de scripts ou d'utiliser GML Code ou GML Visual pour créer des scripts. Pour cela, veuillez consulter les sections suivantes :

  • Scripts de code GML
  • Scripts visuels GML

Lors de la création d'une ressource script, il peut vous être demandé de choisir entre GML Visual et GML Code. Consultez la rubrique Langage GameMaker pour plus d'informations.

L'éditeur de script est l'endroit où vous écrivez votre code GameMaker Language, qui est le langage de programmation intégré utilisé par GameMaker (voir la section Langage GameMaker pour plus de détails). Lorsque vous vous serez familiarisé avec GameMaker et que vous souhaiterez l'utiliser au maximum, il est conseillé de commencer à apprendre à utiliser ce langage, car il élargit considérablement vos possibilités lors de la création de jeux.

Un actif de script peut être renommé en cliquant avec le bouton droit de la souris sur dans le navigateur d'actifs et en sélectionnant Renommer, mais notez que le nom de script doit être conforme aux règles de script pour les actifs, donc il ne peut être qu'alpha-numérique, doit commencer par une lettre et le seul symbole qu'il peut contenir est le symbole "_" sous la barre. L'éditeur de script dans d'autres endroits ne peut pas être renommé et aura un nom spécifique à ce qui est édité, comme un événement d'objet par exemple.

Lorsque vous ouvrez la fenêtre de l'éditeur de script, elle présente les options et la disposition suivantes :

Onglet Script / Événement

L'éditeur de script s'ouvre dans une fenêtre avec des onglets en haut pour vous permettre d'avoir plusieurs scripts ou événements dans une seule fenêtre (bien que ce comportement puisse être modifié à partir des préférences pour donner une nouvelle fenêtre à chaque script ou événement). Vous pouvez cliquer sur sur un onglet et le faire glisser pour changer l'ordre des onglets, ou - si vous préférez - vous pouvez le sortir de la fenêtre actuelle et le placer dans une nouvelle fenêtre. Si vous préférez, vous pouvez l'extraire de la fenêtre actuelle et le placer sur l'espace de travail pour créer une nouvelle fenêtre pour ce script (ou l'ajouter à une fenêtre différente), et vous pouvez également agrandir l'éditeur de script pour créer un nouvel espace de travail. Si vous faites glisser un onglet de script hors de la fenêtre de l IDE, une nouvelle IDE sera créée pour contenir cette ressource de script, et elle peut être utilisée comme vous le feriez avec la fenêtre principale.

Notez que si vous modifiez le code d'un événement d'objet dans l'éditeur de script et que vous avez maximisé l'éditeur de script ou que vous l'avez placé dans une fenêtre ou un espace de travail séparé, le menu de l'éditeur accessible par un clic droit comportera quelques options supplémentaires :

  • Restaurer l'espace de travail: Ceci fera sortir l'éditeur de script d'un état maximisé/espace de travail et le relier à l'objet dans l'espace de travail.
  • Allez à l'objet: Cela vous amènera à l'espace de travail où se trouve l'objet contenant le code et vous permettra de vous concentrer sur cet objet.
  • Ajouter /Ouvrir un événement: Ceci vous permet d'ajouter un nouvel événement à l'objet auquel le script actuel appartient, et ouvrira un nouvel onglet de code dans l'éditeur de script pour l'événement ajouté. Si l'événement sélectionné contient déjà du code, celui-ci sera ouvert dans un nouvel onglet. sera ouvert dans un nouvel onglet.

Pour plus d'informations sur les autres options du menu accessible par un clic droit sur , reportez-vous à la section " Édition", ci-dessous.

Rechercher / Remplacer

Lorsque vous travaillez dans l'éditeur de script, vous pouvez appuyer sur / + "F" pour ouvrir le champ de recherche ou sur / + "H" pour ouvrir le champ de recherche et de remplacement :

Ici, vous pouvez effectuer une recherche locale pour le mot-clé que vous avez entré et, une fois que vous avez saisi votre terme de recherche, vous pouvez alors utiliser les flèches situées en haut à droite de la fenêtre de recherche pour passer d'un terme trouvé à un autre dans le script. Vous pouvez modifier la façon dont l'opération de recherche est effectuée en basculant les boutons suivants :

Lorsque vous activez cette option, la fonction de recherche ne met en évidence que les chaînes de caractères qui correspondent à l'ensemble de la chaîne de caractères saisie. Par exemple, si cette option n'est pas activée, une recherche de "random" affichera tous les mots qui contiennent cette chaîne - comme irandom(), ou randomise() - alors que si vous l'activez, vous n'obtiendrez que la fonction random().
Lorsque cette option est activée, vous demandez à GameMaker de vérifier non seulement le contenu de la chaîne de recherche, mais aussi la casse. Par exemple, si vous avez un sprite appelé "spr_Dog" et que vous faites une recherche pour " dog" alors que cette option est désactivée, la chaîne sprite sera mise en évidence, mais si l'option est activée , elle ne le sera pas puisque "Dog" n'est plus considéré comme identique à "dog".

En cliquant sur ce bouton, la prochaine chaîne de recherche trouvée dans le script sera remplacée par la chaîne de remplacement donnée. Notez que le terme "suivant" est considéré comme le suivant après la position actuelle du curseur, et que vous pouvez passer à des termes différents en utilisant les flèches en haut à droite de la fenêtre de recherche.
En cliquant sur ce bouton, tous les exemples de la chaîne de recherche dans le script seront remplacés par la chaîne de remplacement indiquée.

Notez que si vous souhaitez effectuer une recherche globale (c'est-à-dire une recherche dans l'ensemble du projet plutôt que dans le script ou l'événement en cours), vous pouvez appuyer sur / + + "F".

Editeur de code principal

L'éditeur de code principal est l'endroit où vous écrirez tout votre code pour créer la ressource script ou remplir un événement d'objet. L'édition proprement dite de votre code est couverte dans la section sur l'édition plus bas sur cette page.

Information

La barre d'information au bas de IDE vous indique le numéro de la ligne sur laquelle vous vous trouvez et la position sur la ligne. C'est également là que vous pouvez voir l'aide de code, qui est une ligne de texte indiquant la fonction que vous êtes en train d'éditer ainsi que les arguments qu'elle requiert. avec les arguments qu'elle requiert. Au fur et à mesure que vous remplissez la fonction dans l'éditeur, les arguments sont mis en évidence pour vous montrer celui que vous êtes en train d'éditer.

Notez que tous les arguments facultatifs dans vos fonctions de script seront affichés avec [] parenthèses autour d'eux. Si vous avez utilisé les commentaires de script JSDoc dans une ressource de script, les informations que vous avez fournies seront également affichées ici.
