Le panneau des pistes

Les actions de vos séquences sont régies par les actifs et les paramètres qui ont été ajoutés au panneau des pistes. Vous pouvez ajouter une piste à l'éditeur de deux façons :

  • faire glisser un actif dans le Canvas dela séquence
  • faire glisser un actif dans le panneau de suivi lui-même
  • en cliquant sur le bouton "Add Asset" au bas du Track Panel.

Cette dernière méthode ouvrira l'Asset Explorer pour vous, où vous pourrez choisir la ressource à ajouter.

Quelle que soit la méthode que vous choisissez, une nouvelle piste d'asset sera créée pour l'asset. Cette piste est la piste principale de l'asset dans la séquence et une piste clé d'asset correspondante sera créée pour l'asset sur le site Dope Sheet à la position de la tête de lecture:

Notez que la couleur de la clé de l'actif dans le Dope Sheet peut être définie à partir du panneau des pistes en cliquant sur la barre colorée à l'extrémité. Cela ouvrira le sélecteur de couleurs où vous pourrez choisir la couleur à utiliser pour les pistes clés dans Dope Sheet.

Chaque actif de l'éditeur de pistes peut également être associé à un ensemble de pistes de paramètres. Ces pistes permettent de contrôler les différentes propriétés (paramètres) qui peuvent être modifiées pour la piste au fil du temps. Les différents paramètres disponibles sont traités en profondeur dans la section "Ajout de pistes de paramètres" plus bas sur cette page, mais pour vous donner une idée de ce qui est disponible, voici une brève liste :

  • Tous les actifs: Position et Rotation.
  • Lessons: Hauteur et volume.
  • Séquences: Origine et échelle.
  • Sprites et objets: Origine, Échelle, Index d'image, Vitesse d'image et Multiplication des couleurs.
  • Texte: Origine, Échelle, Multiplication des couleurs, Espacement des caractères, Taille du cadre, Espacement des lignes et Espacement des paragraphes.

Pour ajouter une piste de paramètres dans l'éditeur, vous pouvez activer l'option "Automatically Record Changes" (enregistrer automatiquement les modifications) dans le Track Panel, qui ajoutera alors des clés de paramètres pour toutes les modifications apportées à un actif dans le Canvas à la position actuelle de la tête de lecture, comme illustré ici :

Vous pouvez également cliquer sur le bouton "Add Parameter Track" , qui ouvrira un menu vous présentant les différentes options disponibles :

La sélection d'une option dans ce menu ou l'enregistrement direct à partir de IDE crée une ou plusieurs pistes de paramètres sous la piste principale du poste et ajoute également une clé de paramètre à Dope Sheet, de la même couleur que la piste principale du poste :

Une fois que vous avez ajouté une piste de paramètres à votre piste d'actif principale, le site dope sheet comporte une barre de couleur qui indique la longueur de la piste d'actif en images, et des clés de paramètres (images clés) sont ajoutées aux endroits où vous souhaitez modifier le paramètre. Ces clés peuvent être vues dans l'image ci-dessus comme de simples "points" sur une image. Lorsqu'une piste possède de telles clés de paramètre, le paramètre auquel elles se réfèrent sera interpolé (en utilisant l'interpolation linéaire) par défaut entre chacun des points indiqués. Par exemple, si vous avez un paramètre de rotation et qu'à l'image 0 il est réglé sur 90°, puis à l'image 30 il est réglé sur 180°, alors à l'image 15 il sera réglé sur 135°. Vous pouvez réellement voir le changement des valeurs dans le temps en sélectionnant la piste du paramètre et en "frottant" la tête de lecture dans la feuille de dope :

Notez que pour toutes les pistes de paramètres, vous pouvez désactiver l'interpolation linéaire par défaut entre différentes touches à partir du menu du bouton droit de la souris sur la piste de paramètres, ce qui signifie que les transitions entre les valeurs de paramètres se produiront instantanément sur l'image donnée plutôt que progressivement dans le temps :

Outre la possibilité de choisir le type de transition pour les pistes de paramètres, le menu RMB vous offre également les options suivantes :

  • Couper, Copier et Coller: Ces options vous permettent de couper/copier/coller des pistes d'actifs ou de paramètres.
  • Dupliquer une ou plusieurs pistes: Cette option crée une copie de la ou des pistes actuellement sélectionnées. S'il s'agit d'une piste d'actif, elle dupliquera également les pistes de paramètres qui lui sont associées.
  • Supprimer la ou les pistes: Cette opération permet de supprimer la ou les pistes sélectionnées. Si vous supprimez une piste d'actif, les pistes de paramètres qui lui sont associées seront également supprimées, mais vous pouvez supprimer des pistes de paramètres individuellement.
  • Groupe: L'utilisation de cette option sur un espace vide dans le Track Panel créera un dossier de groupe que vous pourrez ensuite utiliser pour conserver vos ressources dans un format plus ordonné. Vous pouvez ajouter une ressource à un groupe en cliquant simplement dessus et en la faisant glisser vers le dossier. Si vous avez déjà sélectionné une ou plusieurs pistes, cette option permet non seulement de créer un dossier de groupe, mais aussi d'y ajouter la ou les pistes.
  • Renommer: Vous pouvez utiliser cette option pour renommer n'importe quel dossier de piste ou de groupe d'actifs.

Une caractéristique importante des pistes d'actifs et de paramètres est la possibilité de parenter les pistes et les groupes. Si vous faites glisser une piste d'asset sur une autre dans le panneau des pistes, le premier asset devient un enfant de l'asset sur lequel il a été déposé, qui devient à son tour le parent. Toutes les pistes de paramètres affectées à la piste d'actif parent seront alors automatiquement appliquées aux pistes d'actif enfant, de sorte que vous pouvez avoir plusieurs sprites (par exemple) qui utilisent toutes une seule piste d'actif de position pour se déplacer :. Vous pouvez faire de même avec les groupes. Vous pouvez créer un dossier de groupe en cliquant sur le bouton situé en bas du panneau des pistes, puis sur GML. Visualisez les pistes d'actifs que vous souhaitez regrouper dans le dossier de groupe. Ces actifs conserveront toujours leurs propres pistes de paramètres, mais vous pouvez ensuite attribuer des pistes de paramètres supplémentaires au groupe et tous les actifs du groupe seront affectés par ces pistes de paramètres ainsi que par leurs propres pistes de paramètres individuelles.

Lorsque vous travaillez avec le panneau des pistes, vous pouvez également utiliser les raccourcis clavier habituels, à savoir : / et le bouton gauche de la souris pour sélectionner une ou plusieurs pistes dans n'importe quel ordre, et le bouton gauche de la souris pour sélectionner des pistes consécutives, et vous pouvez également les couper, les copier et les coller en utilisant les combinaisons de touches / +"X", / + "C" et / + "V".

C'est la base de l'utilisation de l'éditeur de piste, mais ci-dessous nous allons entrer dans un peu plus de détails sur ce que chacune des différentes sections sont pour et comment les utiliser :

Orientation du changement

Le bouton de changement d'orientation peut être utilisé pour modifier la façon dont l'éditeur de séquence affiche les différentes sections. Par défaut, la vue Cnavas se trouve en haut et l'éditeur de piste et Dopesheet en bas, mais cela peut être modifié en cliquant sur le bouton pour faire défiler les différentes dispositions, ou en cliquant sur le bouton Options pour ouvrir un menu et sélectionner la disposition que vous souhaitez. Les mises en page suivantes sont disponibles :

Fenêtre divisée

Avec cette option, l'éditeur de séquence sera divisé en deux fenêtres distinctes, où la fenêtre principale IDE affichera la vue Canvas sur le bureau complet et la nouvelle fenêtre affichera l'éditeur de piste et Dope Sheet. En cliquant à nouveau sur le bouton, la disposition sera à nouveau modifiée et la fenêtre supplémentaire sera supprimée.
Snap Top

Cette option permet d'accrocher l'éditeur de piste et Dopesheet au sommet du site IDE.
Snap Right

Cette option permet d'accrocher l'éditeur de piste et le site Dopesheet à droite du site IDE.
Bouton-pression

Cette option permet d'accrocher l'éditeur de piste et Dopesheet au bas de la page IDE (il s'agit de la vue par défaut).
Snap Left

Cette option permet de placer l'éditeur de piste et Dopesheet à gauche de IDE.

Vous pouvez également réinitialiser la disposition de l'éditeur de séquence à tout moment en cliquant sur l'option Reset . Cela rétablira l'éditeur dans son état par défaut et rouvrira toutes les sections qui avaient été fermées.

Recherche de piste

Cette zone de recherche vous permet de rechercher rapidement parmi les différentes pistes de la séquence pour trouver celle que vous souhaitez modifier. Si vous tapez un texte dans cette zone, vous saurez combien de pistes d'assets contenant cette chaîne ont été trouvées et les flèches situées sur le côté de la zone de recherche seront mises en évidence. En cliquant sur ces flèches, vous passerez au premier actif et aux actifs consécutifs qui répondent aux critères de recherche.

Suivi des actifs

Chaque ressource ajoutée à une séquence se voit attribuer sa propre piste, qui est répertoriée ici. Si la piste de l'asset a des paramètres définis, ceux-ci sont également affichés en dessous en tant que pistes de paramètres. Vous pouvez développer ou réduire les pistes de paramètres en cliquant sur sur l'icône de flèche à gauche du nom de la piste, et vous pouvez modifier le nom de la piste en utilisant un double-clic lent sur la piste, puis en lui donnant le nom de votre choix. Notez qu'il n'y a aucune restriction sur le nom que vous donnez et que vous pouvez utiliser des espaces, des caractères non alphanumériques et les faire commencer par un chiffre si vous le souhaitez.

Les différents types de pistes de paramètres disponibles sont expliqués plus en détail dans la section "Ajout de pistes de paramètres" plus bas dans cette page.

Contrôles clés

Une fois que vous avez ajouté une piste de paramètres, vous pouvez ajouter et supprimer des clés de paramètres à des images spécifiques pour définir la valeur de ce paramètre. Vous pouvez le faire à l'aide de la "Touche d'enregistrement" ou de la "Touche d'effacement". (voir la section "Commandes de touches de piste" pour plus d'informations), ou vous pouvez le faire en utilisant les commandes de touches sur le paramètre lui-même. Pour utiliser ces commandes, vous devez positionner la tête de lecture dans le Dope Sheet sur le cadre que vous voulez éditer, puis vous pouvez cliquer sur l'icône circulaire "clé" à gauche de la piste du paramètre. Si aucune clé n'est assignée à l'image, le cercle sera "vide" et cliquer dessus ajoutera une nouvelle clé et vous permettra de modifier la valeur du paramètre. Si le cadre a déjà une clé, alors le cercle sera "rempli" et cliquer dessus supprimera la clé du cadre :

Vous remarquerez également que dans l'image ci-dessus, vous pouvez utiliser les boutons fléchés situés de part et d'autre du marqueur de clé de paramètre pour passer rapidement d'une image clé à l'autre sur le site dope sheet, sans avoir à positionner manuellement la tête de lecture.

Valeurs des paramètres

Lorsqu'une piste de paramètres a été ajoutée, vous pouvez modifier les valeurs de certains d'entre eux en plaçant simplement la tête de lecture dans le site Dope Sheet sur l'image comportant la clé que vous souhaitez modifier, puis en utilisant les différents widgets de canevas pour modifier la position, la rotation, l'échelle ou l'origine de l'actif. Ces modifications seront reflétées dans la valeur de la clé pour cette image. Cependant, tous les paramètres ne peuvent pas être modifiés de cette façon, ou cela peut ne pas être assez précis pour vos besoins, et dans ces cas, vous pouvez modifier la valeur du paramètre directement en cliquant dessus dans la piste et en le réglant sur la valeur requise pour le type de paramètre.

Notez que certaines pistes de paramètres peuvent avoir une icône "lien" à côté de leurs valeurs, ce qui signifie qu'elles peuvent être liées entre elles. Cela se fait en cliquant sur l'icône pour activer/désactiver le lien, et lorsqu'il est activé, l'édition d'une valeur définira également l'autre.

Événements

De la même manière que les objets ordinaires, les séquences peuvent avoir des événements qui peuvent exécuter un code qui leur est attribué. Le code est assigné sous la forme d'une fonction scriptée qui ne peut prendre aucun argument et qui sera appelée lorsque l'événement sera déclenché.

Les événements sont ajoutés en cliquant sur le bouton Ajouter un événement qui ouvrira la fenêtre suivante :

Les événements énumérés ici sont les suivants :

  • Créer - Cet événement se produit lorsqu'une instance de la séquence est créée pour la première fois, et c'est la toute première chose qui se produit dans une séquence placée dans la pièce par l'intermédiaire de l'éditeur de pièce lorsqu'on entre dans une pièce.
  • Destroy - Cet événement est exécuté lorsqu'une instance de la séquence est détruite, et sera exécuté avant l'événement Clean Up (voir ci-dessous).
  • Clean Up - Cet événement sera appelé après tout événement qui supprime une instance de la séquence de la pièce. Il sera donc déclenché si la séquence est détruite, si la pièce se termine, ou si le jeu se termine, et il est conçu pour effectuer toute tâche dont vous avez besoin lorsqu'une instance de séquence est retirée du jeu pour une raison quelconque. Si vous avez détruit l'instance de la séquence en utilisant, par exemple, layer_sequence_destroy(), alors cet événement sera appelé après l'événement Destroy (voir ci-dessus).
  • Begin Step / Step / End Step - L'événement Step est un événement qui est vérifié à chaque étape (frame) du jeu tant que l'instance de séquence existe, et est divisé en trois parties : begin, step et end. Pour la plupart des choses, l'événement step standard conviendra, mais parfois, vous souhaitez avoir un peu plus de contrôle sur le code qui s'exécute et à quel moment. Pour cela, vous disposez des événements step Begin et End, qui seront toujours déclenchés dans le même ordre à chaque step (frame) du jeu. Important ! Si la séquence est mise en pause, ces événements ne seront pas déclenchés, et lorsque le jeu reprendra, ils seront déclenchés à l'image suivante après que la séquence ait recommencé à être jouée. Notez également que l'ordre des événements n'est pas influencé par le sens de la tête de lecture, et même si la séquence est lue à l'envers, les événements seront toujours exécutés comme Début, Étape et Fin.
  • Événement asynchrone: C'est l'équivalent de l'objet Événement système asynchrone. Important ! Si la séquence est en pause, cet événement ne sera pas déclenché.

Chaque événement peut être associé à une fonction unique qui sera appelée lorsque l'événement sera déclenché. Vous pouvez attribuer la fonction en utilisant le champ de saisie de l'événement, et en cliquant sur le bouton flèche, vous ouvrirez l'éditeur de script pour la fonction à modifier. Vous pouvez également cliquer sur le bouton Créer nouveau en bas pour créer une nouvelle ressource de script avec des fonctions "passe-partout" déjà définies et prêtes à être remplies. Notez que vous pouvez modifier les noms des fonctions selon vos besoins et ne pas utiliser les noms prédéfinis, et que vous pouvez également supprimer toute définition de fonction dont vous n'avez pas besoin. Il est important de noter que les fonctions utilisées pour les événements de la séquence ne peuvent pas prendre d'arguments.

Les événements de la séquence peuvent également être ajoutés et modifiés à l'aide de code. Pour plus d'informations, voir ici.

Contrôles des images clés

Les boutons de contrôle des images clés sont l'un des moyens d'ajouter, de supprimer ou de modifier des clés spécifiques dans le site dope sheet. Les boutons sont les suivants :

  • - Il s'agit du bouton "Split selected keys". Lorsque vous avez sélectionné une ou plusieurs clés de paramètre dans le dope sheet, vous pouvez cliquer sur ce bouton et elles seront divisées en deux clés distinctes à la position de la tête de lecture de la timeline :
  • - Il s'agit du bouton "Supprimer les clés sélectionnées". Lorsque vous avez sélectionné une ou plusieurs clés de paramètres sur le site dope sheet, vous pouvez cliquer sur ce bouton pour les supprimer :
  • - Il s'agit du bouton "Enregistrer une nouvelle clé". Lorsque vous avez sélectionné une clé de poste dans dope sheet et que vous appuyez sur ce bouton, de nouvelles clés de paramètres seront ajoutées à la piste de poste en tant que pistes de paramètres, et les clés de paramètres seront ajoutées en tant que points dans la timeline dope sheet à la position de la tête de lecture :
  • - Il s'agit du bouton "Automatically Record Changes". Lorsque cette option est activée, toutes les modifications apportées à l'asset dans le canevas sont automatiquement enregistrées et les pistes de paramètres et les clés de paramètres appropriées sont ajoutées à la position de la tête de lecture sur le site dope sheet. Par exemple, si vous déplacez la tête de lecture de l'image 0 à l'image 10, puis déplacez l'asset de 100 pixels vers la droite dans le canevas, une piste de paramètres est ajoutée pour la position et les clés de paramètres sont ajoutées à l'image 0 (position initiale) et à l'image 10 (position de la tête de lecture) et lorsque vous appuyez sur "Play" dans la séquence, l'asset se déplace de 100 pixels vers la droite sur dix images.

Commutateurs Visible / Verrouillé

Les pistes d'actifs et les pistes de paramètres disposent toutes deux d'un bouton permettant de basculer la visibilité de la piste et de verrouiller la piste . Si vous désactivez la visibilité de la piste, elle ne s'affichera pas lorsque la séquence est exécutée dans une salle, ni dans le canevas de la séquence lors des tests, et le fait de basculer la visibilité d'une piste d'actif basculera également la visibilité de ses pistes de paramètres. Le verrouillage d'une piste signifie simplement que cette piste ne pourra plus être modifiée de quelque manière que ce soit. Notez que les pistes invisibles seront dessinées en fondu sur le site dope sheet, tandis que les pistes verrouillées seront dessinées avec une superposition, de sorte que vous pouvez immédiatement voir dans quel état se trouve chaque piste :

Couleur de la piste

Chaque piste d'actif que vous créez se verra attribuer une couleur aléatoire par le site IDE pour faciliter son identification sur le site Dope Sheet. Cette couleur peut être modifiée à tout moment en cliquant simplement sur la barre de couleur de la piste, puis en sélectionnant la couleur que vous souhaitez utiliser dans la fenêtre du sélecteur de couleur qui s'ouvre. Toute couleur sélectionnée pour une piste d'actif sera automatiquement appliquée à toutes les pistes de paramètres associées, mais vous pouvez ensuite modifier la couleur de chaque piste de paramètres individuellement pour qu'elle soit différente si nécessaire. Notez que si vous modifiez à nouveau la couleur de la piste d'actif, les modifications apportées aux pistes de paramètres seront annulées.

Ajout de pistes de paramètres

Le bouton Add Parameter permet d'ajouter une seule piste de paramètres à la piste de poste sélectionnée. Comme indiqué précédemment sur cette page, les pistes de paramètres peuvent être de différents types selon le poste auquel elles se rapportent. Vous trouverez ci-dessous une liste des différentes pistes de paramètres disponibles et de ce à quoi elles font référence, ainsi que des postes auxquels elles peuvent être appliquées :

  • Position: Cette piste de paramètre peut être ajoutée à l'un des quatre types d'actifs de séquence(sprites, objets, sons ou séquences) et est utilisée pour définir la position spatiale de l'actif dans la séquence. Elle prend une valeur X et une valeur Y.
  • Rotation: Comme le paramètre de position, le paramètre de rotation est disponible pour toutes les ressources de la séquence et peut être utilisé pour définir la rotation en degrés de la ressource. Gardez à l'esprit que GameMaker calcule la rotation dans le sens inverse des aiguilles d'une montre, en commençant par 0° à droite, donc 90° correspond au haut, 180° à la gauche et 270° au bas.
  • Origine: Le paramètre d'origine est disponible pour les ressources sprites, objets et séquences et peut être utilisé pour définir la position X et Y de l'origine de la ressource. La modification de l'origine aura une incidence sur le positionnement, la rotation et la mise à l'échelle de la ressource.
  • Échelle: Le paramètre scale est disponible pour les ressources de type sprite, objet et séquence et permet de définir l'échelle horizontale et verticale de la ressource. Il affecte le rendu de la ressource dans IDE et runtime et est calculé comme une valeur scalaire, où 100 % est la taille par défaut de la ressource, 50 % correspond à la moitié de la taille et 200 % correspond au double de la taille. Notez que vous avez la possibilité de lier les valeurs d'échelle horizontale et verticale en cliquant sur le bouton "link" . Cela signifie que la modification d'une valeur entraîne automatiquement la modification de l'autre.
  • Index d'image et vitesse d'image: Ces paramètres ne sont disponibles que pour les actifs de type sprite et objet et peuvent être utilisés pour modifier l'image d'animation actuelle (voir la page sur l'élément image_index pour plus d'informations) ainsi que la vitesse d'animation actuelle, qui est définie à l'aide de FPS et fonctionne de la même manière que la valeur que vous avez définie pour le site sprite utilisé dans l'éditeur sprite.
  • Multiplication de la couleur: Ce paramètre n'est disponible que pour les sprites et les objets et peut être utilisé pour modifier la couleur de mélange actuelle pour le site sprite en cours de dessin. C'est l'équivalent de la valeur image_blend et toute couleur définie ici sera "multipliée" (c'est-à-dire mélangée) avec la couleur du site sprite; une valeur de blanc (#ffffffff) indique qu'aucun mélange ne doit avoir lieu. Notez que lorsque vous travaillez avec ce type de piste, vous pouvez définir la couleur en saisissant manuellement la valeur hexadécimale, ou vous pouvez cliquer sur le nuancier pour ouvrir le sélecteur de couleurs de GameMaker. Notez que les couleurs sont données au format ARGB, vous pouvez donc définir une valeur pour l'alpha en utilisant ce type de piste de paramètres (le sélecteur de couleurs vous permet également de définir la valeur de l'alpha). Cela équivaut à définir le paramètre image_alpha pour le sprite.
  • Volume: Le paramètre de volume s'applique uniquement aux pistes de ressources sonores et peut être utilisé pour définir le gain du son (le gain réel de la ressource sonore ne sera pas modifié, car il s'agit essentiellement d'un émetteur utilisé pour la piste de ressource dans les séquences). La valeur du volume est calculée sous forme de pourcentage, où 0 % correspond à un volume nul et 100 % à un volume maximal.
  • Pitch: Le paramètre pitch s'applique uniquement aux pistes de ressources sonores et peut être utilisé pour définir la hauteur du son (comme pour le volume, le gain réel de la ressource sonore ne sera pas modifié, car il s'agit essentiellement d'un émetteur utilisé pour la piste de ressource dans les séquences). La valeur de la hauteur est calculée sous forme de pourcentage, 100% étant la hauteur "naturelle" du son, et moins de 100% la diminuera tandis que plus de 100% l'augmentera.
  • Falloff: Le paramètre falloff ne s'applique qu'aux pistes d'actifs sonores et n'est actuellement pas implémenté.
  • Il existe quelques paramètres supplémentaires uniquement disponibles pour les pistes de texte. Veuillez lire Texte dans les séquences pour obtenir des informations à leur sujet.

Suivi des actions

Les boutons au bas de l'éditeur de piste sont les différentes actions de la piste et peuvent être utilisés pour les raisons suivantes :

  • - Ce bouton ouvre l'explorateur de ressources pour vous permettre de choisir une ressource à utiliser comme nouvelle piste. Une fois sélectionnée, la piste de la ressource sera créée et les images seront ajoutées dans le site Dope Sheet à la position actuelle de la tête de lecture.
  • - Lorsque vous avez sélectionné une ou plusieurs pistes d'actifs dans l'éditeur de pistes, cliquer sur ce bouton les dupliquera (ainsi que toutes les pistes de paramètres qu'elles peuvent contenir).
  • - En cliquant dessus, vous créerez un groupe spécial de masques d'écrêtage dans lequel vous pourrez donner sprites à utiliser comme masque et sprites à masquer. Voir ici pour plus d'informations.
  • - En cliquant sur ce bouton, vous ajoutez un nouveau dossier de groupe à l'éditeur de pistes, qui peut ensuite être utilisé pour mieux organiser les pistes. Vous pouvez ajouter une piste à un groupe en la faisant glisser sur le dossier de groupe et vous pouvez la retirer en la faisant glisser hors du dossier. Notez que tout actif ajouté à un dossier de groupe se verra automatiquement attribuer la couleur du dossier, bien que vous puissiez ensuite modifier la couleur des pistes individuelles pour qu'elle soit différente si nécessaire.
  • - Comme vous pouvez vous y attendre, cliquer sur cette option supprimera la ou les pistes sélectionnées. S'il s'agit d'une piste d'actif, toutes les pistes de paramètres qu'elle contient seront également supprimées, bien que vous puissiez l'utiliser sur des pistes de paramètres sélectionnées pour les supprimer sans supprimer les pistes d'actif auxquelles elles sont associées. Si vous l'utilisez sur un dossier de groupe vide, le dossier sera supprimé, mais si le dossier contient des pistes, un avertissement s'affichera, car la suppression d'un dossier entraînera également la suppression de toutes les pistes qui s'y trouvent.
