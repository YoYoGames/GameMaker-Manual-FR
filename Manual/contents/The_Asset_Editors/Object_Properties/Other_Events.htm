Cette page explique chacun des différents événements qui sont rassemblés dans la catégorie Autres événements de l'éditeur d'objets :

Cet événement est généralement utilisé pour des éléments tels que les balles, qui sont détruites dès qu'elles quittent la pièce afin d'éviter que des millions de balles ne s'envolent à l'infini et ne ralentissent votre jeu. Notez que cet événement n'est déclenché qu'une seule fois lorsque l'instance quitte la pièce initialement.

Fin de l'animation

Dans GameMaker sprites ne sont pas des choses statiques car ils peuvent avoir des sous-images et être animés à différentes vitesses. Chaque image d'une animation (appelée sous-image) possède son propre numéro, commençant à 0, qui peut être vérifié dans le code ou par des actions, Mais parfois, tout ce dont vous avez vraiment besoin est de savoir quand l'animation est terminée. C'est à ce moment-là que cet événement est déclenché, juste à la fin de l'animation, lorsque l'index de la sous-image indique que la dernière image a été atteinte. Cet événement est vraiment utile pour de nombreuses choses, par exemple un objet d'explosion où vous pouvez configurer l'instance pour qu'elle se détruise elle-même lorsque la dernière image de l'animation est atteinte.

Cela vous permet d'apporter des modifications aux données des os en utilisant les fonctions appropriées.

Événement d'animation

Cet événement est conçu uniquement pour être utilisé avec les fonctions d'animation squelettique. Cet événement est déclenché dans un objet auquel est attribuée une animation squelettique sprite, et où l'objet utilise le dessin par défaut pour le sprite attribué, ou le sprite est dessiné via draw_self(). Actuellement, il ne se déclenche pas sur une animation de squelette sprites dessinée d'une autre manière. d'une autre manière. L'événement est conçu pour capturer et transmettre les données qui ont été assignées à des événements particuliers de l'animation sprite, tels que définis dans l'éditeur utilisé pour les réaliser.

Lorsque l'événement d'animation est déclenché, une carte DS spéciale est créée, appelée carte event_data (de la même manière que la carte async_load pour un événement asynchrone). Cette carte contient toutes les paires clé/valeur suivantes, que vous pouvez analyser pour obtenir des informations sur l'événement :

  • "name" : le nom de l'événement (tel que défini dans le programme d'animation)
  • "track: l'index de la piste sur laquelle l'animation est jouée (0 par défaut).
  • "integer" : une valeur entière associée à l'événement (telle que définie dans le programme d'animation, la valeur par défaut est 0)
  • "float: une valeur flottante associée à l'événement (telle que définie dans le programme d'animation, la valeur par défaut est 0,0).
  • "string: une valeur de chaîne de caractères associée à l'événement (telle que définie dans le programme d'animation, la valeur par défaut est une chaîne vide "").

Événements des utilisateurs

Il s'agit d'événements spéciaux qui ne sont pas déclenchés automatiquement par GameMaker, mais que vous devez appeler implicitement en utilisant du code ou des actions pendant le déroulement du jeu. Il y a 12 événements utilisateur au total et ils peuvent contenir du code ou des actions et ils peuvent être appelés à tout moment dans la boucle du jeu, ce qui les rend très utiles pour créer vos propres événements qui se produisent quand vous le décidez.

Message de diffusion

Cet événement est déclenché lorsqu'une sprite ou une séquence génère une chaîne de messages de diffusion. Pour plus d'informations, veuillez consulter la page sur les messages de diffusion.
