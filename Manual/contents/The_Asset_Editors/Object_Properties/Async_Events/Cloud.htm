Nuage

L'événement Cloud est celui qui est déclenché par le rappel d'une des cloud_*() fonctions, comme cloud_synchronise(). Il génère en fait une carte DS qui est exclusive à cet événement et qui est stockée dans la variable spéciale async_load (voir les fonctions individuelles pour des exemples de code expliquant l'utilisation de cet événement de manière plus détaillée). événement de manière plus détaillée). Cette carte DS a la structure suivante :

  • "status" : Il s'agit du code d'état, où une valeur négative indique une erreur, dont la description sera contenue dans le "errorString". Une valeur de 0 (ou une valeur positive) indique un succès (voir ci-dessous pour les valeurs exactes), et le "resultString" contiendra les données retournées ou un message de succès.
  • "id" : L'identifiant qui a été renvoyé par la fonction appelée. Si vous envoyez une série de requêtes cloud_, vous devez savoir à laquelle vous recevez une réponse. la valeur que vous avez stockée lors de l'envoi initial de la demande pour trouver la bonne.
  • "description" : La description du dernier fichier téléchargé.
  • "resultString" : Ceci contient une chaîne de caractères qui est le blob de données renvoyé par le nuage.
  • "errorString" : renvoie une chaîne d'erreur pour toute erreur.

La signification exacte de l'entrée de carte "status" retournée est expliquée dans le tableau suivant :

Valeur du statut errorString / resultString Description
-1 errorString = "Non connecté à <SERVICE>" You have not successfully logged in to the given Cloud Service
0 resultString = données récupérées New game data downloaded from the cloud (following a cloud_synchronise() appel)
1 resultString = "AlreadySynchronized" (déjà synchronisé) No new data since you last called cloud_synchronise
2 resultString = "ConflictDeferral" (renvoi de conflit) A conflict was encountered, but the gamer chose to ignore it
3 resultString = "GameUploadSuccess" (succès) data from cloud_string_save() ou cloud_file_save() a été téléchargé avec succès sur le cloud.
-n errorString = Description de l'erreur Any other negative number means a synchronisation failure
