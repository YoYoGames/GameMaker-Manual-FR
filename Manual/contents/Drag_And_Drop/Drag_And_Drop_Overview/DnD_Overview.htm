Aperçuvisuel deGML

GML Visual est un outil de script visuel qui peut être utilisé pour créer vos jeux sans avoir à taper le moindre code. Cela ne veut pas dire que vous ne programmez pas de code lorsque vous utilisez GML Visual, car vous le faites, mais que votre code est créé et présenté de manière visuelle en utilisant des "blocs" d'actions plutôt que du texte. Les actions sont de simplesexpressions oudéclarations qui peuvent être "enchaînées" pour qu'une instance d'un objet fasse quelque chose lorsqu'elle est placée dans une pièce. Par exemple, vous pouvez avoir une action dans un événement de création pour déplacer une instance dans une direction aléatoire, et une autre action dans un événement de collision pour qu'elle rebondisse sur les objets avec lesquels elle entre en collision (pour plus d'informations sur les différents événements disponibles et leur fonctionnement conjoint, consultez la section sur les événements d'objets).

Onglets d'événements

Lorsque vous ajoutez un événement à votre objet, un nouvel onglet est ouvert dans la fenêtre de code pour afficher le contenu de cet événement. Vous pouvez ajouter plusieurs événements et avoir plusieurs onglets ouverts en même temps, passer de l'un à l'autre et les modifier comme vous le souhaitez. Vous pouvez également fermer les onglets à tout moment pour réduire l'encombrement (en sélectionnant à nouveau l'événement dans l'objet, vous les rouvrirez), et vous pouvez cliquer sur et faire glisser l'onglet hors de la fenêtre dans sa propre fenêtre de code, auquel cas la fermeture de l'objet ou de la fenêtre de code ne fermera pas la nouvelle fenêtre. Les onglets peuvent également être déplacés vers la gauche ou la droite pour être réorganisés.

Aperçu de l'action

Au fur et à mesure que vous ajoutez des blocs d'action à votre événement, ils apparaissent sous une forme "abrégée" comme un aperçu en anglais de ce qui se passe. L'ordre de l'aperçu est l'ordre dans lequel chacun des blocs d'action sera exécuté pour cet événement, et vous pouvez cliquer sur n'importe quel élément de la liste pour que la vue du bloc d'action s'agrandisse jusqu'à cette position pour l'édition.

Espace de travail du bloc d'action

C'est dans cette zone de l'espace de travail que vous faites glisser les actions de la boîte à outils pour créer votre code GML Visual. Cliquez sur sur une action et faites-la glisser dans cette zone pour l'ajouter à la liste des actions et la modifier. La plupart des actions ont plusieursarguments (paramètres) que vous pouvez définir, et la plupart prennent également des mots-clés, des variables d'instance ou même des fonctions GML. Par exemple, vous pouvez appeler l'action Définir la direction et utiliser la fonction GML random(val) pour définir une direction aléatoire.

Lorsque vous ajoutez des blocs d'action à votre code, ils sont visuellement "enchaînés" les uns aux autres pour indiquer l'ordre dans lequel les différentes actions vont se produire, bien que la façon dont cette chaîne est créée dépende du bloc initial qui a été utilisé. Par exemple, un bloc If Variable aura deux chaînes - une vers les blocs qui doivent être appelés si la condition vérifiée est true et ensuite le bloc suivant. Pour plus d'informations sur la construction de blocs d'action dans le code, consultez les sections liées ci-dessous.

Dans l'espace de travail du bloc d'action, vous pouvez maintenir enfoncé le bouton central de la souris pour effectuer un panoramique et utiliser la molette de la souris pour faire défiler la liste des actions vers le haut ou vers le bas. Si vous maintenez le bouton enfoncé tout en déplaçant la molette de la souris, la vue défilera à gauche et à droite, et si vous maintenez le bouton / enfoncé tout en déplaçant la molette de la souris, la vue fera un zoom avant et arrière, ce qui vous permettra de voir une plus grande partie de l'espace de travail avec les actions. Vous pouvez cliquer sur sur n'importe quelle action de la liste d'actions sur la gauche pour vous concentrer sur cette action, réinitialiser le zoom et la position. Notez également que vous pouvez redimensionner les blocs d'action en déplaçant la souris vers les bords du bloc, puis - lorsque le curseur change - en la faisant glisser dans la direction souhaitée.

Vous pouvez également cliquer avec le bouton droit de la souris sur n'importe où dans l'espace de travail de GML Visual pour faire apparaître le menu RMB, qui est expliqué en détail ici.

Boîte à outils

La boîte à outils est l'endroit où toutes les actions de GML Visual sont stockées, avec différentes sections contenant des collections (bibliothèques) d'actions qui sont similaires ou liées. Chaque collection d'actions est liée à un thème spécifique ou à un objectif commun. Il vous suffit de cliquer sur et de faire glisser l'action requise de la boîte à outils vers l'espace de travail du bloc d'actions pour l'ajouter à l'événement en cours. Pour simplifier les choses - car il y a beaucoup d'actions - vous pouvez utiliser la barre "Recherche" en haut pour rechercher une action spécifique ou pour filtrer les actions visibles (tapez "dessiner" par exemple pour obtenir toutes les actions de dessin), et vous pouvez également cliquer sur et faire glisser les icônes de n'importe quelle bibliothèque dans votre dossier Favoris pour garder ensemble les actions utilisées couramment. Cette fonction est particulièrement utile si vous avez l'habitude de faire certaines choses. Le dossier des favoris peut être vidé en cliquant sur la petite icône "poubelle".

Pour une liste complète des actions disponibles, ainsi que des arguments qu'elles prennent et de la façon dont elles peuvent être enchaînées, veuillez consulter la section Référence visuelle GML.

Vous pouvez utiliser le bouton droit de la souris sur n'importe quelle action de la boîte à outils pour ouvrir le menu suivant :

Vous disposez ici des options suivantes :

  • Utiliser<action>: Ceci ajoutera l'action qui était sous la souris lorsque le bouton droit de la souris a été cliqué à l'espace de travail de l'événement actuel.
  • Créer / Supprimer un favori: Ceci ajoutera ou supprimera l'action dans la bibliothèque des favoris en haut de la boîte à outils. Cette bibliothèque vous permet de créer un groupe personnalisé d'actions que vous utilisez régulièrement, ce qui réduit le temps nécessaire pour les rechercher dans leurs bibliothèques respectives.
  • Effacer les favoris: cette option permet de supprimer toutes les actions de la bibliothèque des favoris.
  • Effacer les actions récemment utilisées: En sélectionnant cette option, vous supprimez toutes les actions de la bibliothèque Utilisé récemment.
  • Réduire tout: Cette option réduit toutes les bibliothèques de GML Visual de sorte que seuls les titres des bibliothèques soient visibles, et non les nœuds d'action. Chaque bibliothèque peut être développée à nouveau en cliquant dessus.
