Créer un tampon

Cette action peut être utilisée pour allouer une portion de mémoire en tant que buffer dans votre jeu, la fonction retournant la valeur unique de l'ID de la mémoire tampon qui doit être stockée dans une variable et utilisée pour tous les appels de fonction ultérieurs vers le buffer. Vous pouvez fournir une variable globale, une variable d'instance ou demander à l'action de créer une variable locale temporaire pour stocker l'ID de buffer. Notez que vous pouvez créer des buffers supplémentaires en cliquant sur l'icône plus à côté de l'action, et en sélectionnant une autre variable pour contenir l'ID buffer.

REMARQUE: il est important de supprimer de la mémoire toute ressource créée dynamiquement comme celle-ci lorsque vous n'en avez plus besoin afin d'éviter les fuites de mémoire. Ainsi, lorsque vous en avez terminé avec le site buffer que vous avez créé, vous devez le libérer à nouveau en utilisant l'action Delete Buffer ( dans l'événement Other - Clean Up, par exemple). Il est particulièrement important de s'en souvenir lorsque vous utilisez une variable locale temporaire. car si vous ne supprimez pas le buffer avant la fin de l'événement ou du script d'action, vous ne pourrez plus utiliser le access et vous risquez de rencontrer des problèmes par la suite.
