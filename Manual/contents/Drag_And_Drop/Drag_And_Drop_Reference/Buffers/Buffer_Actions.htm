Bibliothèque d'action des tampons

Un tampon est une région de la mémoire physique utilisée pour contenir temporairement des données pendant qu'elles sont déplacées d'un endroit à un autre ou manipulées d'une manière ou d'une autre. Par exemple, vous pouvez utiliser un buffer pour contenir des données sur l'état du jeu. GameMaker comporte un certain nombre d'actions spécifiques liées à la création, à la modification et au chargement/à la sauvegarde de buffers, mais avant de les utiliser, veuillez noter les points suivants :

  • Lorsqu'un buffer est créé, il est automatiquement effacé et rempli à 0.
  • L'argument "taille" des fonctions buffer fait toujours référence à la taille en octets du site buffer.
  • GML Visual buffers est toujours aligné sur 1 octet. L'"alignement" fait référence à la manière dont les données sont stockées dans un buffer et est utilisé lors de l'utilisation des fonctions GML buffer , mais pour GML Visual, il n'est pas pertinent.
  • Les données dans buffers sont lues et écrites de manière séquentielle (c'est-à-dire que si vous appelez l'action Write Buffer trois fois, elle écrira les premières données, avancera, écrira les deuxièmes données, avancera et écrira les dernières données), avec quelque chose appelé la valeur "seek" pour vous indiquer où dans le buffer la prochaine lecture/écriture aura lieu.
  • Toutes les données sont écrites sur le site buffer et renvoyées par le site buffer sous la forme d'une chaîne de caractères. Si vous avez stocké des nombres réels, vous devrez utiliser l'une des actions de type de données pour les convertir de la chaîne de caractères en valeur.

La mémoire utilisée pour créer buffers est la mémoire système. Ainsi, même lorsque le jeu n'est pas actif, toutes les données stockées dans buffer sont en sécurité, mais elles sont perdues si l'application est fermée ou redémarrée. Il convient également de noter que buffers peut également être sauvegardé en tant que fichier puis rechargé dans le jeu à l'aide des actions de fichier.
