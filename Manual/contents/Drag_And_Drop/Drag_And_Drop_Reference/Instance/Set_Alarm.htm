Régler le compte à rebours de l'alarme

Ce bloc d'action déclenche un compte à rebours pour une alarme donnée, et lorsque l'alarme atteint zéro, l'événement d'alarme associé est déclenché. Vous fournissez le numéro d'alarme de 0 à 11, puis la valeur à laquelle l'alarme doit être réglée. puis la valeur à laquelle l'alarme doit être réglée. La valeur utilisée est le nombre de "pas de jeu" que vous souhaitez voir s'écouler avant le déclenchement de l'événement d'alarme, et vous pouvez la définir comme relative à la valeur d'alarme actuelle (en ajoutant ou en soustrayant à la valeur actuelle). actuelle). Notez que la valeur doit être un nombre entier et que vous pouvez la définir sur -1 pour arrêter l'alarme. Notez que lorsqu'une alarme effectue un décompte, l'événement est exécuté lorsqu'il atteint 0, mais l'alarme décompte ensuite jusqu'à -1 pour montrer qu'elle est terminée. Si vous réglez une 0 ne lancera pas l'événement, car le compte à rebours passera à -1 à l'étape suivante, donc pour lancer une alarme immédiatement, il faut la régler sur 1. Notez que vous pouvez définir des alarmes supplémentaires dans la même action en cliquant sur l'icône plus à côté de l'action.

Syntaxe de l'action :

Arguments :

Arguments Description
Alarm L'alarme à définir (l'événement d'alarme correspondant sera déclenché lorsqu'il atteindra 0).
Countdown La valeur du décompte en pas de jeu.

Exemple :

Le code du bloc d'action ci-dessus crée une variable locale (temporaire) et stocke la valeur actuelle de l'alarme [0]. Cette valeur est ensuite vérifiée et si elle est inférieure à 0, l'alarme est réglée sur 60.
