Si l'expression

Cette action est utilisée pour vérifier qu'une expression est soit true soit false, et vous pouvez vérifier n'importe quelle expression logique ici tant qu'elle évalue l'une ou l'autre valeur. Par exemple, n'importe laquelle des expressions suivantes peut être utilisée :

x < (room_width / 2)
a == b
++val >= 100

Si vous marquez l'argument " Not ", alors les expressions ci-dessus seront des expressions négatives, par exemple "==" devient "not=". Notez également que l'expression n'est pas limitée aux valeurs numériques et que vous pouvez vérifier les chaînes de caractères, les pointeurs, les ID de ressources ou tout autre élément qu'une fonction peut retourner ou utiliser (voir la section sur les types de données pour plus d'informations), mais vous ne pouvez pas comparer deux types de données différents sans obtenir une erreur - par exemple, si vous essayez d'utiliser une chaîne de caractères et un entier dans l'expression, vous obtiendrez une erreur. Pour plus d'informations sur les expressions, voir ici.

REMARQUE IMPORTANTE: En raison de problèmes de précision de la virgule flottante, vérifier si deux valeurs sont exactement égales peut renvoyer false, car l'une peut être exactement 1, tandis que l'autre peut être 1.00000000000001. l'autre peut être 1.00000000000001. Ce problème peut être évité en utilisant l'action Decimal to Integer avant de vérifier ou en utilisant les expressions "supérieur à (>)" ou "inférieur à (<)".

Ces actions seront maintenant exécutées si le bloc "if" donne la valeur true, tandis que toute action déposée ailleurs sera exécutée après le bloc "if". Notez que vous pouvez "empiler" des expressions dans la même action en cliquant sur l'icône plus à côté de l'action, et en donnant une autre expression à vérifier. Cela permettra de vérifier si toutes les expressions données retournent true en une seule fois et de n'exécuter le code d'action ajouté que si c'est le cas (voir l'exemple ci-dessous).
