Fichiers Bibliothèque d'action

Les actions sur les fichiers traitent de deux types de fichiers différents - les fichiers Buffer et les fichiers Ini - et comportent également certaines actions génériques permettant de renommer ou de copier des fichiers existants.

Lesfichiers tampons sont créés en sauvegardant les données d'un buffer que vous avez précédemment créé et ces données peuvent ensuite être rechargées dans un buffer à tout moment dans le futur. Vous pouvez trouver plus d'informations sur buffers ici : Actions tampons.

Lesfichiers Ini sont de petits fichiers légers qui sont compatibles avec la plupart des plateformes. Ils sont idéaux pour stocker de petits éléments d'information, comme les préférences d'interface, les meilleurs scores locaux, les données de niveau, etc. Les fichiers Ini n'ont pas n'ont pas besoin d'avoir été créés au préalable pour utiliser ces actions, et vous pouvez lire un fichier Ini inexistant et vous obtiendrez simplement une valeur de retour default (que vous spécifiez), cependant nous recommandons que vous créiez au moins un fichier ini "de base" pour ouvrir et modifier les actions. pour l'ouvrir et le modifier avant d'utiliser les actions. Ce fichier ini de base peut être créé en appelant simplement l'action Open Ini File suivie de l'action Close Ini File, puisque la fermeture du fichier l'écrira sur le disque. Ce fichier ini de base peut être créé en appelant simplement l'action Ouvrir le fichier Ini suivie de l'action Fermer le fichier Ini, puisque la fermeture du fichier l'écrira sur le disque. Si vous utilisez un fichier inclus dans l'Asset Browser comme Ini de base, vous devez également lire la section du manuel concernant la façon dont l'Ini de base peut être utilisée. la section du manuel concernant le fonctionnement du système de fichiers.

NOTE: Pour les jeux avec localisations, il est très important que les lettres accentuées puissent être lues à partir de fichiers externes. Cela signifie que vous devez d'abord créer le fichier ini au format UTF8, puis l'ajouter dans GameMaker en tant que fichier inclus. afin qu'il soit exporté lors de l'exécution du jeu et utilisé à la place du fichier ini par défaut au format ANSI qui est créé par les fonctions ini lorsqu'aucun fichier n'a été fourni auparavant. De cette façon, vous pouvez lire et écrire correctement dans ce fichier en conservant tous les accents et les lettres non romaines. sont conservés.
