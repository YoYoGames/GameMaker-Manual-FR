Créer un jeu de retour en arrière

Cette action accueille une nouvelle partie multijoueur. Elle prend le nombre de joueurs qui doivent être connectés pour que le jeu commence, et vous permet éventuellement de désactiver le test de synchronisation.

Lorsqu'il est appelé avec le test de synchronisation désactivé, le système attend que le nombre donné de joueurs se joigne au jeu avant de le démarrer. Veillez à ne pas créer ou modifier d'instances gérées avant le début de la partie.

ASTUCE Vous pouvez éventuellement forcer le jeu à démarrer avant que tous les joueurs ne se soient joints à vous, en exécutant Start Game.

Actuellement, vous ne pouvez avoir que 4 joueurs maximum dans une partie.

AVERTISSEMENT Il n'est pas recommandé de définir une région par défaut dans cette action, sauf si vous permettez au lecteur de la sélectionner par le biais d'un menu.

Syntaxe de l'action :

Arguments :

Arguments Description
Count of players Le nombre de joueurs qui doivent être connectés pour que le jeu commence. Si Define Player est utilisé, il s'agit du nombre d'instances qui sont automatiquement créées.
Sync test Spécifie si le test de synchronisation doit être activé, et est activé par défaut. Lorsqu'elle est désactivée, la fonctionnalité en ligne est activée.
Region Spécifie la région où la salle virtuelle sera créée. Pour plus d'informations sur l'obtention de régions, voir rollback_create_game().
