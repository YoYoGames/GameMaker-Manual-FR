Bibliothèque des actions de la souris et du clavier

Le site Souris et clavier est l'endroit où vous pouvez trouver les actions requises pour interroger et traiter les entrées du clavier ou de la souris. Les actions disponibles sont toutes répertoriées ci-dessous :

Si la souris est pressée
Si la souris est en bas
Si la souris est libérée
Si la touche est pressée
Si la clé est enfoncée
Si la clé est libérée

Le reste des actions présentées ici font référence au clavier virtuel. Si vous avez besoin de saisir du texte ou des chiffres dans votre jeu sur un appareil à écran tactile, vous pouvez utiliser les actions de clavier virtuel fournies pour utiliser le clavier du système d'exploitation de l'appareil. Ces actions appelleront ou masqueront le clavier du système d'exploitation et accepteront les entrées de celui-ci de la même manière qu'un clavier physique. Les actions clavier habituelles de GameMaker devraient donc toutes fonctionner avec les claviers virtuels (par exemple, l'action "If Key Released"), et toute entrée provenant du clavier virtuel sera stockée dans les différentes variables du clavier, telles que keyboard_string ou keyboard_last_char, etc...).

L'ouverture ou la fermeture du clavier virtuel générera des événements asynchrones du système, dans lesquels la carte DS async_load  sera remplie avec les paires clé/valeur suivantes :

  • "event_type" - le type d'événement système reçu, qui sera "virtual keyboard status" pour les claviers virtuels.
  • "screen_height" - la hauteur du clavier virtuel (en pixels). Elle sera de 0 si le clavier est invisible.
  • "keyboard_status" - l'état actuel du clavier, renvoyé sous la forme d'une des chaînes suivantes :
    • "caché"
    • "caché"
    • "visible"
    • "visible"

Les actions du clavier virtuel disponibles sont les suivantes :

Afficher le clavier virtuel
Masquer le clavier virtuel
Obtenir la hauteur du clavier virtuel
Si le clavier virtuel est affiché
