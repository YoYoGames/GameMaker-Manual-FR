Afficher le clavier virtuel

Cette action peut être utilisée pour afficher le clavier virtuel sur le périphérique exécutant le jeu. Lorsque vous appelez cette action, il vous sera demandé de fournir l'une des constantes suivantes pour chacun des trois premiers arguments :

  • Type de clavier: Cette constante est utilisée pour définir le jeu de touches qui sera disponible sur le clavier virtuel. Les types de clavier disponibles sont les suivants :
    Type de clavier Description
    kbv_type_default Le type de clavier par défaut pour le système actuel.
    kbv_type_ascii Un clavier uniquement ASCII.
    kbv_type_url Un clavier normal optimisé pour la saisie d'URL. Il comporte généralement un ".com" ou d'autres touches de domaine, ainsi que des touches "/" et ".".
    kbv_type_email Un clavier normal optimisé pour la saisie d'e-mails. Il comporte généralement les caractères "@" et ".".
    kbv_type_numbers Un clavier uniquement numérique, généralement représenté par un pavé numérique.
    kbv_type_phone Un clavier de pavé téléphonique. Généralement composé de chiffres uniquement avec les touches "*" et "#".
    kbv_type_phone_name Un clavier optimisé pour la saisie d'un numéro de téléphone et d'un nom. Généralement similaire à un clavier ASCII, mais avec une sélection limitée de caractères spéciaux.
  • Type de retour: Cette constante est utilisée pour définir ce qui est affiché sur la touche de retour/action du clavier virtuel. Les types de retour disponibles sont les suivants :
    Type de retour Description
    kbv_returnkey_default Le titre de la touche de retour par défaut pour le système actuel.
    kbv_returnkey_go Définit le titre de la touche de retour à "Go".
    kbv_returnkey_google Définit le titre de la touche de retour à "Google", ou à une icône de recherche générique dans certains cas.
    kbv_returnkey_join Définit le titre de la touche de retour à "Aller".
    kbv_returnkey_next Définit le titre de la touche de retour à "Suivant".
    kbv_returnkey_route La touche de retour s'intitule "Itinéraire".
    kbv_returnkey_search Définit le titre de la touche de retour à "Rechercher", ou à une icône de recherche générique dans certains cas.
    kbv_returnkey_send Définit le titre de la touche de retour à "Envoyer".
    kbv_returnkey_yahoo Définit le titre de la touche de retour à "Yahoo", ou à une icône de recherche générique dans certains cas.
    kbv_returnkey_done Définit le titre de la touche de retour à "Done".
    kbv_returnkey_continue La touche de retour s'affiche sous le titre "Continuer".
    kbv_returnkey_emergency Définit le titre de la touche de retour à "Appel d'urgence".
  • Type de capitalisation automatique: Cette constante est utilisée pour déterminer comment les mots tapés via le clavier virtuel doivent être capitalisés automatiquement. Les types de capitalisation automatique disponibles sont les suivants :
    Type de capitalisation automatique Description
    kbv_autocapitalize_none La capitalisation automatique est désactivée.
    kbv_autocapitalize_words Les mots seront mis en majuscules automatiquement.
    kbv_autocapitalize_sentences Les phrases seront mises en majuscules automatiquement.
    kbv_autocapitalize_characters Tous les caractères seront mis en majuscules.

Le dernier argument permet d'activer/désactiver le texte prédictif. Il doit être défini sur true pour l'autoriser et sur false dans le cas contraire. de l'utilisateur de l'appareil. En mode texte prédictif, le clavier virtuel ne génère pas d'événements de pression de touche GameMaker normaux. Au lieu de cela, il mettra uniquement à jour le dernier caractère appuyé et les variables de la chaîne de caractères du clavier. Ceci est dû à l'incapacité de détecter si un changement dans le champ de texte interne utilisé pour détecter les pressions sur les touches provient d'une touche réelle du clavier virtuel ou d'une suggestion de texte. Dans ce cas, il est préférable de lire l'entrée keyboard_string plutôt que de lire tout type de saisie brute. plutôt que de lire n'importe quel type d'entrée de touche brute.

Il est également important de noter que l'utilisateur obtiendra des claviers différents avec des capacités différentes selon le système d'exploitation de la plate-forme, avec les réserves suivantes pour une utilisation par cible :

  • UWP: l'implémentation UWP ne prend pas en charge return_key_type, autocapitalization_type ou predictive_text_enabled. Le passage de ces paramètres dans une fonction ne provoquera aucune erreur, mais n'aura aucun effet sur le clavier. effet sur le clavier.
  • Android / Amazon Fire: Sur Android, le type de clavier kbv_type_phone_name n'est pas pris en charge et un clavier ASCII s'affiche à la place, et le type de touche de retour ne peut être que kbv_returnkey_go, kbv_returnkey_search, kbv_returnkey_next ou , kbv_returnkey_send ou kbv_returnkey_done. Si d'autres types de touches de retour sont utilisés sur cette plate-forme, le système utilisera par défaut les touches de retour de remplacement indiquées ci-dessous :
    Clé de retour non disponible Touche de remplacement
    kbv_returnkey_google
    kbv_returnkey_yahoo    		 kbv_returnkey_search
    kbv_returnkey_join
    kbv_returnkey_route
    kbv_returnkey_emergency    		 kbv_returnkey_go
    kbv_returnkey_continue kbv_returnkey_next

  • AndroidTV / FireTV: Les types de touches de retour personnalisés ne sont pas pris en charge sur les claviers ASCII - la touche de retour par défaut sera toujours affichée.
  • tvOS: le texte prédictif et la capitalisation automatique ne sont pas pris en charge sur tvOS, et la hauteur du clavier renvoyée par les événements système et la fonction keyboard_virtual_height() correspondra par défaut à la hauteur de l'écran, étant donné qu'il s'étend sur tout l'écran et que les fonctions de dimension rectale du clavier sont désactivées sur tvOS. Notez également que les événements du clavier physique (Bluetooth) ne seront pas diffusés si le clavier virtuel n'a pas été ouvert. été ouvert.

L'appel de cette fonction génère un événement asynchrone du système, dans lequel la carte DS async_load  est remplie avec les paires clé/valeur suivantes :

  • "event_type" - il s'agira de "l'état du clavier virtuel" lorsqu'il sera déclenché par des actions du clavier virtuel.
  • "screen_height" - la hauteur du clavier virtuel (en pixels). Elle sera de 0 si le clavier est invisible.
  • "keyboard_status" - l'état actuel du clavier, renvoyé sous la forme d'une des chaînes suivantes :
    • "caché"
    • "caché"
    • "visible"
    • "visible"

Arguments :

Arguments Description

predictive_text_enabled Réglez sur true/false pour activer/désactiver la saisie de texte prédictive.
