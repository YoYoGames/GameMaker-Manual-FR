Erreurs de compilation

Le deuxième type d'erreur qui peut être signalé pour votre jeu est l'erreur de compilation. Une erreur de compilation se produit lorsque votre jeu rencontre un type d'erreur que le vérificateur syntaxique n'a pas pu détecter - il est très facile d'oublier de déclarer une variable ou de taper par erreur une chaîne de caractères comme un réel, par exemple - ou lorsqu'une erreur est liée à la façon dont vous avez configuré les outils de compilation dans les préférences de la plate-forme pour la plate-forme cible. Ces erreurs seront détectées par GameMaker lors de la compilation du jeu et ces informations seront également affichées dans la sortie des erreurs de compilation, comme indiqué ci-dessous :

NOTE Ces erreurs n'entraînent généralement pas l'ouverture d'une fenêtre et indiquent simplement "Build Failed" dans la sortie du compilateur. Si une fenêtre s'ouvre, il s'agit généralement d'une erreur Runner, qui est expliquée ici.

Les messages d'erreur du compilateur suivront tous le même format :

[object] - [event] - [Line Number]: [error string]

Si l'erreur se trouve dans un script, elle sera tout simplement :

[script] - [Line Number]: [error string]

Vous pouvez ensuite double-cliquer sur sur l'une des entrées d'erreur de compilation pour ouvrir l'actif donné à la position signalée comme donnant lieu à l'erreur, puis vous pouvez utiliser les informations contenues dans le message pour déterminer exactement où l'erreur s'est produite dans l'objet ou le script. Cependant, ces erreurs peuvent parfois sembler un peu cryptiques, c'est pourquoi vous trouverez ci-dessous une liste complète de toutes les erreurs et une brève explication de leur signification :

Erreur Description

Divers
Aucun programme à compiler En raison d'une erreur de compilation antérieure, l'arbre d'analyse du compilateur n'a pas été créé et le compilateur n'a donc pas de données de programme à compiler.
Erreur fatale lors de la compilation de [nom] - détails de l'échec ci-dessous Ce message signifie simplement qu'une erreur a été découverte, mais que le compilateur ne peut pas dire où et comment (si cela se produit régulièrement, vous pouvez contacter le support technique et fournir les détails que la fenêtre de sortie du compilateur affiche).
L'expansion récursive des macros n'est pas supportée Vous avez utilisé des macros récursives qui se réfèrent les unes aux autres et s'étendent constamment.
Impossible de trouver TextureGroup [group] Cette erreur ne se produit généralement qu'en essayant d'exécuter un fichier de projet malformé (si cela se produit régulièrement, vous pouvez contacter le support technique et fournir les détails que la fenêtre de sortie du compilateur affiche).
Mauvaise conversion de [value] en nombre Vous avez essayé de convertir une valeur donnée, par exemple une chaîne de caractères, en un nombre.
Division par 0 Vous avez essayé de diviser une valeur par 0.
Identifiant d'objet non valide [object ID] utilisé Vous avez essayé de access un objet en utilisant une valeur ID qui est incorrecte
Commentaire non fermé (/*) à la fin du script Vous avez créé un commentaire de script en utilisant
/* [comment] */
mais vous n'avez pas fourni la balise de fermeture du commentaire
Jeton invalide Vous avez un caractère invalide dans votre code de jeu, ce qui peut arriver avec des caractères de langue étrangère ou Unicode.
Numéro [num] dans un format incorrect Le nombre donné - indiqué dans l'erreur - n'est pas le format correct pour la fonction ou l'opération exécutée, par exemple vous fournissez un nombre entier alors qu'un pointeur est requis.
Un EOF inattendu a été rencontré Vous avez atteint la fin d'un fichier de manière inattendue lors de l'utilisation des fonctions de fichier.

Cordes
Impossible de Not a string Vous avez essayé d'utiliser "!" (pas) sur une chaîne de caractères.
Impossible de nier une chaîne de caractères Vous avez essayé de nier une chaîne de caractères, par exemple
string = -string

Scripts, Fonctions et arguments
Mauvais nombre d'arguments pour la fonction [function] Vous avez fourni trop ou trop peu d'arguments pour la fonction donnée.
L'analyse de la fonction action_execute_script() a échoué - nombre incorrect d'arguments reçus. L'action visuelle GML Exécuter le script a trop ou trop peu d'arguments pour le script appelé.
Fonction inconnue [function] vérifier si le script est vide Un script ou une fonction a été appelé(e) que le compilateur ne reconnaît pas (cette erreur est généralement déclenchée par des références de ressources de script vides, car le compilateur élimine ces ressources à la compilation).
La fonction [function] attend des arguments de n, a fourni Une fonction a reçu un nombre erroné d'arguments (n) alors qu'un certain nombre était attendu (a).
Assignation d'une valeur vide (la fonction ne renvoie rien ?) Vous avez essayé d'assigner une valeur de retour d'une fonction alors que cette fonction ne retourne rien.
Impossible de trouver la fonction [function] Vous avez appelé une fonction inconnue
Appel d'une fonction qui nécessite une instance et aucune instance n'est disponible La fonction ou le script appelé est destiné à agir sur une instance, mais au moment de l'exécution, aucune instance n'existe.
Appel d'une fonction qui a besoin d'une autre et aucune autre n'est disponible Vous avez utilisé le mot-clé other en dehors de l'événement Collision ou en dehors d'une déclaration with.
Erreur de dénomination des arguments, n arguments mais aucune référence trouvée à a Vous avez donné un certain nombre d'arguments à un script, mais vous omettez de faire référence à l'un d'entre eux dans le script. Par exemple, votre script prend trois arguments mais vous n'utilisez que les arguments n et a
Impossible de comparer les arguments Vous avez essayé de comparer deux types d'arguments différents, comme une chaîne de caractères et un nombre réel.
Fonction ou script inconnu [functions/script] Le compilateur ne reconnaît pas la fonction ou le script appelé.
Impossible d'utiliser le nom de la ressource [resource] car elle n'est pas exportée. Vous avez essayé d'envoyer à access une ressource qui n'a pas été signalée comme faisant partie du projet pour la plateforme sur laquelle elle est compilée.
Impossible d'utiliser un nom de fonction/script pour une variable, en utilisant [functions/script] Vous avez essayé d'utiliser un nom de fonction ou un nom de script comme variable, par exemple :
string = "Hello World"
Impossible d'utiliser un argument en dehors d'un script Les variables d'argument intégrées ne peuvent être utilisées qu'à l'intérieur d'un script, mais vous avez essayé de les utiliser ailleurs.
Aucune référence à argument[num] mais des références argument[num] Vous utilisez une référence à un argument qui n'existe pas mais qui pourrait correspondre à un autre argument.
Nombre incorrect d'arguments pour l'accesseur Vous avez fourni le mauvais nombre d'arguments lors de l'utilisation d'un accesseur.
Mauvais type d'arguments pour (l'un des éléments ci-dessous) :
/, div, mod, +, -, &&, ||, &, |, ^, <<, >>		 Vous avez fourni le mauvais type d'argument pour l'expression évaluée.

Variables

La variable [var] est en lecture seule Vous avez essayé de changer la variable nommée alors qu'elle ne peut pas l'être.
Impossible de trouver la variable nommée [var]. La variable nommée ne peut être trouvée dans la compilation actuelle
Référence de variable malformée Vous avez essayé de référencer une variable de manière incorrecte.
Essayer de définir une variable en lecture seule [var]. Vous avez essayé de définir une variable qui est en lecture seule.
Impossible de redéclarer une variable intégrée. Vous avez essayé de déclarer une variable locale en utilisant le même nom que l'une des variables intégrées.
Impossible d'utiliser le nom de la ressource pour une variable Vous pouvez utiliser le même nom qu'une ressource de l'arborescence des ressources comme variable locale.
[Variable] est en lecture seule Vous avez essayé d'utiliser une variable qui a été marquée comme étant en lecture seule.

Constantes
Impossible de donner une valeur à une constante [constante]. Vous avez essayé de définir une constante à une valeur différente.
La constante est invalide ici Vous avez essayé d'utiliser une constante qui n'est pas valide pour le code en cours d'exécution.
Impossible d'assigner à [var] - c'est une constante. Vous avez essayé d'attribuer une nouvelle valeur à la constante nommée

Tableaux
L'index du tableau ne doit pas être négatif Vous avez essayé de access, de créer ou de définir une valeur de tableau en utilisant un index de tableau négatif.

Switch / Case
La valeur par défaut ne peut pas être utilisée plusieurs fois dans un commutateur. Dans votre déclaration de commutation, vous avez défini default : plus d'une fois.
Erreur de type d'expression de clause (les expressions de clause doivent toutes être du même type) Dans votre commutateur, vous vérifiez deux types de constantes différents, par exemple une chaîne de caractères dans un cas et un nombre entier dans l'autre.
L'argumentaire doit être une constante Vous avez essayé d'utiliser une valeur qui n'est pas une constante dans l'un de vos cas de commutation.
L'énoncé dans un commutateur DOIT apparaître après la casse ou le défaut. Vous avez ajouté une instruction à un commutateur en dehors d'un appel à case: ou default:.
Duplicate case statement found / original here Vous avez utilisé la même constante de cas à deux endroits différents dans un commutateur (notez que le commentaire "original ici" sera sur une autre ligne et peut être cliqué pour montrer le cas original).

Expressions
Pas de valeur de retour de l'expression (l'expression a-t-elle une valeur de retour ?) Vous avez essayé d'assigner une valeur de retour d'une expression à une variable alors que l'expression ne retourne rien.
Expression attendue après le ". Vous avez utilisé la notation du point "." de manière erronée et n'avez pas ajouté une expression après le mot correctement.
Symbole [symbole] inattendu dans l'expression L'expression utilise un symbole qui ne fait pas partie de l'ensemble des symboles d'expression acceptés - le symbole en question sera affiché.

Opérateur de poste inconnu Vous avez essayé d'utiliser un symbole comme post-opérateur alors que ce symbole n'est pas autorisé, par exemple [var]** ou [var]>> - seuls ++ et -- sont acceptables.
Pragma inconnu [chaîne de caractères gml_pragma]. Vous avez utilisé une chaîne de pragma qui est inconnue du compilateur.
Opérateur unaire inconnu Vous avez essayé d'utiliser un symbole inconnu comme opérateur unaire.
Opérateur binaire inconnu Vous avez essayé d'utiliser un symbole inconnu comme opérateur binaire.
Opérateur inconnu [opérateur] Vous avez essayé d'utiliser un symbole inconnu comme opérateur (le symbole sera affiché dans l'erreur).
