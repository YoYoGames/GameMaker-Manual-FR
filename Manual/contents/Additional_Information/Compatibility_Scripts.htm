Scripts de compatibilité

Pour les utilisateurs qui viennent des versions précédentes de GameMaker, il peut être un peu difficile d'utiliser GameMaker au début, car un certain nombre de changements ont été apportés à la façon dont les actifs, l'édition et le codage des projets sont gérés. Ces changements signifient que ni l'EDI ni le langage GameMaker (GML) ne prennent en charge les éléments énumérés ci-dessous. Toutefois, cela ne signifie pas que vous ne pouvez pas importer dans GameMaker des jeux réalisés avec des versions antérieures, car vous le pouvez. GameMaker convertit automatiquement toutes les fonctions héritées en scripts de compatibilité, qui sont des scripts contenant des fonctions qui convertissent les méthodes héritées en méthodes mises à jour. les méthodes héritées en méthodes mises à jour.

GameMaker analysera votre projet à la recherche de toutes les fonctions et variables obsolètes que vous avez utilisées, puis créera une série de scripts utilisant ces noms de fonctions. Ces scripts contiendront soit la nouvelle fonction d'exécution équivalente, soit une courte fonction de script exécutant la fonction. équivalente ou une courte fonction de script qui effectue la même action que la fonction obsolète.

Lorsque vous importez un projet GameMaker pour la première fois, un rapport de compatibilité semblable à celui-ci s'affiche :

Le corps principal du rapport indique quels objets et scripts ont été affectés et à quel endroit. vous pouvez trouver une liste des scripts qui ont été créés pour votre projet. Vous pouvez cliquer sur le bouton central de la souris ou appuyer sur sur l'un des scripts répertoriés dans le rapport pour l'ouvrir et voir ce qui a été fait.

Notez que si vous voyez des scripts créés avec une double barre sous leur nom (par exemple : __global_object_depths), il s'agit de scripts créés par IDE pour faciliter la compatibilité et non de scripts convertis à partir de fonctions obsolètes. qui ont été convertis à partir de fonctions obsolètes. Ils ne doivent donc pas être modifiés car ils sont nécessaires au fonctionnement des autres scripts de compatibilité.

Lorsque vous avez fini de réviser le rapport de compatibilité, vous pouvez fermer l'espace de travail, mais si vous avez besoin de réviser quoi que ce soit dans le rapport, vous pouvez le retrouver en allant dans la section Notes du Navigateur d'actifs.

Notez que vous pouvez continuer à utiliser ces scripts de compatibilité comme vous l'auriez fait avec les fonctions de l'ancienne version de GameMaker. Toutefois, en raison de leur mode de fonctionnement, cela aura un impact sur les performances. Nous vous recommandons de de mettre à jour les projets afin d'utiliser directement les nouvelles méthodes et de supprimer les scripts de compatibilité si vous le pouvez, car cela améliorera les performances et vous donnera un aperçu du fonctionnement de GameMaker et de GameMaker Language par rapport aux anciens produits.

Notez également que certaines fonctions ont été introduites sur le site GameMaker Language pour assurer la compatibilité avec les anciens projets et qu'elles apparaîtront dans ces scripts de compatibilité. Cependant, ces fonctions ne doivent jamais être utilisées dans vos propres projets. Vous pouvez trouver une liste de ces fonctions dans la section suivante du manuel :
