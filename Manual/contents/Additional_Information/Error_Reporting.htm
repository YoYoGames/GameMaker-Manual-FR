Rapport d'erreur

La première forme de rapport d'erreur est l'erreur de syntaxe. Il s'agit d'une erreur dans votre code que GameMaker a détectée avant que vous n'essayiez de le compiler et qui sera affichée sur le site IDE, soit dans l'éditeur de code, soit dans la fenêtre de sortie des erreurs de syntaxe:

Le niveau suivant de signalement des erreurs est l'erreur de compilation. Ces types d'erreurs sont plus généraux que les erreurs de runner et peuvent être dues à une erreur subtile dans le code du jeu, qui provoque un plantage pendant la compilation, avant même qu'il ait la possibilité de s'exécuter. Elles seront également signalées dans la fenêtre de sortie des erreurs de compilation :

Si le jeu se compile et qu'une erreur se produit, vous obtiendrez une Runner Error. Ce type d'erreur affiche un message à l'écran, comme celui illustré ci-dessous :

La plupart des messages d'erreur de runner suivent le même schéma, avec un message indiquant le type d'erreur, l'instance et l'événement dans lequel l'erreur a été rencontrée, puis une série de détails sur runner qui expliquent l'erreur de manière beaucoup plus approfondie et, enfin, une liste de toutes les variables globales, d'instance et locales déclarées dans votre jeu à ce moment-là. Tout en bas se trouve le bouton Abandonner qui interrompt le jeu, ainsi que le bouton Copier ( qui copie l'erreur dans le presse-papiers) et le bouton Effacer qui efface les messages d'erreur de la fenêtre. Les informations fournies par le rapport d'erreur sont très importantes car elles détectent et signalent les gros bogues liés à la syntaxe qui vont faire planter votre jeu fini, mais notez qu'il ne signalera pas les bogues plus subtils et ne donnera pas de retour sur les performances... pour cela, vous devez utiliser le débogueur.
