Structure de base du code

Le code est écrit en blocs et un bloc de code typique consiste en un ensemble d'instructions, appelées instructions, qui sont ensuite interprétées par GameMaker et utilisées pour que quelque chose se produise dans votre jeu. Ce "quelque chose" peut être aussi simple que d'ajouter 2 et 2 pour obtenir 4, ou aussi complexe que de faire fuir un ennemi lorsque sa santé passe en dessous d'une certaine valeur. La structure réelle du programme peut varier considérablement, en fonction des fonctions qu'il utilise, mais pour l'essentiel, elle ressemble à ceci :

<statement>;
<statement>;
...

Les déclarations doivent être séparées par un symbole " ;" afin d'éviter les erreurs de déclaration de variables et de garder votre code propre et ordonné. Elles peuvent consister en des déclarations de variables, des expressions et des appels à des fonctions spécifiques. Vous pouvez également "regrouper" les déclarations sous forme de bloc à l'aide des accolades {} de manière à ce qu'elles se suivent, comme dans l'exemple conditionnel suivant :

if (<conditional>)
{
    <statement>;
    <statement>;
    ...
}

NOTE Le site GameMaker Language acceptera également begin et end au lieu des accolades {}, bien que ce ne soit pas la façon la plus courante de procéder :

if (<conditional>)
begin
    <statement>;
    <statement>;
    ...
end
