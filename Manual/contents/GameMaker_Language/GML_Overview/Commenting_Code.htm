Code de commentaire

Lorsque vous travaillez en collaboration avec d'autres personnes, lorsque vous vous occupez d'un grand projet, ou même simplement pour votre propre référence et debugging, il est très important de laisser des commentaires dans votre code événementiel et vos scripts. Dans GameMaker, plusieurs mécanismes vous permettent de laisser des notes et des commentaires sur des sections de code, voire des blocs entiers de code, visibles dans la liste des événements de l'objet que vous êtes en train de modifier. De cette façon, vous pouvez laisser des notes à vous-même et à vos collègues, ou expliquer un morceau de code particulièrement délicat, ou même simplement laisser un rappel sur ce que fait quelque chose, ou ce que vous voulez que quelque chose fasse à l'avenir.

Pour laisser un simple commentaire d'une ligne, la première chose à faire est d'utiliser // avant le texte. Par exemple :

//initialize variables
sr = 10;
hp = 100;

Vous pouvez également laisser des commentaires de plusieurs lignes, pour donner du crédit, pour omettre une section complète du code pour debugging, ou même pour laisser des notes sur la façon dont vous voulez modifier le code à l'avenir. Pour cela, vous pouvez utiliser /* ... */ comme ceci :

/*
Setup GIF Variables to initial values (these can be edited so you don't need to use the setter functions all the time if you are always using the same values)

IMPORTANT! Please see the individual "set" scripts for more extensive information about each of the different variables.
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

Outre les commentaires, vous pouvez également utiliser des balises spéciales #region pour marquer une section de code pour le pliage du code, et chaque région peut contenir un commentaire. Le principe est le suivant : vous ajoutez une balise #region - ainsi que tout texte de commentaire - en haut d'une section de code, puis une balise #endregion correspondante en bas de la section, ce qui vous permet de "replier" le code, c'est-à-dire de le cacher afin de réduire le nombre de lignes de code visibles dans l'éditeur à un moment donné :

Le code actuel ressemble à quelque chose comme ceci :

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

Enfin, il convient de mentionner que lorsque vous travaillez avec des événements dans l'éditeur d'objets, vous pouvez utiliser un commentaire spécial "description" au début de l'événement pour indiquer le contenu général de l'événement, ce qui s'affichera dans la fenêtre des événements de l'éditeur d'objets. Ce commentaire doit être placé sur la toute première ligne de l'éditeur de code, et est formaté comme suit :

/// @description <This Is An Event Description>

Cela sera maintenant affiché dans la fenêtre de l'événement, par exemple :

Notez que vous pouvez définir un commentaire par défaut qui sera créé pour tous les événements à partir des préférences de l'éditeur d'objets.

Enfin, si vous écrivez des fonctions de script et que vous souhaitez avoir un certain contrôle sur la façon dont elles sont traitées par le site IDE (c'est-à-dire avoir une complétion automatique ou afficher les paramètres du script), vous devez écrire vos commentaires en utilisant la notation JSDoc comme expliqué ici.
