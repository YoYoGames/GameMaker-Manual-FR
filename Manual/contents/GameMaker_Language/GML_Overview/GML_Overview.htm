Aperçu ducode GML

Cette section du manuel contient toutes les informations nécessaires à la compréhension et à l'utilisation de Code GML. Les fonctions de ce langage, runtime, peuvent être utilisées pour créer vos jeux et sont ajoutées dans les objets à partir de l éditeur d'objetsmais il peut aussi être utilisé avec GML Visual. Vous pouvez voir ci-dessous une image typique d'un objet avec l'éditeur de code ouvert sur un événement :

Chaque événement a son propre onglet dans l'éditeur et vous pouvez ajouter, modifier ou supprimer du code à tout moment (pour plus d'informations sur les événements, voir Événements d'objets). Le code lui-même doit avoir une structure de base et peut contenir des indices de ressources, des variables, des fonctions, des expressions, des mots-clés, etc. qui sont tous expliqués dans les sections ci-dessous. Si vous êtes novice en matière de programmation ou si vous passez de GML Visual à GML, il est recommandé de commencer par la page suivante Structure de code de base puis de lire toutes les autres pages de cette section, en testant le code de chacune d'entre elles au sein de GameMaker lui-même.

Vous pouvez utiliser Feather pour compléter intelligemment le code et signaler les erreurs dans vos scripts de code GML.
