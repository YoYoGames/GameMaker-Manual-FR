uwp_check_privilege

Cette fonction vous permet de vérifier qu'un utilisateur donné dispose d'un privilège choisi. La fonction renvoie true ou false dans l'événement asynchrone du système selon que le privilège est activé ou non, et si vous définissez l'argument attempt_resolution à true et que le privilège n'est pas activé, elle ouvre également une boîte de dialogue système (suspendant le jeu) pour inviter l'utilisateur à mettre à niveau le compte ou tout autre élément requis pour obtenir le privilège. Si l'utilisateur active alors l'option requise, la fonction renvoie true.

La fonction exige que vous utilisiez l'un des éléments suivants Constante de privilège UWPpour la demande de privilège :

Constante de privilège UWP
Constant Description
uwp_privilege_internet_browsing Vérifiez si la navigation sur Internet est autorisée
uwp_privilege_social_network_sharing Vérifiez si le partage sur les réseaux sociaux est autorisé.
uwp_privilege_share_kinect_content Vérifiez si le partage de la manette Kinect est autorisé.
uwp_privilege_video_communications Vérifiez si la communication vidéo est autorisée
uwp_privilege_communications Vérifiez si la communication par Internet est autorisée
uwp_privilege_user_created_content Vérifier si access au contenu créé par l'utilisateur est autorisé
uwp_privilege_multiplayer_sessions Vérifiez si les sessions multijoueurs en ligne sont autorisées.
uwp_privilege_sessions Vérifiez si les sessions en ligne sont autorisées
uwp_privilege_fitness_upload Vérifiez si le téléchargement des données de fitness est autorisé.

Une fois que la fonction a été appelée, sur XBox One elle déclenchera un événement Système Asynchrone lorsque le résultat du callback aura été reçu, qui aura la carte spéciale DS async_load. Cette carte doit ensuite être analysée pour la clé suivante :

  • "event_type" - doit contenir la valeur "check_privilege_result" si l'événement a été déclenché par cette fonction.

Si le type d'événement concerne cette fonction, il y aura également les touches supplémentaires suivantes lorsque la fonction est appelée sur XBox :

  • "Résultat" - Il s'agit d'un ou plusieurs des éléments suivants UWP Privilege Result Constants (valeurs combinées) :
    UWP Privilege Result Constant
    Constant Description Valeur
    uwp_privilege_result_no_issue L'utilisateur ne présente aucun problème de privilège 0
    uwp_privilege_result_purchase_required L'utilisateur doit acheter quelque chose de plus, généralement un abonnement, pour avoir accès à l'application. 1
    uwp_privilege_result_aborted La vérification a été interrompue 2
    uwp_privilege_result_banned L'utilisateur est banni 4
    uwp_privilege_result_restricted L'utilisateur ne peut pas accéder à access, généralement par le biais du contrôle parental. 8

  • "privilège" - sera l'une des constantes énumérées ci-dessus à utiliser dans la fonction, afin que vous puissiez vérifier quel privilège vous avez demandé dans le cas de vérifications multiples.

Si le projet est exécuté sur une plateforme UWP qui n'est pas XBox One, l'événement asynchrone ne sera pas déclenché et la fonction renverra immédiatement -1, 0 ou 1 :

  • -1 signifie que l'utilisateur n'a pas ce privilège.
  • 0 signifie que la fonction n'a pas été capable d'interroger l'utilisateur.
  • 1 signifie que l'utilisateur a le privilège.

Syntaxe :

uwp_check_privilege(user_id, privilege, attempt_resolution);

Arguments Type Description
user_id ID utilisateur Xbox The User ID pointer to check.
privilege Constante de privilège UWP The privilege constant to check (see description, above).
attempt_resolution Booléen Enable attempting a resolution if set to trueou ignorer s'il s'agit de false.

Les retours :

Real

Exemple :

var user_one = xboxlive_get_user(0);
if (xboxlive_user_is_signed_in(user_one))
{
    uwp_check_privilege(user_one, xboxlive_privilege_multiplayer_sessions, true);
}

La procédure ci-dessus vérifie si l'utilisateur est connecté et s'il l'est, elle vérifie l'autorisation des sessions multi-joueurs.
