échelle_globale_de_la_texture

Cette fonction vous permet de contrôler la mise à l'échelle des pages de texture lors du chargement à partir du fichier WAD qui fait partie de l'exécutable de votre jeu final. La valeur d'entrée doit être une puissance de deux et fonctionnera de la manière suivante :

  • Pour définir les pages de texture pour être 1:1, utilisez texture_global_scale(1)
  • Pour les textures de moitié en permanence, utilisez texture_global_scale(2)
  • Pour les écarter, utilisez texture_global_scale(4), etc...

De cette façon, vous pouvez mieux contrôler l'utilisation de la mémoire des pages de textures sur les plateformes ayant des problèmes de mémoire faible. Notez que les pages de textures sont créées à la demande du WAD, et donc vous pouvez soit l'appeler au début du jeu pour charger toutes les textures à l'échelle, ou vous pouvez l'appeler à des moments spécifiques pour charger des textures spécifiques. peut l'appeler à des moments précis pour charger des textures spécifiques, auquel cas vous voudrez probablement appeler la fonction draw_texture_flush() avant d'appeler cette fonction, puis d'utiliser les fonctions de pre-fetch pour obtenir les bonnes les bonnes pages de texture en mémoire.

Syntaxe :

texture_global_scale(pow2integer);

ArgumentsType Description
pow2integer The scale factor to use (1, no scale, 2, half scale, 4, quarter scale, etc...)

Les retours :

Exemple :

draw_texture_flush();
texture_global_scale(2);
sprite_prefetch(spr_Trees);

Le code ci-dessus vide toutes les textures de la mémoire, puis règle l'échelle de texture sur 2 (les pages de texture font donc la moitié de leur taille) et, enfin, prélivre la page de texture dans laquelle se trouve le site sprite "spr_Trees".
