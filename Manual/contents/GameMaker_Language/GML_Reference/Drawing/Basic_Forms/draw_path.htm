draw_path

Avec cette fonction, vous pouvez demander à GameMaker de dessiner un chemin à l'écran. Le chemin sera dessiné comme une simple ligne, et peut être soit relatif à l'instance appelante, soit à la position absolue à laquelle il a été créé dans l'éditeur de chemin ou par le code. Cette fonction est extrêmement utile pour debugging. Les chemins dynamiques (par exemple, ceux créés pour les instances avec la fonction mp_grid_path() ).

REMARQUE: Si vous souhaitez dessiner une forme à l'aide d'un shader, vous devez savoir que la plupart des shaders attendent les entrées suivantes : vertex, texture, couleur. Cependant, en utilisant cette fonction, seules les données de vertex et de couleur sont transmises, et donc le shader peut ne rien dessiner (ou dessiner quelque chose mais pas correctement). Si vous avez besoin de dessiner des formes de cette manière, le site shader doit être personnalisé en conséquence.

Syntaxe :

draw_path(path, x, y, absolute);

ArgumentsType Description
path The path to draw
x The x coordinate of where the path is drawn
y The y coordinate of where the path is drawn
absolute Whether the path is drawn at the absolute position (true) ou la position relative (false)

Les retours :

Exemple :

if mp_grid_path(grid, path, x, y, obj_Player.x, obj_Player.y, 1)
{
    draw_path(path, x, y, false);
}

Le code ci-dessus utilisera la fonction mp_grid_path pour générer un chemin et le stocker dans la variable "path". Si le chemin est créé avec succès, il est ensuite dessiné à l'écran à une position relative à l'instance qui exécute le code.
