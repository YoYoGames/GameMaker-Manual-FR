dessiner_sprite_part

Avec cette fonction, vous pouvez dessiner une partie de n'importe quel sprite à une position donnée dans la pièce. Comme avec draw_sprite() vous pouvez spécifier une sprite et une sous-image à dessiner, puis vous devez donner les coordonnées relatives dans la sprite de la zone à sélectionner pour le dessin. Cela signifie qu'une position gauche de 0 et une position supérieure de 0 correspondent au coin supérieur gauche du site sprite et que toutes les autres coordonnées doivent être prises à partir de cette position. L'image ci-dessous montre un exemple comment cela fonctionne :

Vous devez noter que si la page de texture permet un recadrage automatique, cette fonction peut ne pas fonctionner comme prévu, car l'espace "vide" supplémentaire autour de sprite aura été supprimé pour créer la page de texture. Pour résoudre ce problème, vous devrez définir les paramètres de la page de texture (dans l'éditeur de groupe de texture) pour désactiver l'option Recadrage automatique.

REMARQUE: Lorsque vous dessinez avec cette fonction, les décalages x et y de sprite sont ignorés et la partie sprite sera dessinée avec le coin supérieur gauche à la position x/y spécifiée dans la pièce.

NOTE: Cette fonction n'est utile que pour les bitmaps sprites et ne fonctionne pas avec les SWF ou JSON (Spine) sprites.

Syntaxe :

draw_sprite_part(sprite, subimg, left, top, width, height, x, y);

ArgumentsType Description
sprite The index of the sprite to draw.
subimg The subimg (frame) of the sprite to draw (image_index or -1 correlate to the current frame of animation in the object).
left The x position on the sprite of the top left corner of the area to draw.
top The y position on the sprite of the top left corner of the area to draw.
width The width of the area to draw.
height The height of the area to draw.
x The x coordinate of where to draw the sprite.
y The y coordinate of where to draw the sprite.

Les retours :

Exemple :

draw_sprite_part(sprite_index, image_index, 4, 0, sprite_width-16, sprite_height-16, x, y );

Cela va dessiner les instances assignées à sprite (sprite_index) et son image d'animation actuelle (image_index), mais cela va réduire la largeur de 4 px des deux côtés, et la hauteur de 8 pixels à partir du bas de l'image originale de 24x24 pixel sprite.
