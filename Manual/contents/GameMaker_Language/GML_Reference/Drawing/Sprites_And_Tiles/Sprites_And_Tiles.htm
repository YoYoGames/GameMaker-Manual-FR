Sprites et tuiles

Bien que vous puissiez configurer des arrière-plans dans l'éditeur de salle et que vous puissiez assigner sprites aux instances, souvent cela n'est pas assez flexible pour le jeu que vous créez. Par exemple, si vous avez une instance qui doit dessiner un menu, vous pouvez vouloir afficher plusieurs boutons sprites à la fois, ou si vous avez un jeu avec des arrière-plans à défilement parallaxe, vous devrez être en mesure de les définir par code dans une instance de contrôleur quelconque. Dans ces situations, vous devriez utiliser les fonctions suivantes qui peuvent dessiner sprite et les actifs d'arrière-plan n'importe où dans la pièce. Certaines de ces fonctions vous permettent également de manipuler la forme, la taille, la couleur et l'alpha de sprite pour obtenir des effets spéciaux.

Vous pouvez assigner un sprite à une instance et par défaut, GameMaker la dessinera avec les propriétés données par les différentes variables du sprite. Cependant, si vous ajoutez quelque chose dans l'événement draw, vous "prenez le contrôle" et devez indiquer à GameMaker ce qu'il doit dessiner exactement. Cela signifie que si vous devez utiliser les fonctions suivantes pour définir exactement ce que l'instance doit dessiner et à quel endroit de la room elle doit le faire :

  • draw_self
  • draw_sprite
  • draw_sprite_ext
  • draw_sprite_general
  • draw_sprite_part
  • draw_sprite_part_ext
  • draw_sprite_stretched
  • draw_sprite_stretched_ext
  • draw_sprite_pos
  • draw_sprite_tiled
  • draw_sprite_tiled_ext

Si vous utilisez le format SWF sprites, il existe également des options supplémentaires pour contrôler l'anticrénelage utilisé lors du dessin (voir Importation de sprites non bitmap pour plus d'informations sur ces types de sprites). Notez que ces fonctions fonctionnent à un niveau global et définissent l'AA sur tous les SWF vectoriels sprites dessinés après leur appel.

  • draw_enable_swf_aa
  • draw_set_swf_aa_level
  • draw_get_swf_aa_level

Outre les fonctions énumérées ci-dessus, il existe également deux fonctions spéciales conçues pour être utilisées avec les animations squelettiques (voir Importation de sprites non bitmap pour plus d'informations sur ces types de sprites, et la section Animations squelettiques pour les fonctions spécifiques liées à ces types de sprites) :

  • draw_skeleton
  • draw_skeleton_instance
  • draw_skeleton_collision
  • draw_skeleton_time

Les tuiles sont dessinées automatiquement en fonction des données qu'elles contiennent, du jeu de tuiles utilisé et des propriétés de la carte des tuiles. Vous pouvez toutefois dessiner des tuiles individuelles ou des cartes de tuiles à l'aide des fonctions suivantes :

  • draw_tile
  • draw_tilemap
