orientation_type_partie

Cette fonction est utilisée pour déterminer l'orientation de la particule sprite lors de sa création et peut également être utilisée pour que l'orientation de la particule augmente ou diminue au cours de sa vie. Les valeurs d'orientation minimum et maximum sont par défaut à 0 mais ces valeurs peuvent être modifiées pour rendre l'orientation aléatoire en suivant les directions standards de GameMaker: 0 degré pour la droite, 90 degrés pour le haut, 180 degrés pour la gauche et 270 degrés pour le bas. Si vous les fixez à la même valeur, les particules seront toutes créées avec la même orientation.

Vous pouvez également définir une valeur d'incrémentation qui ajoutera (s'il s'agit d'un nombre positif) ou soustraira (s'il s'agit d'un nombre négatif) une quantité de degrés à l'orientation pendant sa durée de vie. Cette valeur peut être au minimum de (+/-) 0,01.

Vous pouvez également définir le facteur "wiggle", comme pour les autres fonctions de particules. Il s'agit d'une valeur qui sera ajoutée ou soustraite aléatoirement à l'orientation à chaque étape de la vie des particules. Évidemment, les valeurs les plus grandes sont plus prononcées que les plus petites, et cette valeur peut même être négative, la plage maximale étant comprise entre -20 et 20.

Enfin, vous pouvez choisir que l'orientation soit relative ou non, ce qui signifie que si la particule a une direction (et une vitesse), la particule sprite sera orientée autour de ce vecteur. Gardez à l'esprit que si vous avez réglé la vitesse de la particule sur la réduction et qu'elle atteint 0, le manque de vitesse fixe la direction à la valeur par défaut de 0° et donc une orientation relative fera "sauter" la particule sprite à un angle différent.

Syntaxe :

part_type_orientation( ind, ang_min, ang_max, ang_incr, ang_wiggle, ang_relative );

Arguments Type Description
ind Type de particule ID The index of the particle type to change.
ang_min Real The minimum starting angle of the particles.
ang_max Real The maximum starting angle of the particles.
ang_incr Real The increase (or decrease if set negative) of the angle in degrees per step.
ang_wiggle Real The amount the angle wiggles clockwise/counterclockwise per step.
ang_relative Real Whether to set its angle relative to the direction of the particle's motion (true) or not (false).

Les retours :

N/A

Exemple :

part_type_shape(particle2, pt_shape_spark);
part_type_size(particle2, 0.10, 0.50, 0.01, 0);
part_type_scale(particle2, 0.30, 0.30);
part_type_colour1(particle2, 8454143);
part_type_alpha1(particle2, 0.50);
part_type_speed(particle2, 4, 4, -0.07, 1);
part_type_direction(particle2, 0, 359, 0, 20);
part_type_orientation(particle2, 0, 359, 0, 20, 1);
part_type_blend(particle2, 1);
part_type_life(particle2, 1, 5);

Le code ci-dessus définira diverses valeurs de particules, y compris l'orientation qui sera une valeur aléatoire entre 0º et 359º. Une quantité aléatoire de 0 à 20 sera également ajoutée à chaque étape, et l'orientation sera relative à la direction du mouvement.
