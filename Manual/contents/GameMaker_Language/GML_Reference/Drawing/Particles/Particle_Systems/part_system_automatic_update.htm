part_system_automatic_update

Cette fonction contrôle si GameMaker doit mettre à jour le système de particules automatiquement ou non. Normalement, vous n'avez pas besoin d'utiliser cette fonction car la valeur par défaut de true (mise à jour automatique activée) correspond à ce que vous souhaitez. Cependant, pour des effets spéciaux ou pour mettre le jeu en pause, vous pouvez régler cette valeur sur false et cela empêchera toute mise à jour du système de particules donné, à moins qu'elle ne soit explicitement commandée par l'utilisation de la fonction part_system_update() ou si vous utilisez à nouveau cette fonction pour activer les mises à jour automatiques. L'absence de mise à jour signifie que les particules créées dans ce système ne changeront plus de position, de couleur ou de tout autre paramètre et que les émetteurs cesseront également de fonctionner, "gelant" le système au point exact où la mise à jour automatique a été réglée sur false.

Une chose à noter est que si vous utilisez les effets simples créés par les fonctions effect_create_above() ou effect_create_below() alors vous pouvez utiliser les valeurs 0 (pour les effets inférieurs) ou 1 (pour les effets supérieurs) comme index du système de particules et ainsi activer la mise à jour automatique pour ceux-ci également (cela fonctionnera également pour activer la mise à jour pour les effets de particules de GML Visual).

Syntaxe :

part_system_automatic_update(ind, automatic);

Arguments Type Description
ind ID du système de particules The index of the particle system to change.
automatic Booléen Whether automatic updating is on (true) ou non (faux).

Les retours :

N/A

Exemple :

if global.Pause
{
    part_system_automatic_update(global.Sname, false);
}

Le code ci-dessus désactivera les mises à jour des particules si la variable globale "Paused" est vraie.
