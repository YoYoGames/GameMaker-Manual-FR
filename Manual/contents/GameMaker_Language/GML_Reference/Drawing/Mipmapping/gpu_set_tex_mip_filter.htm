gpu_set_tex_mip_filter

Cette fonction vous permet de définir le mode de filtrage actuel de l'IAM sur l'un des trois types pris en charge. Vous donnez la constante qui se réfère au mode de filtrage mip dont vous avez besoin, où vous pouvez choisir entre un filtrage ponctuel (réglage par défaut, signifiant aucun filtrage), un filtrage linéaire (également connu sous le nom de filtrage trilinéaire) ou un filtrage aniostropique (voir le tableau des constantes ci-dessous).

Constant Description
tf_point Cela désactivera le filtrage entre les niveaux de mipmap, ce qui peut provoquer des transitions de texture visibles, mais donne les meilleures performances.
tf_linear Cela permettra d'activer le filtrage linéaire entre les niveaux de mipmap (également connu sous le nom de filtrage trilinéaire), qui lisse les transitions de texture, mais cela aura un impact mineur sur les performances.
tf_anisotropic Cela permet d'activer le filtrage anisotropique entre les niveaux de mipmap, ce qui améliore grandement la qualité de la transition des textures et peut réduire le flou visible avec les autres modes de filtrage, mais cela a le plus grand impact sur les performances.

Syntaxe :

gpu_set_tex_mip_filter(filter);

Arguments Type Description
filter Constante du filtre Mipmapping The mip filter mode to use (a constant, default: tf_point).

Les retours :

N/A

Exemple :

if keyboard_check(vk_enter)
{
    switch(gpu_get_tex_mip_filter())
    {
        case tf_point: gpu_set_tex_mip_filter(tf_linear); break;
        case tf_linear: gpu_set_tex_mip_filter(tf_anisotropic); break;
        case tf_anisotropic: gpu_set_tex_mip_filter(tf_point); break;
    }
}

Le code ci-dessus vérifie le clavier et si la touche spécifiée est pressée, il récupère le filtre mip actuel et passe à la valeur suivante, en passant par les différents modes.
