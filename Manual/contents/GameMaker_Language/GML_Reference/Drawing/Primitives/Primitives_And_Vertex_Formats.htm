Primitives et formats de sommets

Un primitive est un type de dessin réalisé à l'aide de points pour définir ce qui est affiché à l'écran. Ils peuvent être constitués de points, de lignes ou de triangles et peuvent être colorés, avoir un mélange alpha et même une texture, ce qui signifie qu'ils peuvent être utilisés pour créer des effets très intéressants dans un jeu en 2D ! Par exemple, vous pouvez créer un rectangle à l'aide de plusieurs triangles primitives, puis le texturer avec un drapeau sprite et, dans l'événement de dessin d'une instance, faire "onduler" ce drapeau dans le vent et déformer le sprite que vous avez utilisé en modifiant simplement les points qui forment le primitive.

ATTENTION ! Ces fonctions ne fonctionnent pas avec le module HTML5 à moins que vous n'ayez activé OpenGL dans les options du jeu.

Cela peut accélérer considérablement les opérations sur shader ou être utilisé pour étendre leurs capacités et créer des effets surprenants. IMPORTANT ! Le site primitives que vous construisez doit suivre le format que vous avez défini à l'aide des fonctions de format de sommet. Ainsi, si vous avez défini un format de sommet avec uniquement des données positionnelles, il est inutile de construire votre site primitive avec des données de couleur. Vous devez noter que l'ordre dans lequel vous ajoutez les propriétés à l'adresse primitive que vous construisez est défini par l'ordre dans lequel vous avez ajouté ces propriétés lors de la création du format de sommet. Ainsi, si vous avez défini le format vectoriel avec l'ordre position, couleur et coordonnée de texture, vous devez ajouter ces propriétés à l'adresse primitive en cours de construction dans le même ordre, sinon vous obtiendrez des erreurs.
