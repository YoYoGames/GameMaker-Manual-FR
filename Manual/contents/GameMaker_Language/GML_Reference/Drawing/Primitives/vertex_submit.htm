soumettre le vertex_submit

Vous pouvez utiliser cette fonction pour soumettre le contenu d'un sommet buffer au pipeline graphique pour qu'il soit utilisé avec un shader. Vous fournissez l'index buffer à utiliser, le type de base primitive à utiliser (voir les constantes ci-dessous) et la texture qui doit être utilisée. Le type de base primitive est uniquement utilisé pour affecter l'ordre dans lequel les sommets que vous avez stockés dans buffer sont dessinés et connectés, mais les données réelles utilisées pour chacun des sommets seront celles que vous avez définies lors de la création du sommet buffer.

REMARQUE: Cette fonction ne peut être utilisée que dans l'événement "Draw".

Pour un exemple visuel des différentes bases disponibles primitives, voir l'image ci-dessous :

Constant Description
pr_pointlist Un primitive constitué d'une liste de points.
pr_linelist Un primitive composé d'une liste de lignes individuelles.
pr_linestrip Un primitive composé d'une bande de lignes consécutives.
pr_trianglelist Un primitive composé de triangles individuels dans une liste.
pr_trianglestrip Un primitive composé d'une bande consécutive de triangles.

Syntaxe :

vertex_submit(buffer, primitive, texture);

Arguments Type Description
buffer Vertex Buffer ID The buffer to use.
primitive Type primitif Constante The primitive base type.
texture Pointeur de texture The texture to use (or -1 for no texture).

Les retours :

N/A

Exemple :

shader_set(shader_prim);
vertex_submit(buff, pr_trianglelist, sprite_get_texture(sprite_index));
shader_reset();

Le code ci-dessus soumettra le vertex buffer indexé dans la variable "buff" pour être dessiné avec le shader "shader_prim", en utilisant une liste de triangles comme base primitive et la texture du sprite pour l'instance qui exécute le code.
