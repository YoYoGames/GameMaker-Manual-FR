gpu_set_blendmode_ext

Lorsque GameMaker dessine un pixel, il y a une couleur source (la couleur du pixel que l'on va dessiner) et une couleur de destination (la couleur qui se trouve déjà dans le pixel que l'on dessine). Ainsi, pour déterminer la nouvelle couleur du pixel, les couleurs source et de destination sont calculées en fonction du mode de fusion choisi. Chaque composante des couleurs est stockée sous forme de valeur à virgule flottante comprise entre 0 et 1, et la nouvelle couleur est calculée en multipliant chaque composante de la couleur source par un certain facteur et en multipliant chaque composante de la couleur de destination par un autre facteur, puis en additionnant les résultats composante par composante.

Cette fonction vous permet de définir les différents composants qui doivent être pris en compte pour créer un mode de fusion personnalisé. La source et la destination ont toutes deux une composante rouge, verte, bleue et alpha, et dans le tableau suivant, les RGBA de la source sont considérés comme (Rs, Gs, Bs, As) tandis que ceux de la destination sont (Rd, Gd, Bd, Ad). Les onze constantes disponibles peuvent être soit la source, soit la destination (ou les deux) lorsqu'elles sont utilisées dans cette fonction.

Constant Facteur de mélange (rouge, vert, bleu, alpha)
bm_zero (0, 0, 0, 0)
bm_one (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-R, 1-G, 1-B, 1-A)
bm_src_alpha (As, As, As, As)
bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As, 1-As)
bm_dest_alpha (Ad, Ad, Ad, Ad, Ad)
bm_inv_dest_alpha (1-Ad, 1-Ad, 1-Ad, 1-Ad, 1-Ad)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) où f = min(As, 1-Ad)

IMPORTANT ! HTML5 sans WebGL activé ne sera pas en mesure d'afficher correctement les modes suivants :

  • bm_src_colour
  • bm_inv_src_colour
  • bm_dest_colour
  • bm_inv_dest_colour
  • bm_src_alpha_sat

Notez que vous pouvez soit fournir deux arguments individuels à cette fonction, soit fournir un tableau d'arguments (tel que retourné par la fonction gpu_get_blendmode_ext() par exemple). Si vous fournissez un tableau, il doit comporter les deux éléments suivants :

  • [0] = Mode de fusion de la source (par défaut, bm_src_alpha)
  • [1] = Mode de fusion de destination (par défaut, bm_inv_src_alpha)

Pour vous aider à tirer le meilleur parti des modes de fusion et à comprendre comment ils fonctionnent et comment ils affectent l'image finale dessinée à l'écran, nous vous recommandons de lire le guide suivant :

  • Guide d'utilisation des Blendmodes

Syntaxe :

gpu_set_blendmode(src, dest);

Arguments Type Description
src Facteur de mode de mélange Constant Source blend mode factor (see constants above).
dest Facteur de mode de mélange Constant Destination blend mode factor (see constants above)

Les retours :

N/A

Exemple :

gpu_set_blendmode_ext(bm_src_alpha, bm_one);
draw_circle_colour(100, 100, 50, c_white, c_black, 0);
gpu_set_blendmode(bm_normal);

Cela transformera le noir en transparence, créant un effet de " lueur " dû au fait que le blanc est fort à l'extérieur et s'affaiblit progressivement à partir du centre du cercle. Le mode de fusion est réinitialisé après le tracé du cercle afin de s'assurer que la fusion additive n'est pas également appliquée à tout ce qui suit.
