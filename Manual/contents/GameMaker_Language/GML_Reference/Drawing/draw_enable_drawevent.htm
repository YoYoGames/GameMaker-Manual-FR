draw_enable_drawevent

Avec cette fonction, vous pouvez choisir d'activer (true) ou de désactiver (false) l'événement de dessin pour toutes les instances du jeu, vous donnant ainsi le contrôle sur comment et quand les choses sont dessinées, ce qui est utile si vous souhaitez implémenter une technique de "saut de trame". Notez que cela n'empêche pas seulement les instances de s'afficher à l'écran, mais supprime complètement l'événement draw, ce qui signifie qu'il faut être prudent car toute logique de jeu présente dans cet événement ne sera pas exécutée non plus.

Une chose importante à comprendre au sujet de cette fonction est que si vous l'appelez juste au début du jeu, avant que l'image initiale ne soit rendue (c'est-à-dire : l'événement de création du premier objet dans la première pièce du jeu), la fenêtre de jeu ne sera pas rendue. Cela pourrait être utile pour les projets multi-joueurs qui nécessitent un serveur dédié qui n'a pas besoin d'effectuer de rendu. Cependant, gardez à l'esprit que cela ne rend pas le jeu headless car GameMaker ne le supporte pas - vous ne pourrez donc pas l'exécuter sur un serveur qui n'a pas d'interface graphique.

Syntaxe :

draw_enable_drawevent(enable);

ArgumentsType Description
Enable Set to true ou false.

Les retours :

Exemple :

frame_skip ++;
if (frame_skip mod 5) == 0
{
    draw_enable_drawevent(true);
}
else
{
    draw_enable_drawevent(false);
}

Le code ci-dessus vérifie une variable et s'il retourne true, il active l'événement de dessin, sinon l'événement de dessin est désactivé.
