ini_open

Cette opération ouvre un fichier ini_file pour la lecture et l'écriture. Si le fichier ini_file n'existe pas à l'emplacement que vous vérifiez, GameMaker peut en créer un, mais seulement si vous y écrivez des données. Si vous n'avez fait que lire les informations du fichier ini, alors les valeurs par défaut de la fonction de lecture seront renvoyées, mais le fichier ini ne sera pas réellement créé.

Veuillez noter que vous ne pouvez avoir qu'un seul fichier ini ouvert à la fois et n'oubliez pas d'utiliser la commande ini_close() une fois que vous avez fini de lire/écrire dans le fichier .ini, car les informations ne sont pas réellement stockées sur le disque avant ce moment (elles sont également stockées en mémoire jusqu'à la fermeture du fichier). disque (elles sont également stockées en mémoire jusqu'à la fermeture du fichier).

AVERTISSEMENT ! Cette fonction peut ne pas fonctionner comme vous l'attendez, car GameMaker est en sandbox ! Veuillez consulter la section sur le système de fichiers pour plus d'informations.

Syntaxe :

ini_open(name);

ArgumentsType Description
name The filename for the .ini file.

Les retours :

Exemple :

ini_open("Settings/savedata.ini");
score = ini_read_real("save1", "score", 0);
ini_close();

Cela ouvrira 'savedata.ini' et lira la valeur du score dans la section "save1" avec la clé "score", puis refermera le .ini. S'il n'y a pas de valeur sous "save1", "score" ou si le fichier "savedata.ini" n'existe pas n'est présent, score sera mis à 0 (la valeur par défaut). Notez que le fichier ini a été placé dans le sous-répertoire "Settings", qui est le dossier qui contient le fichier ini dans les fichiers inclus dans l'Asset Browser.
