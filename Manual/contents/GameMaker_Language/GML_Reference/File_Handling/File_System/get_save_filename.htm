get_save_filename

Cette fonction ouvre un dialogue et demande au joueur un nom de fichier à sauvegarder avec le filtre donné. Le filtre a la forme "nom1|mask1|nom2|mask2|... ", où le masque peut contenir les différentes options avec un point-virgule entre elles et vous pouvez également utiliser un "*" pour signifier n'importe quelle chaîne. Par exemple : "bitmaps|*.bmp;*.wmf", demanderait à l'utilisateur de sélectionner le nom d'un fichier bitmap du système, mais seulement ceux qui ont les deux extensions spécifiées (bmp et png). Cette fonction ne sauvegarde pas le fichier lui-même ! Elle renvoie seulement une chaîne avec le nom complet et le chemin d'accès du fichier. Si l'utilisateur appuie sur "Annuler", une chaîne vide "" est retournée.

Une chose importante à noter lorsque vous utilisez cette fonction est qu'elle vous accorde certaines permissions pour ce fichier, pour la durée de votre jeu. Ainsi, une fois que vous avez obtenu le chemin d'accès au fichier de la part de l'utilisateur, vous pouvez l'envoyer à access encore et encore sans avoir à le demander, et l'autorisation s'applique aussi bien à la lecture qu'à l'écriture.

NOTE: Cette fonction n'est valable que sur les cibles Windows et macOS.

Syntaxe :

get_save_filename(filter, fname);

ArgumentsType Description
filter The file type filter and/or name.
fname The suggested filename to use.

Les retours :

Exemple :

var file;
file = get_save_filename("screenshot|*.png", "");
if file != ""
{
    screen_save(file);
}

Le code ci-dessus demandera à l'utilisateur de sélectionner un nom de fichier d'image à sauvegarder, puis vérifiera le nom et le chemin du fichier renvoyé, et s'il ne s'agit pas d'une chaîne vide, il prendra une capture d'écran et la sauvegardera en utilisant le nom donné.
