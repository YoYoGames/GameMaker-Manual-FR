fichier_copie

Cette fonction copiera le fichier spécifié, en lui donnant le nouveau nom que vous aurez choisi.

Si vous copiez un fichier sur Android à partir de votre zone de fichiers inclus ( qui se trouve dans votre APK), il sera d'abord copié dans la mémoire. Il faut donc savoir que la copie de fichiers très volumineux à partir de l'APK peut entraîner un manque de mémoire sur les appareils. Sur la cible HTML5, nous utilisons l'API de stockage local HTML5 pour l'enregistrement côté client, et les fonctions ne prennent pas en charge les opérations de fichiers binaires, donc suppose que vous copiez un fichier texte.

La fonction échouera si le fichier newname existe déjà.

AVERTISSEMENT ! Cette fonction peut ne pas fonctionner comme vous l'attendez en raison de la présence d'un bac à sable dans GameMaker! Veuillez consulter la section sur les limites du système de fichiers pour plus d'informations.

Syntaxe :

file_copy(fname, newname);

ArgumentsType Description
fname The name of the file to copy.
newname The name of the new, copied file.

Les retours :

N/A

Exemple :

if file_exists("level1.txt")
{
    file_copy("level1.txt", "level2.txt");
}

Cela permet de rechercher un fichier et s'il existe, il est copié.
