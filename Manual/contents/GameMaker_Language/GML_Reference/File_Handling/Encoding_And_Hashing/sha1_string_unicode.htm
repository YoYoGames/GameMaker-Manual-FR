sha1_string_unicode

En cryptographie, SHA-1 est une fonction de hachage cryptographique conçue par l'Agence nationale de sécurité des États-Unis. Elle est employée dans plusieurs applications et protocoles largement utilisés, comme le célèbre Git, où elle sert à vérifier les modifications apportées aux fichiers. Cette fonction prend une chaîne unicode en entrée et renvoie un résumé de message de 160 bits au format ASCII, par exemple :

sha1 input string = "The quick brown fox jumps over the lazy dog"
sha1 output hash = "2fd4e1c6 7a2d28fc ed849ee1 bb76e739 1b93eb12"

De cette façon, vous pouvez générer une clé sécurisée qui peut être stockée et utilisée pour vérifier l'intégrité des informations envoyées à (ou reçues de) un serveur externe (par exemple).

NOTE: Il existe deux formats pour l'encodage sha1, UTF-8 et unicode. Les deux sont fournis pour faciliter la communication avec différentes configurations de serveur, mais le plus courant à utiliser est unicode.

NOTE: SHA-1 n'est pas complètement sécurisé et peut être cassé. Voir cette page pour plus d'informations.

Syntaxe :

sha1_string_unicode(string)

ArgumentsType Description
string The string to hash.

Les retours :

Exemple :

var hash, str;
str = base64_encode(game_data);
hash = sha1_string_unicode(str);
http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?hash=" + hash); http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?data=" + str);

Le code ci-dessus va coder en base64 une chaîne de caractères, puis générer un hachage sha1. Enfin, le hachage et la chaîne encodée sont envoyés à un serveur.
