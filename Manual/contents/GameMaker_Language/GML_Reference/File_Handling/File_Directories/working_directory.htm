répertoire de travail

Working_directory peut en fait renvoyer deux valeurs différentes selon l'usage que vous en faites.

Si vous écrivez un fichier sur le disque, working_directory pointe vers la zone de stockage local réservée à votre jeu sur le périphérique cible (dans Windows, il s'agit de "%LOCALAPPDATA%/gamedir/" où le " gamedir " est le répertoire portant le nom de votre jeu).

Toutefois, lors de la lecture du disque, working_directory peut être soit le stockage local (game_save_id) , soit la zone où sont stockés les fichiers inclus.

NOTE La variable working_directory renverra le chemin d'accès incluant la barre oblique inverse finale.

Par exemple, si vous avez un fichier par défaut *.txt inclus dans votre jeu et que vous le lisez en mémoire à l'aide de working_directory, ce fichier sera pris dans la zone où sont stockés les fichiers inclus. Si vous choisissez ensuite d'écrire ces informations dans un fichier texte à l'aide de working_directory, le fichier sera écrit dans la mémoire locale de la plate-forme cible, où il sera lu la fois suivante, à moins que vous ne demandiez à GameMaker de supprimer le fichier, auquel cas working_directory indiquera à nouveau la zone où sont stockés les fichiers inclus.

Il convient de noter qu'en général, il n'est pas nécessaire d'utiliser cette fonction et qu'il suffit de spécifier le nom du fichier lui-même (plus toute information supplémentaire sur le chemin), ce qui est suffisant. Par exemple, ceci :

file_text_open_read(working_directory + "my_file.txt");

serait mieux fait comme ça :

file_text_open_read("my_file.txt");

AVERTISSEMENT Il se peut que cette fonction ne fonctionne pas comme vous l'attendez, car GameMaker est protégé par un bac à sable ! Veuillez consulter la section sur le système de fichiers pour plus d'informations.

Syntaxe :

working_directory

Les retours :

String

Exemple :

ini_open(working_directory + "temp_ini.ini");

Cela ouvrira un fichier ini dans le répertoire de travail du jeu (en le créant s'il n'existe pas déjà). Il peut s'agir du stockage local, ou de la zone où se trouvent les fichiers inclus, selon que le fichier recherché existe ou non à l'un ou l'autre endroit.
