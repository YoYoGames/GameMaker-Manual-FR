file_text_close

Une fois que vous avez fini de travailler avec un fichier donné (que ce soit en le lisant ou en l'écrivant), vous devez refermer le fichier, sinon vous risquez de perdre les informations qu'il contient. Cela permet également d'éviter les fuites de mémoire et de s'assurer que vous ne dépassez jamais la limite de fichiers en ayant plus de 32 fichiers ouverts.

Syntaxe :

file_text_close(fileid);

ArgumentsType Description
fileid The id of the file to close.

Les retours :

Exemple :

var file = file_text_open_write(working_directory + "Game_Data.txt");
while (!file_text_eof(file))
{
    file_text_readln(file);
}
file_text_write_string(file, level_data);
file_text_close(file);

Le code ci-dessus ouvre un fichier pour l'écrire, puis passe en boucle les lignes de texte déjà écrites dans le fichier jusqu'à ce qu'il atteigne la fin. À ce stade, il écrit une chaîne de caractères, puis referme le fichier.
