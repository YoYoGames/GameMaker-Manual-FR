Types d'expiration de la source de temps

Constante d'expiration de la source de temps
Constant Description Valeur
time_source_expire_nearest La source de temps expirera sur la trame la plus proche de son heure d'expiration. 0
time_source_expire_after La source de temps expirera sur la première image après son temps d'expiration. 1

Les sources de temps qui utilisent les secondes comme unité n'expirent pas toujours sur une image de jeu. Elles peuvent expirer après qu'une trame de jeu ait été traitée mais avant qu'une nouvelle trame puisse commencer à être traitée.

Cependant, la fonction de rappel d'une source de temps ne peut être exécutée que sur une image de jeu. Le type d'expiration d'une source de temps contrôle donc le moment où sa fonction de rappel est exécutée, au cas où elle expire au milieu de deux images.

Lorsque vous utilisez time_source_expire_nearest, le callback d'une source de temps sera exécuté sur la trame la plus proche de son heure d'expiration, qui peut être la trame précédente ou la trame suivante.

Lorsque vous utilisez time_source_expire_after, le callback d'une source de temps sera toujours exécuté sur la trame après son temps d'expiration.
