création_source_temps

Cette fonction crée une nouvelle source de temps. Lisez Sources de temps pour une vue d'ensemble.

La création d'une nouvelle source de temps ne la lance pas automatiquement ; vous devez appeler time_source_start() pour l'activer.

Une nouvelle source de temps ne s'exécute qu'une seule fois, à moins que l'argument répétitions ne soit spécifié (voir ci-dessous).

Vous devez détruire une Source de temps en utilisant time_source_destroy() lorsque vous n'en avez plus besoin.

Arguments

Voici la répartition des arguments de la fonction :

Source du temps des parents

Il s'agit de la source de temps parente qui contrôle la nouvelle source de temps. Il peut s'agir de time_source_global, time_source_game ou d'une source temporelle personnalisée qui existe déjà.

Voir : Sources temporelles intégrées

Période

Il s'agit d'une période de temps qui peut être exprimée en secondes ou en images, selon l'unité spécifiée dans l'argument suivant.

Lorsque l'unité utilisée est l'image, la période doit être un nombre entier. Les valeurs non entières sont arrondies à l'unité inférieure, à l'exception des valeurs inférieures à 1, qui sont arrondies à l'unité supérieure.

Unité

C'est l'unité dans laquelle la période est exprimée, et peut être time_source_units_seconds ou time_source_units_frames.

Voir : Unités de source de temps

Vous pouvez utiliser une valeur de battements par minute en utilisant time_bpm_to_seconds().

Rappel et arguments

Vous devez spécifier un "callback" (rappel) method qui sera appelé lorsque la source de temps expirera.

Vous pouvez éventuellement spécifier un tableau contenant les arguments qui doivent être passés à la méthode lorsqu'elle est appelée.

Le tableau lui-même ne sera pas transmis à la méthode, mais chaque élément du tableau sera transmis comme un argument distinct.

Par exemple, si votre fonction attend les arguments (x, y, objet), alors votre tableau d'arguments peut ressembler à ceci : [30, 600, obj_player].

Répétitions

Vous pouvez optionnellement spécifier combien de fois votre source de temps doit se répéter. Une valeur de 1 signifie qu'elle ne s'exécute qu'une seule fois, ce qui est la valeur par défaut.

Vous pouvez spécifier le nombre total de fois qu'il doit se répéter, ou -1 pour qu'il se répète indéfiniment.

Par exemple, si vous réglez cette valeur sur 3 et que la période de votre source de temps est de "4 secondes", la source de temps sera active pendant un total de 12 secondes et appellera la méthode de rappel toutes les 4 secondes.

Type d'expiration

Il peut s'agir de la date d'expiration la plus proche ou de la date d'expiration ultérieure.

Voir : Types d'expiration des sources de temps

Syntaxe :

time_source_create(parent, period, units, callback, [args, repetitions, expiry_type]);

Arguments Type Description
parent ID de la source de temps The parent Time Source qui contrôle la nouvelle source de temps
period Real The period that the Time Source fonctionne pour, dans les unités données
units Source de temps Unité Constante The units that the given period is in
callback Méthode ou Fonction de script The method or script function to call when the Time Source expirant à
args Array OPTIONAL Un tableau contenant les arguments à passer dans la méthode
repetitions Real OPTIONAL Le nombre de fois que la source temporelle doit se répéter, ou -1 pour une répétition indéfinie.
expiry_type Constante d'expiration de la source de temps OPTIONAL Si la source de temps expire sur la trame la plus proche de son expiration, ou sur la trame suivante.

Les retours :

Time Source ID

Exemple 1 :

var _my_method = function()
{
    instance_destroy();
}

var _time_source = time_source_create(time_source_game, 300, time_source_units_frames, _my_method);

time_source_start(_time_source);

Dans cet exemple, nous voulons que l'instance se détruise 300 images plus tard.

Le code crée d'abord une méthode qui appelle simplement instance_destroy().

Il crée ensuite une source de temps, héritant de la source de temps du jeu. Il fixe sa période à 300 images.

Enfin, il lance la source de temps.

Exemple 2 :

var _my_method = function()
{
    show_debug_message("A second has passed!");
}

global.time_per_second = time_source_create(time_source_game, 1, time_source_units_seconds, _my_method, [], -1, time_source_expire_after);

time_source_start(global.time_per_second);

Dans cet exemple, nous créons une source de temps globale qui expire une fois par seconde. Ce code serait placé à la racine d'un fichier . script.

Le code crée d'abord une méthode qui imprime un message dans le journal de sortie, disant "Une seconde est passée !".

Il crée ensuite une nouvelle source de temps, héritant de la source de temps du jeu. Elle fixe sa période à 1 seconde.

La méthode est passée dans la source de temps afin qu'elle puisse être appelée à chaque fois qu'elle expire. Un tableau vide est donné pour les arguments.

Le nombre de répétitions est fixé à -1, de sorte que la source de temps ne s'arrête jamais et se répète indéfiniment. Son type d'expiration est défini de manière à ce que le callback soit exécuté sur la première image après son expiration.

Enfin, la source de temps est lancée.
