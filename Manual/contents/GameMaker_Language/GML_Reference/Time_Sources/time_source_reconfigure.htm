temps_source_reconfigurer

Cette fonction est utilisée pour modifier les propriétés de base d'une source de temps, sans avoir à en créer une entièrement nouvelle.

Vous spécifiez une source de temps existante, puis vous définissez les propriétés qui sont également définies dans la section time_source_create()à l'exception du parent. Lisez cette page pour obtenir des informations détaillées sur ces propriétés.

La source de temps spécifiée sera réinitialisée et désactivée, et devra être redémarrée.

Syntaxe :

time_source_reconfigure(id, period, units, callback, [args, repetitions, expiry_type]);

Arguments Type Description
id ID de la source de temps The Time Source pour reconfigurer
period Real The period that the Time Source fonctionne pour, dans les unités données
units Source de temps Unité Constante The units that the given period is in
callback Méthode The method to call when the Time Source expirant à
args Array OPTIONAL Un tableau contenant les arguments à passer dans la méthode
repetitions Real OPTIONAL Le nombre de fois que la source temporelle doit se répéter, ou -1 pour une répétition indéfinie.
expiry_type Constante d'expiration de la source de temps OPTIONAL Si la source de temps expire sur la trame la plus proche de son expiration, ou sur la trame suivante.

Les retours :

N/A

Exemple :

function change_spawn_delay(new_delay)
{
    time_source_reconfigure(obj_game.spawn_time_source, new_delay, time_source_units_frames, obj_game.spawn_method, [], -1);
    time_source_start(obj_game.spawn_time_source);
}

Cela crée une nouvelle fonction qui modifie le délai d'apparition des ennemis dans le jeu.

En supposant que le jeu utilise une source de temps appelée obj_game.spawn_time_source pour faire apparaître les ennemis, cette source de temps devra être mise à jour lorsque le délai d'apparition des ennemis sera modifié.

C'est exactement ce que fait cette fonction, qui reconfigure la source de temps avec le nouveau délai, puis la relance.
