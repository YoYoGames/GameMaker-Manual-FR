curseur_sprite

En définissant cette variable, vous demanderez à GameMaker d'utiliser l'adresse sprite comme curseur (en fait, il s'agit de la position x/y actuelle de la souris à chaque étape). La valeur par défaut est -1, ce qui signifie qu'il n'y a pas de sprite pour le curseur, mais vous pouvez attribuer n'importe quel indice sprite à partir des actifs du jeu ou qui a été importé d'une ressource externe. Veuillez noter qu'il n'y a aucun moyen de contrôler la vitesse de l'animation ou l'index d'image, donc si le site sprite comporte des sous-images, celles-ci seront animées à la même vitesse que celle de la pièce. Pour supprimer le curseur sprite, vous pouvez redonner à cette variable la valeur -1.

Il convient également de noter que cette variable ne remplace pas le curseur de la fenêtre de jeu, et qu'il sera toujours dessiné normalement. Pour éviter cela, vous pouvez utiliser la fonction window_set_cursor() et la définir sur la constante cr_none, ce qui rendra le curseur standard invisible.

Syntaxe :

cursor_sprite;

Les retours :

Sprite Asset

Exemple :

cursor_sprite = spr_CustomCursor;

Le code ci-dessus définira le sprite indexé dans la variable "spr_CustomCursor" comme étant le curseur sprite pour le jeu.
