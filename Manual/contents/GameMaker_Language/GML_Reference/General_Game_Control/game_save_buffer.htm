game_save_buffer

Il s'agit d'une variante de la fonction game_save(). Veuillez donc d'abord lire sa page car elle contient des informations importantes relatives à son utilisation et à celle de cette fonction.

Avec cette fonction, vous pouvez sauvegarder l'état actuel du jeu dans un "grow" buffer créé au préalable (voir Buffers) qui peut ensuite être rechargé à tout moment en utilisant la fonction game_load_buffer() fonction. Cette fonction est conçue pour être utilisée dans une seule pièce à la fois (c'est-à-dire qu'il n'est pas conseillé de faire une sauvegarde de buffer dans une pièce, puis d'essayer de charger le buffer depuis une autre pièce) et ne devrait idéalement être utilisée que pour les points de contrôle ou les redémarrages de niveau.

REMARQUE: Cette fonction est très limitée et est conçue pour les débutants qui souhaitent mettre en place un système de points de contrôle rapidement, mais les utilisateurs plus avancés préféreront peut-être coder leur propre système en utilisant les fonctions du fichier, car le jeu ne sauvegardera pas les ressources dynamiques telles que les structures de données, les surfaces, les ajouts à sprites, etc.

Syntaxe :

game_save_buffer(buffer);

ArgumentsType Description
buffer The buffer id to save to.

Les retours :

Exemple :

if keyboard_check_pressed(ord("S"))
{
    global.Checkpoint = true;
    game_save_buffer(save_buff);
}

Le code ci-dessus définira une variable globale à true puis sauvegardera l'état actuel du jeu sur buffer indexé dans la variable "save_buff" lorsque la touche "S" sera pressée.
