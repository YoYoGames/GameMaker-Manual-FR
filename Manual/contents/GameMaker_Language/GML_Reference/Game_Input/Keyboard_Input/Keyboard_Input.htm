Entrée du clavier

Lorsque vous manipulez le clavier dans GameMaker, vous disposez d'une variété de fonctions qui peuvent être utilisées pour reconnaître les différents états du clavier, comme enfoncé ou relâché. Il y en a aussi qui stockent toutes les pressions de touches sous forme de chaîne ou qui peuvent vous dire quelle était la dernière touche pressée, ainsi que d'autres qui vous permettent d'effacer complètement l'état du clavier.

NOTE Ces fonctions sont conçues pour les plateformes de bureau Windows/Mac/Ubuntu uniquement. Il se peut que certaines des variables et constantes intégrées ne soient pas valables sur d'autres plateformes et que de nombreuses fonctions ne fonctionnent pas sur les mobiles.

Chaque caractère d'entrée d'une touche (ou de plusieurs touches) est défini par son code UTF8, qui est une valeur numérique. Cette valeur peut être récupérée pour n'importe quel caractère à l'aide de la fonction ord() mais, GameMaker dispose également d'une série de constants pour les touches spéciales du clavier les plus utilisées et une fonction spéciale. Typiquement, vous utiliserez une combinaison de ord() avec les fonctions de keyboard_check*(), quelque chose comme ceci :

if keyboard_check(ord("A"))
{
    hspeed = -5;
}

Ainsi, la fonction ci-dessus vérifie la touche "A" et si elle est enfoncée, elle règle la vitesse horizontale de l'objet sur -5. Notez que l'utilisation de ord() de cette manière ne fonctionnera correctement que si la chaîne d'entrée ne comporte qu'un seul caractère et est un nombre de 0 à 9 ou un caractère romain majuscule de A à Z. La fonction ord() renverra une valeur UTF8 complète, mais les fonctions keyboard_check*() ne détecteront que A - Z et 0 - 9. Mais que faire si vous voulez utiliser les touches fléchées ? Ou si vous voulez modifier une action en utilisant la touche "shift" ? Eh bien, pour cela, GameMaker dispose d'une série de constantes vk_ (vk_ signifie " touche virtuelle") que vous pouvez utiliser à la place de ord():

Constante de la clé virtuelle (vk_*)
Constant Description
vk_nokey keycode représentant qu'aucune touche n'est pressée
vk_anykey code de touche représentant l'appui sur une touche quelconque
vk_left code de la touche flèche gauche
vk_right code de la touche flèche droite
vk_up code de la touche flèche vers le haut
vk_down code de la touche flèche vers le bas
vk_enter touche Entrée
vk_escape touche d'échappement
vk_space touche espace
vk_shift l'une ou l'autre des touches majuscules
vk_control l'une des touches de contrôle
vk_alt touche alt
vk_backspace touche retour arrière
vk_tab touche de tabulation
vk_home touche d'accueil
vk_end touche finale
vk_delete touche de suppression
vk_insert insérer la clé
vk_pageup touche pageup
vk_pagedown touche pagedown
vk_pause touche pause/rupture
vk_printscreen touche printscreen/sysrq
vk_f1 ... vk_f12 code des touches de fonction F1 à F12
vk_numpad0 ... vk_numpad9 les touches numériques du clavier numérique
vk_multiply touche multiplicatrice du clavier numérique
vk_divide touche de division sur le clavier numérique
vk_add touche d'ajout sur le clavier numérique
vk_subtract touche de soustraction du clavier numérique
vk_decimal les touches de point décimal du clavier numérique

Les constantes suivantes ne peuvent être utilisées qu'avec keyboard_check_direct():

Constante de la clé virtuelle (vk_*)
Constant Description
vk_lshift touche shift gauche
vk_lcontrol touche de commande gauche
vk_lalt touche alt gauche
vk_rshift touche shift droite
vk_rcontrol touche de contrôle droite
vk_ralt touche alt droite

Ce qui suit est un petit exemple de l'utilisation des vk_ constantes :

if keyboard_check_pressed(vk_tab)
{
    instance_create_layer(x, y, "Controllers", obj_Menu);
}

Le code ci-dessus détectera si la touche "Tab" est pressée et créera une instance de l'objet "obj_Menu" si c'est le cas.

Si vous devez vérifier un caractère clé qui n'est pas 0 - 9, A - Z ou l'une des constantes VK, vous devez alors vérifier l'une des variables keyboard_*, comme keyboard_lastchar par exemple :

var _key = keyboard_lastchar;
if ord(_key) == ord("ç")
{
    show_debug_message("ç key pressed");
    keyboard_lastchar = "";
}

Les fonctions suivantes sont disponibles lorsque vous travaillez avec un clavier :

REMARQUE: ces fonctions ne fonctionneront pas si vous utilisez un clavier virtuel à l'écran.

  • io_clear
  • keyboard_check
  • keyboard_check_pressed
  • keyboard_check_released
  • keyboard_check_direct
  • keyboard_clear
  • keyboard_set_map
  • keyboard_get_map
  • keyboard_unset_map
  • keyboard_set_numlock
  • keyboard_get_numlock

Il existe également certaines fonctions qui peuvent être utilisées pour simuler des pressions de touches :

  • keyboard_key_press
  • keyboard_key_release

Enfin, les variables suivantes sont intégrées à GameMaker et peuvent être consultées pour obtenir des informations sur l'état du clavier et les entrées :

NOTE: Lorsque vous utilisez le clavier virtuel, seule la variable keyboard_string sera mise à jour avec l'entrée du clavier.

  • keyboard_key
  • keyboard_lastkey
  • keyboard_lastchar
  • keyboard_string
