clé_virtuelle_add

Cette fonction vous permet d'associer les "touches" d'une zone de l'écran à des événements clavier. Cela signifie qu'une fois que vous avez affecté une zone à une touche virtuelle, tous les touchers de cette zone déclencheront l'événement clavier correspondant à la touche que vous avez affectée à la zone. Vous pouvez également affecter chaque touche virtuelle que vous définissez à une variable, qui peut ensuite être utilisée dans les autres fonctions de touches virtuelles pour les afficher, les masquer et les supprimer. Ces touches sont attribuées par pièce et seront automatiquement supprimées par GameMaker lors d'un changement de pièce.

La position réelle de la clé virtuelle est basée sur la position de l'écran plutôt que sur la position de la pièce et les valeurs x/y sont donc absolues à l'écran. Cela signifie que vous n'avez pas à vous soucier de l'utilisation des vues ou des coordonnées relatives des pièces, et que vous pouvez simplement définir les touches dans l'événement Create d'un objet (vous ne devez définir une touche virtuelle qu'une seule fois par pièce), puis dessiner votre touche sprites dans l'événement Draw GUI, en dimensionnant la couche GUI pour qu'elle soit identique à l'écran.

Syntaxe

virtual_key_add(x, y, w, h, keycode);

Arguments Type Description
x Real The x coordinate (left side) of the virtual key on the screen
y Real The y coordinate (top side) of the virtual key on the screen
w Real The width of the virtual key
h Real The height of the virtual key
keycode Constante de la clé virtuelle (vk_*) Which keyboard key event should be triggered by touching this area

Les retours :

Virtual Key ID

Exemple :

global.Left = virtual_key_add(32, 32, 64, 64, vk_left);

Le code ci-dessus crée une clé virtuelle carrée de 64x64 pixels, positionnée à l'écran à (32, 32) qui déclenchera l'événement vk_left lorsqu'elle sera touchée et attribue l'index de cette clé virtuelle à une variable globale.
