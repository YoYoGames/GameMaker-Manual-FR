buffer_peek

Avec la fonction buffer_read() vous pouvez lire des données sur le site buffer à la position de recherche actuelle, chaque donnée étant lue en avançant cette position des octets lus ou écrits. Cependant, il se peut que vous deviez lire un élément de données donné sans vouloir modifier la position de recherche actuelle, et c'est dans ce cas que vous utiliserez cette fonction. Il vous suffit de fournir à la fonction un identifiant buffer, puis une position de décalage (à partir du début de buffer ) dans ce buffer à lire, ainsi que le type de données que vous souhaitez lire.

REMARQUE: L'utilisation d'un type de données incorrect pour les données lues entraînera des valeurs erronées !

Si la fonction réussit, une valeur (réel, chaîne, booléen, etc...) sera renvoyée en fonction du type de données, tandis qu'un échec renverra simplement 0.

Syntaxe :

buffer_peek(buffer, offset, type);

Arguments Type Description
buffer ID de la mémoire tampon The index of the buffer to use.
offset Real The offset position (in bytes) within the buffer to read the given data from.
type Type de données de la mémoire tampon Constante The type of data that is to be read from the buffer (see the list of constants here).

Les retours :

Real, Boolean, String or 0 (if it fails)

Exemple :

var red = buffer_peek(buff, 1, buffer_u8);
var green = buffer_peek(buff, 2, buffer_u8);
var blue = buffer_peek(buff, 3, buffer_u8);
image_blend = make_colour_rgb(red, green, blue);

Le code ci-dessus obtiendra trois valeurs à partir de trois positions différentes dans le site buffer indexées dans la variable "buff" et utilisera ensuite ces valeurs pour définir le mélange d'images de l'instance.
