tampon_md5

En cryptographie, MD5 (Message-Digest algorithm 5) est une fonction de hachage cryptographique largement utilisée avec une valeur de hachage de 128 bits et a été employée dans une grande variété d'applications de sécurité. Elle est également couramment utilisée pour vérifier l'intégrité des données. Cette fonction prend les données d'entrée d'un site buffer (ou d'une partie du site buffer) et renvoie le hachage MD5 hexadécimal de 32 caractères qui est unique pour ces données. De cette façon, vous pouvez générer une clé sécurisée qui peut être stockée et utilisée pour vérifier l'intégrité des informations envoyées à (ou reçues de) un serveur externe (par exemple). Lorsque vous appliquez cette fonction à buffers, vous devez spécifier l'identifiant buffer du buffer à utiliser. à utiliser, puis une valeur de décalage (en octets) pour le point de départ, et enfin une taille (toujours en octets) pour la région à hacher.

REMARQUE: MD5 n'est pas complètement sécurisé et peut être cassé. Voir cette page pour plus d'informations.

Syntaxe :

buffer_md5(buffer, offset, size);

ArgumentsType Description
buffer The index of the buffer to use.
offset The data offset value.
size The size of the buffer.

Les retours :

Exemple :

check_string = buffer_md5(buff, 0, buffer_get_size(buff));

Le code ci-dessus créera un hachage md5 pour les données complètes stockées dans le site buffer indexé par la variable "buff", et stockera le hachage renvoyé dans la variable "check_string".
