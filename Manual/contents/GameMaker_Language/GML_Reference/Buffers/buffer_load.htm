buffer_load

Cette fonction est utilisée pour charger un buffer qui a été précédemment enregistré à l'aide des fonctions buffer_save() ainsi que tout Fichier Inclus ou fichier chargé en externe. Elle renvoie l'ID du nouveau tampon qui est créé par la fonction sous forme de "grow" buffer (voir ici) avec un alignement d'octets de 1. Cet ID doit être stocké dans une variable et utilisé dans tous les autres appels de fonction à ce buffer. Si le chargement échoue pour une raison quelconque, la fonction renvoie -1 et un message s'affiche dans la fenêtre de sortie du compilateur indiquant que le chargement a échoué.

REMARQUE Il est important que vous supprimiez de la mémoire toute ressource créée dynamiquement comme celle-ci lorsque vous n'en avez plus besoin afin d'éviter les fuites de mémoire. Ainsi, lorsque vous en avez terminé avec le site buffer que vous avez créé, vous devez le libérer à nouveau à l'aide de la commande buffer_delete().

Notes spécifiques à la plate-forme

  • Sur HTML5, tout buffers chargé depuis le stockage local sera décodé à l'aide de base64 (puisqu'il est enregistré en tant que chaîne de caractères base64). Cependant, l'utilisation de cette fonction sur les fichiers inclus ou tout autre fichier externe n'utilisera pas le décodage base64, puisqu'ils ne sont pas censés être dans un format base64.
  • En HTML5, tous les fichiers inclus sont stockés sous forme de texte codé en UTF-8, ce qui peut entraîner une augmentation de la taille du fichier chargé buffer par rapport au fichier original, car la version codée en UTF-8 du fichier peut avoir une taille différente de celle de votre fichier original (en raison du changement de format).

Syntaxe :

buffer_load(filename);

ArgumentsType Description
filename The name of the file to load from.

Les retours :

Buffer ID

Exemple :

player_buffer = buffer_load("Player_Save.sav");

Le code ci-dessus va charger en mémoire un buffer précédemment enregistré, créant ainsi un nouveau buffer. L'index de ce nouveau buffer est stocké dans la variable "player_buffer".
