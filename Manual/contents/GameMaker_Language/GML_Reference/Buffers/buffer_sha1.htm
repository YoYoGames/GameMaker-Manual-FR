buffer_sha1

En cryptographie, SHA-1 est une fonction de hachage cryptographique conçue par l'Agence de sécurité nationale des États-Unis et est employée dans plusieurs applications et protocoles largement utilisés, comme le populaire système Git, où elle est utilisée pour vérifier les modifications de fichiers. Cette fonction prend les données d'entrée d'un buffer et renvoie un condensé de message de 160 bits au format ASCII. De cette façon, vous pouvez générer une clé sécurisée qui peut être stockée et utilisée pour vérifier l'intégrité de l'information envoyée à (ou à l'étranger). de l'information envoyée à (ou reçue de) un serveur externe (par exemple). Lorsque vous appliquez cette fonction à buffers, vous devez spécifier l'identifiant buffer du buffer à utiliser, puis une valeur de décalage (en octets) pour savoir où commencer, et enfin une taille (toujours en octets) pour la région à hacher.

NOTE: SHA-1 n'est pas complètement sécurisé et peut être cassé. Voir cette page pour plus d'informations.

Syntaxe :

buffer_sha1(buffer, offset, size);

ArgumentsType Description
buffer The index of the buffer to use.
offset The data offset value.
size The size of the buffer.

Les retours :

Exemple :

check_string = buffer_sha1(buff, 0, buffer_get_size(buff));

Le code ci-dessus créera un hachage sha1 pour les données complètes stockées dans le site buffer indexé par la variable "buff", et stockera le hachage renvoyé dans la variable "check_string".
