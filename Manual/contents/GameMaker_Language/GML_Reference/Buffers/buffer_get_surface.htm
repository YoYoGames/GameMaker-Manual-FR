tampon_get_surface

Avec cette fonction, vous pouvez écrire des informations d'une surface vers un buffer donné. Le site buffer doit avoir été créé précédemment et doit être un buffer aligné sur 1 octet suffisamment grand pour stocker les données de la surface que vous allez écrire (si vous n'êtes pas sûr, utilisez un tampon de croissance ou consultez l'exemple au bas de cette page).

Les informations relatives à la surface seront toujours écrites au début du site buffer (quelle que soit la position de recherche) mais vous pouvez choisir une valeur de décalage (en octets) pour déplacer la position d'écriture à partir du début du site buffer.

Notez que la fonction écrit chaque pixel de la surface sur le site buffer en utilisant un formatage RGBA sur la cible Windows, mais sur d'autres cibles, il peut être différent selon le système d'exploitation ou même le périphérique.

REMARQUE: cette fonction a été modifiée à partir de la mise à jour 2.3.1 de GameMaker. Les versions précédentes utilisaient la constante buffer_surface_copy, qui est désormais obsolète.

Syntaxe :

buffer_get_surface(buffer, surface, offset);

Arguments Type Description
buffer ID de la mémoire tampon The index of the buffer to use.
surface ID de la surface The index of the surface to use.
offset Real The data offset value (in bytes).

Les retours :

N/A

Exemple :

buffer = buffer_create(surf_width * surf_height * 4, buffer_fixed, 1);
buffer_get_surface(buffer, surface, 0);

Ce code va créer un buffer de taille fixe qui a la capacité de stocker 4 octets par pixel (pour les composantes R, G, B et A) en fonction de la taille exacte de la surface, puis y copier les données de la surface.
