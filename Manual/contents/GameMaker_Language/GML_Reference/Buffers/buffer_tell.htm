buffer_tell

Lorsque vous lisez ou écrivez des données sur un site buffer à l'aide de l'option buffer_read() ou buffer_write() la position de recherche actuelle est avancée par les octets écrits ou lus. cette fonction vous permet d'obtenir la position de recherche actuelle pour l'utiliser dans d'autres fonctions de buffer. Par exemple, si l'alignement de buffer est fixé à 4 octets et que vous écrivez une seule donnée d'un octet, faites un buffer_tell(), Vous obtiendrez une valeur de retour de 1. Cependant, si vous écrivez un autre morceau de données, également d'un octet, et que vous faites un buffer_tell(), vous obtiendrez une valeur de retour de 5, car l'alignement a "rembourré" les données à cette position.

Syntaxe :

buffer_tell(buffer);

ArgumentsType Description
buffer The index of the buffer to use.

Les retours :

Exemple :

var pos = buffer_tell(buff); buffer_seek(buff, buffer_seek_start, 0);
val[0] = buffer_read(buff, buffer_S16);
val[1] = buffer_read(buff, buffer_S16);
val[2] = buffer_read(buff, buffer_S16);
buffer_seek(buff, buffer_seek_start, pos);

Le code ci-dessus stocke la position de recherche actuelle dans le site buffer, indexée dans la variable "buff", dans la variable locale "pos". La position de recherche de buffer sera ensuite fixée au début de buffer, et trois données seront lues dans un tableau, avant que la position de recherche de ne soit enfin rétablie à l'endroit où elle se trouvait précédemment. dans un tableau, avant de réinitialiser la position de recherche de buffer à l'endroit où elle se trouvait précédemment.
