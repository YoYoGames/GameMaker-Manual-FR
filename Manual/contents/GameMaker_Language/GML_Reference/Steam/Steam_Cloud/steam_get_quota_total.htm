steam_get_quota_total

Lorsque vous utilisez le cloud Steam pour stocker et synchroniser des fichiers, vous devez définir le quota d'espace dont votre jeu aura besoin. Ce quota est appliqué à chaque jeu dans le nuage, par utilisateur et par jeu. Ainsi, par exemple, si le quota pour le jeu X est de 1 mégaoctet, chaque compte Steam qui possède le jeu X peut stocker, au maximum, 1 mégaoctet de données associées à ce jeu dans le nuage. Les données stockées par le jeu X n'affecteront pas les autres jeux compatibles avec le cloud que possède l'utilisateur (par exemple, le jeu Y). Le quota par défaut des nouveaux jeux Steamworks est d'un gigaoctet, mais vous pouvez le modifier à partir du panneau de configuration Steamworks de votre jeu.

NOTE: Une fois le quota épuisé , l'écriture de fichiers échouera. Si vous pensez qu'il est possible que le quota soit épuisé pour l'utilisateur de votre jeu, vous devez créer un code pour le gérer, car en ne faisant rien, vous laissez les utilisateurs dans une situation où ils sont incapables de réparer les choses et cela conduira à une mauvaise expérience de votre jeu.

Syntaxe :

steam_get_quota_total();

Les retours :

Real

Exemple :

if steam_is_cloud_enabled_for_app()
{
    quota = steam_get_quota_total();
}

Le code ci-dessus vérifie si le nuage steam est activé pour le jeu et si c'est le cas, il obtient la taille du quota de stockage et la stocke dans une variable.
