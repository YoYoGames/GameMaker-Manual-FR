steam_is_screenshot_requested

Cette fonction interroge l'API Steam pour savoir si la touche permettant de prendre une capture d'écran du jeu a été enfoncée. La fonction ne retournera vrai que pour une étape (tick du jeu) lorsque la touche est enfoncée, et retournera faux à tous les autres moments.

Veuillez noter que si la touche de capture d'écran est enfoncée, cette fonction ne renverra qu'une seule fois true pour chaque étape où elle est enfoncée, et renverra false pour tout appel ultérieur dans la même étape. Par exemple, si une capture d'écran est demandée dans l'image actuelle et que vous appelez cette fonction dans l'événement Step pour le savoir, vous obtiendrez true; cependant, si vous l'appelez à nouveau dans Draw GUI pour vérifier si une capture d'écran a été demandée, vous obtiendrez false car la fonction a déjà été "utilisée" dans l'événement Step. Pour utiliser la valeur de retour de la fonction plusieurs fois dans la même image, il est recommandé de la stocker dans une variable et de la lire au lieu de rappeler la fonction.

NOTE: Cette fonction ne prend pas de capture d'écran pour vous. Elle signale seulement que la touche a été pressée et que vous devez utiliser les fonctions GameMaker Studio 2 screen_save() ou screen_save_part() pour sauvegarder une copie locale du fichier à télécharger.

Syntaxe :

steam_is_screenshot_requested();

Les retours :

Boolean

Exemple :

if steam_is_screenshot_requested()
{
    var file = "Catch_The_Haggis_" + string(global.scrn_num) + ".png");
    screen_save(file)
    steam_send_screenshot(file, window_get_width(), window_get_height());
    global.scrn_num += 1;
}

Le code ci-dessus interroge l'API Steam pour une demande de capture d'écran et, si c'est le cas, un nom unique pour le fichier image est généré, une capture d'écran est effectuée et le fichier est envoyé à la page communautaire Steam pour l'utilisateur.
