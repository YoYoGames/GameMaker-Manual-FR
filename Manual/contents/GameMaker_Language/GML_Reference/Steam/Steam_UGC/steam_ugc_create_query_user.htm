steam_ugc_create_query_user

Cette fonction peut être utilisée pour interroger la base de données UGC. La fonction utilise automatiquement l'identifiant par défaut de l'application, l'utilisateur, et suppose que la requête est effectuée par le consommateur (plutôt que par le créateur). La fonction exige que vous utilisiez les éléments suivants pour le type de données à interroger, le type d'élément à interroger et l'ordre dans lequel les éléments retournés seront triés, ainsi qu'un numéro de page - notez qu'une requête retournera un nombre maximum de 50 éléments.

La fonction renvoie une valeur unique de poignée de requête qui doit être stockée dans une variable pour être utilisée dans les autres fonctions de requête. Notez que cette fonction ne fait que préparer la requête mais ne l'envoie pas réellement - pour cela, vous devez appeler la fonction steam_ugc_send_query() - et vous pouvez utiliser d'autres fonctions steam_ugc_query_*() pour affiner la demande de recherche avant qu'elle ne soit réellement envoyée.

Syntaxe :

steam_ugc_create_query_user(list_type, match_type, sort_order, page);

Arguments Description
list_type Le type de liste de données à créer (voir les constantes listées ci-dessous).
match_type Le type d'éléments UGC à interroger (voir les constantes énumérées ci-dessous).
sort_order La manière dont les données doivent être ordonnées (voir les constantes énumérées ci-dessous).
page Le numéro de la page à interroger.

Les retours :

Query Handle

Exemple :

query_handle = steam_ugc_create_query_user(ugc_list_Published, ugc_match_Items, ugc_sortorder_TitleAsc, 1);

Le code ci-dessus crée une demande de requête et stocke son handle dans une variable pour une utilisation future.

Retour : UGC
