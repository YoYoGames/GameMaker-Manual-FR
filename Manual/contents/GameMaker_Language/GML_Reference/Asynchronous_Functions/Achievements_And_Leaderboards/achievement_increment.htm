réalisation_incrément

Les jeux peuvent avoir des réalisations sans valeur d'achèvement et vous pouvez donc utiliser cette fonction pour incrémenter ces réalisations d'un montant donné. Sur Android, vous envoyez l'ID de la réalisation sous la forme d'une chaîne de caractères (il s'agit de l'ID unique de la réalisation qui a été attribué lors de la configuration de la réalisation). vous avez configuré l'accomplissement), tandis que sur toutes les autres plates-formes, vous fournissez le nom de l'accomplissement défini comme une chaîne, puis vous donnez la valeur réelle de l'accomplissement à incrémenter.

La fonction déclenchera un événement social asynchrone où la carte DS renvoyée aura les paires de clés et de valeurs suivantes async_load retournée contiendra les paires clé/valeur suivantes :

  • "Type" - Il s'agit du type d'événement qui a été déclenché, qui sera la chaîne "achievement_increment_result" pour cette fonction.
  • "id" - Il s'agit de l'ID de l'événement et, pour cette fonction, elle renverra une constante GML GooglePlayServices_EVENT_ID_INCREMENT_ACHIEVEMENT.
  • "achievement_id" - Il s'agit de l'ID unique de la réalisation qui a été incrémentée.
  • "status" - Ce sera 1 si la réalisation a été incrémentée avec succès, ou 0 si la requête a échoué.
  • "complete" - Ce sera 1 si la réalisation a été complétée par cet incrément, ou 0 si la demande a échoué.

Syntaxe :

achievement_increment(name, value)

Arguments Description
name Le nom/identifiant de la réalisation.
value La valeur à envoyer en tant qu'incrément.

Les retours :

N/A

Exemple :

achievement_increment(global.Achievement[0], score);

Le code ci-dessus ajoute le score actuel du joueur à la réalisation avec l'ID stocké dans le tableau global.
