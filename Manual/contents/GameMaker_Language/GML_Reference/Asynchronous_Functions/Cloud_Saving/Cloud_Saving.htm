Sauvegarde des nuages

Lors de la création de jeux, il est souvent nécessaire de stocker des informations sur l'état du jeu dans un fichier quelconque, mais le stockage de ces informations sur l'appareil n'est pas toujours la meilleure option car, si le joueur supprime le jeu puis le réinstalle, ces informations peuvent être perdues. Pour éviter cela, vous pouvez utiliser différents services en nuage, qui permettent de stocker des données sur Internet afin de les récupérer et de les modifier à tout moment. Il convient de noter que cette fonction est limitée à un seul "blob" de données par jeu, de sorte que chaque fois que vous envoyez une nouvelle sauvegarde au service en nuage, qu'il s'agisse d'une chaîne ou d'un fichier, elle écrasera toutes les données précédemment stockées.

REMARQUE Actuellement, seule la cible Android utilise ces fonctions, et vous devez cocher la case Activer la sauvegarde dans le nuage de Google dans la section sociale des options de jeu de Android. Vous serez invité à télécharger l'extension Google Services, qui est nécessaire au fonctionnement de la sauvegarde dans le nuage sur Android. Pour plus d'informations sur le Cloud Saving, veuillez consulter l'article suivant du service d'assistance : Android Cloud Saving

GameMaker prend en charge cette fonction grâce à quelques fonctions simples qui fonctionnent conjointement avec l'événement nuage asynchrone. Ces fonctions sont expliquées dans les pages suivantes :

  • cloud_synchronise
  • cloud_string_save
  • cloud_file_save
