Sauvegarde du fichier dans le nuage

Cette fonction engage un fichier vers le service en nuage choisi pour le stockage. La fonction renvoie une valeur d'identification unique qui doit ensuite être utilisée dans l'événement asynchrone approprié pour identifier la carte DS renvoyée par le service en nuage. Le fichier doit contenir toutes les informations que vous devez sauvegarder pour votre jeu, car vous ne pouvez stocker qu'un seul "blob de données" dans le nuage, et le fait d'exécuter à nouveau cette fonction écrasera toutes les valeurs précédemment stockées (tout comme l'utilisation de la fonction cloud_string_save() ). La description doit être une courte chaîne d'informations décrivant la sauvegarde, par exemple : "Level2, Stage2".

Pour plus d'informations sur les données asynchrones retournées, veuillez consulter la fonction cloud_synchronise().

Syntaxe :

cloud_file_save(file, description);

ArgumentsType Description
string The file to be uploaded (as a string).
description A brief description of the data being stored.

Les retours :

Async Request ID

Exemple :

var t_str = "";
for (var i = 0; i < 10; i++;)
{
    t_str += string(global.Highscore[i]) + "|"
}
var file = file_text_open_write("Highscores.txt");
file_text_write_string(file, t_str);
file_text_close(file);
save_check = cloud_file_save("Highscores.txt", "Current Highscores");

Le code ci-dessus crée une chaîne de caractères à partir des valeurs stockées dans le tableau global "Highscores", puis écrit cette chaîne dans un fichier pour le stockage local. Le fichier est ensuite envoyé au service en nuage pour le stockage.
