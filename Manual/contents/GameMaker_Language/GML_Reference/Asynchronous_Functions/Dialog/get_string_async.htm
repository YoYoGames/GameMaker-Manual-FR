get_string_async

Cette fonction ouvre une fenêtre et affiche un message ainsi qu'un espace permettant à l'utilisateur de saisir une chaîne de caractères (qui contiendra la chaîne par défaut fournie pour commencer). Il s'agit d'une fonction asynchrone, et en tant que telle, GameMaker ne bloque pas le périphérique sur lequel elle est exécutée en attendant une réponse, mais continue à exécuter les événements normalement. Une fois que l'utilisateur a tapé sa chaîne et appuyé sur le bouton "Okay", un événement asynchrone Dialog est déclenché qui, pour la durée de cet événement uniquement, aura une carte DS stockée dans la variable async_load.

Cette carte contiendra les trois clés suivantes : "id", "status" et "result". "id" est la valeur renvoyée par la fonction lorsqu'elle est appelée, "status" sera soit true si le bouton "Okay" a été pressé, soit false si le message a été annulé (le cas échéant, car toutes les plateformes cibles ne permettent pas d'annuler le message). Enfin, "result" renvoie la chaîne de caractères saisie par l'utilisateur (ou une chaîne vide "" si aucune chaîne n'a été fournie).

NOTE Cette fonction est réservée à un usage de débogage et de testet ne doit pas être utilisée dans les jeux publiés. Dans ce cas, vous devez créer une interface utilisateur personnalisée pour recevoir les entrées des joueurs, de manière à avoir un contrôle total sur l'apparence et le comportement des boîtes de dialogue.

Syntaxe :

get_string_async(string, default);

Arguments Type Description
string Chaîne de caractères The message to show in the dialog.
default Chaîne de caractères The default string.

Les retours :

Async Request ID

Exemple étendu :

L'événement de pression de la souris gauche (par exemple) de l'objet qui affiche le message aurait le code suivant :

msg = get_string_async("What's your name?","Anon");

L'exemple ci-dessus affiche un message demandant à l'utilisateur de saisir une chaîne de caractères et d'appuyer sur "Okay". L'identifiant de la fonction est stocké dans la variable "msg" et sera utilisé dans l'événement asynchrone Dialogs, comme indiqué ci-dessous :

var i_d = ds_map_find_value(async_load, "id");
if (i_d == msg)
{
    if (ds_map_find_value(async_load, "status"))
    {
        if (ds_map_find_value(async_load, "result") != "")
        {
            global.Name = ds_map_find_value(async_load, "result");
        }
    }
}

Le code ci-dessus vérifie la clé "id" de l'adresse DS Map retournée par rapport à la valeur stockée dans la variable "msg". S'ils sont identiques, il vérifie ensuite si l'on a appuyé sur "Okay" (plutôt que de fermer/annuler la fenêtre) et s'il renvoie true, il vérifie ensuite le "résultat" d'une chaîne pour s'assurer qu'aucune chaîne vide n'a été renvoyée avant de définir une variable globale.
