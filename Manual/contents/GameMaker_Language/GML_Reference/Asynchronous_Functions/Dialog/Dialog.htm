Dialogue

La plupart des ordinateurs et des appareils mobiles modernes n'apprécient pas qu'un jeu soit arrêté alors qu'une fenêtre de dialogue s'affiche. Ils ont tendance à interpréter cela comme un arrêt du jeu runner, et peuvent considérer cela comme une erreur et fermer l'application prématurément. Pour éviter cela, GameMaker dispose de fonctions de dialogue asynchrones qui vous permettent de demander certaines formes de retour ou de saisie de l'utilisateur, mais sans bloquer le jeu runner à aucun moment, ce qui signifie que - lorsqu'il est appelé - le jeu se déroulera normalement en arrière-plan pendant que l'utilisateur répondra à la fenêtre de dialogue affichée. Une fois que l'utilisateur aura répondu, un événement asynchrone de dialogue sera appelé pour que vous puissiez traiter les résultats.

REMARQUE: à l'exception de shop_leave_rating()ces fonctions sont réservées à un usage de débogage et de testet ne doivent pas être utilisées dans les jeux publiés. Dans ce cas, vous devez créer une interface utilisateur personnalisée pour recevoir les entrées des joueurs, de manière à avoir un contrôle total sur l'apparence et le comportement des boîtes de dialogue.

Vous trouverez ci-dessous une liste de toutes les fonctions de dialogue asynchrone disponibles :

  • get_login_async
  • get_string_async
  • get_integer_async
  • show_message_async
  • show_question_async
  • shop_leave_rating
