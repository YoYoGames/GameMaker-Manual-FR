Débogage

Lorsque vous créez un jeu, il est souvent très important que vous connaissiez certains détails de la mécanique du jeu à des moments précis, ou que vous puissiez saisir certaines valeurs clés pour certaines fonctions afin de tester différentes méthodes et idées pour votre jeu. À cette fin, GameMaker vous fournit un certain nombre de fonctions de débogage qui afficheront des messages et demanderont des entrées en mode de débogage (ou sur une cible Windows ), mais qui seront ignorées lors de la compilation pour d'autres modules cibles, car ces fonctions placent le jeu dans une boucle fermée jusqu'à ce que l'entrée soit résolue.

NOTE Bien qu'ils puissent fonctionner sur certaines plateformes, ils ne doivent pas être utilisés pour autre chose que debugging.

  • debug_mode
  • debug_event
  • debug_get_callstack
  • exception_unhandled_handler
  • get_integer
  • get_string
  • show_error
  • show_message
  • show_question
  • show_debug_message
  • show_debug_overlay
  • code_is_compiled
  • fps
  • fps_real

En dehors de ces fonctions spécialisées, qui vous aident à créer votre jeu, GameMaker dispose également d'une longue liste de messages d'erreur qui peuvent être affichés lors de l'écriture du code du jeu, de la compilation du jeu ou de l'exécution du jeu lui-même. Ces messages indiquent l'erreur spécifique en question et peuvent être utilisés pour localiser un problème dans un objet ou une fonction de script spécifique, ce qui facilite grandement le processus de debugging. Cependant, la signification exacte du message d'erreur n'est pas toujours évidente. Les sections suivantes énumèrent tous les messages possibles et expliquent leur signification pour vous faciliter la vie :

  • Erreurs de syntaxe
  • Erreurs de compilation
  • Erreurs des coureurs
