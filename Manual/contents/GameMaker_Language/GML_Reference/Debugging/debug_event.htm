débogage_événement

Cette fonction génère un événement de débogage personnalisé qui s'affiche dans la vue graphique du débogueur lorsqu'un jeu est exécuté en mode débogage. Si vous souhaitez que les messages s'affichent en dehors du mode de débogage, utilisez la fonction show_debug_message().

La fonction prendra également deux chaînes réservées pour faciliter l'exécution de debugging en utilisant des applications externes comme Visual Studio. Ces chaînes sont :

  • "OutputDebugOn" - Ceci permet un appel à OutputDebugString pour la cible Windows, ce qui signifie que toute la sortie de débogage - tout ce que vous voyez dans la fenêtre de sortie - peut être visualisée par Visual Studio ou par des applications tierces.
  • "OutputDebugOff" - Désactive le comportement décrit ci-dessus.
  • "BreakOnError" - Cette option est réservée aux versions YYC de Windows et signifie que les projets n' afficheront pas l'écran d'erreur de code habituel si runtime détecte une erreur, mais qu'ils continueront et se planteront. Cela vous permet de voir la trace de la pile dans Visual Studio si debugging.
  • "ResourceCounts- Liste toutes les ressources actuellement actives, telles que les structures de données, les sources de temps, les surfaces, etc.
  • "DumpMemory" - Donne des informations sur l'utilisation actuelle de la mémoire.

Syntaxe :

debug_event(string)

Arguments Type Description
string Chaîne de caractères The custom debug event string to use.

Les retours :

N/A

Exemple :

if !surface_exists(global.EffectsSurface)
{
    debug_event("Recreating Effects Surface");
    global.EffectsSurface = surface_create(room_width, room_height);
}

Le code ci-dessus vérifie si une surface existe et si ce n'est pas le cas, un événement de débogage est déclenché dans la vue graphique du débogueur (le jeu doit avoir été exécuté en mode débogage pour que cela soit visible) et la surface est recréée.
