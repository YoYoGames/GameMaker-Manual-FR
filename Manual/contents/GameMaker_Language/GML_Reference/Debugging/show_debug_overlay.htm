show_debug_overlay

Cette fonction peut être utilisée pour activer et désactiver la superposition de débogage standard lors du test de votre jeu. Elle est désactivée par défaut. La superposition de débogage affiche une barre d'utilisation graphique du CPU/GPU dans la fenêtre de jeu elle-même, ainsi que la valeur réelle actuelle de l'image par seconde, le nombre d'échanges de textures et le nombre de lots de sommets (notez que les échanges de textures et les lots de sommets ne seront jamais nuls et afficheront normalement des valeurs de 2 ou 3, car même avec une pièce vide et sans objets, GameMaker doit toujours dessiner et traiter les choses).

Cette barre est divisée en sections, chaque section représentant 1/60e de seconde. Comme vous pouvez le voir sur l'image ci-dessus, la barre est composée de différentes couleurs :

  • Vert - Traitement des entrées/sorties (clavier, souris, manette de jeu, réseau, etc...)
  • Rouge - La vitesse de mise à jour de l'événement de l'étape
  • Jaune - Le temps nécessaire pour l'événement de tirage
  • Orange -L'heure de mise à jour du débogage, qui n'est normalement visible que lorsque vous utilisez le module de débogage.
  • Blanc - Temps d'attente du GPU, c'est-à-dire le temps passé à attendre que le GPU termine le rendu de l'image avant que la suivante puisse commencer.
  • Cyan - Le temps de rendu du texte
  • Gris - Le temps nécessaire pour effacer l'écran à chaque étape du tirage au sort.
  • Bleu - Le temps nécessaire à l'exécution du Garbage Collector.
  • Rouge foncé - Le flush du GPU, c'est-à-dire le temps que prend le GPU pour vider la mémoire des images.

Grâce à cette fonction, vous pouvez ajouter la superposition, que vous soyez en mode débogage ou non. De cette façon, vous pouvez voir l'efficacité de votre jeu et obtenir un indice visuel sur la façon dont il utilise les ressources disponibles, sans avoir à utiliser le débogueur en parallèle.

REMARQUE: Cette fonction ne fonctionnera pas sur la cible HTML5 en raison de son mode de fonctionnement différent de celui de toutes les autres cibles.

Syntaxe :

show_debug_overlay(enable);

ArgumentsType Description
enable switch on (true) ou éteint (false) the debug overlay.

Les retours :

Exemple :

if global.debug
{
    show_debug_overlay(true);
}
else
{
    show_debug_overlay(false);
}

Le code ci-dessus va activer ou désactiver le débogage en fonction de la valeur d'une variable globale.
