Cordes

À un moment ou à un autre de la création de votre jeu, vous aurez besoin d'utiliser du texte. Dans les jeux, le texte est traité à l'aide des fonctions de chaîne de caractères (une chaîne de caractères est une autre façon de dire une ligne de texte) et GameMaker possède un ensemble complet de fonctions qui vous permettent de manipuler les chaînes de caractères de nombreuses façons, y compris l'insertion d'une chaîne de caractères dans une autre, la copie de chaînes de caractères et la possibilité d'analyser les chaînes de caractères pour trouver les chiffres ou les lettres qu'elles contiennent. En général, une chaîne ne peut être créée qu' en ajoutant du texte entre guillemets doubles " " et les chaînes entre guillemets simples ne sont pas acceptées. Vous ne pouvez pas non plus diviser la chaîne sur plusieurs lignes et attendre de GameMaker qu'il la rende comme si les sauts de ligne étaient des nouvelles lignes (à moins d'utiliser un identificateur de chaîne littérale @, comme expliqué ci-dessous).

Il convient de noter qu'il existe certaines conventions que vous pouvez utiliser lors de la création de chaînes de caractères, principalement en ce qui concerne l'utilisation des caractères d'échappement. Il s'agit de caractères précédés du symbole "\". Ainsi, par exemple, si vous vouliez mettre des guillemets dans une chaîne de caractères, vous auriez quelque chose comme ceci :

str = "Hello\"World\"";

GameMaker prend également en charge les caractères Unicode sur quatre octets, ce qui vous permet de décoder et de coder les caractères Unicode dans les limites supérieures de la norme (y compris, mais sans s'y limiter, les emoji). Pour traiter les caractères Unicode, vous pouvez utiliser le "\" pour précéder tout littéral Unicode - des chiffres hexadécimaux précédés d'un "u", par exemple "\u00e2" pour " á " - où les chiffres sont le numéro du caractère Unicode. Lorsque vous travaillez avec Unicode de cette manière, vous devez être conscient du fait que GameMaker interprétera tous les chiffres suivant le "u", donc si vous voulez écrire " áa " par exemple, vous devez utiliser :

"\u00e2\a"

ou

"\u00e2\u61"

ou

"\u00e2" + "a"

car le simple fait d'utiliser "\u00e2a" donnerait en fait le caractère Unicode "ส" (qui devient essentiellement "\ue2a").

GameMaker peut également traiter n'importe quel littéral hexadécimal - normalement écrit sous forme de chiffres hexadécimaux à la suite de "0x", par exemple "0xff", où les chiffres correspondent au numéro du caractère à utiliser. Dans GameMaker, ces caractères sont écrits en utilisant "\x", puis la valeur hexadécimale. Ces caractères d'échappement et d'autres caractères prédéfinis sont répertoriés dans le tableau ci-dessous :

Constant Description
\n Nouvelle ligne
\r Retour chariot
\b Retour arrière (0x08)
\f Saut de page (0x0c)
\t Onglet horizontal (0x09)
\v Onglet vertical (0x0b)
\\ Backslash lui-même (0x5c)
\a Alerte (0x07)
\u[Hex Digits] Insérer un caractère Unicode
\x[Hex Digits] Insérer un caractère littéral hexagonal
\[Octal Digits] Insérer un caractère octal Unicode

NOTE: Les chaînes de caractères supportent le saut de page, la tabulation verticale, etc... mais cela ne veut pas dire que le rendu le fait, et lors du dessin des chaînes de caractères, ces caractères peuvent être ignorés.

Vous pouvez également créer des chaînes de caractères littérales en faisant précéder la chaîne entière du caractère @:

var test = @"
Line breaks
over multiple
lines
";

Le code ci-dessus rendra la chaîne sur plusieurs lignes comme s'il y avait un caractère d'échappement de saut de ligne inclus. Une chaîne littérale verbatim est similaire aux chaînes littérales de la version précédente GameMaker, mais elle utilise également des guillemets doubles ou simples et doit être préfixée par un symbole @. Elle peut être interrompue sur plusieurs lignes dans le fichier de code et ne prend PAS en charge les caractères échappés, c'est-à-dire que @"Hello\World" n' essaiera pas d'échapper le W de World et sera stockée verbatim. Notez cependant que lorsque vous utilisez des chaînes littérales comme celle-ci, vous devrez rompre la chaîne si vous souhaitez inclure des guillemets dans la chaîne, par exemple :

var test = @"Hello " + "\"" + @"World" + "\""

Une autre chose à noter est que le caractère Unicode 9647 (▯) est utilisé pour substituer tout glyphe manquant que vous pourriez avoir dans votre police désignée lors du rendu dans l'événement draw. Ainsi, si votre police ne possède pas, par exemple, le symbole °, l'écriture de 90° produira en fait 90▯.
