gc_get_stats

Cette fonction permet de récupérer des informations sur l'état actuel du garbage collector. La fonction renvoie une structure contenant les variables suivantes (notez que le terme "objets" désigne ici tout ce qui peut être récupéré par le ramasseur de déchets et non les instances générales d'objets telles que définies dans le navigateur d'actifs) :

Structure des statistiques du GC
Variable Type Description
objects_touched Real Il s'agit du nombre d'objets actifs que le ramasseur d'ordures a trouvé dans la trame précédente. Il varie en fonction de la génération collectée.
objects_collected Real Le nombre d'objets qui, selon le ramasseur d'ordures, n'étaient pas actifs dans la trame précédente, et qui ont donc pu être supprimés.
traversal_time Real Il s'agit du temps en microsecondes (sur le thread principal) que le ramasseur d'ordures a pris pour déterminer quels objets étaient actifs.
collection_time Real C'est le temps en microsecondes (sur un thread séparé) que le ramasseur d'ordures a pris pour nettoyer les objets jugés inactifs.
gc_frame Real Il s'agit d'un compteur qui est incrémenté chaque fois qu'un passage de garbage collection se produit. Si le ramassage des déchets est désactivé, il n'augmentera pas.
generation_collected Real Il s'agit de l'indice de la génération qui a été collectée en dernier. 0 est la génération la plus jeune et 3 est actuellement la plus ancienne.
num_generations Real Il s'agit du nombre total de générations de collecte d'ordures.
num_objects_in_generation Array de Véritables Il s'agit d'un tableau (de taille num_generations) contenant le nombre d'objets dans chaque génération.

REMARQUE: sur la plate-forme cible HTML5, la collecte des déchets est gérée par le moteur JavaScript. Par conséquent, les variables membres de l'adresse struct ci-dessus renverront toutes 0 si cette fonction est utilisée sur cette plate-forme.

Lorsque vous utilisez cette fonction, veuillez noter que les informations affichées pour les objets ne sont mises à jour que lorsque le traitement d'une génération complète est terminé, ce qui peut prendre plusieurs images en fonction du réglage de la durée de l'image (voir ici pour plus d'informations sur la durée de l'image).

Syntaxe :

gc_get_stats();

Les retours :

GC Stats Struct

Exemple :

if (global.debug == true)
{
    var _s = gc_get_stats();
    var _t = _s.traversal_time;
    var _c = _s.collection_time;
    show_debug_message("Traversal time = " + string(_t))
    show_debug_message("Collection time = " + string(_c))
}

Le code ci-dessus vérifie une variable globale et s'il s'agit de true, il obtient des informations du collecteur de garage et les envoie à la console sous forme de messages de débogage.
