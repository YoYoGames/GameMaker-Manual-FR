Fonctions des numéros

Les nombres réels dans GameMaker sont considérés comme des nombres à virgule flottante en double précision - c'est-à-dire qu'ils comportent un point décimal sans qu'il y ait un nombre fixe de chiffres avant ou après le point - ou comme des nombres entiers - c'est-à-dire qu'il s'agit de nombres entiers sans valeur décimale. 2, par exemple, est un nombre entier mais 2,01 est un réel à virgule flottante.

NOTE: Sur la cible HTML5, tous les nombres réels sont des doubles.

Cette distinction entre les entiers et les flottants est très importante lorsqu'il s'agit de développement multiplateforme, car la précision des calculs effectués sur un PC Windows n' est pas la même que celle de ces mêmes calculs lorsqu'ils sont effectués sur un appareil mobile. Cela signifie que vous devez faire particulièrement attention lorsque vous effectuez des comparaisons, par exemple :

if (image_index == 1)
{
    // Do something...
}

Dans l'exemple ci-dessus, si nous avons fixé la valeur de image_speed à 0,25 - par exemple - alors après quatre étapes, vous pouvez supposer que la valeur de image_index sera 1, mais il peut s 'agir d'une valeur comme 1,0000002 en raison de la façon dont les mathématiques à virgule flottante fonctionnent et l'évaluation ne sera donc pas true. Ces types d'erreurs peuvent être assez difficiles à déboguer et il est donc toujours bon d'en être conscient et de prévoir d'autres moyens de vérifier les valeurs ou d'utiliser les fonctions de plancher ou d'arrondi appropriées (listées ci-dessous) pour convertir le nombre à vérifier en un entier (pour plus d'informations sur les mathématiques en virgule flottante et pourquoi cela pose problème, veuillez voir ici). Par exemple, le code ci-dessus pourrait être écrit comme suit :

if (floor(image_index) == 1)
{
    // Do something...
}

Il convient également de noter que lors de l'utilisation des cibles YoYo Compiler, toutes les expressions et fonctions sont évaluées de gauche à droite, alors que sur toutes les autres plateformes cibles, elles sont évaluées de droite à gauche, ce qui signifie que ceci - par exemple - donnera des résultats différents selon la plateforme :

val = max(num, ++num, num++);

NOTE: Pour plus d'informations, voir la section sur l'ordre d'évaluation.

Vous pouvez également utiliser une fonction spéciale disponible dans GameMaker pour définir la valeur epsilon pour les mathématiques en virgule flottante. Lorsqu'un nombre réel est arrondi au nombre à virgule flottante le plus proche, l'epsilon (également appelé "epsilon machine") constitue une limite supérieure de l'erreur relative. Vous pouvez obtenir et définir la valeur de l'epsilon à l'aide des fonctions suivantes :

  • math_set_epsilon
  • math_get_epsilon

REMARQUE Lorsque vous utilisez les fonctions aléatoires, GameMaker conserve la même graine aléatoire à chaque fois que vous démarrez le jeu. Cela facilite grandement l'accès à debugging car vous avez la garantie que les fonctions aléatoires renverront toujours la même valeur au départ. Toutefois, si vous ne souhaitez pas que cela se produise, vous devez d'abord définir une nouvelle graine aléatoire au tout début du jeu, soit en utilisant randomise() ou random_set_seed().
