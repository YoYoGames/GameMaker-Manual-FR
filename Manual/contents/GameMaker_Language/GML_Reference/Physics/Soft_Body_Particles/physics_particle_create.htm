physique_particule_créer

Avec cette fonction, vous pouvez créer une particule unique n'importe où dans votre salle de jeu, en définissant certaines flags et propriétés. Les "drapeaux" sont la valeur de retour d'une combinaison des constantes suivantes :

Physique Particule Flag Constant
Constant Description
phy_particle_flag_water Les propriétés par défaut d'une particule de corps mou.
phy_particle_flag_zombie Une particule zombie est une particule qui sera détruite après une seule étape, toutes les autres étant signalées de cette manière.
phy_particle_flag_wall Cela définit la particule comme statique, la créant essentiellement comme un objet inamovible dans la simulation physique, car elle restera dans une position fixe, peu importe ce qui entre en collision avec elle. Vous devriez utiliser cet indicateur plutôt que de définir la densité à 0.
phy_particle_flag_spring Les particules à ressort produisent l'effet d'être attachées les unes aux autres, comme par un ressort. Les particules créées avec ce drapeau sont "connectées" par paires, chaque particule étant connectée à celle qui était la plus proche d'elle au moment de la création. Une fois appariées, les particules ne changent pas de "partenaires", et plus une force extérieure les éloigne l'une de l'autre, plus la puissance avec laquelle elles entreront en collision lorsque cette force extérieure sera supprimée sera grande. Notez que quelle que soit la distance qui sépare les particules appariées, le lien qui les unit ne se rompt pas.
phy_particle_flag_elastic Les particules élastiques se déforment et peuvent également rebondir lorsqu'elles entrent en collision avec d'autres corps rigides dans la simulation physique.
phy_particle_flag_viscous Une particule visqueuse est une particule qui présente un caractère "collant", comme l'huile. Les particules visqueuses s'agglutinent et collent davantage les unes aux autres.
phy_particle_flag_powder Les particules de poudre produisent un effet de dispersion tel que celui que l'on peut observer avec le sable ou la poussière.
phy_particle_flag_tensile Les particules de tension sont utilisées pour produire l'effet de la tension de surface, ou la courbure tendue à la surface d'un corps liquide. Elles peuvent être utilisées, par exemple, pour créer la tension de surface que l'on observe sur une goutte d'eau. Une fois la tension rompue, les particules rebondissent comme si elles étaient élastiques, mais continuent également à s'attirer les unes les autres. Par conséquent, les particules ont tendance à former des amas lorsqu'elles rebondissent.
phy_particle_flag_colourmixing Les particules dont la couleur est mélangée prennent une partie de la couleur des autres particules avec lesquelles elles entrent en collision. Notez que si une seule des deux particules qui entrent en collision est une particule à couleur mixte, l'autre particule conserve sa couleur d'avant la collision.

Ces flags utilisent le masquage de bits pour créer une valeur de sortie finale qui est ensuite vérifiée pour définir les différentes propriétés de base de la particule (la propriété de base étant toujours celle de phy_particle_flag_water). Par exemple, si vous voulez simuler un liquide visqueux avec une tension superficielle, vous utiliserez la fonction "bitwise" ou "|" pour marquer les bits appropriés comme indiqué dans l'exemple ci-dessous. De cette façon, vous pouvez définir différentes propriétés (autres que les propriétés globales) pour chaque particule individuelle créée.

Outre le site flags, vous pouvez également définir la position dans la pièce où créer la particule, la vitesse horizontale et verticale initiale, la couleur et l'alpha initiaux, ainsi que la catégorie de l'utilisateur. Les valeurs de couleur et d'alpha ne sont utilisées que pour dessiner la particule à l'aide de la fonction physics_particle_draw()mais la valeur de la catégorie est utilisée dans de nombreuses autres fonctions. Cette valeur est une valeur entière arbitraire que vous donnez à chaque particule que vous voulez voir avoir des propriétés similaires, et elle peut être utilisée pour définir flags plus tard pour cette catégorie spécifique ainsi qu'un certain nombre d'autres choses. Par exemple, si vous avez défini une particule "eau" et une particule "huile", vous donnerez à toutes les particules marquées comme "eau" la catégorie d'utilisateur 1 et à toutes les particules créées comme "huile" la catégorie d'utilisateur 2, ce qui vous permettra de cibler ultérieurement une seule catégorie pour la modifier flags ou obtenir des données spécifiques.

REMARQUE: Vous pouvez utiliser n'importe quelle valeur entière pour la catégorie , sauf 0 qui est réservé par la simulation et est utilisé pour sélectionner toutes les catégories dans d'autres fonctions (comme pour le dessin).

La fonction renvoie une valeur d'index (ou d'ID) unique pour la particule, qui peut être stockée dans une variable pour être utilisée dans d'autres fonctions faisant directement référence à la particule.

Syntaxe :

physics_particle_create(flags, x, y, xv, yv, col, alpha, category)

Arguments Type Description
flags Physique Particule Flag Constante(s) The flags to set on the particle.
x Real The x position to create the particle at.
y Real The y position to create the particle at.
xv Real The initial horizontal velocity.
yv Real The initial vertical velocity.
col Couleur The base colour to use for the particle.
alpha Real The base alpha to use for the particle.
category Real The user defined category to which the particle belongs.

Les retours :

Physics Particle ID

Exemple :

var flags = phy_particle_flag_water | phy_particle_flag_viscous | phy_particle_flag_tensile;
var x_vel = lengthdir_x(5, image_angle);
var y_vel = lengthdir_y(5, image_angle);
var p = physics_particle_create(flags, x, y, x_vel, y_vel, c_white, 1, 1);

Le code ci-dessus crée des variables pour définir le type de particule et sa vitesse initiale, puis les utilise pour créer une particule à la position x/y de l'instance.
