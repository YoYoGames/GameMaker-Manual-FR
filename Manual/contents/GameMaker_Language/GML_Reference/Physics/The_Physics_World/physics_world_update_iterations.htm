physics_world_update_iterations

Tout dans GameMaker est basé sur des étapes et, normalement, chaque événement ne se produit qu'une fois par étape. Cependant, pour obtenir la précision nécessaire avec les fonctions physiques, celles-ci sont amenées à effectuer plusieurs calculs à chaque étape, appelés "itérations", dont le nombre est contrôlé par cette fonction. Le nombre par défaut pour le système physique est de 10, mais il peut être modifié à la hausse ou à la baisse en fonction du nombre de fois que vous voulez que la physique calcule chaque étape, mais il est recommandé de ne pas le fixer à moins de 5 et à plus de 30. Vous devez également savoir que cette fonction dépend de la fonction physics_world_update_speed() fonction.

Syntaxe :

physics_world_update_iterations(iterations)

Arguments Type Description
y1 Real The iterations (per step) that the physics system will perform

Les retours :

N/A

Exemple :

physics_world_update_iterations(20);

Le code ci-dessus configure le monde physique dans la pièce actuelle pour effectuer des calculs 20 fois par étape.
