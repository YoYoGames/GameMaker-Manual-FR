physics_world_draw_debug

Cette fonction va dessiner une représentation du système physique de la pièce pour vous aider avec debugging lorsque vous avez des difficultés à faire fonctionner correctement la scène physique. Parfois, le problème peut se situer dans la partie rendu de votre jeu, par exemple un sprite peut être dessiné à la mauvaise position ou rotation, donnant l'apparence d'une physique incorrecte, ou il se peut que vos calculs ne soient pas corrects. Dans tous les cas, cette fonction vous aidera à trouver et à résoudre le problème en dessinant ce qui se passe dans votre système physique. Pour ce faire, vous devez lui indiquer quelle partie du système dessiner, et cela se fait en passant une valeur créée à partir des bits stockés dans diverses constantes "flag" :

Drapeau de débogage de la physique
Drapeau Description
phy_debug_render_aabb Cela montre le périmètre absolu de chaque appareil par rapport à l'axe de la pièce.
phy_debug_render_collision_pairs Cela montrera tous les appareils qui sont actuellement en collision.
phy_debug_render_coms Cela marque le centre de masse de chaque appareil dans la pièce.
phy_debug_render_core_shapes Montre les formes de base qui composent les appareils de la pièce.
phy_debug_render_joints Cela va dessiner chacun des joints de tous les appareils de la pièce.
phy_debug_render_obb Cela montre la boîte de délimitation relative pour les appareils dans la pièce.
phy_debug_render_shapes Cela montre les formes réelles qui constituent les appareils dans la pièce.

Ainsi, pour dessiner deux de ces options, nous devrions faire un "ou" au sens du bit (dans GameMaker, cela est représenté par | ) sur les deux options et stocker la variable résultante pour que la fonction l'utilise. Voici un exemple de la façon dont vous pouvez définir une variable à utiliser avec la fonction qui dessinera uniquement les centres de masse et les articulations de tous les appareils d'une pièce :

flag = phy_debug_render_coms | phy_debug_render_shapes;

Ce qui vous donnerait quelque chose comme ça :

Comme vous pouvez le constater avec le code ci-dessus, seuls les formes et les centres de masse sont dessinés à l'écran (ainsi que les couleurs représentant leurs états - pour plus d'informations, veuillez consulter la documentation Box 2D en ligne). De la même manière, vous pouvez afficher plus, voire toutes les constantes que vous souhaitez déboguer... Il suffit de les "ou" toutes ensemble ! Une autre chose à noter : cette fonction ne doit être appelée que dans l'événement de dessin d'une instance qui a une profondeur inférieure à celles que vous souhaitez déboguer, sinon vous ne serez pas en mesure de voir les informations de débogage.

Syntaxe :

physics_world_draw_debug(flag)

Arguments Type Description
flag Drapeau de débogage de la physique(s) The flag value for showing different debug features

Les retours :

N/A

Exemple :

flag = phy_debug_render_aabb | phy_debug_render_collision_pairs | phy_debug_render_obb;
physics_world_draw_debug(flag);

Le code ci-dessus dessine toutes les boîtes de délimitation absolues, les boîtes de délimitation relatives ainsi que les appareils en collision dans le système physique actuel.
