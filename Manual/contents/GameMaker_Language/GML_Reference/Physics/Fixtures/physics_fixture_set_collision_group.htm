physics_fixture_set_collision_group

Une fois que vous avez créé votre dispositif, vous pouvez le définir comme faisant partie d'un groupe de collision. Cela signifie que vous pouvez dire aux instances avec ce fixture de toujours entrer en collision ou de ne jamais entrer en collision avec d'autres instances qui ont des fixtures liées à elles avec le même groupe de collision. Les valeurs qui peuvent être attribuées sont basées sur des nombres positifs ou négatifs, 0 étant la valeur par défaut (ce qui signifie que l'instance avec ce paramètre de fixation est ignorée, sauf si un événement de collision est créé pour elle). Voici une image pour vous aider à comprendre :

Comme vous pouvez le constater, la définition de groupes de collision peut être très utile pour créer des corps physiques complexes qui interagissent avec le monde physique qui les entoure, mais pas entre eux. Toutefois, il convient de noter qu'en raison de la complexité du calcul des collisions sur la base de groupes de ce type, il est recommandé d'essayer de limiter au maximum le nombre de groupes utilisés dans une même pièce.

Les valeurs du groupe de collision peuvent aller de -32 768 à 32 767 (les deux inclus).

Syntaxe :

physics_fixture_set_collision_group(fixture, group);

Arguments Type Description
fixture ID du dispositif de physique The fixture that is to be bound
group Real The group that this fixture is assigned to (either a positive value, 0, or a negative value)

Les retours :

N/A

Exemple :

physics_fixture_set_collision_group(fix_Wheel, -1);

Le code ci-dessus fera en sorte que le dispositif indexé dans la variable "fix_Wheel" fasse partie du groupe "-1".
