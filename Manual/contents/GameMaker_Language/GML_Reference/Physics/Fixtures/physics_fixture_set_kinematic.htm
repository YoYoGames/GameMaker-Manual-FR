physics_fixture_set_kinematic

Dans certains cas, dans un jeu basé sur la physique, vous voudrez vous déplacer sans être soumis à des forces telles que la gravité ou à des forces résultant de collisions avec des objets dynamiques (pensez aux plates-formes mobiles dans un jeu de plates-formes, par exemple). par exemple). Pour ces objets, il suffit de régler la densité d'un accessoire sur 0 pour que la physique considère que l'objet est censé être statique. Cependant, si vous définissez le paramètre phy_speed_x, phy_speed_y ou phy_angular_velocity pour une instance à laquelle un tel dispositif est lié, elle sera convertie en objet cinématique et commencera à bouger. en objet cinématique et commencera à se déplacer. Changer l'instance de statique à cinématique au milieu de la simulation peut être coûteux en termes de traitement (selon l'état des instances dans le monde physique). pour qu'il soit cinématique en utilisant cette fonction avant de le lier à une instance, afin qu'il ne soit pas affecté par les collisions et les forces (comme la gravité) mais qu'il puisse se déplacer avec une vitesse linéaire et/ou angulaire constante.

Syntaxe :

physics_fixture_set_kinematic(fixture)

ArgumentsType Description
fixture the index of the fixture

Les retours :

Exemple :

physics_fixture_set_kinematic(fix_Cloud);

Le code ci-dessus indique que le dispositif indexé dans "fix_Cloud" doit être marqué comme cinématique.
