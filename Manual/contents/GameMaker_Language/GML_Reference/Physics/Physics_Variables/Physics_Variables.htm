Variables de physique

Comme pour le système "traditionnel" de collisions et de mouvements, le système physique dispose également d'une série de variables d'instance attribuées à chaque instance d'un objet lors de sa création. Certaines de ces propriétés peuvent être définies par l'utilisateur sur le site GML, mais beaucoup sont en lecture seule et d'autres encore ne sont applicables que lors de l'événement de collision.

Les variables suivantes existent dans tous les objets de physique et peuvent être lues et écrites (c'est-à-dire modifiées) à l'aide du code des instances et des fonctions de script :

  • phy_active
  • phy_angular_velocity
  • phy_angular_damping
  • phy_linear_velocity_x
  • phy_linear_velocity_y
  • phy_linear_damping
  • phy_speed_x
  • phy_speed_y
  • phy_position_x
  • phy_position_y
  • phy_position_xprevious
  • phy_position_yprevious
  • phy_rotation
  • phy_fixed_rotation
  • phy_bullet

Certaines variables sont des variables en lecture seule, ce qui signifie que les informations qu'elles contiennent ne peuvent pas être modifiées par le code d'instance ou les fonctions de script (mais elles peuvent être modifiées en interne par GameMaker, c'est pourquoi ce sont des variables et non des constantes). La liste suivante montre les variables en lecture seule qui existent dans tous les objets de physique :

  • phy_speed
  • phy_com_x
  • phy_com_y
  • phy_dynamic
  • phy_kinematic
  • phy_inertia
  • phy_mass
  • phy_sleeping

Enfin, il existe un certain nombre de variables temporaires qui existent dans les objets activés par la physique et qui sont créées uniquement lors de l'événement de collision. Ces variables sont semblables à celles énumérées ci-dessus et sont en lecture seule:

  • phy_collision_points
  • phy_collision_x
  • phy_collision_y
  • phy_col_normal_x
  • phy_col_normal_y
