os_get_info

Cette fonction renvoie un DS Map contenant des informations détaillées sur le système d'exploitation sur lequel le jeu est exécuté. Les informations exactes renvoyées dépendent du système d'exploitation et de l'appareil, il est donc recommandé de tester cette fonction sur toutes les plates-formes que vous souhaitez cibler et d'afficher les valeurs renvoyées à l'aide d'une fonction comme show_debug_message(). Notez que la carte DS n'est pas automatiquement effacée de la mémoire et que vous devez utiliser la fonction ds_map_destroy() lorsque vous n'en avez plus besoin.

Vous trouverez ci-dessous quelques détails sur les données supplémentaires qui seront mises à la disposition de la carte DS spécifique aux plates-formes suivantes. Il convient de noter que sur toutes les plates-formes, à l'exception de HTML5, la carte aura la clé "is64bit" qui sera soit true, soit false selon que runner fonctionne en mode 64 bits ou non.

Windows / UWP

Sur Windows Desktop et Windows UWP, vous obtiendrez quelques données supplémentaires de DX11 avec les touches suivantes disponibles dans la carte DS (utiles pour les extensions) :

  • udid: Un identifiant qui est unique à la machine
  • video_d3d11_device: pointeur vers le périphérique DX11
  • video_d3d11_context: pointeur vers le contexte DX11
  • video_d3d11_spawnchain: pointeur vers la swapchain DX11
  • video_adapter_vendorid: chaîne de caractères contenant l'identifiant du fournisseur de l'adaptateur
  • video_adapter_deviceid: chaîne de caractères contenant l'identifiant du dispositif de l'adaptateur
  • video_adapter_subsysid: chaîne de caractères contenant l'identifiant du sous-système de l'adaptateur
  • video_adapter_revision: chaîne de caractères contenant le numéro de révision de l'adaptateur
  • video_adapter_description: chaîne de caractères contenant la description de l'adaptateur vidéo
  • video_adapter_dedicatedvideomemory: chaîne de caractères indiquant la taille de la mémoire vidéo
  • video_adapter_dedicatedsystemmemory: chaîne de caractères indiquant la taille de la mémoire du système (utilisée par l'adaptateur DX11)
  • video_adapter_sharedsystemmemory: chaîne de caractères indiquant la taille de la mémoire partagée du système

macOS

La fonction renvoie quelques informations OpenGL limitées sur macOS :

  • udid: Un identifiant qui est unique à la machine
  • gl_vendor_string: GL_VENDOR de OpenGL, exemple : "Intel Inc.".
  • gl_version_string: GL_VERSION à partir d'OpenGL, exemple : "2.1 INTEL-16.2.16".
  • gl_renderer_string: GL_RENDERER à partir d'OpenGL, exemple : "Intel(R) Iris(TM) Graphics 6100".

Ubuntu

Il contient les mêmes informations que sur macOS.

Android

Sur la plateforme Android, vous obtiendrez des données supplémentaires à partir de différentes API avec les clés suivantes disponibles dans la carte DS :

  • android_tvce qui renvoie true si le périphérique est détecté comme étant un périphérique TV Android, ou false sinon.
  • GL_VERSIONla version d'OpenGL telle que rapportée par le pilote.
  • GL_VENDORle vendeur d'OpenGL tel que rapporté par le pilote.
  • GL_RENDERER: le moteur de rendu d'OpenGL tel que rapporté par le pilote
  • GL_EXTENSIONS: les extensions d'OpenGL qui sont disponibles, telles que signalées par le pilote.
  • GL_SHADING_LANGUAGE_VERSIONversion du GLSL supportée par le pilote OpenGL.
  • GL_MAX_TEXTURE_SIZE: la taille maximale de la texture supportée par ce pilote OpenGL
  • SDK_INT: valeur de android.os.Build.VERSION.SDK_INT (voir note ci-dessous)
  • RELEASE: valeur de android.os.Build.VERSION.RELEASE
  • MODEL: valeur de android.os.Build.MODEL
  • DEVICE: valeur de android.os.Build.DEVICE
  • MANUFACTURER: valeur de android.os.Build.MANUFACTURER
  • CPU_ABI: valeur de android.os.Build.CPU_ABI
  • CPU_ABI2: valeur de android.os.Build.CPU_ABI2
  • BOOTLOADER: valeur de android.os.Build.BOOTLOADER
  • BOARD: valeur de android.os.Build.BOARD
  • VERSION: valeur de os.version à partir de la méthode Android System.getProperty.
  • REGION: valeurs de user.region à partir de la méthode Android System.getProperty
  • VERSION_NAME: valeur du nom de la version de ce paquet - voir ici pour plus d'informations.
  • PHYSICAL_KEYBOARD: si nous pensons qu'un clavier physique est disponible, alors la chaîne "TRUE" sinon "FALSE"

REMARQUE: pour plus d'informations sur les variables android.os.Build, consultez la documentation Android Developer.

iOS et tvOS

Voici les clés retournées sur iOS et tvOS (pour des informations détaillées, veuillez vous référer à la documentation Apple Developer) :

  • name: chaîne de caractères telle que rapportée par [[UIDevice currentDevice] name]
  • systemName: chaîne de caractères telle que rapportée par [[UIDevice currentDevice] systemName]
  • systemVersion: chaîne de caractères telle que rapportée par [[UIDevice currentDevice] systemVersion]
  • model: chaîne de caractères telle que rapportée par [[UIDevice currentDevice] model]
  • localisedModel: chaîne de caractères telle que rapportée par [[UIDevice currentDevice] localizedModel]
  • uiIdiom: chaîne de caractères telle que rapportée par [[UIDevice currentDevice] userInterfaceIdiom]
  • platform: hw.machine chaîne de caractères recueillie auprès de sysctlbyname
  • hwModel: hw.model chaîne de caractères recueillie auprès de sysctlbyname
  • freeDiskSpace: L'espace disque libre
  • totalDiskSpace: L'espace disque total
  • cpuFrequency: La fréquence du CPU
  • busFrequency: La fréquence du bus
  • cpuCount: Le nombre de CPU
  • totalMemory: La mémoire totale de l'appareil
  • userMemory: La mémoire de l'utilisateur
  • maxSocketBufferSize: La taille maximale de la prise buffer
  • Clés supplémentaires contenant des informations graphiques OpenGL

Opéra GX

  • mobile: Ce sera 1.0 si le jeu fonctionne sur un navigateur mobile, et 0.0 si ce n'est pas le cas.

Renvoie à -1.

Nintendo Switch

PlayStation 4

Veuillez vous reporter à la documentation du PS4 pour obtenir des informations sur les fonctions mentionnées ici.

Cette fonction renvoie les touches suivantes sur une console PlayStation 4 :

  • display_safe_area_ratio: Une valeur à virgule flottante telle que retournée par sceSystemServiceGetDisplaySafeAreaInfo().
  • is_neo_mode: Ce sera 1 si le jeu fonctionne sur une Neo (PS4 Pro) et 0 s'il fonctionne sur une PS4 ordinaire.
  • enter_button_assign: Une valeur entière telle que retournée par sceSystemServiceParamGetInt(SCE_SYSTEM_SERVICE_PARAM_ID_ENTER_BUTTON_ASSIGN, &enterassign); une valeur de 0 signifie que le bouton Cercle est utilisé comme "assign" et 1 signifie que le bouton X est utilisé à la place.

PlayStation 5

La fonction renvoie les touches suivantes sur une console PlayStation 5 :

  • display_safe_area_ratio: Même chose que pour la PS4 (voir ci-dessus).
  • display_resolution: "HD", "4K" ou "inconnu".
  • display_dynamic_range: "HDR", "SDR" ou "inconnu".
  • display_refresh_rate: "120Hz", "60Hz" ou "Inconnu".

Xbox One

Les informations renvoyées sur la Xbox One sont en grande partie les mêmes que celles de Windows, mais les clés video_adapter_* et udid sont 0 (à l'exception de video_adapter_description qui est une chaîne vide ""). Il n'y a pas non plus de clé video_d3d11_swapchain.

Cette plateforme renvoie également les informations suivantes :

  • is_xbox_one_x: Ce sera 1 si le jeu fonctionne sur une Xbox One X et 0 s'il fonctionne sur une Xbox One de base.

Xbox Série X/S

Les informations renvoyées sur les consoles Xbox Series X/S sont pour la plupart identiques à celles de la Xbox One, mais les touches video_d3d11_* ont été remplacées par video_d3d12_*.

Voici quelques clés que cette plateforme renvoie en plus de celles listées sous Xbox One :

  • video_d3d12_cmdqueue: pointe vers la file d'attente des commandes DX12
  • video_d3d12_cmdlist: Pointe vers la liste des commandes DX12
  • video_d3d12_currentrt: pointe vers la cible de rendu DX12 actuelle.

Syntaxe :

os_get_info()

Les retours :

DS Map ID

Exemple :

if (os_type == os_android)
{
    var _info = os_get_info();
    
    if (_info[? "android_tv"])
    {
        global.android_tv = true;
    }
}

Le code ci-dessus vérifie si le système d'exploitation actuel est Android, obtient les informations sur le système d'exploitation et lit la carte renvoyée pour vérifier si le jeu fonctionne sur un téléviseur Android; dans ce cas, il définit la variable personnalisée global.android_tv sur true.
