Système d'exploitation et compilateur

Lorsque vous créez des jeux multiplateformes, il est souvent vital d'obtenir les détails du système d'exploitation utilisé par l'appareil qui exécute votre jeu. Des éléments tels que la langue, la version ou l'état du réseau peuvent tous être utilisés pour adapter votre jeu au système qui l'exécute et rendre l'expérience de l'utilisateur final la meilleure possible. GameMaker possède donc un certain nombre de fonctions spéciales qui peuvent être utilisées pour obtenir les informations nécessaires.

Les fonctions et variables suivantes vous permettent d'obtenir certains détails sur le système d'exploitation ou le navigateur qui exécute votre jeu :

  • os_browser
  • os_device
  • os_type
  • os_version
  • os_is_paused
  • os_is_network_connected
  • os_get_config
  • os_get_language
  • os_get_region
  • os_get_info
  • os_powersave_enable
  • os_lock_orientation
  • os_check_permission
  • os_request_permission

Certaines fonctions sont précompilées lorsque vous exécutez votre jeu (c'est-à-dire qu'elles sont utilisées pour définir comment le jeu final sera compilé), et d'autres sont également à utiliser lorsque le jeu a été compilé pour obtenir des détails spécifiques sur l'environnement runtime. Vous trouverez ici la liste complète de ces fonctions :

  • GM_build_date
  • GM_build_type
  • GM_version
  • GM_runtime_version
  • Nom de fichier du projet GM
  • gml_release_mode
  • gml_pragma
  • parameter_count
  • parameter_string
  • environment_get_variable

Sur Windows, la résolution du thread scheduler Windows est fixée à 1ms par défaut lors de l'exécution de votre jeu. Les fonctions suivantes permettent de récupérer et de modifier la résolution du thread scheduler de Windows au moment de l'exécution :

  • scheduler_resolution_get
  • scheduler_resolution_set

Enfin, lorsque vous travaillez avec des extensions, il se peut que vous deviez être en mesure de définir et d'appeler des fonctions externes à l'adresse suivante : runtime:

REMARQUE: Il s'agit de fonctions héritées et vous devez définir toute fonction d'extension dans l'éditeur d'extension lui-même.

REMARQUE: ces fonctions sont destinées aux plateformes cibles Windows et macOS uniquement.

  • external_define
  • external_call
  • external_free
