os_check_permission

Cette fonction vous permet de demander une autorisation spécifique. Vous devez fournir l'autorisation à demander sous la forme d'une chaîne de caractères au format "android.permission.<permission>". Ainsi, pour demander l'autorisation RECORD_AUDIO (par exemple), vous devez appeler la fonction suivante

os_request_permission("android.permission.RECORD_AUDIO");

Le retour de la fonction déclenchera un événement asynchrone du système dans lequel la carte DS intégrée contiendra les paires de clés/valeurs suivantes async_load DS map contiendra les paires clé/valeur suivantes :

  • "Type": Il s'agit du type d'événement qui a été déclenché et la valeur sera la chaîne de caractères "permission_request_result".
  • "<permission>" : Il s'agira de la permission demandée par la fonction sous forme de chaîne de caractères, qui contiendra l'une des constantes suivantes :
Constant Description
os_permission_granted Cela indique que l'autorisation a été accordée
os_permission_denied Cela indique que l'autorisation n'a pas été accordée

Il convient de noter que les autorisations suivantes sont prises en charge nativement par GameMaker, mais sont considérées comme "dangereuses" par Google et doivent donc être explicitement demandées (notez également que certaines autorisations peuvent être demandées à l'aide des options de jeu d'Android sans avoir besoin de cette fonction) :

  • android.permission.WRITE_EXTERNAL_STORAGE
  • android.permission.READ_PHONE_STATE
  • android.permission.RECORD_AUDIO

Pour plus d'informations sur les autorisations des applications, veuillez consulter la documentation Android.

IMPORTANT ! Cette fonction est réservée à la cible Android.

Syntaxe :

os_request_permission(permission)

Arguments Type Description
permission Chaîne de caractères The permission to request (a string)

Les retours :

N/A

Exemple :

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

Le code ci-dessus vérifie le type de système d'exploitation et s'il s'agit de Android, il effectue une vérification des permissions et si la permission "INTERNET" n'a pas encore été accordée, il la demande.
