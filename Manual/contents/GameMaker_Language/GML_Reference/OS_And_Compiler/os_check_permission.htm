os_check_permission

Cette fonction vous permet de vérifier si une permission spécifique a été accordée au jeu par l'utilisateur. Vous fournissez l'autorisation à vérifier sous la forme d'une chaîne de caractères au format "android.permission.<permission>". Ainsi, pour vérifier l'autorisation RECORD_AUDIO (par exemple), vous devez appeler la fonction suivante

os_check_permission("android.permission.RECORD_AUDIO");

La fonction renverra l'une des trois constantes - indiquées ci-dessous - pour indiquer au jeu l'état de la permission. Pour plus d'informations sur les permissions des applications, veuillez consulter la documentation Android.

IMPORTANT ! Cette fonction est réservée à la cible Android.

Syntaxe :

os_check_permission(permission)

Arguments Type Description
permission Chaîne de caractères The permission to check (a string)

Les retours :

Permission State Constant

Constante d'état de permission
Constant Description
os_permission_granted Cela indique que l'autorisation a été accordée
os_permission_denied Cela indique que l'autorisation n'a pas été accordée
os_permission_denied_dont_request Cela indique que l'autorisation a été bloquée par les paramètres du téléphone, ou que l'utilisateur a précédemment refusé la demande et sélectionné "Ne plus demander".

Exemple :

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

Le code ci-dessus vérifie le type de système d'exploitation et s'il s'agit de Android, il effectue une vérification des permissions et si la permission "INTERNET" n'a pas encore été accordée, il la demande.
