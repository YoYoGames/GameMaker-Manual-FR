network_send_udp

Avec cette fonction, vous pouvez envoyer des données sur le réseau en utilisant UDP à un serveur. La fonction prend l'identifiant de la socket à laquelle se connecter, l'URL à laquelle se connecter et le port à utiliser. Vous devez ensuite fournir l'identifiant du tampon qui contient les données à envoyer (pour plus d'informations sur buffers, voir Référence - Tampons) et enfin la taille (en octets) des données. UDP est "sans connexion" dans le sens où vous ne faites pas réellement de connexion, vous envoyez juste un paquet directement à une IP, et le serveur reçoit les données entrantes d'une adresse IP et doit les traiter "telles quelles". La fonction renvoie le nombre d'octets de données envoyés, ou un nombre inférieur à 0 si l'envoi a échoué. Il convient de noter que la taille finale des données envoyées qui est renvoyée par cette fonction comprendra également les informations de l'en-tête GameMaker, qui représentent 12 octets supplémentaires.

Cette fonction ne fonctionnera pas lorsqu'elle est utilisée dans un projet fonctionnant sur la cible HTML5, et les projets HTML5 ne pourront pas non plus recevoir d'UDP.

Syntaxe :

network_send_udp(socket, url, port, bufferid, size);

Arguments Type Description
socket ID de la prise réseau The id of the socket to use.
url Chaîne de caractères The url or IP to connect to (a string).
port Real The port to connect to.
bufferid ID de la mémoire tampon The id of the buffer to get the data from.
size Real The size (in bytes) of the data.

Les retours :

Real

Exemple :

network_send_udp(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

Le code ci-dessus va envoyer un paquet UDP au serveur défini par l'URL sur le port 6510. Les données sont extraites du site buffer indexé dans la variable "buff".
