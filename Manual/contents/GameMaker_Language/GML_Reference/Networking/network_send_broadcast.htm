réseau_send_broadcast

Cette fonction permet de diffuser localement les données d'un buffer vers une plage d'adresses IP (pour plus d'informations sur buffers, voir Référence - Tampons). La plage est limitée à celle du périphérique exécutant le serveur. Ainsi, si le périphérique a une adresse IP de 92.168.11.130, les données seront diffusées sur la plage 92.168.11.*. La fonction renvoie le nombre d'octets de données envoyés, ou un nombre inférieur à 0 si l'envoi a échoué.

Cette fonction ne fonctionne que lorsqu'elle est utilisée avec UDP - votre serveur doit être TCP et votre client doit avoir une socket client UDP créée avec network_create_socket_ext() afin de recevoir les diffusions envoyées par le serveur.

Cette fonction ne fonctionnera pas si elle est utilisée dans un projet fonctionnant sur la cible HTML5.

Syntaxe :

network_send_broadcast(socket, port, bufferid, size);

Arguments Type Description
socket ID de la prise réseau The id of the socket to use.
port Real The port that the server will use.
bufferid ID de la mémoire tampon The id of the buffer to get the data from.
size Real The size (in bytes) of the data.

Les retours :

Real

Exemple :

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.server_name);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

Le code ci-dessus écrit la chaîne de nom du serveur actuel (stockée dans "global.ServerName"), puis l'écrit dans un fichier binaire buffer avec l'id "broadcast_buffer". Ces données sont ensuite diffusées localement vers une plage d'IP (l'IP du périphérique est actuellement impliquée comme plage de base de diffusion) sur le port 6511.
