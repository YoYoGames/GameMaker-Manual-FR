réseau_send_raw

Avec cette fonction, vous pouvez envoyer un paquet de données "brut" à travers le réseau. La fonction prend l'identifiant de la socket pour se connecter, puis vous devez fournir l'identifiant du buffer qui contient les données à envoyer (pour plus d'informations sur buffers voir Référence - Buffers) et enfin la taille (en octets) du paquet de données. Les données envoyées ne sont en aucun cas formatées par GameMaker et les appareils récepteurs recevront les données sous forme de flux, ce qui signifie que vous devrez les gérer vous-même. La fonction retournera le nombre d'octets de données envoyés, ou un nombre inférieur à 0 si l'envoi a échoué.

Argument des options

Le dernier argument est facultatif, et n'est utilisé qu'avec les WebSockets. Il vous permet de choisir entre l'envoi de données binaires ou textuelles. L'une ou l'autre de ces constantes peut être spécifiée dans cet argument :

Type d'envoi réseau Constant
Constant Description
network_send_binary Envoyer un message binaire
network_send_text Envoyer un message texte

Les API de certaines plateformes n'acceptent que les messages texte lors de l'utilisation de WebSockets (par exemple, Twitch), la constante network_send_text peut donc être utilisée dans ces cas. Si cet argument n'est pas spécifié, des données binaires sont envoyées par défaut.

Syntaxe :

network_send_raw(socket, bufferid, size, [options]);

Arguments Type Description
socket ID de la prise réseau The id of the socket to use.
bufferid ID de la mémoire tampon The id of the buffer to get the data from.
size Real The size (in bytes) of the data.
options Type d'envoi réseau Constant OPTIONAL Utilisé pour les WebSockets pour choisir entre les données textuelles et binaires ; si non spécifié, les données binaires sont envoyées.

Les retours :

Real

Exemple :

buff = buffer_load("player_save.dat");
network_send_raw(sock, buff, buffer_get_size(buff));

Les informations ci-dessus chargent en mémoire des données précédemment enregistrées sur buffer et renvoient l'identifiant de buffer à stocker dans la variable "buff". Ce buffer complet est ensuite envoyé comme un paquet de données brutes sur le réseau en utilisant le socket identifié par la variable "sock".
