instance_création_profondeur

Avec cette fonction, vous pouvez créer une nouvelle instance de l'objet spécifié à n'importe quel point de la pièce et à la profondeur spécifiée. La profondeur peut être n'importe quelle valeur, où plus la profondeur est faible, plus les objets seront dessinés "près" de la caméra et plus la profondeur est élevée, plus ils seront éloignés. Ainsi, une instance à la profondeur -200 sera dessinée par rapport à une instance à la profondeur +300 (par exemple).

Notez que cette fonction va en fait créer un calque de room pour l'instance, puisque toutes les instances doivent se trouver sur un calque de la room, mais qu'il s'agit d'un calque géré (c'est-à-dire qu'il ne s'agit pas d'un calque que vous avez créé par le code ou dans la room, mais d'un calque que GameMaker a créé automatiquement). Vous ne pouvez pas access ce calque, et la variable d'instance layer retournera -1.

IMPORTANT Il existe une profondeur de couche minimale et maximale de -16000 à 16000. Tout ce qui est placé sur une profondeur en dehors de cette plage ne sera pas dessiné, mais tous les événements se dérouleront normalement.

Cette fonction renvoie le id de la nouvelle instance qui peut alors être stockée dans une variable et utilisée pour access cette instance. Notez que cette fonction appelle également l'événement Create de l'instance en cours de création avant de poursuivre le code ou les actions de l'événement qui a appelé la fonction.

Structure facultative

Le dernier argument, var_struct, est facultatif et prend une struct contenant des variables supplémentaires pour la nouvelle instance.

Les variables de ce site struct sont appliquées à la nouvelle instance avant l' exécution de son événement Create, mais après la définition de ses variables.

Cela signifie que les valeurs de cette struct sont lisibles dans l'événement Create de la nouvelle instance. Voir l'exemple 2 en bas de page.

Les valeurs appliquées à la nouvelle instance par le biais de cette struct peuvent être de n'importe quel type, y compris des method variables. Les variables intégrées peuvent également être modifiées.

REMARQUE Les variables du site struct sont "copiées superficiellement" dans la nouvelle instance, ce qui signifie que tous les tableaux, structs et autres ressources sont copiés par référence et non dupliqués.

Syntaxe :

instance_create_depth(x, y, depth, obj);

Arguments Type Description
x Real The x position the object will be created at
y Real The y position the object will be created at
depth Real The depth to assign the created instance to
obj Objet Asset The object index of the object to create an instance of
var_struct Structure OPTIONAL Une structure avec des variables à assigner à la nouvelle instance

Les retours :

Instance ID

Exemple 1 :

var inst = instance_create_depth(x, y, -100, obj_Bullet);
with (inst)
{
    speed = other.shoot_speed;
    direction = other.image_angle;
}

Le code ci-dessus crée une nouvelle instance de l'objet obj_bullet à la profondeur -100, et stocke l'ID de l'instance dans une variable. Cette variable est ensuite utilisée pour assigner speed et direction à la nouvelle instance.

Cela va d'abord créer l'instance, exécuter son événement Create, puis attribuer des valeurs à ses variables.

Si vous souhaitez affecter certaines variables avant l' exécution de l'événement Create, consultez l'exemple ci-dessous.

Exemple 2 :

var inst = instance_create_depth(x, y, -100, obj_bullet,
{
    speed : shoot_speed,
    direction : image_angle
});

Le code ci-dessus crée une instance de obj_bullet, et passe un struct comme dernier argument.

Cette struct possède des variables pour la vitesse et la direction. Elle tire ses valeurs de l'instance appelante, sans avoir besoin d'en utiliser d'autres.

Ces variables sont appliquées à la nouvelle instance avant l'exécution de son événement Create.

Vous n'êtes pas limité à un littéral struct, car vous pouvez également transmettre une variable qui stocke un struct existant, ou créer une structure à partir d'une structure. new à partir d'un constructor.
