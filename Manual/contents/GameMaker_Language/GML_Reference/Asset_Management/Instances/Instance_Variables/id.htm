id

Cette variable en lecture seule contient le numéro d'identification unique de l'instance. Chaque instance que vous créez - que ce soit par le biais du code ou en l'ajoutant à une pièce dans l'éditeur de pièce - reçoit un numéro qui est utilisé en interne pour identifier cette instance et la variable id est ce que vous pouvez utiliser pour la référencer. La variable id est également retournée (et peut être stockée dans une variable) lorsqu'une instance est créée à l'aide de instance_create_layer() ou instance_create_depth(), ainsi que d'autres fonctions d'instance.

Cette variable est maintenant une référence à une instance, au lieu d'être simplement un nombre réel. Vous pouvez obtenir le numéro d'identification de l'instance en appelant real() ou int64() sur cette variable.

NOTE La valeur de cette variable n' est pas la même que l'identifiant que l'instance reçoit dans l'éditeur de salle et est également différente du instance_id qui contient tous les identifiants de toutes les instances actuellement actives.

Syntaxe :

id;

Les retours :

Instance ID

Exemple :

for (var i = 0; i < instance_count; i++;)
{
    if (instance_id[i] != id)
    {
        instance_id[i].scr += 5;
    }
}

Le code ci-dessus ajoute 5 à la variable "scr" pour chaque instance active dans la room, sauf celle qui exécute le code. Pour ce faire, il parcourt en boucle TOUTES les instances actives (en utilisant le tableau "instance_id" pour retourner l'ID de chaque instance active) et les compare à la variable "id" intégrée, qui est l'ID de l'instance originale exécutant le code.
