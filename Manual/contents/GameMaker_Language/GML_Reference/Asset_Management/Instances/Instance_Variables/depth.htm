profondeur

Lorsque vous créez un objet, vous pouvez lui attribuer une profondeur initiale qui définit la manière dont les instances de cet objet seront dessinées dans la room lorsque le jeu est en cours. Cette variable peut être utilisée pour obtenir et modifier cette valeur de profondeur pendant le déroulement du jeu. Normalement, vous n'avez pas besoin d'utiliser cette variable, car vous devez régler les instances pour qu'elles soient ajoutées à des couches discrètes, qui à leur tour sont réglées sur une profondeur spécifique, mais il se peut que vous souhaitiez modifier la profondeur d'une instance en utilisant cette valeur, auquel cas une "couche temporaire" sera créée spécifiquement pour l'instance à la profondeur donnée. Notez que si aucune instance ne se trouve à la même profondeur, cette couche temporaire sera supprimée de la mémoire (contrairement aux couches régulières qui resteront même si elles ne contiennent rien).

IMPORTANT ! Lorsque vous modifiez la variable depth et que GameMaker gère les calques, la variable intégrée renvoie -1 et non l'ID du calque. layer renverra -1 et non l'ID du calque, puisque les calques gérés ne peuvent pas être manipulés par le code.

Dans GameMaker, plus la valeur de profondeur d'une instance est faible, plus cette instance sera dessinée "près de la caméra", tandis qu'une valeur de profondeur plus élevée signifie que l'instance sera dessinée "plus loin de la caméra", c'est-à-dire que -1000 est dessiné au-dessus de -100, qui est dessiné au-dessus de 0, et ainsi de suite.

Notez que les instances d'un objet qui ont la même profondeur peuvent être dessinées au-dessus ou au-dessous les unes des autres, même si elles semblent être placées les unes au-dessus des autres dans l'éditeur de pièce. Si vous voulez garantir qu'un objet est dessiné au-dessus ou au-dessous de tout le reste, vous devez toujours définir la profondeur (ou le calque) explicitement. Notez également qu'il existe une valeur de profondeur minimale (-16000) et maximale (+16000) en dehors de laquelle les instances ne seront pas dessinées, bien qu'elles existent toujours et traitent les événements.

NOTE Vous ne pouvez pas définir la profondeur d'une instance dans son événement Draw (tous les autres événements sont acceptés).

Gardez à l'esprit que la modification de la profondeur d'une instance peut changer les filtres et les effets qui lui sont appliqués, car si vous modifiez la profondeur pour qu'elle soit inférieure à celle d'une couche FX, l'effet ne sera plus appliqué à l'instance.

Syntaxe :

depth;

Les retours :

Real (single precision floating point value)

Exemple :

if y != yprevious
{
    depth = -y;
}

Le code ci-dessus vérifiera si la position y a changé et si c'est le cas, la profondeur sera également définie pour y correspondre.
