Variables d'instance

Lorsque vous créez un nouvel objet, il est livré avec certaines variables déjà initialisées avec des valeurs par défaut. Ces variables sont appelées variables d'instance, car elles seront créées pour chaque instance que vous placerez de l'objet dans vos salles de jeu, et une fois qu'une instance est créée, les valeurs de ces variables sont uniques pour cette instance et seulement cette instance. Vous utiliserez beaucoup certaines de ces variables, comme x et y, tandis que d'autres seront moins nécessaires en fonction de ce que vous voulez faire, mais en général, elles sont très utiles et, dans la mesure du possible, vous devriez chercher à les utiliser plutôt que de créer vos propres variables personnalisées.

La plupart des variables d'instance peuvent être définies et lues, ce qui vous permet de modifier les propriétés et le comportement d'une instance simplement en modifiant la valeur d'une certaine variable - vous pouvez empêcher une instance de dessiner, par exemple, en définissant simplement la variable intégrée visible sur false. Vous trouverez ci-dessous les différentes variables qui sont initialisées pour toutes les instances de tous les objets de votre jeu.

IMPORTANT Si une instance est dans un sequence alors certaines de ces variables - comme image_xscale / image_yscale / image_angle / x / y - seront écrasées lorsque la séquence mettra à jour chaque pas après avoir commencé à être jouée.

Ces variables traitent des propriétés générales de l'instance :

  • id
  • visible
  • solid
  • persistent
  • depth
  • layer
  • alarm

Ces variables concernent la position et le mouvement de l'instance :

  • direction
  • friction
  • gravity
  • gravity_direction
  • hspeed
  • vspeed
  • speed
  • xstart
  • ystart
  • x
  • y
  • xprevious
  • yprevious

Propriétés des objets

La variable suivante contient l'ID d'index unique de l'objet à partir duquel l'instance a été créée :

  • object_index

Ces variables sont toutes liées à l'adresse sprite attribuée à l'instance et peuvent être utilisées pour modifier ce qui est dessiné et comment :

  • sprite_index
  • sprite_width
  • sprite_height
  • sprite_xoffset
  • sprite_yoffset
  • image_alpha
  • image_angle
  • image_blend
  • image_index
  • image_number
  • image_speed
  • image_xscale
  • image_yscale

Ces variables concernent le masque de collision :

  • mask_index
  • bbox_bottom
  • bbox_left
  • bbox_right
  • bbox_top

Ces variables sont liées aux chemins et à la façon dont l'instance interagit avec l'un d'eux, s'il est assigné :

  • path_index
  • path_position
  • path_positionprevious
  • path_speed
  • path_scale
  • path_orientation
  • path_endaction

Ces variables permettent de configurer une instance pour utiliser une ligne de temps :

  • timeline_index
  • timeline_running
  • timeline_speed
  • timeline_position
  • timeline_loop

Séquences

Les variables suivantes stockent des informations concernant la séquence qui peut contrôler l'instance à un moment donné :

  • in_sequence
  • sequence_instance

Il y a un grand nombre de variables intégrées à utiliser avec les fonctions physiques de GameMaker, et donc pour garder les choses plus claires, elles peuvent être trouvées dans la section du manuel qui couvre tout ce qui concerne la simulation physique :

  • Physique
