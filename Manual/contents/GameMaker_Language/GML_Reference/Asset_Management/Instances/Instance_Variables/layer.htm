couche

Cette variable intégrée est créée pour chaque instance d'un room et contient la valeur de l'ID de la couche à laquelle l'instance est assignée. Cette valeur peut ensuite être utilisée dans d'autres fonctions telles que layer_get_depth() ou elle peut être modifiée pour déplacer l'instance vers un autre layer, mais notez que si le layer assigné n'existe pas dans la room actuelle, vous obtiendrez une erreur qui forcera votre jeu à se fermer. Lorsque vous assignez une couche, vous devez fournir l'ID unique de la couche tel que retourné par la fonction layer_get_id() (lorsque vous utilisez des couches de pièce nommées), ou tel que renvoyé par la fonction layer_create() (lorsque vous créez vos propres couches au moment de l'exécution).

IMPORTANT Si vous avez créé l'instance à l'aide de la fonction instance_create_depth() ou que vous avez modifié manuellement la variable depth la couche affectée à l'instance devient une couche "gérée", c'est-à-dire une couche que GameMaker contrôle et gère automatiquement. Dans ce cas, la variable layer renverra -1.

Syntaxe :

layer;

Les retours :

Layer ID

Exemple :

layer = layer_create(-1000);

Le code ci-dessus créera une nouvelle couche avec une profondeur de -1000 et définira ensuite la variable layer de l'instance avec l'ID de la couche retournée, déplaçant l'instance de la couche sur laquelle elle se trouve actuellement vers la nouvelle couche créée.
