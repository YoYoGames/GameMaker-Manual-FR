alarme

Ce tableau à une dimension est utilisé pour obtenir la valeur actuelle de toutes les alarmes que l'instance peut avoir, ou il peut être utilisé pour définir ces alarmes. Il y a douze alarmes intégrées dans chaque instance d'un objet, et chacune d'entre elles a sa propre valeur. event qui se déclenchera lorsque cette variable atteindra 0.

Les alarmes sont décomptées au début de chaque étape, après l'événement Begin Step mais avant l'événement Main Step, et avant que quoi que ce soit ne soit rendu pour l'image. Cela signifie que si vous définissez une alarme à 1 dans l'événement Begin Step, son événement sera exécuté dans la même image (car l'alarme sera décomptée à 0 après la fin de Begin Step), mais si vous définissez une alarme à 1 dans l'événement Step, elle sera exécutée dans l'image suivante.

Il faut noter que l'alarme n'est pas terminée lorsqu'elle atteint 0 (bien que l'événement ait été déclenché) car à l'étape suivante, elle passera à -1, donc si vous devez arrêter une alarme pour une raison quelconque, vous devez définir sa valeur de tableau à -1 et non à 0. Les temps d'alarme sont calculés en pas de jeu, avec une valeur de 30 étant 30 pas, 60 étant 60 pas, etc...

REMARQUE Une alarme qui ne contient aucune action ou code ne fonctionnera pas. Toutefois, même avec un simple commentaire et sans code, l'alarme effectuera un décompte.

Vous pouvez définir le tableau d'alarmes directement (et récupérer la valeur actuelle de l'alarme en utilisant également le tableau), mais dans certaines circonstances, ce n'est pas la méthode la plus appropriée, vous pouvez donc également utiliser la fonction alarm_set() pour définir une alarme, et la fonction alarm_get() pour obtenir la valeur d'une alarme.

Syntaxe :

alarm[val];

Les retours :

(-1 if the alarm isn't running)

Exemple :

if canshoot
{
    if keyboard_check_pressed(vk_space)
    {
        canshoot = false;
        alarm[0] = room_speed;
        instance_create_layer(x, y, "Bullets", obj_Bullet);
    }
}

Le code ci-dessus vérifie une variable et si elle est vraie, il vérifie si une touche a été pressée et si c'est vrai, il met finalement la variable à faux, déclenche une alarme (où elle sera à nouveau mise à vrai) et crée une instance.
