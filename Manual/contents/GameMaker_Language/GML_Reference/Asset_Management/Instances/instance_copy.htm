instance_copy

Avec cette fonction, vous pouvez "cloner" une instance, ce qui créera une nouvelle version de l'instance exécutant le code à la même position. L'argument "perf" est utilisé pour indiquer à cette nouvelle instance d'exécuter ou non l'événement de création. Cette fonction renvoie le id de la nouvelle instance qui peut alors être stockée dans une variable ou utilisée pour access cette instance.

NOTE: Si vous choisissez de ne pas exécuter l'événement de création, vous risquez de rencontrer des erreurs si l'instance dépend de variables initialisées dans cet événement.

Syntaxe :

instance_copy(perf);

Arguments Type Description
perf Booléen Whether to perform the new instance's Create event (true) or not (false).

Les retours :

Instance ID

Exemple :

var inst = instance_number(object_index);
if inst < 10
{
    instance_copy(true);
}

Le code ci-dessus crée une variable locale et l'utilise pour stocker le nombre d'instances de l'objet exécutant le code dans la pièce. Si le nombre est inférieur à 10, l'instance fait alors une copie d'elle-même.
