instance_find

Toutes les instances ont un identifiant unique (id) qui peut être utilisé pour les modifier et les manipuler pendant le déroulement d'un jeu, mais vous ne savez pas toujours quel est l'identifiant d'une instance spécifique et cette fonction peut vous aider car vous pouvez l'utiliser pour itérer à travers toutes les instances pour trouver ce dont vous avez besoin. Vous spécifiez l'objet dont vous voulez trouver l'instance et un nombre, et s'il y a une instance à cette position dans la liste des instances, la fonction renvoie l'id de cette instance, et sinon elle renvoie le mot clé spécial noone. Vous pouvez également utiliser le mot-clé all pour parcourir toutes les instances d'une pièce, ainsi qu'un objet parent pour parcourir toutes les instances qui font partie de cette hiérarchie parent/enfant, et vous pouvez même spécifier une instance (si vous avez son id) pour vérifier si elle existe réellement dans la pièce actuelle. Veuillez noter qu'étant donné que les instances sont triées de manière arbitraire, il n'y a pas d'ordre spécifique à la manière dont les instances sont vérifiées par cette fonction, et toute instance peut se trouver dans n'importe quelle position.

La valeur maximale de "n" dans cette fonction serait de

  • Pour le mot-clé all: instance_count - 1
  • Pour un index d'objet : instance_number(OBJ) - 1

Syntaxe :

instance_find(obj, n);

Arguments Type Description
obj Objet Asset The object to find the nth instance of
n Real The number of the instance to find.

Les retours :

Instance ID or noone

Exemple :

for (var i = 0; i < instance_number(obj_Enemy); ++i;)
{
    enemy[i] = instance_find(obj_Enemy,i);
}

Le code ci-dessus utilise une boucle for pour parcourir toutes les instances de "obj_Enemy" et stocker leur identifiant dans le tableau "enemy[]".
