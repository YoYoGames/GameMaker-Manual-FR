instance_place

Avec cette fonction, vous pouvez vérifier la présence d'une collision entre une position et une autre instance ou toutes les instances d'un objet en utilisant le masque de collision de l'instance qui exécute le code de vérification. Lorsque vous utilisez cette fonction, vous demandez effectivement à GameMaker de déplacer l'instance vers la nouvelle position, de vérifier s'il y a une collision, de revenir en arrière et de vous dire si une collision a été trouvée ou non.

Cela fonctionnera pour les collisions précises, mais seulement si l'instance et l'objet contrôlé ont tous deux des masques de collision précis sélectionnés, sinon seules les collisions de la boîte englobante sont appliquées.

Cette fonction renvoie l'instance unique id de l'objet en collision, mais si cela n'est pas nécessaire, il est légèrement plus rapide d'utiliser la fonction place_meeting(). Cette fonction accepte également les mots-clés spéciaux all et other et renvoie le mot-clé noone si aucune collision ne se produit, ou la valeur de l'ID d'instance unique de l'instance trouvée si une collision se produit.

Syntaxe :

instance_place(x, y, obj);

Arguments Type Description
x Real The x position to check for instances.
y Real The y position to check for instances.
obj ID de l'actif ou de l'instance del'objet The object to check for instances of.

Les retours :

Instance ID or noone

Exemple :

var _inst = instance_place(x, y, obj_Enemy);
if _inst != noone
{
    hp -= _inst.dmg;
    instance_destroy(_inst);
}

Le code ci-dessus vérifie s'il y a une collision avec des instances de "obj_Enemy" et s'il y en a une, il réduit la variable "hp" de la quantité stockée dans la variable "dmg" de l'instance en collision, puis détruit l'instance en collision.
