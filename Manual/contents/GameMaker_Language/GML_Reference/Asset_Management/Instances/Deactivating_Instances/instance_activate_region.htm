instance_activate_region

Cette fonction vous permet de définir une région dans la salle pour activer les instances qui ont été précédemment désactivées. Cette région peut être marquée comme "intérieure" ou "extérieure", comme le montre l'image suivante :

Vous pouvez voir dans l'image ci-dessus que l'instance "apple" est toujours active car, même si le site sprite lui-même ne chevauche pas la région, sa boîte englobante le fait. Ainsi, les instances sont considérées comme étant dans la région spécifiée lorsque leur boîte englobante la chevauche, et l'état du masque de collision (précis ou non) n'est pas pris en considération. Notez que l'activation n'est pas instantanée, et une instance qui a été activée de cette manière ne sera pas considérée comme active avant la fin de l'événement dans lequel la fonction a été appelée.

Syntaxe :

instance_activate_region(left, top, width, height, inside);

Arguments Type Description
left Real The x coordinate of the left of the rectangular region to activate.
top Real The y coordinate of the top of the rectangular region to activate.
width Real The width of the region to activate.
height Real The height of the region to activate.
inside Booléen Whether to activate instances on the inside of the region (true) or the outside (false).

Les retours :

N/A

Exemple :

instance_deactivate_all(true);
var _vx = camera_get_view_x(view_camera[0]);
var _vy = camera_get_view_y(view_camera[0]);
var _vw = camera_get_view_width(view_camera[0]);
var _vh = camera_get_view_height(view_camera[0]);
instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false);

Le code ci-dessus désactive toutes les instances, sauf celle qui exécute le code, puis active une région dans la pièce.
