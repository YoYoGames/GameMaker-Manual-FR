Désactivation des instances

GameMaker vous offre la possibilité de "désactiver" des instances afin qu'elles ne soient plus traitées d'aucune manière. Techniquement, elles n'existent plus vraiment, sauf en tant que pointeur dans le processus de désactivation lui-même, ce qui signifie qu'une instance désactivée ne peut pas être manipulée ou modifiée de quelque manière que ce soit jusqu'à ce qu'elle soit réactivée à nouveau. Ces fonctions doivent donc être utilisées avec beaucoup de précaution car elles peuvent causer des problèmes lorsqu'elles ne sont pas utilisées correctement, en particulier avec les objets persistants, car un objet persistant qui a été désactivé ne sera pas déplacé dans la pièce suivante à moins d'être réactivé auparavant (ce qui le supprime effectivement du jeu). Notez également que l'activation et la désactivation ne sont pas instantanées, et une instance dont l'état a été modifié de cette manière ne sera pas considérée comme active/inactive jusqu'à la fin de l'événement dans lequel la fonction a été appelée.

Il n'est normalement pas nécessaire de désactiver les instances à chaque étape de votre jeu et cela peut même entraîner des problèmes de performance. Il est plutôt recommandé de n'exécuter ces fonctions que toutes les quelques étapes d'une alarme (par exemple), ou si la vue de la caméra a changé de position, et il est particulièrement important de ne pas utiliser ces fonctions dans l'événement draw car cela peut entraîner de graves erreurs dans votre jeu.

Une instance désactivée cesse effectivement d'exister dans le jeu, mais les instances individuelles peuvent toujours avoir accès à leurs variables. Vous ne pouvez cependant pas utiliser with(object) ou with(instance), mais vous devez plutôt access l'instance directement en utilisant son ID unique sous la forme suivante :

val = inst.variable;

Dans l'exemple ci-dessus, "inst" serait une variable qui contient l'ID de l'instance désactivée, et c'est la seule façon de référencer une instance désactivée sans qu'elle ait été activée auparavant.

NOTE Si vous désactivez une instance au démarrage de la room (c'est-à-dire à partir du code de création de la room, ou à partir d'un événement de création d'une instance dans la room), toutes les instances qui sont placées dans la room à partir de l'éditeur de room exécuteront toujours leur événement de création avant d'être désactivées.

AVERTISSEMENT : La désactivation d'instances dont la physique est activée n'empêchera PAS leurs appareils d'interagir dans la simulation physique. Pour cela, vous devez définir leur variable phy_active sur true ou false lorsque vous activez/désactivez les instances.

Les fonctions suivantes permettent de désactiver les instances :

  • instance_activate_all
  • instance_activate_object
  • instance_activate_region
  • instance_activate_layer
  • instance_deactivate_all
  • instance_deactivate_object
  • instance_deactivate_region
  • instance_deactivate_layer
