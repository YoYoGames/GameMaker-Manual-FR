Instances

LesInstances sont créées à partir d'Objets et, une fois créées, elles peuvent être modifiées et manipulées à travers le code GML et les actions GML Visual. Les principales fonctions liées aux instances qui sont disponibles et le plus souvent utilisées sont :

  • instance_create_layer
  • instance_create_depth
  • instance_destroy
  • instance_exists
  • instance_change
  • instance_copy
  • instance_find
  • instance_furthest
  • instance_nearest
  • instance_number
  • instance_place
  • instance_place_list
  • instance_position
  • instance_position_list

Notez qu'il y a deux variables supplémentaires intégrées (ce sont des variables de portée globale ) et une fonction liée à l'utilisation des instances qui peut être utile pour récupérer certains détails :

  • instance_id_get
  • instance_id
  • instance_count

Il y a un certain nombre de variables intégrées disponibles dans chaque instance d'un objet qui peuvent être utilisées pour contrôler différents aspects de la façon dont l'instance est affichée, comment elle utilise d'autres ressources, et comment elle réagit aux collisions, etc... qui sont toutes listées ici :

  • Variables d'instance

Les instances possèdent une variable de type tableau qui peut être utilisée pour obtenir ou définir les valeurs des événements d'alarme, mais il existe également deux fonctions qui peuvent être utilisées pour faire la même chose :

  • alarm_get
  • alarm_set

Enfin, vous avez également la possibilité de désactiver des instances, ce qui signifie qu'elles ne traiteront plus aucun de leurs événements, bien qu'elles existeront toujours dans le jeu. Les fonctions relatives à cela sont disponibles à partir de la page suivante :

  • Désactivation des instances
