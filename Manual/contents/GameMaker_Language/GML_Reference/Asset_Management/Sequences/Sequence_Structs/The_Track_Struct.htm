La structure de la piste

La structure de piste struct peut être utilisée soit pour lespistes d'actifsde niveau supérieur , soit pour lespistes de paramètres de niveau inférieur . Le comportement de la piste est défini par deux éléments, son nom et son type. Les structures de piste sont créées à l'aide de la fonction sequence_track_new() et peuvent être récupérées à partir des actifs de la séquence en utilisant la propriété activeTracks de la Sequence Instance Struct ou la propriété tracks de la Sequence Object Struct.

Les propriétés disponibles dans la piste struct sont les suivantes :

Séquence Track Struct
Variable Type Description
type Séquence Type de piste Constante Cette constante contient un Constante de type de piste de séquence qui décrit le type de piste dont il s'agit. Il peut s'agir de l'une des constantes indiquées dans la section "Type" ci-dessous.
name Chaîne de caractères Lors de la création d'une piste d'actif de "premier niveau", le nom que vous donnez ici peut être n'importe quelle chaîne de caractères dont vous avez besoin pour identifier la piste. Cependant, pour les pistes de paramètres, vous devez spécifier des chaînes de caractères spécifiques pour indiquer à GameMaker le type de piste de paramètres que vous créez. Ce point est développé dans la section "Nom" ci-dessous.
tracks Array de Séquence Piste Structures Cette propriété permet à access d'accéder à la liste des pistes qui sont les enfants de cette piste. Lorsque vous obtenez cette propriété, un tableau de Séquence Track Structest retourné, et lorsque vous définissez cette propriété, un tableau de Séquence Track Structs doit être spécifié.
visible Booléen Ceci indique si cette piste est visible (la valeur est true) ou non (la valeur est false). Vous pouvez obtenir ou définir cette valeur et si une piste n'est pas visible, aucune de ses pistes enfants ne sera dessinée non plus.
keyframes Tableau de Structure des images clés de la séquences Cette propriété permet d'accéder à access à la liste des structures d'images clés pour la piste. Lorsque vous obtenez cette propriété, un tableau d'images clés structs est renvoyé, et lorsque vous définissez cette propriété, un tableau d'images clés structs doit être spécifié.
enabled Boolean This indicates whether this track is enabled. If the track is an asset track it isn't drawn when disabled, if it's a parameter track the track's values aren't applied.

Nom

La propriété name peut être l'une des chaînes de caractères suivantes :

  • "Position" - lorsqu'il est placé sur une piste de type seqtracktype_real, il indique que le contenu de cette piste doit être utilisé comme données de position.
  • "scale" - lorsqu'il est défini sur une piste de type seqtracktype_real, il indique que le contenu de cette piste doit être utilisé comme données de mise à l'échelle.
  • "rotation" - lorsqu'il est défini sur une piste de type seqtracktype_real, il indique que le contenu de cette piste doit être utilisé comme données de rotation.
  • "blend_multiply" ou "image_blend" : lorsqu'il est défini sur une piste de type seqtracktype_colour, il indique que le contenu de cette piste doit être utilisé comme données de multiplication des couleurs. Les données d'image clé d'une telle piste doivent être un tableau de quatre valeurs au format [A, R, G, B]. Notez que les valeurs de chaque composant sont exprimées entre 0 et 1, où 0 correspond à la valeur HEX #00 et 1 à la valeur HEX #FF (0 - 255 comme indiqué dans le sélecteur de couleur pour les pistes de mélange d'images dans l'éditeur de séquences).
  • "Image_speed" - lorsqu'elle est définie sur une piste de type seqtracktype_real, elle indique que le contenu de cette piste doit être utilisé comme valeur de vitesse de l'image.
  • "image_index" - lorsqu'il est défini sur une piste de type seqtracktype_real, il indique que le contenu de cette piste doit être utilisé comme valeur d'index d'image.
  • "Image_angle" - lorsqu'il est défini sur une piste de type seqtracktype_real, il indique que le contenu de cette piste doit être utilisé comme valeur d'angle de l'image.


    Noms des paramètres des pistes de texte (uniquement utilisés sous les pistes de texte) :
  • "frameSize" - lorsqu'il est défini sur une piste seqtracktype_real, il indique que le contenu de cette piste doit être utilisé comme taille d'image pour le texte.Chaque image clé de cette piste doit avoir deux canaux, 0 et 1, où 0 stocke la taille X et 1 stocke la taille Y de l'image.
  • "characterSpacing" - également utilisé avec les pistes seqtracktype_real, utilisé comme espacement des caractères pour le texte. Chaque image clé de cette piste doit avoir un seul canal contenant la valeur d'espacement.
  • "lineSpacing" - également utilisé avec les pistes seqtracktype_real, utilisé comme espacement de ligne pour le texte. Chaque image clé de cette piste doit avoir un seul canal contenant la valeur de l'espacement.
  • "paragraphSpacing" - également utilisé avec les pistes seqtracktype_real, utilisé comme espacement de paragraphe pour le texte. Chaque image clé de cette piste doit avoir un seul canal contenant la valeur d'espacement.
