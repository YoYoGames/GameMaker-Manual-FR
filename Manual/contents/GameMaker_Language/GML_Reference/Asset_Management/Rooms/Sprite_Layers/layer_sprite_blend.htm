couche_sprite_blend

Cette fonction contrôle le mélange (ou "teinte") de sprite sprite et la valeur par défaut est -1 (qui représente la constante c_white, qui peut également être utilisée). Toute autre valeur (y compris les constantes de couleur internes comme c_red, ou c_aqua) permet de mélanger la couleur spécifiée avec la couleur originale sprite. Vous donnez l'ID de l'élément sprite (que vous obtenez lorsque vous créez un élément sprite à l'aide de la fonction layer_sprite_create() ou lorsque vous utilisez la fonction layer_sprite_get_id()), puis définissez la couleur de fusion à utiliser. Vous pouvez voir ci-dessous un exemple de sprite qui a été mélangé avec différentes couleurs :

Veuillez noter que vous devez essayer de limiter les mélanges sur la plateforme HTML5 (sauf si vous utilisez WebGL), car chaque mélange sprite doit être mis en cache séparément et le fait de disposer de nombreux mélanges sprites peut nuire aux performances (vous pouvez également définir la taille de cache à l'aide de la fonction sprite_set_cache_size()).

Syntaxe :

layer_sprite_blend(sprite_element_id, blend);

ArgumentsType Description
sprite_element_id The unique ID value of the sprite element to change
blend The colour to blend with the sprite sprite (default is c_white)

Les retours :

Exemple :

var lay_id = layer_get_id("Asset_sky");
var spr_id = layer_sprite_get_id(lay_id, "Clouds");
layer_sprite_blend(spr_id, c_gray);

Le code ci-dessus récupère la valeur d'identification de l'adresse sprite appelée "Clouds" affectée à la couche "Asset_sky", puis la colore d'une couleur.
