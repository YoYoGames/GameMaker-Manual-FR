path_rotate

Vous pouvez utiliser cette fonction pour faire pivoter un chemin donné autour de son centre. Rappelez-vous que dans GameMaker (à moins que vous n'utilisiez la physique), les angles sont calculés dans le sens inverse des aiguilles d'une montre, donc faire pivoter le chemin de 90 degrés le ferait tourner vers la gauche. Cette fonction modifie la ressource réelle du chemin, et affectera donc de façon permanente la façon dont le chemin est utilisé par toutes les instances du jeu à partir du moment où la fonction est utilisée jusqu'à la fin du jeu. Si ce n'est pas ce dont vous avez besoin, vous devriez utiliser une fonction comme path_duplicate() pour créer d'abord une copie du chemin, puis appeler cette fonction sur la ressource dupliquée (n'oubliez pas d'appeler path_delete() sur la ressource lorsqu'elle n'est plus nécessaire).

NOTE: Une fois qu'un chemin a été assigné à une instance à l'aide de la fonction path_start() vous pouvez également utiliser la variable path_orientation pour modifier la rotation du chemin.

Syntaxe :

path_rotate(index, angle);

Arguments Type Description
index Path Asset The index of the path to flip.
angle Real The angle of rotation degrees.

Les retours :

N/A

Exemple :

path_rotate(pth_Patrol, 90);

Cela ferait tourner la donnée de 90 degrés dans le sens inverse des aiguilles d'une montre.
