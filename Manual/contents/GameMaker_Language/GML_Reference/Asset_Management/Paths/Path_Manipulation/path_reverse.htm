chemin_inverse

Cette fonction permet d'inverser l'ordre dans lequel les différents points de la trajectoire sont numérotés. Ainsi, par exemple, si la trajectoire comporte 5 points, le point 0 devient le point 4, le point 1 devient le point 3 et le point 2 ne change pas. La position réelle des points reste la même, seul l'ordre dans lequel ils sont traités est modifié. Cette fonction modifie l'actif réel du chemin, et affectera donc de façon permanente la façon dont le chemin est utilisé par toutes les instances du jeu à partir du moment où la fonction est utilisée jusqu'à la fin du jeu. Si ce n'est pas ce dont vous avez besoin, vous devriez utiliser une fonction comme path_duplicate() pour créer d'abord une copie du chemin, puis appeler cette fonction sur la ressource dupliquée (n'oubliez pas d'appeler path_delete() sur la ressource lorsqu'elle n'est plus nécessaire).

Syntaxe :

path_reverse(index);

ArgumentsType Description
index The index of the path to change.

Les retours :

Exemple :

path_reverse(pth_AI);

Cela inverserait l'ordre dans lequel tous les points du chemin donné sont traités.
