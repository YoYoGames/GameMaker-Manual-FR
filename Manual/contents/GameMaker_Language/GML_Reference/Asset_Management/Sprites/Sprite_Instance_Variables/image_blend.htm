mélange d'images

Cette variable contrôle la "teinte" de l'instance sprite et la valeur par défaut est -1 (mais peut aussi être c_white). Toute autre valeur (y compris les constantes de couleur internes comme c_red, ou c_aqua) mélangera la couleur spécifiée avec la couleur originale sprite. Veuillez noter que pour que les changements de cette variable soient visibles, l'instance ne doit pas avoir d'événement de dessin (et donc GameMaker dessinera par défaut le sprite) ou être dessinée à l'aide d'une des fonctions de dessin étendues telles que draw_self() ou draw_sprite_ext().

Veuillez noter que vous devez essayer de limiter les mélanges sur la plate-forme HTML5 (sauf si vous utilisez WebGL), car chaque mélange sprite doit être mis en cache séparément et le fait d'avoir beaucoup de mélanges sprites peut nuire aux performances (vous pouvez également définir la taille de cache à l'aide de la fonction sprite_set_cache_size()).

Syntaxe :

image_blend;

Les retours :

Colour Constant

Exemple :

image_blend = make_colour_hsv(255, 255, random(255));

Le code ci-dessus définira la couleur aléatoire de l'image_blend du site sprite défini pour l'instance.
