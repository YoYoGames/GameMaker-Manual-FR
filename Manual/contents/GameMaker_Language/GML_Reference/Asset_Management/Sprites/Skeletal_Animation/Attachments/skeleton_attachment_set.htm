jeu_attachement_squelette

Une animation squelettique sprite peut avoir d'autres images ajoutées comme pièces jointes, ces images étant ajoutées à un slot nommé (le nom est donné lorsque vous créez le slot de pièce jointe dans votre programme d'animation) et elles seront dessinées avec l'animation du sprite actuel. Cette fonction permet d'associer un accessoire à un emplacement donné, en indiquant les noms (sous forme de chaînes) de l'emplacement et de l'accessoire. Ces noms sont définis par le programme d'animation utilisé, ou (dans le cas de l'attachement) lorsque vous appelez skeleton_attachment_create().

Notez que vous pouvez également passer un sprite_index en tant que pièce jointe, et que sprite sera utilisé, ou vous pouvez utiliser -1 pour supprimer la pièce jointe du slot. Lorsque vous passez un index sprite comme argument, cela créera un slot d'attachement en utilisant le nom de sprite comme chaîne de caractères pour nommer le slot (donc en utilisant spr_sword, par exemple, cela créera un slot d'attachement "spr_sword"), et le slot utilisera le premier index d'image (0) de sprite, ainsi que son décalage d'origine x et y, avec une échelle de (1,1) et une rotation de 0.

Syntaxe :

skeleton_attachment_set(slot, attachment);

Arguments Type Description
slot Chaîne de caractères The slot name (a string) to get the attachment of.
attachment Chaîne ou Sprite Asset The name (as a string or a sprite_index) of the attachment image.

Les retours :

String

Exemple :

if skeleton_attachment_get("slot_leftHand") == ""
{
    skeleton_attachment_set("slot_leftHand", choose("sword", "spear", "knife"));
}

Le code ci-dessus vérifie le nom de l'attachement actuellement attribué au slot nommé "slot_leftHand" et si une chaîne vide est retournée, un nouveau sprite est attaché.
