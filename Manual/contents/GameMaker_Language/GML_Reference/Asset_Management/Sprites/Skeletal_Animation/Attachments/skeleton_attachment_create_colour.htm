attachement_squelette_créer_couleur

Lorsque vous créez votre animation squelettique sprite, vous pouvez attribuer des emplacements d'attachement et des attachements à y placer. Il s'agit simplement d'images (sprites) qui sont séparées de l'animation mais qui, lorsqu'elles sont attachées, se déplacent avec l'os auquel elles sont attachées. Normalement, vous devez attribuer ces pièces jointes dans votre programme d'animation (Spine), mais cette fonction vous permet de créer les vôtres au moment de l'exécution en utilisant une ressource sprite de votre jeu. La fonction nécessite que vous donniez un nom à la pièce jointe (sous forme de chaîne) et que vous définissiez les paramètres sprite_index et image_index à utiliser, ainsi que l'origine x et y (qui peut être différente de celle définie par sprite dans les propriétés de sprite ), et vous pouvez également définir toutes les transformations que vous souhaitez appliquer à l'image lorsqu'elle est jointe, ainsi que la couleur à mélanger avec l'image et sa valeur alpha (transparence). Si la pièce jointe a été créée avec succès, la fonction renvoie 1. Dans le cas contraire (vous avez fourni un index sprite invalide ou la base sprite n'est pas un Spine sprite), elle renvoie -1.

Il convient de noter que l'instance sprite peut avoir une couleur de fusion et un paramètre alpha (image_blend et image_alpha), tout comme le slot d'attachement (voir la fonction skeleton_slot_colour_set()), et donc la couleur et l'alpha finaux que l'attachement assigné sprite pour le slot aura seront une combinaison de toutes ces valeurs.

Syntaxe :

skeleton_attachment_create_colour(name, sprite, ind, xorigin, yorigin, xscale, yscale, rot, colour, alpha);

Arguments Type Description
name Chaîne de caractères The name (as a string) of the attachment to create.
sprite Sprite Asset The sprite_index to get the attachment image from.
ind Real The image_index to get the attachment image from.
xorigin Real The x origin for the image being used.
yorigin Real The y origin for the image being used.
xscale Real The horizontal scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
yscale Real The vertical scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
rot Real The rotation of the image: 0 = normal, 90 = turned 90° counter-clockwise etc.
Colour Couleur The colour value to use (A constant, integer or hex value).
Alpha Real The alpha value to use (from 0 to 1).

Les retours :

Real (1 if successful, -1 if not)

Exemple :

skeleton_attachment_create_colour("sword", spr_Weapons, 0, 0, 80, 1, 1, 90, c_red, 1);
skeleton_attachment_create_colour("knife", spr_Weapons, 1, 0, 45, 1, 1, 90, c_yellow, 1);
skeleton_attachment_create_colour("crossbow", spr_Weapons, 0, 10, 30, 1, 1, 0, c_white, 0.5);
skeleton_attachment_set("slot_leftHand", choose("sword", "knife", "crossbow"));

Le code ci-dessus vérifie le nom de l'accessoire actuellement attribué au slot nommé "slot_leftHand" et si une chaîne vide est renvoyée, un nouveau sprite est attaché.
