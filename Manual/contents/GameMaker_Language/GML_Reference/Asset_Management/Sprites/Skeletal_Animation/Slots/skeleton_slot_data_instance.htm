données_instance_slot_squelette

Cette fonction vous permet de remplir une liste de DS (pré-créée) avec des données pour chacun des emplacements d'attachement disponibles pour le site Spine sprite associé à l'instance actuelle (y compris toute modification d'attachement). Ces données sont retournées dans le forum d'une carte DS qui contient les paires clé/valeur suivantes :

  • "name" : Le nom de l'emplacement.
  • "bone" : Le nom de l'os.
  • "attachment" : Le nom de la pièce jointe s'il y en a une ou "(none)" s'il n'y en a pas.

Les valeurs de chaque clé seront des chaînes de caractères et pourront ensuite être utilisées dans les autres fonctions de rattachement du squelette pour ces types de sprite. Notez que les DS Map créés ne sont pas détruits, vous devrez donc parcourir en boucle la liste DS et détruire vous-même chacune des cartes créées.

Syntaxe :

skeleton_slot_data_instance(list);

Arguments Type Description
list ID de la liste DS The ID of the DS list to populate with the DS maps.

Les retours :

N/A

Exemple :

var list = ds_list_create();
var open = true;
slot_name = "";
skeleton_slot_data_instance(list);
for (var i = 0; i < ds_list_size(list); i++)
{
    var map = list[| i];
    if open
    {
        if map[? "attachment"] == "(none)"
        {
            open = false;
            slot_name = map[? "name"];
        }
    }

    ds_map_destroy(map);
}

ds_list_destroy(list);

Le code ci-dessus crée une liste de DS et la remplit ensuite avec les données de slot pour l'instance sprite. Ces données sont ensuite analysées pour extraire les cartes DS individuelles avec les données de slot. On vérifie ensuite s'il existe un slot vide et, si c'est le cas, on attribue à la variable "slot_name" le nom du slot vide avant de détruire la carte DS. Enfin, nous détruisons la liste des DS car elle n'est plus nécessaire.
