skeleton_animation_mix

Vous pouvez facilement changer de jeu d'animations à l'aide de la fonction skeleton_animation_set() mais cela peut provoquer un saut ou un bégaiement lorsqu'une animation est remplacée par une autre. Pour éviter cela, vous pouvez définir la valeur de mixage entre deux jeux d'animation et le site sprite les interpolera. Normalement, vous devriez effectuer cette opération dans l'événement Create de l'instance avec l'animation du squelette, car elle ne doit être définie qu'une seule fois, et GameMaker interpolera toutes les modifications ultérieures du site sprite en utilisant les jeux d'animation de cette instance. Notez que la valeur de la durée est comprise entre 0 et 1, alors qu'une valeur de 0,5 entraînerait une interpolation "moitié-moitié" d'un jeu à l'autre.

REMARQUE Spine 4.0 modifie le fonctionnement de l'animation rotationnelle, car il n'interpole plus la rotation en utilisant le chemin le plus court possible, mais utilise les valeurs absolues de vos images clés pour l'interpolation. Cela peut entraîner un comportement inattendu si vous mettez à jour votre animation Spine à partir d'une version plus ancienne, et des modifications dans votre projet Spine peuvent être nécessaires pour que les rotations se comportent comme prévu.

Syntaxe :

skeleton_animation_mix(animfrom, animto, duration);

Arguments Type Description
animfrom Chaîne de caractères The name (a string) of the first animation set to interpolate from.
animto Chaîne de caractères The name (a string) of the second animation set to interpolate to.
duration Real The duration of the interpolation (from 0 to 1)

Les retours :

N/A

Exemple :

skeleton_animation_set("walk");
skeleton_animation_mix("walk", "jump", 0.2);
skeleton_animation_mix("jump", "walk", 0.4);

Le code ci-dessus s'insère dans l'événement de création d'une instance avec une animation squelettique sprite et définit la durée du mélange d'animations pour interpoler entre les deux animations "walk" et "jump".
