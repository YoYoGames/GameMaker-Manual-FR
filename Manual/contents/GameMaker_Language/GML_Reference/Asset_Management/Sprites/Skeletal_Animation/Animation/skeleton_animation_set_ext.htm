squelette_animation_set_ext

Lorsque vous définissez un jeu d'animation pour votre animation de squelette sprite, la piste 0 lui est attribuée en interne. Cependant, vous pouvez affecter d'autres jeux à d'autres pistes (1 et plus) et ainsi faire en sorte que votre sprite joue deux (ou plus) animations en même temps. Notez que le site sprite doit avoir été conçu dans votre programme d'animation en gardant cela à l'esprit, de sorte que l'animation "de base" de la piste 0 ne comporte que des images clés pour certains os, tandis que les pistes suivantes comportent des images clés pour les autres. Par exemple, si vous avez une animation de "marche" et que ses images clés n'affectent que les jambes de sprite, vous pouvez alors avoir un ensemble d'animations de tête et d'animations de arm que vous pouvez placer dans les pistes 1 et 2 respectivement, ce qui vous permet d'avoir un mouvement de base et de l'adapter en temps réel pendant le jeu.
L'animation est jouée en boucle, ce qui est le cas par défaut.

Syntaxe :

skeleton_animation_set_ext(animname, track, [loop]);

Arguments Type Description
animname Chaîne de caractères The name (a string) of the animation set to use.
track Real The track for the animation set to use.
loop Booléen Si l'animation doit être mise en boucle ou non (la valeur par défaut est true)

Les retours :

N/A

Exemple :

if mouse_check_button(mb_left)
{
    skeleton_animation_set_ext("bodyfight", 1);
}
else
{
    skeleton_animation_set_ext("bodywalk", 1);
}

Le code ci-dessus changera le jeu d'animation utilisé par la piste 1 en jeu "bodyfight" ou "bodywalk" lorsque le bouton gauche de la souris est pressé.
