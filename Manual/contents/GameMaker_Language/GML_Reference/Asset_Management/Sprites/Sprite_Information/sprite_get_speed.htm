sprite_get_speed

Cette fonction peut être utilisée pour récupérer la vitesse de sprite telle que définie pour la ressource sprite dans l'éditeur de Sprite. La valeur renvoyée peut alors être utilisée, par exemple, pour calculer le nombre d'images qui peuvent être dessinées pour différents types de sprites. combien d'images peuvent être dessinées pour différentes image_speed. Notez que la valeur de retour sera très différente en fonction du type de vitesse qui a été appliquée dans l'éditeur Sprite, soit Frames Per Second, soit Frames Per Game Frame. Les deux exemples suivants illustrent ce point :

  • Si vous avez un sprite qui dessine 1 image par seconde et que vous réglez la vitesse de l'image sur 0,5, il dessinera à 0,5 image par seconde. Si la fréquence d'images de votre jeu est de 60 images par seconde, le site sprite affichera une image toutes les 120 images de jeu. images du jeu.
  • Si vous avez un sprite qui dessine 1 image par image de jeu et que vous réglez la vitesse de l'image sur 0,5, il dessinera 0,5 image par image de jeu. Si la fréquence d'images de votre jeu est de 60 images par seconde, le site sprite affichera 30 images pour 60 images de jeu. toutes les 60 images du jeu.

Vous pouvez savoir quel type de vitesse d'animation a été utilisé lors de la définition du site sprite en utilisant la fonction sprite_get_speed_type()et vous pouvez définir la vitesse et le type d'animation à l'aide de la fonction sprite_set_speed().

Syntaxe :

sprite_get_speed(index)

ArgumentsType Description
index The index of the sprite to get the speed of

Les retours :

Exemple :

s_speed = sprite_get_speed(sprite_index);
s_type = sprite_get_speed_type(sprite_index);

Le code ci-dessus récupère la vitesse de sprite et le type d'animation de sprite et les stocke dans des variables d'instance pour une utilisation ultérieure.
