masque_collision_spirituel

Cette fonction vous permet de définir les propriétés du masque de collision que doit avoir un sprite. Si vous sélectionnez le mode de boîte englobante automatique (0) ou image complète (1), les valeurs individuelles de la boîte englobante peuvent être définies sur 0. Cependant, pour un masque défini par l'utilisateur (2), vous devrez définir ces valeurs. Les différentes valeurs de la boîte englobante sont toujours relatives au coin supérieur gauche du site sprite (indépendamment de l'origine x et y) qui est considéré comme la position (0, 0).

La définition du type de masque définit la forme générale du masque lui-même, mais notez que tout autre type de masque qu'un masque rectangulaire nécessitera plus de puissance de traitement lors de la résolution des collisions, ce qui entraînera une baisse des performances. En général, vous ne devriez utiliser des types de masque autres que rectangulaires qu'en cas de nécessité absolue.

REMARQUE: Cette fonction ne permet pas le type de masque de collision du rectangle tourné.

Le type de masque qui peut être défini sera l'une des constantes suivantes :

Type de boîte englobante (forme) Constante
Constant Description
bboxkind_rectangular Forme de masque de collision rectangulaire (non rotatif)
bboxkind_ellipse Une forme de masque de collision elliptique
bboxkind_diamond Un masque de collision en forme de diamant
bboxkind_precise Un masque de collision précis, où le masque se conformera aux pixels non transparents du site sprite, en fonction de la valeur de tolérance indiquée (voir ci-dessous)).


Enfin, la tolérance est utilisée pour définir le degré de précision du masque (lorsqu'il est utilisé avec un masque "image complète", cela n'aura aucun effet). Une tolérance de 0 signifie que le masque suivra chaque pixel dont la transparence est supérieure à 0, tandis que d'autres valeurs décalent le périmètre du masque de collision en fonction de la transparence des pixels.

NOTE: Cette fonction n'est utile que pour les bitmaps sprites et ne fonctionne pas avec les SWF ou JSON (Spine) sprites.

NOTE: Cette fonction ne fonctionnera que sur les ressources ajoutées sprites ou dupliquées sprites et pas directement sur les ressources pré-fabriquées. Vous pouvez dupliquer sprites en utilisant la fonction sprite_duplicate().

Syntaxe :

sprite_collision_mask(ind, sepmasks, bboxmode, bbleft, bbtop, bbright, bbbottom, kind, tolerance);

Arguments Type Description
ind Sprite Asset The index of the sprite to set the bounding box of.
sepmasks Booléen Whether to create collision masks for each sub-image of the sprite (true), ou un seul masque pour tous (false).
bboxmode Real What kind of bounding box to use. 0 = automatic, 1 = full image, 2 = user defined.
bbleft Real The maximum left position of the bounding box.
bbtop Real The maximum top position of the bounding box.
bbright Real The maximum right position of the bounding box.
bbbottom Real The maximum bottom position of the bounding box.
kind Type de boîte englobante (forme) Constante The kind of mask, a constant (see the table in the description).
tolerance Real Indicates the tolerance in the transparency value (0=no tolerance, 255=full tolerance).

Renvoie à

N/A

Exemple :

spr = sprite_add("player_5.png", 16, true, true, 0, 0);
sprite_collision_mask(spr, true, 1, 0, 0, 0, 0, 0, 0);

Le code ci-dessus charge un sprite à partir d'une source externe et enregistre le nouvel indice dans la variable "spr". Le code configure ensuite le nouveau sprite pour qu'il ait un masque de collision précis pour chacune de ses sous-images.
