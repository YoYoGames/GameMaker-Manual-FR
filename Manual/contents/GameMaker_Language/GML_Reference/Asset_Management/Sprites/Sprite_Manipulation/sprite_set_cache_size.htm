sprite_set_cache_size

Lorsqu'il effectue le mélange d'images, HTML5 ne peut pas le faire dynamiquement de la même manière qu'un exécutable. GameMaker doit donc enregistrer temporairement une copie mélangée de l'image et la charger. Cette fonction définit le nombre de copies mélangées de l'adresse sprite peut être mis en cache avant que les anciennes ne soient écrasées. La valeur par défaut est 4. Ceci est appliqué à toutes les sous-images de sprite.

Syntaxe :

sprite_set_cache_size(ind, max);

ArgumentsType Description
ind The index of the sprite to change the cache size of.
max The maximum number of cached copies of the sprite that can be stored.

Renvoie à

Exemple :

sprite_set_cache_size(sprite0, 2);

Cela va définir le sprite cache de toutes les sous-images de sprite0 à 2 copies.
