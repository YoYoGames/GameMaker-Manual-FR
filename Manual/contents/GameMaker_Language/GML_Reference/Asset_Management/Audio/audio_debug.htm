audio_debug

Cette fonction peut être utilisée pour afficher des informations de débogage sur le système audio, avec true pour l'activer et false pour le désactiver. Lorsque vous utilisez debugging audio de cette manière, la superposition affichera les informations suivantes :

  • Handle - Le handle de la source OpenAL (unique pour chaque son joué)
  • Data Handle - Le handle d'OpenAL buffer (montre la même valeur pour plusieurs sons avec le même handle source)
  • Buffers traités - combien de buffers ont été joués.
  • Buffers en attente - combien de buffers ont été mis en attente (et attendent toujours de jouer)
  • Gain - La valeur du gain pour l'audio (0 - 65536)
  • Play Offset - la position de l'échantillon (position de lecture actuelle) dans le tampon actuel.
  • Barres de la file d'attente Buffers - un petit diagramme en forme de boîte du site buffers traité/en file d'attente avec le décalage de lecture (position actuelle) dessiné comme une ligne bleue.

Pour les groupes audio synchronisés, vous disposez également de la fonction audio_sync_group_debug().

NOTE Cette fonction ne fonctionne pas sur la plateforme cible HTML5.

Syntaxe :

audio_debug(enable);

ArgumentsType Description
enable Enable (set to true) ou de désactiver (false) l'audio debugging.

Les retours :

Exemple :

if (debug_mode)
{
    audio_debug(true);
}
else
{
    audio_debug(false);
}

Le code ci-dessus permet d'activer ou de désactiver la superposition de débogage audio selon que le jeu fonctionne en mode débogage ou non.
