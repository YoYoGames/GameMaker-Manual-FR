enregistrement_début_audio

Cette fonction démarre l'enregistrement audio à partir de la source d'enregistrement indexée. Vous pouvez obtenir le nombre de sources d'enregistrement à l'aide de la fonction audio_get_recorder_count()Une fois l'enregistrement commencé, l'audio sera stocké dans un fichier temporaire buffer et déclenchera un événement asynchrone d'enregistrement audio. Cet événement est déclenché à chaque étape de l'enregistrement et crée la carte DS spéciale dans la variable async_load avec les paires clé/valeur suivantes :

  • "buffer_id" - l'ID de l'adresse temporaire buffer que vous pouvez utiliser pour récupérer les données audio
  • "channel_index" - l'index du canal d'enregistrement (tel que renvoyé par la fonction) d'où proviennent les données.
  • "data_len" - la longueur des données (en octets) que vous avez reçues.

Notez qu'après le déclenchement de l'événement asynchrone, le contenu audio de l'adresse temporaire buffer sera effacé. Vous devrez donc copier ses données dans une adresse personnalisée buffer que vous aurez préalablement créée.

NOTE La plupart des plates-formes prennent en charge l'enregistrement audio sous une forme ou une autre, mais cela ne signifie pas que tous les périphériques l'autoriseront, même si la plate-forme le fait. Vous devez donc toujours vérifier que la fonction audio_get_recorder_count() renvoie une valeur supérieure à 0 pour vérifier que les périphériques d'enregistrement sont disponibles avant d'utiliser le reste des fonctions d'enregistrement.

Syntaxe :

audio_start_recording(recorder_index);

Arguments Type Description
recorder_index Real The index of the recorder source to use.

Les retours :

Buffer ID

Exemple :

audio_record = audio_start_recording(0);

Le code ci-dessus lance l'enregistrement à partir de la source 0 de l'enregistreur, en stockant l'index du canal de l'enregistrement dans la variable "audio_record" pour l'utiliser dans l'événement d'enregistrement audio asynchrone.
