file d'attente_audio_créer_lecture

Cette fonction prépare une file d'attente buffer pour l'audio. Vous passez le format des données (seuls buffer_u8 ou buffer_s16 sont actuellement supportés), la fréquence d'échantillonnage (qui peut être comprise entre 1000hz et 48000hz), et les canaux que l'audio requiert à partir d'une des constantes listées ci-dessous :

Constant Description
audio_mono Audio mono (un seul canal).
audio_stereo Audio stéréo (double canal).
audio_3D Audio 3D (5.1).


Notez que vous devez libérer toutes les files d'attente audio créées avec cette fonction lorsqu'elles ne sont plus nécessaires afin d'éviter les fuites de mémoire en utilisant la fonction audio_free_play_queue().

Cette fonction déclenchera un événement asynchrone de lecture audio lorsque la file d'attente aura atteint la fin, et dans cet événement, une carte DS spéciale sera créée dans la variable async_load avec les paires clé/valeur suivantes :

  • "queue_id" - la file d'attente qui a terminé la lecture des données.
  • "buffer_id" - le buffer qui n'est plus lu à partir de
  • "queue_shutdown" - cette valeur est fixée à 0 pendant la lecture normale et à 1 lorsque l'événement est reçu car audio_free_play_queue() a été appelé. Lorsque cette valeur est 1, vous ne voulez pas mettre en file d'attente d'autres données.

Syntaxe :

audio_create_play_queue(queueFormat, queueRate, queueChannels);

Arguments Type Description
queueFormat Type de données de la mémoire tampon Constante The format of the buffer data to use (buffer_u8 ou buffer_s16).
queueRate Real The sample rate of the data in the buffer queue.
queueChannels Type de canal audio The channels to use from one of the constants listed above.

Les retours :

Audio Queue ID

Exemple :

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono);

Le code ci-dessus crée une nouvelle file d'attente audio et attribue son ID à une variable pour référence future.
