audio_free_play_queue

Cette fonction est utilisée pour libérer la mémoire associée à la file d'attente audio donnée. L'index de la file est la valeur renvoyée lorsque vous avez créé la file à l'aide de la fonction audio_create_play_queue()Cette fonction doit être appelée lorsque la file n'est plus nécessaire afin d'éviter les fuites de mémoire. Libérer la file d'attente arrêtera tout son en cours de lecture, et vous ne pouvez pas supprimer le site buffer à partir duquel un son est diffusé tant que la file d'attente à laquelle il est affecté n'a pas été libérée. Cette fonction déclenche un événement asynchrone de lecture audio et, dans cet événement, une carte DS spéciale est créée dans la variable async_load avec les paires clé/valeur suivantes :

  • "queue_id" - la file d'attente qui a terminé la lecture des données.
  • "buffer_id" - le buffer qui n'est plus lu à partir de
  • "queue_shutdown" - ce paramètre est défini à 0 pendant la lecture normale et à 1 lorsque l'événement est reçu parce que vous avez libéré la file d'attente avec cette fonction. Lorsque cette valeur est 1, vous ne voulez pas mettre en file d'attente d'autres données.

Syntaxe :

audio_free_play_queue(queueIndex);

Arguments Type Description
queueIndex ID de la file d'attente audio The index of the queue to free.

Les retours :

N/A

Exemple :

var q_id = async_load[? "queue_id"];
if (q_id == audio_queue)
{
    audio_free_play_queue(audio_queue);
    buffer_delete(audio_buffer);
}

Le code ci-dessus s'insère dans l'événement de lecture audio asynchrone et vérifie si l'audio qui a fini de jouer provient de la "file d'attente audio". Si c'est le cas, il libère la file d'attente et le site buffer qui lui est associé.
