Tampons audio

Vous pouvez utiliser des tampons audio pour charger, sauvegarder et manipuler des fichiers audio dans vos jeux. Ces buffers sont créés au préalable à l'aide des fonctions de tampon appropriées. Vous pouvez ensuite utiliser les fonctions suivantes pour créer un son à utiliser dans votre projet :

  • audio_create_buffer_sound
  • audio_free_buffer_sound

Notez qu'une fois que vous avez créé un son à partir d'un fichier audio buffer, vous ne créez pas un nouveau son en mémoire, mais vous indiquez la position dans le fichier buffer où le son est stocké. Vous ne devez pas modifier ce buffer après qu'un son lui ait été associé, sinon vous obtiendrez des résultats inattendus, et vous ne pouvez pas supprimer le buffer pendant qu'un son lui est associé. Au lieu de cela, libérez d'abord le son (ou les sons) puis supprimez le buffer. Une fois que vous avez créé votre nouvelle ressource audio à partir d'un buffer, vous pouvez utiliser sa valeur d'index dans n'importe quelle fonction audio ordinaire pour la lire, modifier son gain ou sa hauteur, la mettre en pause, etc... Notez que le site buffer utilisé pour charger l'audio ne peut supporter que des sons PCM, donc des formats comme OGG ne seront pas supportés.

Une autre caractéristique de l'audio en mémoire tampon est que vous pouvez utiliser un buffer pour créer une file d'attente audio qui peut ensuite être diffusée sur le périphérique exécutant votre projet. Une file d'attente audio prend une adresse régulière buffer que vous avez préalablement remplie de données audio, et vous permet de "pointer" vers des parties de celle-ci et de demander à GameMaker de diffuser l'audio dans un ordre donné à partir de cette adresse buffer. Les fonctions disponibles pour les files d'attente audio sont les suivantes :

  • audio_create_play_queue
  • audio_free_play_queue
  • audio_queue_sound

Une fois qu'une file d'attente a été créée, vous pouvez utiliser l'index de la file d'attente renvoyée comme vous le feriez avec l'index d'un son normal pour le lire, le mettre en pause, le redémarrer ou modifier son gain et sa hauteur. La seule chose que vous ne pouvez pas faire est d'obtenir la longueur de la piste ou de définir la position de la piste, mais à part cela, une file d'attente audio peut être utilisée comme n'importe quel autre son. Vous devez également noter que certaines des fonctions de la file d'attente audio déclencheront également un événement asynchrone de lecture audio.

Audio buffers vous permettra également d'enregistrer du son à partir de n'importe quelle source disponible (comme des microphones ou le système d'exploitation), qui pourra ensuite être mis en file d'attente et diffusé en utilisant les fonctions ci-dessus (par exemple). L'enregistrement audio déclenche un événement asynchrone d'enregistrement audio qui contient des informations spécifiques sur ce qui est enregistré.

Pour l'enregistrement audio, les fonctions suivantes sont disponibles :

  • audio_start_recording
  • audio_stop_recording
  • audio_get_recorder_count
  • audio_get_recorder_info

NOTE La plupart des plates-formes , à l'exception de HTML5, prennent en charge l'enregistrement audio sous une forme ou une autre, mais cela ne signifie pas que tous les périphériques le permettront, même si la plate-forme le fait. Vous devez donc toujours vérifier que la fonction audio_get_recorder_count() renvoie une valeur supérieure à 0 pour vérifier que les périphériques d'enregistrement sont disponibles avant d'utiliser le reste des fonctions d'enregistrement.
