audio_listener_set_velocity

Cette fonction peut être utilisée pour donner à l'auditeur donné des effets Doppler et simuler un mouvement audio basé sur le vecteur qui est résolu à partir des positions relatives x, y et z données. L'indice par défaut de l'auditeur est 0, mais vous pouvez utiliser la fonction audio_get_listener_info() pour obtenir les différents indices disponibles pour la plateforme cible.

Si l'écouteur donné ne bouge jamais, ou si le mouvement n'est pas constant, vous n'aurez normalement pas besoin de définir ces valeurs, mais, par exemple, si vous créez un jeu à défilement où le joueur a un mouvement constant de bas en haut et les ennemis un mouvement constant de haut en bas, vous définirez les vitesses de l'écouteur et de l' émetteur (pour les émetteurs vous utiliserez audio_emitter_velocity()) aux vecteurs appropriés pour simuler l'effet Doppler correct lorsqu'ils passent devant l'instance du joueur.

L'image ci-dessous montre comment cela pourrait être configuré pour l'exemple de jeu donné ci-dessus :

Syntaxe :

audio_listener_set_velocity(index, x, y, z);

Arguments Type Description
index ID de l'auditeurréel ou audio The listener to change the velocity of (default 0).
x Real The x velocity of the listener (default 0).
y Real The y velocity of the listener (default 0).
z Real The z velocity of the listener (default 0).

Les retours :

N/A

Exemple :

var num = audio_get_listener_count();
for(var i = 0; i < num; ++i;)
{
    var info = audio_get_listener_info(i);
    var data = audio_listener_get_data(info[? "index"]);
    if data[? "vx"] != 0
    {
        audio_listener_set_velocity(info[? "index"], 0, 0, 0);
    }
    ds_map_destroy(info);
    ds_map_destroy(data);
}

Le code ci-dessus vérifie le nombre d'auditeurs disponibles, puis boucle à travers eux et si leur vitesse x est différente de 0, il définit leurs valeurs de vitesse à 0, 0, 0.
