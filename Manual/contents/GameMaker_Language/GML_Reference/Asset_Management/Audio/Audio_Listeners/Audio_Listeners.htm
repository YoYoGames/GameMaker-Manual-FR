Auditeurs audio

Lorsque vous traitez des situations audio complexes, il peut être utile de définir l'emplacement de l'auditeur dans l'environnement audio. Toutefois, lorsque vous utilisez des émetteurs pour générer un son réaliste dans l'espace audio 3D, il est important que l'auditeur soit modifié et déplacé correctement, sinon le son ne sera pas dans la bonne position pour la personne qui joue à votre jeu.

Par défaut, il y a un écouteur automatiquement créé dans le jeu, qui utilise l'ID 0 (pour référence dans les fonctions de l'écouteur).

GameMaker fournit les fonctions suivantes pour les écouteurs :

  • audio_listener_position
  • audio_listener_velocity
  • audio_listener_orientation
  • audio_listener_get_data
  • audio_listener_set_position
  • audio_listener_set_velocity
  • audio_listener_set_orientation

Certaines plates-formes cibles peuvent autoriser plus d'un écouteur, chacun correspondant à un périphérique de sortie différent. Par exemple, votre plate-forme cible peut autoriser deux écouteurs, un pour le casque et un pour le moniteur. C'est pourquoi GameMaker vous fournit un certain nombre de fonctions supplémentaires qui vous permettent de trouver les écouteurs disponibles et de les définir pour les sons joués :

  • audio_get_listener_count
  • audio_get_listener_info
  • audio_get_listener_mask
  • audio_set_listener_mask

Il existe également des fonctions permettant d'obtenir et de régler les données de masque sur les sons individuels :

  • audio_sound_get_listener_mask
  • audio_sound_set_listener_mask

Ainsi que pour obtenir et définir les données de masque sur les émetteurs :

  • audio_emitter_get_listener_mask
  • audio_emitter_set_listener_mask
