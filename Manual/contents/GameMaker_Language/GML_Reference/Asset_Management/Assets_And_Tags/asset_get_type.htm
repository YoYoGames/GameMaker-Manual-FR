type d'actif_get

Cette fonction permet d'obtenir le type de poste référencé à partir de son nom (une chaîne de caractères). L'une des constantes listées ci-dessous sera retournée.

Syntaxe :

asset_get_type(name);

Arguments Type Description
name Chaîne de caractères The name of the game asset to get the type of (a string).

Les retours :

Asset Type Constant

Constant Description
asset_object Le nom donné fait référence à un objet.
asset_sprite Le nom donné fait référence à un sprite.
asset_sound Le prénom fait référence à un son.
asset_room Le prénom fait référence à un room.
asset_tiles Le nom donné fait référence à un ensemble de carreaux.
asset_path Le nom donné fait référence à un chemin.
asset_script Le prénom fait référence à un script.
asset_font Le nom donné fait référence à une police de caractères.
asset_timeline Le prénom fait référence à une ligne de temps.
asset_shader Le nom donné fait référence à un shader.
asset_animationcurve Le nom donné fait référence à une courbe d'animation.
asset_sequence Le nom donné fait référence à une Séquence.
asset_unknown Le nom donné fait référence à un actif qui n'existe pas ou qui n'est pas l'un de ceux énumérés ci-dessus.

Exemple :

if asset_get_type("pth_Path_" + string(global.Game)) == asset_unknown
{
    show_debug_message("Path doesn't exist!!!");
}
else
{
    path_index = asset_get_index("pth_Path_" + string(global.Game));
}

Le code ci-dessus vérifie un nom d'asset créé dynamiquement pour voir si l'asset est du bon type. Si ce n'est pas le cas, un message de débogage est affiché, sinon le nom de l'actif est utilisé pour affecter l'actif à l'instance.
