asset_get_tags

Cette fonction vous permet de récupérer toutes les étiquettes affectées à un poste dans le navigateur de postes. Vous fournissez soit le nom du bien (sous forme de chaîne), soit son index, et la fonction renvoie un tableau de tags pour ce bien. Si aucune balise n'est trouvée ou s'il y a une erreur (par exemple, la chaîne de nom donnée n'existe pas), le tableau retourné sera vide. Si vous fournissez une valeur d'index de poste, vous devrez fournir l'argument optionnel de type de poste (une constante), car des postes de types différents peuvent avoir le même index, même s'ils ne peuvent pas avoir le même nom. Les types de biens disponibles sont répertoriés dans le tableau ci-dessous :

Constant Description
asset_object Le nom donné fait référence à un objet.
asset_sprite Le nom donné fait référence à un sprite.
asset_sound Le prénom fait référence à un son.
asset_room Le prénom fait référence à un room.
asset_tiles Le nom donné fait référence à un ensemble de carreaux.
asset_path Le nom donné fait référence à un chemin.
asset_script Le prénom fait référence à un script.
asset_font Le nom donné fait référence à une police de caractères.
asset_timeline Le prénom fait référence à une ligne de temps.
asset_shader Le nom donné fait référence à un shader.
asset_animationcurve Le nom donné fait référence à une courbe d'animation.
asset_sequence Le nom donné fait référence à une Séquence.

Syntaxe :

asset_get_tags(name_or_index, [asset_type]);

Arguments Type Description
name_or_index Chaîne ou actif The name of the asset (a string) or its index.
[asset_type] Type d'actif Constant OPTIONAL! The type of asset to get the tags for, only used when supplying an index for the first argument.

Les retours :

Array

Exemple :

my_tags = asset_get_tags(object_get_name(object_index));

Le code ci-dessus récupère toutes les balises attribuées à l'objet à partir duquel l'instance qui exécute le code a été créée.
