tag_get_asset_ids

Avec cette fonction, vous pouvez obtenir tous les actifs d'un type donné qui ont les balises données qui leur sont assignées. Vous fournissez soit une seule balise (sous forme de chaîne) soit un tableau, où chaque élément du tableau est une balise (sous forme de chaîne), ainsi que le type de bien à vérifier. Le type de bien doit être l'une des constantes suivantes :

Constant Description
asset_object Le nom donné fait référence à un objet.
asset_sprite Le nom donné fait référence à un sprite.
asset_sound Le prénom fait référence à un son.
asset_room Le prénom fait référence à un room.
asset_tiles Le nom donné fait référence à un ensemble de carreaux.
asset_path Le nom donné fait référence à un chemin.
asset_script Le prénom fait référence à un script.
asset_font Le nom donné fait référence à une police de caractères.
asset_timeline Le prénom fait référence à une ligne de temps.
asset_shader Le nom donné fait référence à un shader.
asset_animationcurve Le nom donné fait référence à une courbe d'animation.
asset_sequence Le nom donné fait référence à une Séquence.

La fonction renvoie un tableau, où chaque élément du tableau est une valeur d'index de bien unique. S'il n'y a pas de biens du type ayant la ou les étiquettes données, un tableau vide sera renvoyé.

Syntaxe :

tag_get_asset_ids(tags, asset_type);

Arguments Type Description
tags Chaîne ou tableau de chaînes de caractères A single asset tag string or an array with various asset tags.
asset_type Type d'actif Constant An asset type constant (listed above)

Les retours :

Array

Exemple :

var _paths = tag_get_asset_ids("enemy", asset_path);
var _num = irandom(array_length(_paths) - 1);
path_start(_paths[_num], 1, path_action_reverse, false);

Le code ci-dessus utilise la balise "enemy" pour trouver toutes les ressources de chemin avec cette balise, avant d'en choisir une au hasard et de l'attribuer à l'instance qui exécute le code.
