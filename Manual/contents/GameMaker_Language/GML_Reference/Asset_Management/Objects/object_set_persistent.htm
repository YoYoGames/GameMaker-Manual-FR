objet_set_persistant

Cette fonction vous permet de définir la persistance d'un objet spécifique. Un objet persistant est "reporté" d'une pièce à l'autre et conserve ses propriétés à tout moment (pour plus d'informations sur la persistance, voir la section sur l'éditeur d'objets). Le fait de définir la persistance de l'objet signifie que toutes les instances de cet objet qui sont créées après la modification de la persistance seront créées de manière persistante, tandis que les instances qui sont déjà dans la salle peuvent ne pas être affectées.

Lorsque vous créez une instance avec la persistance activée dans une room, un layer ou une profondeur lui est attribué (selon la fonction utilisée pour créer l'instance de l'objet). Lorsque la room est changée, et si la room suivante n'a pas de layer avec le même nom ou la même profondeur que celui attribué, alors un nouveau layer sera créé pour l'instance qui persiste à travers les rooms. Si vous avez donné un nom à la couche lors de la création de l'instance, alors la nouvelle couche portera également le même nom que la couche originale, tandis que si vous avez attribué une profondeur à l'instance, alors la nouvelle couche sera nommée "_layer_XXX", où "XXX" est une valeur hexadécimale utilisée pour donner un nom unique à la couche. Ces couches seront supprimées de la pièce à la fin de la pièce. Une autre chose à noter à propos des couches et des instances persistantes est que si vous avez assigné à l'instance une couche nommée à la création, et qu'il y a une autre couche dans les salles suivantes avec le même nom, alors l'instance persistée sera assignée à la couche avec le même nom indépendamment de la profondeur de la couche. Enfin, si une instance persistante se déplace dans une pièce avec une couche à la même profondeur que celle où l'instance a été créée, elle ne sera pas assignée à cette couche, mais une nouvelle couche sera créée à la même profondeur (en suivant la convention de dénomination expliquée précédemment).

Veuillez noter qu'il ne s'agit pas d'une fonction d'instance ! Vous pouvez définir l'indicateur de persistance d'instances individuelles à l'aide de la variable persistent et ainsi avoir dix instances toutes marquées comme persistantes même si l'objet en question a l'indicateur défini à false dans le IDE, et même si vous changez l'indicateur persistant de l'objet à true en utilisant cette fonction, toutes les instances qui sont actuellement dans la pièce resteront comme elles étaient, et seulement les instances créées après l'appel de la fonction commenceront avec persistent flagged as true.

Syntaxe :

object_set_persistent(index, pers);

Arguments Type Description
index Objet Asset The index of the object to change.
pers Booléen Whether the object should be flagged as persistent (true) ou non (false).

Les retours :

N/A

Exemple :

if (room == rm_final)
{
    object_set_persistent(obj_Player, false);
}
instance_create_layer(32, 32, "Instances", obj_Player);

Le code ci-dessus vérifie dans quelle salle l'instance se trouve actuellement et si elle se trouve dans la salle spécifiée "rm_final", il définit la persistance de l'objet "obj_Player" à false et crée ensuite l'instance.
