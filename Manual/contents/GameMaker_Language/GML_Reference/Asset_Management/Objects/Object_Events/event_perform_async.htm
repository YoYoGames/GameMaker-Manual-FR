événement_perform_async

Cette fonction est utilisée pour exécuter l'un des événements asynchrones proposés par GameMaker. Vous fournissez la constante d'événement Async (indiquée dans le tableau ci-dessous) et une carte DS qui sera disponible dans l'événement Async appelé dans la variable async_load.

Type d'événement asynchrone Constant
Constant Description
ev_async_web_image_load Événement Image Loaded
ev_async_web Événement HTTP
ev_async_dialog Événement de dialogue
ev_async_web_iap Événement d'achat In-App
ev_async_web_cloud Événement dans les nuages
ev_async_web_networking Événement de réseautage
ev_async_web_steam Événement vapeur
ev_async_social Événement social
ev_async_push_notification Événement de notification push
ev_async_save_load Sauvegarde/chargement d'un événement
ev_async_audio_recording Événement d'enregistrement audio
ev_async_audio_playback Événement de lecture audio
ev_async_system_event Événement système

Les événements non-asynchrones peuvent être appelés en utilisant event_perform().

Notez que la carte DS spécifiée dans le second argument ne doit pas être détruite manuellement car elle sera automatiquement détruite par la fonction après l'exécution de l'événement Async.

Syntaxe :

event_perform_async(type, ds_map);

Arguments Type Description
type Type d'événement asynchrone Constant The type of event to perform (see the table above).
ds_map ID de la carte DS The DS map to use as async_load dans l'événement appelé.

Les retours :

N/A

Exemple :

var _map = ds_map_create();

_map[? "id"] = "custom_async_event";
_map[? "result"] = true;
_map[? "data"] = { a: 13, b: 16 };

event_perform_async(ev_async_social, _map);

Le code ci-dessus crée une carte DS et la remplit d'entrées personnalisées à lire dans l'événement asynchrone. Il exécute ensuite l'événement social asynchrone avec la carte nouvellement créée passée en tant que async_load pour l'événement appelé.
