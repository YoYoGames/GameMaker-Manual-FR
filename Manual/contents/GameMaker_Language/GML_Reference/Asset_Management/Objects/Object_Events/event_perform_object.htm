événement_perform_objet

Cette fonction fonctionne de la même manière que event_perform() sauf que cette fois, vous pouvez spécifier les événements d'un autre objet. De nombreuses options permettent de simuler tous les événements possibles, mais notez que cette fonction ne fait qu'exécuter tout le code de l'événement et que le jeu ne modifiera rien pour qu'il se déclenche manuellement. Par exemple, si vous choisissez d'exécuter un événement de pression du clavier, l'événement sera déclenché mais la touche correspondante ne sera pas reconnue comme ayant été pressée. Ou encore, si vous déclenchez un événement d'alarme, le compteur d'alarme ne sera pas mis à -1 mais continuera à décompter. Vous pouvez trouver une liste complète des constantes disponibles que cette fonction requiert sur la page de la fonction event_perform().

NOTE Les actions de l'événement appelé avec cette fonction sont appliquées à l'instance courante, et non aux instances de l'objet donné.

Syntaxe :

event_perform_object(obj, type, numb);

Arguments Type Description
obj Objet Asset The object that should have its event triggered.
type Type d'événement Constant The type of event to perform.
numb Nombreréel ou d'événements Constante The specific event, if one is necessary (otherwise, just use 0).

Les retours :

N/A

Exemple :

event_perform_object(obj_Player, ev_keypress, ord("W"));

Cela permettrait d'exécuter l'événement associé à la touche Keyboard Check Pressed > W de l'objet "obj_Player" dans l'instance actuelle.
