Génération d'événements d'objets

GameMaker est entièrement piloté par les événements, et toutes les actions se produisent à la suite de certains événements qui sont déclenchés dans un ordre spécifique ou par une occurrence spécifique dans le jeu. Il existe un certain nombre d'événements différents, chacun ayant une tâche ou un timing spécifique, et certains d'entre eux sont même divisés en "sous" événements distincts. Normalement, ces événements sont lancés par GameMaker lorsqu'il détecte un déclencheur dans le jeu, ou à chaque tick de jeu (comme c'est le cas pour les événements step et draw), mais vous pouvez également appliquer un événement à une instance de la room actuelle à partir d'un morceau de code.

Les fonctions suivantes permettent de générer des événements :

  • event_inherited
  • event_perform
  • événement_perform_async
  • event_perform_object
  • event_user

Il existe également un certain nombre de variables en lecture seule qui peuvent être utilisées avec les fonctions énumérées ci-dessus et qui sont particulièrement utiles pour le débogage :

  • event_action
  • event_number
  • event_object
  • event_type

Pour plus d'informations sur les événements, veuillez consulter la section Événements relatifs aux objets.
