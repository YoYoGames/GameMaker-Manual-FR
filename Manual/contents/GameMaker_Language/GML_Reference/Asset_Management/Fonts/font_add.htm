font_add

Cette fonction peut être utilisée pour ajouter une police à votre jeu à partir d'un fichier de police présent dans les fichiers inclus. Vous pouvez définir la taille de la police (en points), ainsi que si la police doit être en gras ou en italique, et vous pouvez également définir la gamme de caractères à inclure. La fonction renvoie une valeur d'index qui doit être stockée dans une variable car elle sera nécessaire dans tout code ultérieur faisant référence à cette police, ou elle renverra -1 si la fonction échoue pour une raison quelconque.

Notez que le fichier de police doit être un fichier *.ttf ou *.otf, ce qui peut également être utile pour ajouter des polices non standard comme les polices asiatiques ou arabes.

ATTENTION ! Si vous incluez une telle police dans les fichiers inclus, elle doit faire l'objet d'une licence pour être distribuée avec le jeu.

Pour le module cible HTML5, cette fonction ne peut être utilisée que pour ajouter des polices Web, et ne fonctionnera que si WebGL n' est pas activé - si WebGL est activé, vous devez utiliser une ressource de police générale (c'est-à-dire utiliser draw_set_font() avec une police du navigateur de ressources) ou des polices bitmap (cf. font_add_sprite()).

IMPORTANT ! En HTML5, le texte en italique et en gras n'est pris en charge que si la ressource de police ou la police Web elle-même contient ces caractères. Ils ne seront pas synthétisés à partir de la police de base.

Lors du chargement d'une police depuis un fichier, la taille de la police est exprimée en pixels et les première et dernière valeurs sont ignorées, ce qui signifie que tous les glyphes de la police seront ajoutés. Les polices ajoutées de cette manière sont assignées à leur propre page de texture, il faut donc être prudent lors de l'utilisation de cette fonction car elle augmentera le nombre de changements de texture lors du dessin. Il convient également de noter que les polices peuvent apparaître légèrement plus grandes lorsqu'elles sont dessinées, car les glyphes peuvent avoir des parties qui sont dessinées en dehors de la boîte de délimitation définie pour la police. Vous devez également savoir qu'il y a une limitation d'environ 200 glyphes qui peuvent être rendus dans une seule image pour une police particulière.

Par défaut, l'anticrénelage est activé pour les polices ajoutées. Cependant, vous pouvez vérifier l'état de l'AA à tout moment en utilisant la fonction font_add_get_enable_aa() et vous pouvez activer/désactiver l'AA en utilisant la fonction font_add_enable_aa().

NOTE Lorsque vous chargez une police dans GameMaker, vous devez penser à la supprimer (avec la touche font_delete()) lorsqu'elle n'est plus nécessaire, sinon il y a un risque de fuite de mémoire qui ralentira et finira par faire planter votre jeu.

Syntaxe :

font_add(name, size, bold, italic, first, last);

Arguments Type Description
name Chaîne de caractères The path of the font file in the Included Files (*.ttf ou *.otf).
size Real The size of the font - points for Web Fonts, pixels for file fonts.
bold Booléen Whether the font is bold (true) ou non (false).
italic Booléen Whether the font is italic (true) ou non (false).
first Real The first character to include (if you're unsure, go for 32).
last Real The last character to include (if you're unsure, go for 128).

Les retours :

Font Asset

Exemple :

new_font = font_add("gamefont.ttf", 24, true, true, 32, 128);

Cela chargera le fichier "gamefont.ttf" présent dans les fichiers inclus du jeu avec une taille de 24pt. La police sera en gras et en italique, et la gamme de polices comprendra les lettres majuscules et minuscules, les chiffres et toute la ponctuation courante. Une fois la police chargée, son ID sera stocké dans la variable new_font.
