font_get_first (obsolète)

Lorsque vous définissez une police dans GameMaker, vous pouvez définir une plage de caractères à inclure. En effet, la police elle-même n'est pas incluse dans votre jeu (pour des raisons juridiques), mais une image de la police est incluse dans une page de texture. la police est incluse dans une page de texture et c'est ce que votre jeu utilisera (comme tout autre élément graphique). Cela signifie que vous devrez limiter au maximum le nombre de caractères utilisés et ne spécifier que la gamme de caractères dont votre jeu aura besoin, afin de ne pas nuire à la qualité du jeu. dont votre jeu aura besoin, afin d'optimiser au maximum la mémoire des textures. Cette fonction peut être utilisée pour trouver le caractère de départ (sous forme de valeur ASCII) qui a été utilisé lorsque votre police a été ajoutée à votre jeu.

NOTE En raison de changements dans la façon dont les polices sont gérées, cette fonction retournera toujours 32, car GameMaker impose le caractère espace (32) dans la police et n'autorise pas non plus les caractères dont le code est inférieur à 32. n'autorise pas non plus les caractères dont le code est inférieur à 32. De nouvelles fonctions pour effectuer des tâches similaires seront ajoutées dans une future mise à jour.

Syntaxe :

font_get_first(ind);

ArgumentsType Description
ind Index of the font to check.

Les retours :

Exemple :

num = font_get_first(fnt_Main);

Le code ci-dessus stockera la valeur ASCII de la première lettre de la gamme de polices pour la police indexée dans "fnt_Main".
