font_add_sprite_ext

Avec cette fonction, vous pouvez utiliser une "bande de sprites" (le site sprite doit être un actif sprite du navigateur d'actifs ou un sprite que vous avez ajouté aux ressources du jeu à l'aide de sprite_add) pour créer un nouvel actif de police, où chaque sous-image est un symbole ou une lettre individuelle. Contrairement à la police normale font_add_sprite() normale qui a un ordre spécifique pour les sous-images de sprite, cette fonction mettra en correspondance les sous-images de sprite sur la base de l'argument "string_map" de la fonction. Cet argument est une chaîne de caractères que vous pouvez utiliser pour indiquer à GameMaker l'ordre dans lequel les sous-images de la police sprite sont placées. Ainsi, par exemple, si vous avez un mappage de chaîne de caractères de "AaBbCcDdEeFfGgHh", votre police sprite doit avoir les sous-images ordonnées de cette façon. Vous pouvez définir l'"espace" comme étant n'importe quel caractère, par exemple une ligne unique de la taille que vous souhaitez pour l'espace, et tous les espaces dans le texte seront rendus à cette largeur (mais l'image choisie ne sera jamais rendue), ou si vous ne fournissez pas de sprite pour l'espace, la largeur du caractère le plus large dans la police sprite sera utilisée à la place.

Vous pouvez également définir l'espacement de la police pour qu'elle soit proportionnelle (true) ou non (false). Une police proportionnelle est espacée en fonction de la largeur réelle des lettres (la lettre "i" prend moins de place que la lettre "w", par exemple), tandis qu'une police non proportionnelle sera espacée en fonction de la largeur de la sous-image, comme une police monospace. Enfin, vous pouvez définir l'espace à laisser entre chaque lettre lors de l'écriture. Il peut s'agir d'une valeur entière, 0 correspondant à l'absence d'espace (les lettres se "toucheront" si elles sont proportionnelles).

La fonction renvoie une valeur d'index qui doit être stockée dans une variable car elle sera nécessaire dans tous les codes ultérieurs faisant référence à cette police, ou elle renvoie -1 si la fonction échoue pour une raison quelconque.

Il convient de noter que pour que les fonctions d'alignement des polices fonctionnent (comme la fonction draw_set_halign()), l'origine de la police sprite doit être définie dans le coin supérieur gauche. Si vous utilisez d'autres valeurs, vous devrez prendre en compte le décalage de l'origine lorsque vous dessinerez du texte avec cette police.

NOTE Lorsque vous chargez une police dans GameMaker, vous devez penser à la supprimer (avec la touche font_delete()) lorsqu'elle n'est plus nécessaire, sinon il y a un risque de fuite de mémoire qui ralentira et finira par faire planter votre jeu.

Syntaxe :

font_add_sprite_ext(spr, string_map, prop, sep);

Arguments Type Description
spr Sprite Asset The sprite to add a font based on.
string_map Chaîne de caractères String from which sprite sub-image order is taken.
prop Booléen Set as proportional font or not.
sep Real The space to leave between each letter.

Les retours :

Font Asset

Exemple :

global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2);

Le code ci-dessus va créer une nouvelle police à partir de sprite indexée dans la variable "spr_CalcFont" et stocker l'index de la nouvelle police dans la variable "global.Font". Les sous-images de la police sprite seront mises en correspondance avec la chaîne de caractères spécifiée.
