font_texture_page_size

Cette variable intégrée peut être utilisée pour obtenir ou définir la taille de la page de texture lors de l'utilisation de la fonction font_add(). Lors de l'ajout d'une police à l'aide de cette fonction, GameMaker créera une page de texture cache contenant les glyphes requis jusqu'à la taille définie par cette variable. Le processus est le suivant :

  • Chaque caractère que vous utilisez est placé dans une page de texture à partir de la police.
  • Lorsque la page de textures est pleine (parce que vous utilisez des personnages de grande taille ou beaucoup de personnages différents), GameMaker retire les personnages du site cache et les remplace par les nouveaux dont vous avez besoin.
  • GameMaker essaie de ne pas rendre la texture trop grande parce que tous les appareils ne peuvent pas supporter de grandes textures, et il essaie de ne pas rendre la texture trop petite parce que la suppression et le réajout de caractères à la texture prennent du temps au CPU et font que votre jeu tourne lentement.

Si vous pensez avoir besoin d'une taille de page de texture plus grande ou plus petite que les 1024px utilisés par défaut par GameMaker, vous pouvez définir la valeur en utilisant cette variable intégrée. Si vous n'utilisez pas la fonction font_add(), ce paramètre n'aura aucun effet sur les performances de votre jeu, car les polices ajoutées dans IDE sont mises en cache sur les pages de texture normales (que vous pouvez contrôler à partir de l'éditeur de polices et des options du jeu pour la plate-forme cible).

Syntaxe :

font_texture_page_size

Les retours :

Real (maximum width/height in pixels)

Exemple :

font_texture_page_size = 512;
newfont = font_add("Arial", 24, true, true, 32, 128);

Le code ci-dessus définit la page de texture de la police cache à une taille maximale de 512px, puis ajoute une nouvelle police au jeu.
