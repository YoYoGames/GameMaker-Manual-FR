font_set_cache_size

Lorsqu'il s'agit de mélanger et de colorer des images, HTML5 ne peut pas le faire dynamiquement de la même manière qu'un exécutable. GameMaker doit donc sauvegarder temporairement une copie mélangée des images et la charger en cas de besoin. et les charger en cas de besoin. Cette fonction définit combien de copies mélangées de la police donnée peuvent être mises en cache avant que les anciennes ne soient écrasées. La valeur par défaut est 4.

NOTE pour les polices sprite vous devez utiliser la fonction équivalente pour sprites, sprite_set_cache_size().

Syntaxe :

font_set_cache_size(ind, max);

ArgumentsType Description
ind The index of the font to change the cache size of.
max The maximum number of cached copies of the font that can be stored.

Les retours :

Exemple :

font_set_cache_size(fnt_MainMenu, 2);

Cela va définir la police cache de la police indexée dans la variable "fnt_MainMenu" à 2 exemplaires.
