font_replace

Cette fonction peut être utilisée pour remplacer une police dans votre jeu à partir des polices qui sont installées sur le système sur lequel il fonctionne. Notez que la police remplacée ne peut pas être une police du navigateur d'actifs et doit être une police ajoutée précédemment à l'aide de la fonction font_add().

Lorsque vous remplacez une police de cette manière, vous pouvez définir la taille de la police (en points), ainsi que si la police doit être en gras ou en italique, et vous pouvez également définir la plage de caractères à inclure. Si la fonction échoue, elle échouera silencieusement et la police donnée ne sera pas remplacée.

Pour le module cible HTML5, cette fonction ne peut être utilisée que pour remplacer les polices Web, et ne fonctionnera que si WebGL n' est pas activé - si WebGL est activé, vous devez utiliser une ressource de police générale (c'est-à-dire utiliser draw_set_font() avec une police du navigateur de ressources) ou des polices bitmap (cf. font_add_sprite()).

IMPORTANT En HTML5, le texte en italique et en gras n'est pris en charge que si la ressource de police ou la police Web elle-même contient ces caractères. Ils ne seront pas synthétisés à partir de la police de base.

Sur toutes les autres cibles (non Web), vous pouvez utiliser cette fonction pour remplacer une police à partir d'un fichier. Le fichier doit être inclus dans le paquet de jeu à l'aide de la fonctionnalité Fichiers inclus de GameMaker, et doit être un fichier de police au format *.ttf ou *.otf, utile pour ajouter des polices non standard comme les polices asiatiques ou arabes.

CONSEIL: Si vous incluez un tel fichier de police dans un jeu, assurez-vous qu'il bénéficie d'une licence appropriée pour sa distribution avec le jeu.

Lorsque vous chargez une police à partir d'un fichier de cette manière, la taille de la police est en pixels et les premières et dernières valeurs sont ignorées, ce qui signifie que tous les glyphes de la police seront ajoutés. Les polices ajoutées de cette manière sont assignées à leur propre page de texture, il faut donc faire attention en utilisant cette fonction car elle augmentera le nombre de changements de texture lors du dessin. Il convient également de noter que les polices peuvent apparaître légèrement plus grandes lorsqu'elles sont dessinées, car les glyphes peuvent avoir des parties qui sont dessinées en dehors de la boîte de délimitation définie pour la police. Vous devez également savoir qu'il y a une limitation d'environ 200 glyphes qui peuvent être rendus dans une seule image pour une police particulière.

NOTE Lorsque vous chargez une police dans GameMaker, vous devez penser à la supprimer (avec la touche font_delete()) lorsqu'elle n'est plus nécessaire, sinon il y a un risque de fuite de mémoire qui ralentira et finira par faire planter votre jeu.

Syntaxe :

font_replace(ind, name, size, bold, italic, first, last);

Arguments Type Description
ind Font Asset The index of the (previously added) font to be replaced.
name Chaîne de caractères The name of the font to be used (eg 'Arial'), or the file path if the font is an included *. Fichierttf ou *.otf.
size Real The size of the font - points for Web Fonts, pixels for file fonts.
bold Booléen Whether the font is bold (true) or not (false).
italics Booléen Whether the font is italic (true) or not (false).
first Real The first character to include (if you're unsure, go for 32).
last Real The last character to include (if you're unsure, go for 128).

Les retours :

N/A

Exemple :

font_replace(myfont, "Arial", 24, true, true, 32, 128);

Cela remplacera la police indexée dans la variable "myfont" par une nouvelle police de taille 24pt, utilisant "Arial" et qui est en gras et en italique. La gamme de polices comprend les lettres majuscules et minuscules, les chiffres et toute la ponctuation courante.
