font_add_sprite

Avec cette fonction, vous pouvez utiliser une "bande de sprites" pour créer une nouvelle ressource de police, où chaque sous-image serait un symbole ou une lettre individuelle. L'index est le numéro de carte UTF8 du premier caractère à utiliser et les plages à utiliser sont les mêmes que pour une police normale, en commençant par " !", mais n'oubliez pas que vous ne devez inclure que ce dont vous avez besoin et que vous devez utiliser la fonction ord() pour obtenir la valeur UTF8 correcte pour la lettre de départ. La police sprite créée aura alors les sous-images définies à un glyphe individuel de la police, où le "premier" glyphe aura l'image_index de 0. Vous pouvez définir "espace" comme étant n'importe quel caractère que vous voulez, par exemple une ligne unique de la taille que vous voulez que l'espace soit, et tous les espaces dans le texte seront rendus à cette largeur (mais l'image choisie ne sera jamais rendue), ou si vous ne fournissez pas un sprite pour l'espace alors la largeur du caractère le plus large dans la police sprite sera utilisée à la place.

Vous pouvez également définir l'espacement de la police pour qu'elle soit proportionnelle (true) ou non (false). Une police proportionnelle est espacée en fonction de la largeur réelle des lettres (la lettre "i" prend moins de place que la lettre "w", par exemple), tandis qu'une police non proportionnelle sera espacée en fonction de la largeur de la sous-image, comme une police monospace. Enfin, vous pouvez définir l'espace à laisser entre chaque lettre lors de l'écriture. Il peut s'agir d'une valeur entière, 0 correspondant à l'absence d'espace (les lettres se "toucheront" si elles sont proportionnelles).

La fonction renvoie une valeur d'index qui doit être stockée dans une variable car elle sera nécessaire dans tous les codes ultérieurs faisant référence à cette police, ou elle renvoie -1 si la fonction échoue pour une raison quelconque.

Il convient également de noter qu'il ne peut y avoir d'espace dans la carte de caractères, de sorte que si, par exemple, vous utilisez '.' et '0' mais pas '/', vous devez soit inclure '/' de toute façon, soit laisser un sous-image vide entre les deux caractères utilisés.

Le site sprite lui-même peut être un actif sprite provenant du navigateur d'actifs ou un actif chargé à partir d'un fichier en utilisant d'abord sprite_add pour obtenir l'index de l'actif chargé sprite (qui est ensuite utilisé dans la fonction). Il est important de noter que, pour que les fonctions d'alignement des polices (telles que draw_set_halign()) fonctionnent comme prévu lorsqu'elles utilisent une police sprite, l'origine de sprite doit être réglée sur le coin supérieur gauche. Si vous utilisez d'autres valeurs, vous devrez prendre en compte le décalage de l'origine lorsque vous dessinerez du texte avec la police.

NOTE Lorsque vous chargez une police dans GameMaker, vous devez penser à la supprimer (avec la touche font_delete()) lorsqu'elle n'est plus nécessaire, sinon il y a un risque de fuite de mémoire qui ralentira et finira par faire planter votre jeu.

Syntaxe :

font_add_sprite(spr, first, prop, sep);

Arguments Type Description
spr Sprite Asset The sprite to add a font based on.
first Real The index of the first available character.
prop Booléen Set as proportional font or not.
sep Real The space to leave between each letter.

Les retours :

Font Asset

Exemple :

global.Font = font_add_sprite(spr_HUD_Font, ord("!"), true, 2);

Le code ci-dessus crée une nouvelle ressource de police à partir d'une ressource sprite et stocke l'index de la nouvelle police dans la variable "global.Font" pour une utilisation ultérieure.
