display_set_gui_size

Cette fonction n'affecte que les éléments dessinés dans l'événement Draw GUI et peut être utilisée pour définir une largeur et une hauteur spécifiques pour tous les composants GUI dans cet événement. d'affichage ou de fenêtre utilisée pour afficher l'interface graphique, celle-ci sera mise à l'échelle pour s'adapter correctement. La position de l'interface graphique (c'est-à-dire la position (0,0)) dépend de l'utilisation ou non de la fonction display_set_gui_maximise() ou non. Si vous n'utilisez pas cette fonction (paramètre par défaut), la couche GUI sera positionnée avec son point (0,0) au même endroit que la surface de l'application (la zone où votre jeu est dessiné) et la largeur et la hauteur seront étirées pour s'adapter à cette surface. Ainsi, si vous définissez une largeur d'interface graphique de 256 pixels avec cette fonction alors que la surface d'application est de 512 pixels, les 256 pixels seront étirés pour s'adapter, ce qui aura pour effet de doubler la taille des pixels. C'est idéal pour les jeux dont la correction du rapport hauteur/largeur est définie dans les paramètres globaux du jeu.

Si vous avez utilisé la fonction display_set_gui_maximise(), la position (0,0) de la couche de l'interface graphique correspondra par défaut au coin supérieur gauche de la fenêtre du jeu ou de l'écran (en plein écran ou sur les périphériques), avec le facteur d'échelle défini par cette fonction, et la largeur et la hauteur définies par celle-ci. Vous pouvez réinitialiser la couche GUI à tout moment à la configuration par défaut en appelant cette fonction en utilisant -1 comme largeur et hauteur - ceci définira la position (0, 0) à la surface de l'application et la largeur et la hauteur seront réinitialisées à 1. et la hauteur seront également réinitialisées à 1:1 avec la surface.

Syntaxe :

display_set_gui_size(width, height);

ArgumentsType Description
width The width of the GUI
height The height of the GUI

Les retours :

Exemple :

display_set_gui_size(768, 1024);

Le code ci-dessus verrouille l'événement draw GUI sur la largeur et la hauteur données, mettant à l'échelle tous les composants pour qu'ils s'adaptent soit à la surface de l'application, soit à l'écran, en utilisant cette proportion.
