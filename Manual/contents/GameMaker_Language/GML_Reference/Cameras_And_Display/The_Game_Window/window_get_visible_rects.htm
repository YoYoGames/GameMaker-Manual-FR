window_get_visible_rects

Avec cette fonction, vous pouvez trouver la région de chevauchement du rectangle défini par (x1,y1) à (x2,y2) sur chacun des écrans joints. La fonction renvoie un tableau de 8 valeurs par écran (c'est-à-dire que si vous avez deux écrans, le tableau aura une longueur de 16 indices). longueur de 16 indices), où les valeurs [0 ... 3] correspondent aux valeurs overlapx1, overlapy1, overlapx2, overlapy2 - définissant la région de chevauchement sur cet affichage et seront définies à 0,0,0,0 si aucun chevauchement - et les valeurs [4 ... 7] correspondent aux valeurs monitorx1, monitory1, monitorx2, monitory2 - les coordonnées de l'affichage dans l'espace d'affichage virtuel. Ceci peut être utilisé pour tester si une position de fenêtre sauvegardée va être visible ou non (l'utilisateur peut avoir déconnecté un moniteur externe ou déplacé la fenêtre hors de l'écran, ce qui a laissé la fenêtre visible). déplacé la fenêtre hors de l'écran, ce qui fait que la position de la fenêtre qui a été sauvegardée n'est pas valide), par exemple.

Syntaxe :

window_get_visible_rects(x1, y1, x2, y2);

ArgumentsType Description
x1 The left edge of the rectangle to check
y1 The top edge of the rectangle to check.
x2 The right edge of the rectangle to check
y2 The bottom edge of the rectangle to check.

Les retours :

Exemple :

var wx = window_get_x();
var wy = window_get_y();
var ww = window_get_width();
var wh = window_get_height();
display_data = window_get_visible_rects(wx, wy, wx + ww, wy + wh);
display_num = array_length(display_data) / 8;

Le code ci-dessus va générer un tableau 1D contenu dans la variable display_data contenant les informations sur les écrans, et créer la variable display_num pour contenir le nombre d'écrans actifs trouvés.
