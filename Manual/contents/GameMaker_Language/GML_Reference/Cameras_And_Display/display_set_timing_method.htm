Méthode d'affichage de la durée d'utilisation

Cette fonction peut être utilisée pour définir la méthode de timing à utiliser pour le rendu de votre jeu. La méthode peut être l'une des constantes suivantes :

Constant Description
tm_sleep Utiliser la valeur de la marge de sommeil comme méthode principale de chronométrage
tm_countvsyncs Utiliser la synchronisation vsync comme méthode principale de synchronisation (par défaut pour toutes les plateformes supportées)

La méthode de synchronisation vsync utilise la prise en charge de la synchronisation verticale par la plate-forme cible pour fournir un point d'ancrage pour les calculs de synchronisation de rendu du jeu, tandis que le réglage du mode sur la marge de sommeil essaiera simplement de s'assurer que chaque image dure le temps correct (disons 1/30e ou 1/60e de seconde) en attendant ou en dormant. En général, le timing vsync par défaut donnera les résultats les plus fluides, mais notez que lors de l'utilisation de la méthode vsync, les valeurs de la marge de sommeil sont toujours pertinentes, bien qu'elles aient un impact réduit et nous vous recommandons de les garder à leur valeur par défaut lorsque vous utilisez cette méthode.

Par défaut, sur toutes les plateformes, à l'exception de la PS4, Ubuntu et HTML5, GameMaker utilisera la méthode de synchronisation vsync, tandis que sur les plateformes non prises en charge, seule la synchronisation par marge de sommeil peut être utilisée.

Si vous souhaitez définir la marge de sommeil, vous pouvez le faire en utilisant la fonction display_set_sleep_margin() et vous pouvez trouver quelle méthode de synchronisation est actuellement utilisée avec la fonction display_get_timing_method().

Syntaxe :

display_set_timing_method(method);

Arguments Type Description
method Méthode de temporisation de l'affichage Constante The timing method to use (see the list of constants, above)

Les retours :

N/A

Exemple :

if display_get_timing_method() != tm_sleep
{
    display_set_timing_method(tm_sleep);
    if (display_get_sleep_margin() != 20)
    {
        display_set_sleep_margin(20);
    }
}

Le code ci-dessus vérifiera la méthode de synchronisation et si elle n'est pas définie sur tm_sleep, elle sera définie sur cette méthode et la marge de sommeil sera définie sur 20.
