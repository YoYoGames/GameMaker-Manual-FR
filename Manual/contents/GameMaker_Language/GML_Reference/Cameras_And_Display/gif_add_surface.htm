gif_add_surface

Cette fonction permet d'enregistrer une image dans un fichier GIF à partir d'une surface. Vous fournissez l'ID du fichier GIF (tel que renvoyé par la fonction gif_open()) et un ID de surface (tel que renvoyé par la fonction surface_create()), ainsi qu'un temps de retard entre les images. Le délai est calculé en 1/100e de seconde, mais notez qu'une fois le GIF créé, si vous l'utilisez à des fins promotionnelles par exemple, différents navigateurs peuvent interpréter le délai entre les images de manière légèrement différente - voir ici pour plus de détails. Vous pouvez (facultativement) fournir des valeurs de décalage X et Y pour la surface utilisée pour l'ajout au GIF, auquel cas la partie dessinée au GIF commencera à partir du décalage et non de la position par défaut (0,0), ce qui vous permet de sélectionner uniquement une section de la surface à ajouter. Vous pouvez également fournir une valeur de quantification, qui est un nombre entier compris entre 0 et 3, où 0 correspond à une quantification complète et 3 à une quantification nulle (les valeurs inférieures auront une dégradation ultérieure de la qualité, mais créeront un GIF plus petit). Notez qu'il n'y a pas beaucoup de différence entre la qualité par défaut de 2, et la plus élevée de 3, et l'utilisation de 3 ralentira significativement la création du GIF, donc il faut faire attention en utilisant la valeur maximale.

Notez qu'il n'y a pas de limites intégrées sur le nombre d'images ou sur la taille de ces images dans votre GIF, mais si vous utilisez trop de mémoire, la fonction échouera et retournera -1, sinon elle retournera 0.

Syntaxe :

gif_add_surface(gif_index, surface, delay_time, [xoffset], [yoffset]);

Arguments Type Description
gif_index ID GIF The ID of gif to add the surface data to
surface ID de la surface The ID of the surface to use as the added frame
delay_time Real The delay time for the frame (in 100ths of a second)
[xoffset] Real OPTIONAL Le décalage le long de l'axe X de la surface à ajouter à partir de
[yoffset] Real OPTIONAL Le décalage le long de l'axe Y de la surface à ajouter à partir de
[quantization] Real OPTIONAL La quantité de quantification de 0 à 3 (0 qualité la plus faible, 3 qualité la plus élevée, la valeur par défaut est 2).

Les retours :

Real

Exemple :

if save_gif == true
{
    if count == 0
    {
        gif_image = gif_open(room_width, room_height);
    }
    else if count < 30
    {
        gif_add_surface(gif_image, application_surface, 6/100);
    }
    else
    {
        gif_save(gif_image, "GameCapture.gif");
        count = 0;
        save_gif = false;
    }
    count++;
}

Le code ci-dessus va créer un fichier image GIF avec 30 images prises sur la surface de l'application, puis l'enregistrer.
