camera_apply

Cette fonction appliquera les paramètres de caméra donnés à la vue caméra en cours de rendu immédiatement. Bien que vous puissiez définir n'importe quelle vue de caméra à l'aide des fonctions spécifiques de camera_set_*, les nouveaux paramètres ne seront pas utilisés pour le rendu avant la prochaine image du jeu, mais avec cette fonction, vous pouvez effectuer ces changements immédiatement. Si vous utilisez plusieurs vues, la fonction doit être utilisée dans l'événement Draw et vous pouvez utiliser la variable view_current pour appliquer sélectivement la caméra donnée uniquement à la vue de la caméra en cours de rendu. Notez que cela ne modifie PAS la valeur de l'ID de la caméra, mais uniquement les paramètres de cette caméra, et que si vous utilisez cette fonction sur la caméra par défaut (c'est-à-dire la caméra interne que GameMaker utilise lorsqu'aucune caméra de vue n'est active), les modifications seront écrasées dès la prochaine image de jeu par les paramètres internes de dessin de la pièce.

Syntaxe :

camera_apply(camera_id)

ArgumentsType Description
camera_id The unique camera ID value returned when you created the camera.

Les retours :

Exemple :

if view_current == 0
{
    camera_apply(cutscene_cam);
}

Le code ci-dessus (dans un événement de dessin) vérifie quelle vue de caméra est actuellement rendue et s'il s'agit de la vue de caméra [0], les paramètres de la caméra référencée par la variable "cutscene_cam" lui sont appliqués.
