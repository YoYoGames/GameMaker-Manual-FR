mp_potential_path

Cette fonction calcule un chemin pour l'instance depuis sa position et son orientation actuelles jusqu'à la position spécifiée par les arguments xgoal,ygoal. Elle utilise la taille de pas indiquée tout en essayant d'éviter les collisions avec toutes les instances ou seulement avec les avec toutes les instances ou seulement avec celles signalées comme étant solides. Comme dans la fonction mp_potential_step() elle utilise des pas de champ potentiel pour y parvenir et la manière dont elle le fait peut être modifiée par la fonction mp_potential_settings(). Le chemin indiqué doit déjà exister et il sera écrasé par le nouveau chemin. La fonction renvoie si un chemin a été trouvé (true) ou non (false).

Pour éviter que la fonction ne continue à calculer indéfiniment, vous devez fournir un facteur supérieur à 1. La fonction s'arrêtera et signalera un échec si elle ne peut pas trouver un chemin plus court que ce facteur multiplié par la distance entre le départ et le but. Un facteur de 4 est normalement suffisant, mais si vous prévoyez de longs détours, vous pouvez l'augmenter. En cas d'échec, un chemin est tout de même créé dans la direction approximative du but, mais il ne l'atteindra pas.

NOTE: Cette fonction ne déplace pas l'instance. Elle définit uniquement un chemin, et vous devez utiliser les fonctions Path pour le déplacement.

Syntaxe :

mp_potential_path(path, xgoal, ygoal, stepsize, factor, checkall)

ArgumentsType Description
path index of the path that is to be used by the function
xgoal The target x position.
ygoal The target y position.
stepsize The speed the instance moves in pixels per step.
factor This number is used to prevent an infinite loop. It should be larger than 1. See the function description for more information.
checkall Whether to check all instances (true) or just solid ones (false).

Les retours :

Exemple :

path = path_add();
mp_potential_path(path, obj_Player.x, obj_Player.y, 3, 4, 0);
path_start(path, 3, 0, 0);

Le code ci-dessus crée une nouvelle ressource de chemin et stocke son index dans la variable "path". Il génère ensuite un chemin en utilisant mp_potential_path entre l'objet avec ce code et l'objet "obj_Player", en vérifiant qu'il n'y a pas de collisions avec toutes les instances de la pièce. Enfin, elle lance l'objet le long de ce chemin, même si la fonction n'a pas réussi à renvoyer un chemin complet vers l'"obj_Player".
