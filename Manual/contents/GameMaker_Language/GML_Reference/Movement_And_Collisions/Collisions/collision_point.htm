point de collision

Collision_point vérifie que le point spécifié par les arguments x1,y1 n'entre pas en collision avec une instance de l'objet spécifié par l'argument "obj". Cette vérification peut être précise ou non, mais pour que les collisions précises soient activées, l'objet ou l'instance que vous vérifiez doit également avoir des collisions précises activées pour son sprite. Si ce n'est pas le cas, la vérification par défaut est basée sur les boîtes englobantes. L'image suivante illustre ce fonctionnement :

N'oubliez pas que pour que les collisions précises soient prises en compte, l'objet sprite et la fonction de collision doivent tous deux avoir la précision activée. Il convient également de noter que la valeur de retour de la fonction peut être l'identifiant de n'importe laquelle des instances considérées comme étant en collision. Ainsi, si trois instances se chevauchent à cet endroit, n'importe lequel de leurs identifiants peut être la valeur de retour de la fonction.

Notez qu'au lieu d'un indice d'objet, vous pouvez fournir une instance id pour rechercher une instance spécifique, ou les mots-clés d'instance all, ou other (en fonction de l'événement et de la portée actuelle du code).

Syntaxe :

collision_point(x, y, obj, prec, notme);

Arguments Type Description
x Real The x coordinate of the point to check.
y Real The y coordinate of the point to check.
obj ID de l'actif ou de l'instance del'objet The object to check for instance collisions.
prec Booléen Whether the check is based on precise collisions (truece qui est plus lent) ou sa boîte de délimitation en général (false, plus rapide).
notme Booléen Whether the calling instance, if relevant, should be excluded (true) ou non (false).

Les retours :

Instance ID or noone

Exemple :

if collision_point(x, y, obj_Cursor, false, true)
{
    score += 10S;
}

Ici, nous vérifions le point à la position de l'objet qui a le code de l'objet "obj_Cursor". S'il y en a un, alors nous ajoutons 10 à la variable de score.
