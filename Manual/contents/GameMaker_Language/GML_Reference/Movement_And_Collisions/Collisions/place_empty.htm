place_vide

Vous pouvez utiliser cette fonction pour vérifier si l'instance appelante entre en collision avec une autre instance d'un objet ou avec toutes les instances de votre jeu. Pour que cette collision soit résolue correctement, l'instance qui exécute le code doit disposer d'un masque de collision valide (soit pour le site sprite lui-même, soit par le biais de la fonction mask_index) et elle n'enregistrera les collisions qu'avec les instances qui disposent également d'un masque valide.

La fonction teste s'il n'y a pas de collisions si l'instance appelante est placée à une position spécifique, et vous pouvez fournir un argument optionnel pour affiner la vérification et ne vérifier que si une position est exempte de collisions avec des instances du type donné. Notez que si aucun ID d'objet optionnel n'est fourni, la vérification sera effectuée sur toutes les instances de la pièce. La vérification des collisions sera soit précise, soit basée sur la boîte englobante de l'instance, en fonction du type de masque de collision sélectionné, mais pour que les collisions précises fonctionnent correctement, toutes les instances de la vérification doivent avoir des masques de collision précis.

Notez que les coordonnées x/y données seront ramenées à l'entier le plus proche avant que la vérification ne soit effectuée.

Syntaxe :

place_empty(x, y, [object_id]);

Arguments Type Description
x Real The x position to check.
y Real The y position to check.
[object_id] Objet Asset OPTIONAL L'objet à vérifier.

Les retours :

Boolean

Exemple :

if place_empty(mouse_x, mouse_y, obj_Enemy)
{
    x = mouse_x;
    y = mouse_y;
}

Le code ci-dessus vérifie la présence d'une collision avec toute autre instance de l'objet "obj_Enemy", comme si l'instance appelante était placée à la même position que la souris. Si aucune collision n'est détectée, les coordonnées x/y de l'instance sont fixées à celles de la souris.
