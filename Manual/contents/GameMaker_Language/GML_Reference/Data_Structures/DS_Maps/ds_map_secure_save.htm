ds_map_secure_save

Cette fonction enregistre le contenu de la carte DS donnée dans un fichier lié au périphérique sur lequel il a été créé (ce qui signifie qu'il ne peut pas être lu s'il est transféré sur un autre périphérique). Le fichier lui-même peut avoir presque n'importe quelle extension (par exemple, *.dat, *.json, *.bin, etc...) et sera obscurci et stocké sur le stockage local de la plate-forme cible. Vous pouvez ensuite recharger le ds_map en utilisant la fonction ds_map_secure_load(). Notez que si la carte DS en cours de sauvegarde contient un tableau, ce tableau sera converti en une liste DS lors de la sauvegarde.

IMPORTANT ! L'une des caractéristiques d'un fichier sauvegardé sécurisé est qu'il est verrouillé sur l'appareil sur lequel il a été créé. Vous ne pouvez donc pas charger un fichier sauvegardé sur un appareil dans un projet exécuté sur un autre appareil.

Syntaxe :

ds_map_secure_save(map, filename);

ArgumentsType Description
map The id of the data structure to use
filename The name of the file to save the map to

Les retours :

Exemple :

ds_map_secure_save(purchase_map, "p_data.dat");

Le code ci-dessus enregistrera la carte DS indexée dans la variable "p_data" dans le fichier donné pour une récupération ultérieure.
