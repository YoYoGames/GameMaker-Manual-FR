ds_list_mark_as_list

Cette fonction "marque" (ou "signale") une position donnée dans une liste DS précédemment créée comme contenant une autre liste DS. Cette fonctionnalité est nécessaire lors de l'encodage de chaînes JSON (que vous pouvez créer à l'aide de la fonction json_encode()), mais peut également être utile lors de l'imbrication de structures de données, car les éléments marqués de cette manière seront automatiquement récupérés (détruits) lorsque la liste DS parente sera détruite. Cela signifie que vous n'avez pas besoin de parcourir manuellement le contenu de la liste et de détruire les structures de données marquées individuellement avant de détruire la liste "parent". Toutefois, si vous supprimez la position de la liste individuellement, la liste DS qu'elle contenait ne sera pas récupérée et vous devrez appeler la fonction de destruction de la liste DS appropriée avant de supprimer l'élément de la position de la liste parent. Notez également que si vous appelez la fonction ds_list_clear() sur une liste, tous les éléments marqués comme des listes seront également détruits lorsque la liste sera effacée.

Il convient également de noter que si la position qui a été marquée change dans la liste, la "marque" se déplacera avec elle. Ainsi, si vous avez marqué la position 3 dans la liste (par exemple) et que vous insérez ensuite 2 éléments supplémentaires en dessous pour qu'elle passe à la position 5, elle sera toujours marquée comme une liste.

Syntaxe :

ds_list_mark_as_list(id, pos);

ArgumentsType Description
id The id of the list to mark.
pos The position within the list to mark.

Les retours :

DS List ID or -1 (if it fails)

Exemple :

var j_list = ds_list_create();
var sub_list = ds_list_create();
ds_list_add(sub_list, health);
ds_list_add(sub_list, lives);
ds_list_add(sub_list, score);
ds_list_add(j_list, sub_list);
ds_list_mark_as_list(j_list, 0);

Le code ci-dessus crée deux listes DS, puis remplit l'une d'elles avec diverses valeurs provenant de variables globales. Cette liste est ensuite ajoutée à la deuxième liste, et la position est "marquée" comme telle afin qu'elle puisse être correctement encodée par la suite.
