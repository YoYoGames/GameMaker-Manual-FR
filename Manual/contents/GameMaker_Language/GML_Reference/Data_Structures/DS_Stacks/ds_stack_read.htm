ds_stack_read

Avec cette fonction, vous pouvez recréer une pile DS sauvegardée (une pile qui a été précédemment écrite sous forme de chaîne de caractères à l'aide de la fonction ds_stack_write()). Vous devez d'abord créer une nouvelle pile DS pour y lire la chaîne. Si la pile DS existe déjà et contient des informations, celles-ci seront effacées avant la lecture. Cette fonction est d'une importance vitale lors de la création de mécanismes de sauvegarde/chargement pour votre jeu.

Notez que si la chaîne DS spécifiée a été écrite par GameMaker : Studio 1.2.x runtime (ou plus ancien), vous devez spécifier l'argument facultatif "legacy", en lui attribuant la valeur true car le format de la chaîne a changé après cela.

Syntaxe :

ds_stack_read(id, str [, legacy]);

Arguments Type Description
id ID de la pile DS The id of the data structure to read into.
str Chaîne de caractères The string to read from.
legacy Booléen OPTIONAL Peut être soit true ou false, soit être complètement omis.

Les retours :

N/A

Exemple :

stack = ds_stack_create();
ini_open("save.ini");
var str = ini_read_string("Stacks", "0", "");
if (str != "")
{
    ds_stack_read(stack, str);
}
ini_close();

Le code ci-dessus crée une pile et stocke l'index dans la variable "stack". Il ouvre ensuite un fichier ini et y lit une chaîne de caractères, en vérifiant d'abord que la chaîne n'est pas renvoyée comme vide. Cette chaîne est ensuite lue dans la pile DS nouvellement créée.
