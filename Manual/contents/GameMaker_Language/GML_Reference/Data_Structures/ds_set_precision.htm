ds_set_precision

Lors de la comparaison de valeurs, par exemple lors d'une recherche dans une carte ou du tri d'une liste, GameMaker doit décider si deux valeurs sont égales. Pour les chaînes de caractères et les valeurs entières, c'est clair, mais pour les nombres réels, à cause des erreurs d'arrondi en virgule flottante, des nombres apparemment égaux peuvent facilement devenir inégaux. Par exemple, il est possible que (5 / 3) * 3 ne soit pas égal à 5 ! Pour éviter cela, une valeur de précision est utilisée pour toutes les fonctions de nombres réels. Lorsque la différence entre deux nombres est inférieure à cette précision, ils sont considérés comme égaux. Par défaut, une précision de 0,0000001 est utilisée pour toutes les fonctions de structure de données, sauf si elle est modifiée par cette fonction.

NOTE: Cette précision est utilisée dans toutes les structures de données mais pas dans les autres comparaisons dans GML!

Syntaxe :

ds_set_precision(prec);

Arguments Type Description
prec Real The precision value (default 0.0000001)

Les retours :

N/A

Exemple :

ds_set_precision(0.0001);

Le code ci-dessus changera le paramètre de précision par défaut pour toutes les fonctions de structure de données.
