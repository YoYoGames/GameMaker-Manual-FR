joueur_utilisateur_id

Cette variable stocke l'ID GX.games de l'instance de joueur actuelle, pour une utilisation avec des appels API directs au serveur.

Cette variable n'existe que dans les instances de joueur créées par rollback_define_player()et est différente pour chaque joueur.

Lesjoueurs invités se verront attribuer une chaîne vide dans cette variable.

Syntaxe :

player_user_id;

Les retours :

String
