player_local

Lorsque vous utilisez rollback_define_player() pour définir un objet joueur, le système crée automatiquement une instance de cet objet pour chaque joueur. Chaque instance de cet objet reçoit la variable player_local, qui indique si cette instance de joueur est locale ou non.

Si vous n'utilisez pas rollback_define_player(), cette variable ne sera créée pour aucun objet.

Vous pouvez utiliser cette variable pour indiquer quel joueur est local. Il n'est pas recommandé d'utiliser cette variable d'une manière qui affecte votre logique de jeu, qui doit être la même pour les joueurs locaux et distants.

Syntaxe :

player_local

Les retours :

Boolean

Exemple :

// Draw event
if (player_local)
{
    draw_sprite(spr_arrow, 0, x, y - 60);
}

Le code ci-dessus s'exécute dans l'événement Draw, et vérifie si l'instance du joueur est locale. Dans ce cas, il dessine une flèche sprite au-dessus de l'instance, indiquant au joueur quelle instance il contrôle.
