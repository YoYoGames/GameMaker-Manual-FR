Erreurs de retour en arrière

Lorsque vous utilisez le système de retour en arrière, certaines erreurs peuvent se produire. Ces erreurs sont des erreurs Runner et sont donc affichées pendant le déroulement du jeu :

Le tableau suivant répertorie les erreurs possibles que vous pouvez rencontrer lorsque vous travaillez avec le système Rollback, ainsi qu'une description. Il fournit également une solution pour chaque message d'erreur spécifique.

Erreur Description
Impossible d'obtenir la couche pour créer des joueurs. Vous devez passer le nom de la couche à la fonction rollback_define_player ou créer la couche Instances. Si l'objet joueur a été défini avec la fonction rollback_define_player, le système de rollback essaiera de créer l'objet joueur sur la couche, dont le nom peut éventuellement être passé comme second paramètre (ou sur la couche avec le nom "Instances" si elle n'est pas définie). Si la couche n'a pas pu être trouvée par son nom, cette erreur est déclenchée. La solution est soit de définir explicitement le nom de la couche sur laquelle créer les joueurs, soit de créer une couche avec le nom "Instances" si vous ne définissez pas explicitement le nom de la couche.
Les objets des joueurs doivent être gérés lors de l'exécution de jeux multi-joueurs. Si l'objet lecteur a été défini avec la fonction rollback_define_player et que l'objet lecteur n'est pas marqué comme géré, cette erreur est déclenchée. La solution consiste à marquer l'objet lecteur comme géré dans l'éditeur d'objets.
Échec de la récupération des informations utilisateur pour le joueur_id N. Si le système de retour en arrière n'a pas pu obtenir du serveur des informations sur le joueur, cette erreur est déclenchée. Vérifiez la console du développeur pour voir si des requêtes ont été bloquées.
Impossible d'appeler rollback_chat avant d'être entièrement connecté au jeu. Essayez d'attendre l'événement rollback_game_info. La fonction de chat n'est pas disponible tant que le joueur n'a pas rejoint la salle. Solution : Désactivez le chat jusqu'à la réception de l'événement rollback_game_info (voir Événements Rollback).
rollback_use_manual_start doit être appelé avant rollback_create_game et rollback_join_game. Ces erreurs sont déclenchées si un appel de fonction viole les contraintes internes de la machine d'état. Le message d'erreur suggère une solution.
rollback_use_player_prefs doit être appelé avant rollback_create_game et rollback_join_game.
rollback_use_player_prefs doit être appelé avant rollback_set_player_prefs.
rollback_set_player_prefs doit être appelé avant rollback_start_game.
rollback_define_player doit être appelé avant rollback_create_game et rollback_join_game.
rollback_define_input doit être appelé avant rollback_create_game et rollback_join_game.
rollback_define_mock_input doit être appelé avant rollback_create_game et rollback_join_game.
Ne pas lire les informations sur le joueur avant le début de la partie. Appel à rollback_get_info().
Ne pas lire les entrées du joueur avant le début du jeu. Appel à rollback_get_input().
rollback_define_input_frame_delay doit être appelé avant rollback_create_game et rollback_join_game
rollback_define_input_frame_delay doit être appelé avant rollback_create_game et rollback_join_game.
rollback_set_player_prefs n'a pas réussi à définir les préférences Le serveur a rejeté les préférences du joueur. Cela se produit généralement dans les cas suivants : 1. Un utilisateur essaie de réinitialiser les préférences du joueur alors que l'autre utilisateur a déjà commencé le jeu. Dans ce cas, le jeu doit utiliser les dernières préférences utilisateur définies avec succès, nous garantissons que chaque joueur doit définir des préférences utilisateur avant que le jeu puisse être lancé. 2. La taille des préférences du joueur sérialisées dépasse la taille maximale des préférences du joueur (actuellement 800 octets).
rollback_create_game, nombre de joueurs fixé à N mais devant être compris entre 1 et 4 La fonction rollback_create_game a été appelée avec un nombre de joueurs incorrect. La solution consiste à corriger le premier paramètre de la fonction rollback_create_game pour qu'il soit compris entre 1 et 4.
rollback_create_game n'a pas réussi à définir les préférences par défaut du joueur. Voir la description de "rollback_set_player_prefs failed to set the preferences".
Le retour en arrière multi-joueurs n'est supporté que dans la cible operagx. La fonction rollback_create_game est appelée et la cible n'est pas définie sur OperaGX. Actuellement, le multijoueur n'est supporté que sur la plateforme OperaGX. La solution consiste à définir la cible sur OperaGX.
Échec de la création d'une partie (type de session inconnu). Il s'agit d'une erreur interne du système de retour en arrière. Signalez un problème à l'équipe GameMaker.
Impossible de trouver le player_id sur cette instance, vous devez le passer à rollback_get_input(). La fonctionrollback_get_input peut accepter l'identifiant du joueur comme paramètre. Si aucun paramètre n'a été fourni à la fonction, elle prendra le champ player_id d'un objet joueur pour obtenir des entrées. Si le paramètre player_id n'est pas défini, cette erreur est déclenchée. Si l'objet joueur a été créé avec la fonction rollback_define_player, le paramètre player_id doit être défini. Solution : 1. Passez explicitement le paramètre player_id à la fonction 2. Créez le lecteur avec rollback_define_player.
Info player_id N est hors de portée. rollback_get_info est appelé avec un mauvais player_id (ou l'objet joueur a une mauvaise valeur dans le champ player_id). Solution : Corrigez le paramètre player_id pour qu'il soit compris entre 1 et 4.
Impossible de trouver l'instance <id> de l'objet <id> (<name>) référencé dans les données du rollback. Erreur interne du système de retour en arrière. Signalez un problèmeà l'équipe GameMaker.
Aucune idée de la façon de créer OBJECT_KIND=<kind>.
RefID <id> n'a pas d'objet (entrée malformée ?).
Invalid ReferenceKind <id> (entrée malformée ?).
Le mappage d'instance est activé mais nous avons essayé de désérialiser un index hors de portée. <id> n'est pas dans [0, <id>].
Valeur RV invalide lors de la désérialisation.
Essayer de ressusciter une instance qui a été supprimée !
Impossible de sérialiser RValue avec kind=<id> (<name>).
Définition d'entrée vide. rollback_define_input a été appelé avec une entrée vide.
La définition d'entrée contient la constante <code clé> deux fois, ce qui n'est pas autorisé. Un code clé ne peut être associé qu'à une seule entrée. Vérifiez si un code clé est défini plus d'une fois dans la définition de l'entrée.
La définition d'entrée avec l'étiquette <label> n'est pas une valeur réelle ou un tableau de réels. Toutes les valeurs décrivant les valeurs d'entrée doivent avoir le type Real.
Définition d'entrée fictive vide pour le joueur_id N. Voir "Définition des entrées vides".
La définition fantaisie contient une valeur non réelle. Voir "La définition d'entrée avec l'étiquette <label> n'est pas une valeur réelle ou un tableau de réels".
