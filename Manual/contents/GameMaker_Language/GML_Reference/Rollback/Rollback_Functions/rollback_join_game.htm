rollback_join_game

Cette fonction tente de rejoindre une partie. Vous devez être sur une URL qui contient l'ID de la salle virtuelle à rejoindre.

Si une salle a été jointe, elle renvoie true, et si aucune partie n'a été jointe, elle renvoie false.

Lorsqu'un nouveau joueur rejoint une partie dans laquelle vous êtes déjà présent, l'événement Rollback Event est déclenché. Voir Événements de retour en arrière pour plus d'informations.

L'argument dry_run est facultatif, et lorsqu'il vaut true, il vous permet d'exécuter la fonction sans rejoindre une partie. Ceci est utilisé pour vérifier si un jeu peut être rejoint sans le rejoindre réellement.

Syntaxe :

rollback_join_game([dry_run]);

Arguments Type Description
dry_run Booléen Disabled by default (false). Lorsqu'elle est activée(true), la fonction ne rejoint pas la partie, mais renvoie simplement true si une partie peut être rejointe, ou false sinon.

De cette façon, vous pouvez utiliser cette fonction pour vérifier uniquement si le joueur a été invité.

Les retours :

Boolean

Exemple :

rollback_define_player(obj_player);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_create_game(2, false);
}

Le code ci-dessus définit un objet joueur, et tente ensuite de rejoindre une partie. S'il n'a pas été rejoint, il crée une nouvelle partie à la place, avec un maximum de 2 joueurs et un test de synchronisation désactivé.

Pour un exemple détaillé, voir Créer un jeu multi-joueurs.
