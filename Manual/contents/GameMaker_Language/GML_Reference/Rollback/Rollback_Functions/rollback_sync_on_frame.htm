rollback_sync_on_frame

Cette fonction vérifie si tous les joueurs sont synchronisés sur la trame actuelle, ce qui signifie que leurs états ne sont pas basés sur des prédictions mais sur les données réelles reçues de leur part. Dans ce cas, elle renvoie true.

Si les joueurs ne sont pas synchronisés sur la trame actuelle, il renvoie false, et tente de synchroniser tous les joueurs.

Tenter de synchroniser des lecteurs avec cette fonction peut provoquer un gel. Vous devez vous assurer que cette fonction n'est pas appelée en permanence dans un événement Step.

Utilisez-le plutôt comme une bascule. Voir l'exemple au bas de la page.

La variable rollback_confirmed_frame est mise à jour automatiquement lorsque tous les joueurs sont synchronisés.

Syntaxe :

rollback_sync_on_frame();

Les retours :

Boolean

Exemple :

if (obj_game.winning_player == -1 && points >= obj_game.points_required_for_win)
{
    if (rollback_sync_on_frame())
    {
        obj_game.winning_player = player_id;
    }
}

Ce code s'exécute dans l'événement Step d'un objet joueur.

Il vérifie s'il n'y a pas de joueur gagnant, et si le joueur actuel a les points requis pour gagner.

Si les deux sont vrais, cela signifie que le joueur devrait gagner. Cependant, avant de faire cela, il vérifie si tous les joueurs sont synchronisés, pour s'assurer que l'état est correct.

Si tous les joueurs sont synchronisés, il définit la variable winning_player. La première condition devient alors fausse, ce qui garantit que rollback_sync_on_frame() ne sera pas appelé à nouveau.

Cela signifie que la fonction est utilisée comme une bascule, au lieu de fonctionner en continu à chaque image.
