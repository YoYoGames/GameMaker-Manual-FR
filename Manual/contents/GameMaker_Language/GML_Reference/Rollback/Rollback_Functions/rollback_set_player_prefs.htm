rollback_set_player_prefs

Après avoir activé les préférences, cette fonction est utilisée pour définir les préférences du lecteur local. Cela signifie que la fonction doit s'appuyer sur l'entrée locale et non sur l'entrée Rollback.

Les préférences peuvent être de n'importe quel type. Si des préférences par défaut sont définies, assurez-vous d'utiliser le même type de données et la même structure.

Lorsqu'un joueur définit ses préférences, tous les participants au jeu reçoivent un événement de retour en arrière.

Si elle est utilisée, cette fonction doit être exécutée avant la fonction "joindre/créer".

Veuillez lire : Préférences des joueurs

Syntaxe :

rollback_set_player_prefs(preferences);

Arguments Type Description
preferences Tout The preferences to set for the local player.

Les retours :

N/A

Exemple :

rollback_set_player_prefs({
character : obj_customizer.character_selected,
couleur : obj_customizer.color_selected
}) ;

Le code ci-dessus définit les préférences du lecteur, en lisant les valeurs d'un objet "customizer", qui serait non géré (local).
