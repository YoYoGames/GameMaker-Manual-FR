délai de définition de l'image d'entrée du retour arrière

Cette fonction définit le délai d'image fixe pour les entrées du lecteur local.

Par défaut, votre jeu utilisera un délai d'entrée adaptatif, basé sur la latence entre les joueurs. Cela fonctionnera pour presque tous les jeux, mais certains types de jeux spécifiques (par exemple les jeux de combat) préféreront utiliser un délai d'entrée fixe plutôt qu'un délai adaptatif qui peut changer à tout moment.

Il n'est pas recommandé de le modifier, sauf si vous savez exactement ce que vous faites. Vous pouvez appeler cette fonction à tout moment avant et pendant une partie multi-joueurs.

La valeur du délai spécifiée doit être en nombre d'images. Si vous spécifiez -1, le jeu utilisera un délai adaptatif. Le délai maximal que vous pouvez définir est de 10 images.

Syntaxe :

rollback_define_input_frame_delay(frames);

Arguments Type Description
frames Real The number of frames to set as the fixed input delay, or -1 pour le remettre à sa valeur par défaut.

Les retours :

N/A

Exemple :

// 'Options' script
global.frame_delay = 5;

// obj_game: Create event
rollback_define_input_frame_delay(global.frame_delay);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_start_game(2, false);
}

Vous pouvez définir votre valeur de délai d'image comme une variable globale, et l'utiliser en appelant rollback_define_input_frame_delay() avant qu'une partie de rollback ne soit lancée.
