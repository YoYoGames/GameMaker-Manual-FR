rollback_define_player

Cette fonction définit l'objet qui doit être utilisé pour les joueurs en mode multi-joueurs. Elle est facultative, et lorsqu'elle est utilisée, le système crée automatiquement des instances pour les joueurs qui se connectent, et détruit les instances pour les joueurs qui se déconnectent.

Par défaut, cela crée des instances de joueurs dans la couche "Instances", qui doit exister dans votre salle de jeu. Si vous souhaitez utiliser une autre couche, passez son nom comme deuxième argument.

Pour plus d'informations sur la création d'un jeu avec ou sans cette fonction, voir Définition d'un objet joueur. Si elle est utilisée, cette fonction doit être exécutée avant la fonction join/create.

Chaque instance créée avec cette fonction reçoit les variables d'instance listées sur cette page.

Syntaxe :

rollback_define_player(object, [layer_name]);

Arguments Type Description
object Objet Asset The object that will be used for creating player instances.
layer_name Chaîne de caractères OPTIONAL Le nom de la couche où les instances de joueurs seront créées. "Instances" par défaut.

Les retours :

N/A

Exemple :

rollback_define_player(obj_player);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_start_game(2, false);
}

Le code ci-dessus définit un objet joueur, et tente ensuite de rejoindre une partie. S'il n'a pas été rejoint, il démarre une nouvelle partie à la place, avec un maximum de 2 joueurs et un test de synchronisation désactivé.
