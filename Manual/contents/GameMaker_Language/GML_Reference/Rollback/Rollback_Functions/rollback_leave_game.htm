annuler_sortir_du_jeu

Cette fonction quitte la session multijoueur en cours, permettant au joueur de rejoindre ou de créer à nouveau une nouvelle partie.

Lorsque l'hôte d'une salle quitte, la propriété de cette salle est transférée à un autre joueur, de sorte que le jeu peut continuer. Le jeu ne se termine que lorsque tous les joueurs sont partis.

Éviter les erreurs

Lorsque le joueur quitte une session multijoueur, GameMaker continue d'exécuter tout le code du jeu, mais il revient à l'état "pré-démarrage" dans lequel vous ne pouvez pas créer ou modifier les instances gérées.

Cela signifie que vous devez détruire manuellement vos instances de joueur lorsque vous quittez une partie, et mettre en pause toute autre logique de jeu en utilisant rollback_game_running dans une condition, pour éviter les erreurs.

Voir : État avant le début du rollback

Événement déclenché

Lorsqu'un joueur quitte une partie dans laquelle vous êtes présent, l'événement Rollback Event est déclenché. Voir Événements de retour pour plus d'informations.

Syntaxe :

rollback_leave_game();

Les retours :

N/A

Exemple :

if (keyboard_check_pressed(vk_escape))
{
    rollback_leave_game();
}

Le code ci-dessus permet au joueur de quitter le jeu lorsqu'il appuie sur la touche Echap.
