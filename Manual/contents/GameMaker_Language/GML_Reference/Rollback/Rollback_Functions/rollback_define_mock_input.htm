rollback_define_mock_input

Cette fonction peut être utilisée pendant le test de synchronisation pour attribuer une entrée temporaire "factice" au lecteur spécifié. Cela remplace les contrôles définis pour le lecteur et toute entrée aléatoire provenant de Sync Test.

Pour plus d'informations sur son utilisation, voir Mock Input.

Syntaxe :

rollback_define_mock_input(player_id, input_struct);

Arguments Type Description
player_id Real The ID of the player to assign mock input to.
input_struct Structure A struct containing input names as variables, and their assigned inputs as values.

Les retours :

N/A

Exemple :

rollback_define_input({
    fire: mb_left,
    interact: vk_space,
    left: ord("A"),
    right: ord("D")
});

rollback_define_mock_input(1, {
    fire: vk_control,
    interact: vk_shift,
    left: ord("J"),
    right: ord("L")
});

Le code ci-dessus définit d'abord les entrées pour les joueurs. Il définit ensuite une entrée fictive pour le joueur 1, en conservant les mêmes noms d'entrée mais en attribuant des entrées différentes. De cette façon, deux personnes peuvent jouer au jeu localement en utilisant le clavier.
