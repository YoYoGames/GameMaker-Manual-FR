Préférences de retour en arrière

Vous souhaitez peut-être permettre à chaque joueur de sélectionner un personnage ou de personnaliser ses propriétés. Ces propriétés peuvent être définies comme "préférences du joueur".

Comment utiliser les préférences ?

Tout d'abord, activez les préférences en appelant rollback_use_player_prefs() / Use Player Prefs. Cette fonction vous permet de définir les préférences par défaut pour tous les joueurs.

Une fois que vous avez appelé cette fonction, le "démarrage automatique" est désactivé, ce qui signifie que le jeu ne démarrera pas, même si tous les joueurs ont rejoint la partie, et que vous devrez le démarrer manuellement avec la commande rollback_start_game() / Start Game.

Avant que le jeu ne commence, vous pouvez définir les préférences d'un joueur en appelant rollback_set_player_prefs() / Set Player Prefs sur le client de ce joueur. Celles-ci peuvent être lues plus tard par le biais de rollback_get_player_prefs() / Get Player Prefs, ou la variable player_prefs dans une instance de joueur.

Événement de retour en arrière

Lorsqu'un joueur modifie ses préférences, tous les participants au jeu reçoivent un événement de retour en arrière rollback_player_prefs. Le joueur doit être entièrement connecté pour que l'événement soit envoyé.

Si les préférences ont été définies avant la connexion, les dernières préférences définies avant la connexion sont envoyées à tous les joueurs à la fin de la connexion.

Condition de départ

Si vous avez activé les préférences et qu'aucune préférence par défaut n'a été définie, le jeu ne démarrera pas tant que les préférences de tous les joueurs n'auront pas été définies.

Dans ce cas, vous pouvez soit appeler rollback_start_game() de façon répétée dans une alarme pour essayer de démarrer le jeu, ou l'appeler à chaque fois que l'événement Rollback_player_prefs est reçu.

Quels types puis-je utiliser ?

Une préférence peut être de n'importe quel type, par exemple un nombre, une chaîne, un tableau, etc., mais il est recommandé d'utiliser une structure, qui est le moyen le plus simple de stocker une collection de variables.
