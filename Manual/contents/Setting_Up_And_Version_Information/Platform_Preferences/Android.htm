Préférences Android

Les préférences d'Android sont divisées en plusieurs sections énumérées ci-dessous :

En haut des préférences, vous pouvez définir la taille maximale du tas de mémoire du JDK (2 Go par défaut). Ensuite, vous devez configurer les différents chemins d'accès afin que GameMaker sache où chercher les différents outils nécessaires pour construire le paquet exécutable final et tester votre jeu. Ceci est divisé en trois sections pour le SDK Android, le NDK Android et le JDK Java. Ces chemins devraient avoir été remplis automatiquement pour vous, mais si vous avez des problèmes, vous pouvez cliquer sur le bouton à droite pour ouvrir l'explorateur de fichiers et sélectionner manuellement les emplacements de chaque élément. Vous pouvez trouver des détails sur les versions requises pour chacun des éléments du SDK ici. Les chemins d'accès devraient être validés au fur et à mesure que vous les appliquez et un message indiquera "Found" ou vous informera de toute erreur (auquel cas vous devrez revoir toutes les informations pour vous assurer qu'elles sont correctes).

En dessous de cela, vous avez les différentes options d'emballage qui sont :

  • Toujours faire une installation complète de l'APK: si vous marquez cette option, GameMaker supprimera de votre appareil toute version précédente de votre jeu ( runners ), ainsi que tous les fichiers associés (par exemple, les fichiers *.ini ) de votre appareil de test avant d'installer une nouvelle version du jeu.
  • Installer sur le paquet: Lorsque cette option est cochée, la compilation d'un paquet final exécutable *.apk l'installera aussi automatiquement sur le dispositif cible actuellement sélectionné.

Enfin, nous arrivons au KeyStore. Il s'agit d'un fichier qui sera utilisé pour "signer" toutes vos applications Android. Veuillez donc remplir tous les détails correctement et gardez à l'esprit que ce fichier sera nécessaire pour créer et mettre à jour toutes vos applications Android à l'avenir, ce qui signifie qu'après l'avoir configuré, . il est conseillé de faire une capture d'écran des paramètres utilisés et de faire une sauvegarde du fichier final.. Si vous perdez ce fichier, vous ne serez pas en mesure de mettre à jour les jeux existants qui ont été téléchargés sur la boutique.

Pour créer un nouveau fichier keystore, les informations suivantes sont nécessaires :

  • Filename: Il s'agit du nom du fichier keystore.

REMARQUE: il ne s'agit pas de votre nom ou du nom de votre société, mais du nom du fichier à générer ! Si vous disposez déjà d'un fichier keystore provenant de projets précédents, vous pouvez cliquer sur le bouton du navigateur de fichiers ici et diriger GameMaker vers le fichier keystore précédemment créé (ou cliquer sur le bouton Importer ). Dans ce cas, vous devez toujours remplir le reste des informations requises (exactement comme lorsque vous avez généré le fichier pour la première fois) mais vous ne devez pas cliquer sur le bouton Générer le keystore ! Cela générera un nouveau fichier keystore, qui écrasera le précédent.

  • Nom commun: Il s'agit normalement de votre nom.
  • Mot de passe: votre mot de passe de sécurité pour le fichier keystore, qui doit comporter au moins six caractères.
  • Alias: Il s'agit d'un autre nom qui peut être utilisé pour le fichier keystore et qui doit être différent du nom saisi ci-dessus.
  • Mot de passe alias: votre mot de passe de sécurité alias pour le fichier keystore, qui doit comporter au moins six caractères.

    REMARQUE Si votre version de Android Studio est 4.2 ou supérieure, vous devez utiliser le même mot de passe pour les champs "Password" et "Alias Password" ci-dessus. L'utilisation d'un mot de passe et d'un mot de passe alias différents entraînera l'échec de vos constructions.
  • Unité organisationnelle: Le département dont vous faites partie au sein de l'entreprise.
  • Organisation: Le nom de votre entreprise.
  • Localisation: Le nom de la ville où vous êtes basé.
  • Pays: Le code standard de deux lettres pour le pays où vous êtes basé.

Une fois que ces informations sont remplies et que vous en êtes satisfait, cliquez sur le bouton " Generate Keystore for GameMaker " pour créer le fichier keystore nécessaire. Cela peut prendre un moment, mais une fois terminé, il n'est normalement pas nécessaire de le modifier à nouveau. Notez que vous pouvez cliquer sur le bouton Importer pour importer un fichier keystore précédemment créé (voir la section Nom de fichier , ci-dessus) et que vous pouvez également récupérer le hachage SHA1 du keystore en cliquant sur le bouton Générer le hachage de la clé . En appuyant sur ce bouton, vous générerez deux clés de hachage de sécurité spéciales directement liées à votre fichier keystore, que certains sites Web exigent pour fonctionner correctement (par exemple, Facebook ou Google Play pour le hachage Sha1). Si vous en avez besoin, appuyez simplement sur le bouton, puis copiez et collez les clés de hachage générées.
