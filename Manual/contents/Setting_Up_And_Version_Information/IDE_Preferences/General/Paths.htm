Préférences du chemin

Les préférences Path contrôlent les différents chemins de fichier requis par les éléments de GameMaker IDE ainsi que pour les éditeurs externes dont vous avez besoin. Les options suivantes existent pour l'IDE :

  • Répertoire temporaire: L'emplacement pour sauvegarder tous les dossiers temporaires. Par défaut, sur Windows, il s'agit de

    \Users\<Username>\AppData\Local\Temp\GameMaker

    Et sur macOS c'est :

    /var/folders/<Path_Hash>/GameMaker

  • Répertoire de l'IDE cache : L'emplacement pour la sauvegarde de IDE cache. Par défaut, sur Windows, il s'agit de

    \Users\<Username>\AppData\Roaming\GameMaker\Cache

    Et sur macOS c'est :

    /Users/<Username>/.config/GameMaker/Cache

  • Répertoire des actifs cache : L'emplacement pour sauvegarder l'actif cache pour chaque projet. Par défaut, sur Windows, il s'agit de

    \Users\<Username>\AppData\Roaming\GameMaker\Cache

    Et sur macOS c'est :

    /Users/<Username>/.config/GameMaker/Cache

  • Emplacement " Mes projets ": L'emplacement où GameMaker créera initialement les nouveaux projets. Par défaut, sur Windows, il s'agit de

    \Users\<Username>\Documents\GameMaker

    Et sur macOS, c'est

    /Users/<Username>/GameMaker

  • Supprimer automatiquement le répertoire temporaire à la fermeture: Si vous cochez cette option, GameMaker supprimera automatiquement le dossier temporaire qu'il crée par projet pour la compilation, etc... Ce paramètre est activé par défaut, et le décocher le désactivera (ce qui signifie que vous devrez supprimer manuellement tous les fichiers temporaires plus tard).
  • Supprimer automatiquement le dossier asset cache à la fermeture: Si cette option est cochée, le dossier du compilateur d'actifs cache sera supprimé lorsque vous quitterez GameMaker. Cette option est désactivée par défaut, et l'activer signifiera qu'à chaque fois que vous chargerez et exécuterez un projet, le dossier cache devra être reconstruit (ce qui peut prendre du temps en fonction de la taille du jeu).
  • Supprimer automatiquement IDE cache à la fermeture: Si cette option est cochée, le dossier du compilateur IDE cache sera supprimé lorsque vous quitterez GameMaker. Cette option est désactivée par défaut, et si vous l'activez, le dossier cache devra être reconstruit à chaque fois que vous démarrez GameMaker.
  • Supprimer le dossier temporaire: En cliquant sur ce bouton, vous supprimez le dossier temporaire du projet.
  • Supprimer le cache des actifs: Cliquer sur ce bouton supprimera le compilateur d'actifs cache pour le projet.
  • Supprimer le cache de IDE : En cliquant sur ce bouton, le cache de IDE sera supprimé.

Les options suivantes existent pour définir les chemins d'accès aux éditeurs externes :

  • Chemin d'accès à l'éditeur/visualisateur externe pour les fichiers SWF: Si vous travaillez avec le format SWF sprites, vous pouvez définir le chemin de votre visionneur/éditeur préféré et lorsque vous cliquez sur le bouton Modifier l'image dans l'éditeur Sprite, il ouvrira le programme donné. La valeur par défaut est de ne pas avoir de chemin d'accès.
  • Chemin d'accès à un éditeur/visualisateur externe pour les fichiers Spine : Si vous travaillez avec le format Spine sprites , vous pouvez indiquer ici le chemin d'accès à votre visualiseur/éditeur préféré et lorsque vous cliquez sur le bouton Modifier l'image dans l'éditeur Sprite, le programme indiqué s'ouvrira. La valeur par défaut ici est de ne pas avoir de chemin.
