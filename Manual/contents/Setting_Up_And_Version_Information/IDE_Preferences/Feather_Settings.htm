Options de base

Les paramètres de base de la plume contiennent les options suivantes :

  • Activer Plume: Activez ou désactivez Feather ici. Lorsque cette option est désactivée, aucune fonctionnalité de Feather ne sera disponible sur le site IDE.
  • Délai d'analyse du document: Le temps que Feather attend pour vérifier votre script après que vous y ayez apporté une modification.
  • Info-bulles compactes: En activant cette option, vous supprimez certaines informations de l'infobulle qui s'affichent lorsque vous passez la souris sur des mots clés, comme les descriptions des paramètres des fonctions, ce qui permet de réduire la taille de vos infobulles.
  • Le nombre maximum de champs struct qui sont affichés par les infobulles: Lorsque vous survolez une référence struct, une infobulle affiche les variables contenues dans cette struct, jusqu'au nombre indiqué ici.
  • Refactoriser les références au nom de l'actif lorsque l'actif est renommé dans le navigateur d'actifs: Si vous activez cette option, lorsque vous renommez un actif dans l'explorateur d'actifs, toutes les mentions de cet actif dans votre code sont également modifiées afin que ces références ne soient pas rompues.
  • Préférer le style de commentaire JSDoc multi-lignes lors de la création à partir d'une fixation rapide: Lorsque vous générez des commentaires JSDoc pour une fonction par le biais du menu Quick Fixes, le JSDoc généré utilisera des commentaires à une ou plusieurs lignes, en fonction de cette option.
  • Activez le mode Strict Type: Lorsque cette option est activée, Feather est plus strict pour s'assurer que vous utilisez les bons types de données dans votre code.

Mise en évidence

Cette section contient des paramètres permettant de modifier les couleurs utilisées pour souligner une partie de votre code afin de signaler une erreur, un avertissement ou une suggestion :

  • Couleur des erreurs: La couleur utilisée pour les erreurs.
  • Couleur des avertissements: La couleur utilisée pour les avertissements.
  • Couleur des suggestions: La couleur utilisée pour les suggestions.

Gravité du message

Cette section contient toutes les règles que Feather utilise pour vérifier votre code, et vous permet d'ajuster la sévérité de chaque règle.

Profil

Dans ce menu déroulant, vous pouvez choisir entre plusieurs profils prédéfinis pour les règles GM*. Les profils suivants sont disponibles :

  • Aucun: Ce profil n'affichera aucun message d'erreur.
  • SyntaxErrors: Si vous utilisez ce profil, les erreurs de syntaxe seront affichées, en plus de ce qui est affiché dans le profil Aucun.
  • TypeErrors: Ce profil affiche également les erreurs de type en plus du profil SyntaxErrors .
  • Tout: Ce profil montre tout ce que Plume peut analyser.

Règles de GM

Vous pouvez définir une règle comme étant une "erreur", un "avertissement" ou une "suggestion", ce qui affecte la manière dont le code enfreignant la règle vous est signalé dans l'éditeur de code et la fenêtre des messages de la plume.

Vous pouvez définir une règle sur "Ignorer", ce qui aura pour effet d'ignorer tous les cas de violation de cette règle.

Toutes les règles commençant par "GM1000" vérifient votre syntaxe pour détecter d'éventuelles erreurs fatales, et celles commençant par "GM2000" sont des règles de bonne pratique qui vous aident à éviter les bogues dans votre jeu.

Règles de dénomination

Cette section vous permet de définir des règles de dénomination pour les actifs, les variables et diverses autres parties du code GML.

La règle GM2017 sous "Gravité du message" doit être activée pour que les règles de nommage prennent effet. Par défaut, elle est désactivée.

Les paramètres de cette section sont les suivants :

  • Liste de blocage des identificateurs: Il s'agit d'une liste d'identifiants séparés par des espaces qui seront ignorés pour toutes les règles de nommage.

    Par exemple, disons que vous avez une règle pour utiliser obj_ comme préfixe pour les objets, mais vous voulez que votre gestionnaire d' objets et votre réseau restent tels quels et n'utilisent pas le préfixe obj_.

    Dans ce cas, vous écrirez manager network dans le champ Indentifier Blocklist. Feather ignorera ces objets et tout autre identifiant (variables, enums, paramètres, etc.) portant le même nom.
  • <identifiant> Règle de dénomination: Vous pouvez définir des règles de dénomination pour chaque type d'identifiant de la liste, comme les actifs, les macros, les enums, les paramètres de fonction, les variables, etc.

    Chaque liste déroulante de règle de nommage dispose des options suivantes :
    • Préfixe: Texte qui doit apparaître avant le nom de l'identifiant, qui peut être obj_ pour les objets, _ pour les variables locales, etc. selon vos préférences.
    • Suffixe: Texte qui doit apparaître après le nom de l'identifiant.
    • Style de dénomination: Choisissez le style de dénomination de l'identifiant. Vous avez le choix entre les options suivantes :
      • Non contraint: Aucun formatage n'est imposé à ce type d'identifiant.
      • UpperCamelCase: MaSanté, ObjGrappleHook, MaxInventoryItemsAllowed, etc.
      • lowerCamelCase: myHealth, objGrappleHook, maxInventoryItemsAllowed, etc.
      • ALL_UPPER: MY_HEALTH, OBJ_GRAPPLE_HOOK, MAX_INVENTORY_ITEMS_ALLOWED, etc.
      • all_lower: my_health, obj_grapple_hook, max_inventory_items_allowed, etc.
