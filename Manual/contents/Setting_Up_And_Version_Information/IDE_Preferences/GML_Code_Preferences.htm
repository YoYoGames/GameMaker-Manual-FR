Préférences ducode GML

Les préférences relatives au code GML permettent de définir la manière dont certaines fonctions du langage de codage seront affichées sur le site IDE de GameMaker. Les options pour le code GML sont les suivantes :

  • Afficher les types d'arguments dans la barre d'état: Cette option concerne le cas où vous utilisez vos propres scripts personnalisés et que vous avez utilisé la notation JSDoc pour identifier les différents arguments, etc... (voir ici pour plus d'informations). Lorsque cette option est cochée, et que vous avez spécifié les types d'arguments dans le script, ils seront affichés dans la barre d'état en bas de l'éditeur, entourés de {}. Cette option est désactivée par défaut.
  • Afficher les descriptions des arguments dans la barre d'état: Si cette option est cochée, toutes les descriptions que vous avez données à vos scripts personnalisés en utilisant la notation JSDoc seront affichées dans la barre d'état en bas de la fenêtre de l'éditeur. Cette option est désactivée par défaut.
  • Afficher l'orthographe américaine pour les fonctions système: Par défaut, GameMaker utilise l'orthographe anglaise de la Grande-Bretagne pour les fonctions, par exemple draw_text_colour(), mais cela peut ne pas être exactement ce à quoi vous êtes habitué et vous pouvez donc passer à l'orthographe américaine à l'aide de cette option, de sorte que les fonctions deviennent draw_text_color(), par exemple. Cette option est désactivée par défaut.
  • Nombre maximal d'erreurs de syntaxe à afficher: Cette valeur définit le nombre maximum d'erreurs de syntaxe qui seront affichées à un moment donné. Si vous trouvez que le site IDE est en retard, vous pouvez essayer de réduire cette valeur. Cette valeur est fixée à 50 par défaut.
  • Activez les avertissements de référence de variable: Lorsque cette option est activée, elle signale tout problème potentiel avec les variables, par exemple lorsqu'une variable a été définie quelque part mais n'est pas réellement utilisée dans le code. Lorsqu'il est désactivé, ces avertissements ne seront plus affichés. Par défaut, cette option est activée.
  • Activez le pliage des instructions comme des régions: Lorsque cette option est activée, les parenthèses de votre code peuvent être repliées comme des régions. Cette option est activée par défaut.
