Préférencesvisuelles GML

Le site GML Visual sont utilisées pour définir l'aspect et la convivialité de l'éditeur Visual. Les différentes options disponibles sont listées ci-dessous avec une description de ce que fait chacune d'entre elles.

  • Emplacement par défaut de la boîte à outilsGML Visual: Par défaut, la boîte à outils de GML Visual (où se trouvent toutes les différentes bibliothèques d'actions) est placée à droite de l'éditeur, mais vous pouvez utiliser cette option pour qu'elle soit placée par défaut à gauche.
  • Mode de visibilité de l'aperçu des nœuds deGML Visual: Un "aperçu" de nœud est le court texte descriptif qui apparaît à côté de certaines actions dans l'espace de travail de l'action, par exemple cette action Détruire l'instance :

    Vous pouvez choisir de toujours les activer, de les activer uniquement si l'action est réduite (paramètre par défaut) ou de les désactiver complètement.
  • Réponse automatique à la conversion de GML Visual en GameMaker Language : Lorsque vous convertissez un GML Visual ou un script en code GML, vous obtiendrez par défaut un message vous avertissant de ce que vous êtes sur le point de faire et vous demandant si vous voulez continuer. Si vous réglez cette option sur "OK", la modification sera simplement acceptée sans afficher le message.
  • Réponse automatique à la suppression d'un nœud GML Visual: Lorsque vous supprimez un nœud GML Visual vous recevrez par défaut un message vous avertissant de ce que vous êtes sur le point de faire et vous demandant si vous voulez continuer. En réglant cette option sur "OK", la suppression sera simplement acceptée sans afficher le message.
  • Taille maximale des vignettes d'action de la boîte à outils GML Visual: Au bas de la boîte à outils d'action, vous disposez d'un curseur permettant de modifier la taille des GML Visual icônes. Ce paramètre sera utilisé pour définir la taille maximale autorisée (64px par défaut).
  • Taille minimale des vignettes d'action de la boîte à outils GML Visual: En bas de la boîte à outils d'action, vous disposez d'un curseur permettant de modifier la taille des GML Visual icônes. Ce paramètre sera utilisé pour définir la taille minimale autorisée (16px par défaut).
  • Taille par défaut des vignettes d'action de la boîte à outils GML Visual: Ce paramètre sera utilisé pour définir la taille par défaut de toutes les icônes d'action de la boîte à outils (32px par défaut).
  • Nombre maximal d'actions récemment utilisées à afficher: Dans la boîte à outils des actions, il y a une bibliothèque en haut qui contient les actions que vous avez utilisées le plus récemment. Vous pouvez utiliser cette option pour modifier le nombre maximum d'actions récentes qui seront affichées (5 par défaut).
  • Épaisseur de la chaîne qui relie les actions de GML Visual: Définit l'épaisseur de la chaîne graphique qui relie les différentes actions dans un bloc de code. GML Visual bloc de code (3px par défaut).
  • Temps de pression de la souris pour commencer à faire glisser les nœuds (ms): Lorsque vous cliquez sur un nœud et que vous le faites glisser, vous repositionnez le nœud dans le code de l'action. Cependant, si vous maintenez la souris enfoncée pendant un moment et que vous bougez ensuite, vous déplacerez la position du nœud dans l'espace de travail mais pas dans l'ordre du code. Ce paramètre définit le nombre de millisecondes à attendre avec le bouton de la souris enfoncé avant que le nœud puisse être déplacé (pas repositionné). La valeur par défaut est 350.
  • Masquer le panneau de présentation du nœud GML Visual: Vous pouvez choisir ici de masquer le panneau de présentation de l'éditeur de code d'action. Par défaut, il est désactivé et son activation supprimera la liste en texte brut des chaînes d'action de l'éditeur.
