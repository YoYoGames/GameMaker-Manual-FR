Préférences du flux d'exécution

Les préférences Runtime Feeds sont utilisées pour afficher et modifier le Runtime actuellement installé ainsi que le runtime en cours d'utilisation. Plusieurs outils runtime différents peuvent être installés sur GameMaker à un moment donné et vous pouvez définir lequel utiliser à partir de cette préférence. Idéalement, vous souhaitez utiliser le plus récent runtime, mais il peut arriver qu'il y ait un problème, ou que vous soyez obligé d'utiliser un outil plus ancien pour des raisons de comparabilité, etc... et vous pouvez donc modifier les outils runtime que GameMaker utilise pour construire votre jeu.

La section initiale de cette préférence est liée à l'endroit d'où vous tirez les outils runtime et vous demande d'entrer une URL vers un flux RSS. Ce flux sera ensuite analysé pour détecter les mises à jour des outils runtime, qui pourront ensuite être téléchargées et installées. Cette section affichera par défaut l'URL du flux Master runtime, mais vous pouvez cliquer sur le bouton Ajouter pour créer un nouveau flux Runtime dans lequel vous indiquerez le nom et l'URL du flux RSS, puis cliquer sur Appliquer pour demander à GameMaker d'en faire un access. Vous devrez fermer et rouvrir la fenêtre des Préférences pour que le nouveau flux s'affiche.

Une fois qu'un flux Runtime a été créé, il sera répertorié dans cette section comme une sous-catégorie, et vous devriez toujours avoir au moins un flux répertorié (par défaut, il s'agit du flux principal ). Les préférences de Runtime Feed sont utilisées pour sélectionner, installer et désinstaller les différents outils de runtime. En haut, vous avez les outils runtime actuellement utilisés, la date de leur création et l'emplacement où ils sont stockés sur le disque dur. Sous ces détails, vous pouvez trouver une liste de tous les runtimes disponibles, et vous pouvez choisir n'importe lequel de cette liste pour qu'il devienne le runtime "courant" utilisé pour compiler vos projets.

Pour sélectionner un autre runtime à utiliser, il suffit de le trouver dans la liste et de double-cliquer sur . Si le site runtime n'a pas été téléchargé et installé, vous serez invité à le faire, et une fois qu'il aura été installé, il vous sera demandé de fermer et de redémarrer le site IDE afin que le nouveau runtime puisse être initialisé.

Cette liste détaille tous les runtimes disponibles, leur numéro de version, la date de leur publication et, s'ils sont installés, le chemin d'accès à l'endroit où ils ont été installés. Le côté droit affiche également les différentes icônes d'état suivantes :

Ceci marque le site runtime actuel qui a été sélectionné pour être utilisé lors de la construction de vos projets.
En cliquant sur ce bouton, vous lancerez le téléchargement et l'installation du site runtime sélectionné (une barre d'état s'affichera en haut à droite de l'IDE).
En cliquant sur ce bouton, vous supprimez le site runtime sélectionné de votre disque dur.

Vous pouvez mettre à jour la liste runtime à tout moment à partir du flux RSS donné en cliquant sur le bouton d'actualisation , et vous pouvez nettoyer votre disque dur des vieux runtimes en cliquant sur le bouton "Nettoyer Runtimes maintenant" .
