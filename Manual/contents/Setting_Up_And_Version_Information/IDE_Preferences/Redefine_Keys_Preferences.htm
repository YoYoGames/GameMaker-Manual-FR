Redéfinir les préférences des touches

La section Redéfinir les touches des Préférences vous permet de modifier les mappages des différents raccourcis clavier fournis dans l'IDE GameMaker.

Recherche

En haut, vous avez la barre de recherche, que vous pouvez utiliser pour rechercher des commandes spécifiques en saisissant un nom (par exemple, "Copier"), une combinaison de touches (par exemple, "CTRL+C") ou un emplacement (par exemple, "Éditeur de texte").

En activant le bouton à côté du champ de recherche, vous pouvez effectuer une recherche en saisissant un raccourci. Toutes les touches sur lesquelles vous appuyez lorsque cette option est activée seront saisies comme un raccourci au lieu d'un texte normal.

À côté de la barre de recherche, vous avez le menu Filtre qui est décrit dans sa propre section plus loin sur cette page.

Liste des commandes

La "Liste des commandes" affiche tous les raccourcis disponibles qui peuvent être remappés, séparés en différentes catégories basées sur les emplacements de ces raccourcis. Cliquez sur n'importe quel emplacement dans cette liste pour afficher les commandes qui peuvent être utilisées dans cet emplacement :

Les informations relatives à chaque commandement sont séparées en deux colonnes :

  • Commande: C'est le nom de la commande/du raccourci et il décrit l'action qu'il exécute.
  • Touches: Ceci affiche le mappage actuel du clavier pour la commande. Vous pouvez double-cliquer sur n'importe quel élément de cette colonne pour réaffecter cette commande en appuyant simplement sur la combinaison de touches appropriée du clavier (elles doivent être maintenues simultanément, comme si le raccourci était réellement utilisé). Dès que vous relâcherez votre nouvelle combinaison de touches, elle sera enregistrée et appliquée à la commande. Si vous appuyez sur pendant l'édition d'une commande, vous quitterez le mode d'édition et ne modifierez pas la commande sélectionnée.

Vous pouvez modifier le tri de la liste des commandes en cliquant sur sur l'en-tête d'une colonne (soit "Commande", soit "Clés") pour trier la liste en utilisant les valeurs de cette colonne, en alternant entre les valeurs croissantes et décroissantes à chaque clic.

Fenêtre d'affrontement

Si vous essayez de définir une combinaison de touches qui est déjà utilisée par d'autres commandes, vous verrez apparaître une fenêtre spéciale "Clash" :
Cette fenêtre affichera l'emplacement et le nom de la commande qui utilise déjà la combinaison de touches que vous essayez d'utiliser. Si la combinaison de touches est utilisée par plusieurs commandes, elle les affichera toutes dans une liste.

Deux options s'offrent à vous : "Annuler" et "Ignorer (TOUT) et réaffecter":

  • Le fait d'appuyer sur "Annuler" fermera la fenêtre Clash et n'apportera aucun changement aux commandes.
  • En appuyant sur "Override (ALL) and Reassign", la nouvelle combinaison de touches sera attribuée à la commande sélectionnée et les liens de toutes les autres commandes qui utilisaient la même combinaison seront effacés .

Après avoir utilisé "Remplacer et réaffecter", GameMaker met en surbrillance la commande remplacée dans la liste des commandes pour que vous puissiez lui attribuer une nouvelle combinaison de touches. Si plusieurs commandes ont été remplacées, le filtre sera modifié pour n'afficher que ces commandes afin que vous puissiez leur appliquer de nouveaux raccourcis. Ce filtre peut être supprimé en cliquant sur "Effacer".

Si vous souhaitez utiliser une combinaison différente pour la commande sélectionnée, vous pouvez développer la section "Suggested Free Keys" (Touches libres suggérées) pour afficher une liste de combinaisons de touches qui ne sont utilisées nulle part dans le site IDE. En cliquant sur "Use" (Utiliser) à côté de l'une de ces combinaisons, celle-ci sera affectée à la commande sélectionnée et le conflit sera résolu.

Menu Filtre

Le menu Filtre vous permet de spécifier quel type de commandes doit apparaître dans la liste des commandes. En haut, vous avez deux options :

  • "Show Unbound Commands": En activant cette option, vous afficherez toutes les commandes qui ont été déliées, c'est-à-dire que leurs liens ont été effacés. Toutes les commandes qui restent non liées ne peuvent pas être utilisées car aucune combinaison de touches ne leur est attribuée.
  • "Show Only Redefined Commands": En activant cette option, vous n'afficherez que les commandes qui ont été modifiées manuellement, c'est-à-dire que leurs liens sont différents de leurs liens par défaut.

En dessous, vous disposez de nombreuses options vous permettant de spécifier les emplacements qui sont affichés dans la liste des commandes. Par défaut, l'option "All Locations" est cochée, ce qui signifie que vous verrez les commandes de tous les emplacements sur le site IDE. En décochant l'option "All Locations", vous décochez tous les autres éléments situés en dessous, ce qui vous permet de sélectionner manuellement ceux que vous souhaitez voir ; par exemple, en décochant tous les éléments et en cochant "Image Editor", vous ne verrez que les commandes qui peuvent être utilisées dans l'éditeur d'images.

Commandes d'effacement

En double-cliquant sur une combinaison de touches existante, vous pourrez la modifier, mais pendant ce temps, vous pouvez cliquer sur le bouton "Effacer" situé sous la liste des commandes pour effacer la liaison de cette commande.

Si vous avez accidentellement écrasé certains raccourcis clavier et que vous souhaitez rétablir les paramètres par défaut, vous pouvez cliquer sur le bouton "Restaurer les valeurs par défaut" en haut de la fenêtre des préférences pour réinitialiser toutes les commandes à leurs mappages par défaut, ou consulter la page Raccourcis clavier pour vérifier les liaisons par défaut et rétablir vos commandes manuellement.
