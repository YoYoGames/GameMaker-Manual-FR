Préférences des plugins

Les préférences des plugins sont utilisées pour contrôler tous les plugins supplémentaires qui ont été ajoutés à GameMaker IDE. Par défaut, une section est consacrée au plugin Source Control pour l'intégration des projets GameMaker avec Git.

Contrôle des sources (Git)

Cette section traite des préférences requises pour que le plugin Git SCM fonctionne avec GameMaker.

  • Chemin de l'exécutable Git: Il s'agit de l'emplacement de l'exécutable Git que GameMaker utilise pour toutes les commandes de contrôle de source. Comme le montre l'image ci-dessus, il sera situé dans le dossier cmd de votre installation Git.

En dessous, vous pouvez configurer l'outil de fusion et l'outil de diffusion pour le contrôle de la source avec ces paramètres(facultatifs):

  • Fusion (Emplacement de l'outil): vous indiquez ici le chemin d'accès complet au fichier de l'outil de fusion Git.
  • Fusionner (options de l'outil): Dans ce champ, vous pouvez ajouter toute commande supplémentaire à exécuter chaque fois que vous utilisez l'outil de fusion.
  • Diff (emplacement de l'outil): Indiquez ici le chemin d'accès complet au fichier de l'outil de comparaison Git.
  • Diff (Options de l'outil): Dans ce champ, vous pouvez ajouter toute commande supplémentaire à exécuter chaque fois que vous utilisez l'outil diff.

Finally there are two more options that affect how Git is set up in new projects: 

  • Add skeleton .git defaults to new projects: When this setting is enabled, GameMaker will add .gitignore and .gitattributes Files to new projects that you create. It is enabled by default.
  • Add skeleton .git defaults to imported projects: When this setting is enabled, GameMaker will add .gitignore and .gitattributes Files to projects that you import from a .yyz file. It is enabled by default.
