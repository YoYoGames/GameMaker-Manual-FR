Objets et instances

Avec les actifs que vous avez vus jusqu'à présent, vous pouvez ajouter de belles images et de beaux sons dans le navigateur d'actifs du jeu, mais ils ne font encore rien dans le jeu lui-même ! Pour cela, nous avons besoin de l'actif le plus important de tous dans GameMaker, l'actif Objet . Les objets sont des ressources spéciales que nous utilisons pour contrôler certains aspects d'un jeu et pour faire des choses spécifiques. La plupart du temps, ils sont associés à une adresse sprite afin que vous puissiez les voir dans les salles de jeu, mais parfois, ils sont utilisés comme des objets de contrôle "en coulisses" pour faire des choses comme le suivi des statistiques, ou définir le timing des cut-scenes, etc... On peut donner aux objets des comportements et ils peuvent réagir à certains événements ainsi qu'entre eux, et la plupart des choses que vous voyez dans un jeu sont basées sur les objets et leurs interactions avec l'utilisateur et entre eux.

On ne peut pas parler d'objets sans parler d'instances. Dans un jeu réalisé avec GameMaker, les personnages, monstres, balles, murs, etc... que vous voyez dans le jeu sont en fait tous considérés comme des instances. Une instance est essentiellement une "copie" d'un objet que l'on trouve dans l'Asset Browser et que l'on place dans une pièce (pensez aux objets comme à des modèles d'instances). Ainsi, lorsque nous parlons de quelque chose qui affecte ou modifie une instance, nous voulons dire qu'une copie particulière d'un objet dans une salle est affectée alors que toutes les autres ne le sont pas, mais lorsque nous parlons d'affecter ou de modifier un objet, nous voulons dire que tout ce que nous faisons sur lui sera reflété dans toutes les instances créées à partir de ce point également. Ainsi, l'objet est le modèle de l'instance, et l'instance est ce que nous plaçons dans une salle pour créer notre jeu.

Pour créer un nouvel objet, il suffit de cliquer avec le bouton droit de la souris sur le dossier d'objets et de sélectionner Créer ( ou de cliquer avec le bouton droit de la souris n'importe où dans le navigateur d'objets et de sélectionner Créer un objet).

Il s'agit de la fenêtre de l'éditeur d'objets, qui affiche les propriétés actuelles de l'objet. La première chose à faire est de nommer l'objet, en utilisant un préfixe comme " obj_" ou un autre identifiant. En général, pour qu'un objet puisse dessiner quoi que ce soit et enregistrer des collisions avec des instances d'autres objets, il doit se voir attribuer un sprite, qui est ajouté à l'aide du bouton . Cela ouvrira l'Explorateur d'actifs où vous pourrez sélectionner le sprite à utiliser (vous pouvez également faire glisser sprites depuis l'Explorateur d'actifs et les déposer dans l'Éditeur d'objets pour les assigner) :

Si vous ne disposez pas encore d'un sprite approprié, vous pouvez cliquer sur sur le bouton New Sprite pour créer un nouvel actif sprite et le modifier comme vous le souhaitez, comme si vous aviez créé l'actif via les options de création standard sprite, sauf que maintenant il sera automatiquement affecté à l'objet que vous aviez ouvert lorsque vous avez commencé à le créer. Une fois que vous avez sélectionné un sprite, vous pouvez également cliquer sur sur les boutons Edit Sprite ou Edit Image pour modifier les images du sprite lui-même ou de l'actif sprite. Cela peut être plus rapide que de trouver d'abord le sprite dans le navigateur d'actifs et d'indiquer ensuite que vous voulez le modifier.

Une fois que vous avez ajouté ou créé le site sprite, vous pouvez modifier quelques autres propriétés de l'objet, par exemple s'il doit être visible ou non ou quel type de masque de collision il doit avoir. Pour les jeux simples, vous n'avez probablement pas besoin de modifier quoi que ce soit ici et vous pouvez maintenant ajouter des événements à l'objet.

Les événements sont des moments discrets dans la boucle du jeu où les choses se produisent en fonction de ce que vous avez programmé pour elles. GameMaker fonctionne avec des cycles de ces événements - du moment où une pièce est commencée au moment où elle est terminée, une boucle de jeu est en cours d'exécution où, à chaque étape, une série d'événements est exécutée, et vous pouvez choisir de placer du code GML ou des actions visuelles GML dans vos instances qui répondent à ces événements (une étape est un moment du temps de jeu, régi par le paramètre de vitesse du jeu - également appelé "frame").

Pour ajouter un événement, vous cliquez sur sur le bouton Ajouter un événement :

Ci-dessus, vous pouvez voir toutes les différentes catégories d'événements, dont certaines contiennent d'autres sous-catégories. Nous n'allons pas passer en revue tous les événements ici, mais nous allons couvrir brièvement les cinq plus importants :

Créer un événement Cet événement se produit lorsqu'une instance de l'objet est créée pour la première fois, et c'est la toute première chose qui se produit dans une instance qui a été placée dans la room via l'éditeur de room lorsqu'on entre dans une room. Cela signifie que cet événement est l'endroit idéal pour initialiser des variables, démarrer des lignes de temps, définir des chemins, etc... et faire tout ce qui ne doit généralement être fait qu'une fois ou seulement lorsqu'une instance apparaît pour la première fois dans la room.
Evénement d'étape GameMaker divise le temps de jeu en étapes, la vitesse du jeu définissant le nombre d'étapes par seconde. Une étape, c'est en fait la boucle qui tourne constamment avec tous les événements vérifiés et déclenchés si nécessaire pendant le déroulement du jeu, donc comme vous pouvez l'imaginer, l'événement Step est un événement qui est vérifié à chaque étape du jeu pendant que l'instance existe. Notez que la catégorie Step Event a trois sous-catégories pour "affiner" le timing dans l'instance, mais en général vous utiliserez le Step Event principal le plus souvent.
Événement de collision Il est évident que lorsque vous créez un jeu, il est très important que vous sachiez quand deux instances (ou plus) d'un objet sont entrées en collision, et pour cela nous avons l'événement de collision. Il s'agit d'un événement que vous placez dans un objet, puis spécifiez l'autre objet avec lequel vous devez vérifier les collisions. Une fois que vous avez ajouté l'événement et choisi l'objet à vérifier, vous pouvez ajouter le code GML ou GML Visual pour résoudre la collision.
Événement de tirage au sort Il existe plusieurs événements de tirage au sort différents dans la catégorie des événements de tirage au sort, mais celui que vous utilisez le plus est celui qui se trouve en haut et qui est simplement appelé "Tirage au sort". Cet événement général de dessin sera appelé, comme l'événement step, à chaque image de jeu, mais il fonctionne de deux manières différentes. La première est lorsque vous n'ajoutez pas cet événement à l'objet... Si aucun Draw Event n'est spécifié , GameMaker dessinera par défaut le sprite assigné à l'objet, en utilisant toutes les transformations qui ont été appliquées ailleurs dans l'objet (comme pour modifier sa taille ou le mélange alpha). Ainsi, même si aucun Draw Event n'est listé dans la liste des événements, il est toujours exécuté et sera toujours dessiné à l'écran lorsqu'une instance sera placée dans une room. La deuxième façon dont cela fonctionne est lorsque vous ajoutez l'événement lui-même et que vous y incluez n'importe quel visuel GML ou GML. En ajoutant quelque chose à l'événement, vous dites à GameMaker " Je veux contrôler ce que vous dessinez pour les instances de cet objet" et cela remplace complètement le dessin par défaut. Cela signifie que vous pouvez avoir un objet avec un sprite assigné, puis définir l'événement draw pour dessiner du texte et le sprite ne sera pas dessiné car vous n'avez pas demandé à GameMaker de le dessiner, mais le texte le sera.
Détruire l'événement Cet événement est le tout dernier événement à être exécuté lorsqu'une instance est détruite. Il est souvent négligé lors de l'ajout de comportements aux objets, mais il peut être très utile - par exemple, pour créer des effets d'explosion ou de particules lorsqu'un ennemi est tué, ou pour faire réapparaître une nouvelle instance de l'objet dans une autre partie de la pièce, ou même pour ajouter des points à un score.

Pour plus d'informations sur les différents événements, veuillez cliquer ici.

Vous pouvez ensuite cliquer sur un événement de la liste pour l'ajouter à l'objet.

L'ajout d'un événement ouvre l'éditeur GML Code ou GML Visual, selon le type d'événement que vous créez. Dans cette fenêtre, vous pouvez maintenant ajouter le code ou les actions que vous souhaitez exécuter chaque fois que cet événement est déclenché dans la boucle du jeu.
