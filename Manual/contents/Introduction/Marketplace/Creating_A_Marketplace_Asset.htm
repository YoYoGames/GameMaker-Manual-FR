Création d'un actif Marketplace

Lorsque vous sélectionnez l'option Créer un package dans le menu de la place de marché IDE, vous accédez à l'espace de travail suivant : . Ici vous devez d'abord sélectionner l'identité de l'éditeur sous laquelle vous souhaitez publier le package d'actifs (que vous avez dû configurer depuis votre tableau de bord de l'éditeur), puis donner un nom d'affichage et un numéro de version. Un ID de package sera alors créé pour vous sur la base du nom d'affichage et du compte d'éditeur utilisés, mais vous pouvez le modifier pour qu'il soit différent si vous le souhaitez, tant qu'il conserve le format URL inversé. Vous devez ensuite choisir le type de package d'actifs à créer :

  • Actif: Il s'agit d'une collection d'actifs (sprites, scripts, objets, etc...) que vous souhaitez vendre sur la place de marché.
  • Démo: Il s'agit d'une démo pour démontrer un concept ou une idée de jeu.
  • Tutoriel: La ressource téléchargée est un tutoriel, conçu pour être exécuté à partir de GameMaker IDE (voir cette page pour plus de détails).

Une fois que vous avez fait cela, vous pouvez choisir les actifs du projet que vous voulez ajouter dans le paquet d'actifs en utilisant la section sur la droite. Vous pouvez ajouter toutes vos ressources, ou seulement certaines d'entre elles, et supprimer celles que vous auriez ajoutées par erreur. Nous partons du principe que vous avez déjà créé les ressources à télécharger dans GameMaker et que le projet actuel contient toutes les ressources que vous souhaitez télécharger.

RAPPEL ! Pour pouvoir créer ou utiliser des paquets d'actifs, vous devez avoir préalablement créé un certificat de profil utilisateur. Vous pouvez le faire à partir des préférences de Marketplace, ce qui est expliqué ici.

Lorsque vous êtes satisfait de la sélection des actifs et que vous souhaitez télécharger votre package sur le site Marketplace, il vous suffit de cliquer sur le bouton OK en bas de la page, et le package d'actifs sera créé et téléchargé sur le site Marketplace. Une fois le téléchargement terminé, un nouvel espace de travail s'ouvre avec une vue du navigateur de la page d'actifs que vous pouvez modifier. Pour plus d'informations sur la façon de configurer cette fonction, veuillez consulter la page suivante :

  • Inscription d'un actif

Si vous devez mettre à jour un actif Marketplace, vous faites plus ou moins la même chose que lorsque vous avez créé le paquet d'origine, sauf que maintenant vous devez spécifier le paquet à mettre à jour dans une liste déroulante de paquets disponibles :

Lors de la mise à jour, vous ne pourrez pas modifier l'ID du package, et vous devez savoir que le numéro de version affiché sera le numéro de version actuel et que vous devez l'incrémenter comme il convient, sinon la mise à jour échouera (puisque vous ne pouvez pas avoir deux packages de ressources avec le même numéro de version). Il existe également un champ dans lequel vous pouvez ajouter du texte pour le journal des modifications de la ressource, qui sera téléchargé avec la ressource et affiché sur la page Marketplace.

IMPORTANT ! Les ressources créées avec la version 2.3+ de GameMaker ne seront disponibles que pour les utilisateurs qui utilisent la version 2.3 et supérieure. Si vous mettez à jour une ressource d'une version antérieure, cette ressource sera toujours disponible pour les utilisateurs qui n'ont pas fait de mise à jour et coexistera avec la version 2.3+ mise à jour.

Téléchargement via la page Web

Vous pouvez télécharger des ressources via la page Web Marketplace plutôt que via IDE, ce qui peut être nécessaire si le téléchargement via IDE a échoué pour une raison quelconque. Pour commencer, vous devez créer le paquet et essayer de le télécharger via IDE comme indiqué ci-dessus, et si cela a échoué, vous devez alors aller dans la fenêtre de sortie et vérifier le journal pour une ligne qui montre où le paquet que vous créez a été enregistré avant de tenter le téléchargement. Par exemple :

Sous Windows:

Created package file:
C:\Users\WINDOWSUSERNAME\AppData\Roaming\GameMaker\GM_USERNAME\Marketplace\Packages\com.publishername.assetname.yymp

Sur macOS:

Created package file:
/Users/MACUSERNAME/.config/GameMaker/GM_USERNAME/Marketplace/Packages/com.publishername.assetname.yymp

Vous devez vous rendre à l'emplacement de ce fichier et copier le fichier *.YYMP dans un endroit sûr, car il s'agira du fichier que vous téléchargerez sur le site Marketplace.

À ce stade, vous devez ouvrir le site Marketplace dans votre navigateur Web et naviguer jusqu'à la liste des actifs pour l'actif, puis aller dans la section Versions, et enfin cliquer sur le bouton Télécharger la version, comme le montre l'image ci-dessous :

Vous accédez alors à la page de téléchargement de la nouvelle version, où vous pouvez ajouter des informations sur la version à télécharger, le cas échéant, et fournir le fichier en cliquant sur le bouton Choose FIle et en sélectionnant le fichier *.YYMP créé précédemment :

Lorsque le téléchargement est terminé et que vous êtes satisfait des détails, vous pouvez cliquer sur le bouton Save pour enregistrer la ressource dans la liste. Notez que la ressource sera en phase de traitement pendant quelques minutes avant de pouvoir être publiée.
