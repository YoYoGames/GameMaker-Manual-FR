Compiler

Compiler votre jeu peut signifier deux choses : le compiler pour le tester ou le compiler pour créer une version finale du jeu. paquet exécutable pour une plateforme cible spécifique. Cette page a pour but d'expliquer ces deux options en détail.

Compilation pour les tests

Vous pouvez compiler votre jeu pour le tester en appuyant simplement sur le bouton Play en haut de la page IDE, qui lancera le jeu pour le tester en utilisant la cible spécifiée. Vous pouvez également exécuter le jeu en mode débogage en testant à l'aide du bouton Debug . Cela lancera le jeu, mais ouvrira également la fenêtre de débogage, où vous pourrez surveiller les performances de votre jeu et voir les problèmes éventuels (voir la section sur le débogage pour plus d'informations).

Paramètres de la cible

Par défaut, GameMaker s'exécute et débogue en utilisant la machine virtuelle (VM) intégrée, qui est plus ou moins la même que celle du système d'exploitation de bureau utilisé. Cependant, GameMaker est un moteur multiplateforme et vous pouvez tester, déboguer et compiler des paquets exécutables de vos projets sur un certain nombre de plateformes cibles différentes (les plateformes exactes disponibles dépendent des détails de votre licence). Pour modifier la plate-forme cible actuelle, vous pouvez cliquer sur sur le bouton Cibles pour ouvrir la fenêtre Cibles, qui ressemblera à ceci (les détails exacts varieront en fonction de votre type de licence) :

En haut, à côté du bouton Cibles, vous avez les paramètres actuels qui vous indiquent la plate-forme et les paramètres spécifiques actuellement utilisés, puis le reste de la fenêtre est occupé par les détails et les options de toutes les cibles disponibles que vous pouvez sélectionner pour les utiliser. Chaque section de cette fenêtre est expliquée ci-dessous :

Plate-forme

Cette section énumère toutes les plateformes cibles disponibles. Le contenu de cette liste varie en fonction de la licence que vous possédez, mais il y aura toujours au moins la cible "Opera GX". Pour sélectionner une cible, il vous suffit de cliquer sur . Les autres fenêtres d'options seront alors mises à jour pour afficher différents détails en fonction de la plateforme sélectionnée.

Des détails spécifiques sur la compilation pour chaque plate-forme sont donnés dans la section "Création d'un paquet exécutable final" ci-dessous.

Sortie

Chaque plateforme cible peut avoir un ou plusieurs formats de sortie, les principaux étant :

  • VM: La cible VM (Virtual Machine) utilise un runner générique pour chaque plate-forme et interprète ensuite le code pour votre jeu. En général, cette option est utilisée pour les tests en raison de ses temps de construction plus rapides, mais elle n'offre pas le même gain de performance que l'option YYC (si disponible). Vous pouvez utiliser cette option pour compiler des paquets exécutables pour des jeux plus petits ou des jeux pour lesquels les performances ne seront jamais un problème.
  • YYC: Le YYC(YoYo Compiler) génère du code C++ à partir de votre code GML, avant d'utiliser le compilateur C++ de la cible pour le compiler en code natif pour la plate-forme cible. Il supprime les fonctions inutiles et exécute une multitude d'autres techniques d'optimisation pour créer un paquet exécutable plus petit et plus performant.

    Les performances de votre jeu peuvent être multipliées par deux ou trois, en particulier pour les jeux à forte logique, et c'est la solution idéale pour les projets de grande envergure ou exigeants en termes de CPU. Les temps de compilation peuvent être plus longs et vous devez toujours effacer le compilateur cache avant de construire un paquet d'actifs complet final pour une plateforme cible. Notez que la cible YYC peut nécessiter l'installation d'outils supplémentaires pour la plateforme sélectionnée, sinon elle ne fonctionnera pas - vous trouverez de plus amples informations à ce sujet dans le centre d'aide de YoYo Games ainsi que sur les pages de préférences des cibles individuelles détaillées dans ce manuel.
  • JavaScript: La cible JavaScript n'est disponible que pour certaines cibles, comme la cible HTML5, et permet de compiler le jeu en JavaScript pur. Elle utilise ECMAScript 2015 (ou ES6) pour le JS qu'elle produit.

Création d'un paquet final exécutable

Vue d'ensemble

Vous pouvez simplement cliquer sur le bouton Créer un exécutable sur le site IDE pour lancer la construction du compilateur ou sélectionner Créer un exécutable dans le menu de construction. L'un ou l'autre de ces processus lancera le processus de construction qui dépendra de la plate-forme cible sélectionnée.

Sur la cible Opera GX, il ouvrira une fenêtre spéciale vous permettant de compiler et de télécharger votre jeu sur GX.games ; sur toutes les autres cibles, il ouvrira une fenêtre d'exploration de fichiers où vous pourrez donner le nom final que vous souhaitez utiliser pour l'exécutable de votre jeu, avant de cliquer sur Enregistrer pour lancer le processus de compilation et de construction. Une fois que vous avez fait cela, les fichiers nécessaires seront générés afin que vous puissiez le distribuer comme vous le souhaitez.

Pour certaines plates-formes, la compilation de votre projet de jeu nécessitera que vous ayez configuré les bons outils de compilation. outils de compilation (voir ici) et que vous ayez également rempli les préférences de plate-forme appropriées.

En fonction de votre licence, vous pouvez avoir la possibilité de construire votre jeu en ligne de commande, ce qui vous permet de mettre en place une intégration continue pour votre projet et de rationaliser votre processus de construction et de test. Consultez cette page pour plus d'informations sur la construction en ligne de commande.

NOTE Avant d'effectuer une construction finale de votre projet en vue de sa publication, vous devez toujours vider le cache du compilateur d'actifs (en utilisant l'icône "balai" en haut de la page IDE). Pour réduire au maximum la vitesse de compilation, GameMaker enverra à cache un grand nombre de vos fichiers de jeu, qui peuvent parfois être corrompus. Il est donc plus prudent de vider le cache cache avant d'effectuer la compilation finale.

La taille maximale du paquet final du jeu est de 4 Go, sauf pour les jeux 32 bits Windows, où elle est de 2 Go. Voir les conseils pour réduire la taille du jeu final.

Formats de la cible

Chaque option cible est enregistrée dans un format spécifique à la plateforme, listé ci-dessous :

  • Opera GX - Lorsque vous compilez pour la cible "Opera GX", vous verrez la fenêtre "Opera GX Packaging" qui vous demandera d'abord de vous connecter à l'aide de votre compte Opera :. Notez que vous n'aurez pas besoin de vous connecter ici si vous vous êtes déjà connecté à GameMaker à l'aide de votre compte Opera.

    Une fois que vous vous serez connecté, la compilation de votre jeu commencera et vous pourrez le télécharger sur GX.games. Vous pouvez ensuite cliquer sur "Edit Game on Opera" pour le modifier sur GX.games DevCloud et le publier :
    Veuillez vous reporter à cette page pour en savoir plus sur le processus de téléchargement d'Opera GX, et à cette page pour configurer les différentes exportations, telles que YYC et Mobile.
  • Windows - La compilation pour le système d'exploitation général Windows vous demandera d'abord de choisir entre la création d'un package Installer ou Zip , où l'installateur sera un exécutable unique qui installera votre jeu, et le fichier Zip sera un fichier compressé unique au format *.zip contenant tous les fichiers de votre jeu (les fichiers devront être extraits pour que le jeu fonctionne). Si vous cochez la case " Remember Packaging Option", GameMaker se souviendra de ce choix pour toutes les compilations futures (cette option peut être réinitialisée ou modifiée dans les préférences Windows). Vous pouvez en savoir plus en consultant le centre d'aide de YoYo Games. Veuillez noter que la cible Windows peut compiler des exécutables 32 ou 64 bits en fonction de ce que vous avez sélectionné dans les Options générales du jeu. Nous recommandons fortement à tous les nouveaux projets d'utiliser la cible x64 Runtime pour créer des exécutables 64bit car la cible 32bit runtime sera dépréciée et supprimée dans le futur.
  • Ubuntu (Linux) - La compilation pour Ubuntu vous demandera de choisir entre la création d'un fichier .AppImage ou .zip, contenant un exécutable x64 ou ARM (cependant AppImage ne supporte pas ARM). Vous pouvez en savoir plus en consultant le centre d'aide de YoYo Games.
    Le fichier .AppImage est utilisé pour le partage en tant que paquet via n'importe quel distributeur sauf Steam, tandis que le fichier .zip est utilisé uniquement pour le téléchargement vers Steam (car il utilise spécifiquement le Steam Runtime ). Pour cette raison, le .zip créé ne peut pas être ouvert comme un fichier zip normal.
  • HTML5 - Si vous avez choisi de créer un jeu HTML5, un fichier index.html (nom par défaut, mais vous pouvez aussi donner votre propre nom dans les préférences de la plate-forme) ainsi qu'un dossier contenant vos fichiers de jeu seront créés et enregistrés à l'emplacement spécifié. Pour que votre jeu fonctionne, ces deux fichiers doivent être téléchargés sur un serveur. Le site index.html peut également être personnalisé pour afficher votre jeu avec une couleur d'arrière-plan différente, ou à une position différente, etc... mais une connaissance du HTML est nécessaire pour cela, et vous pouvez également spécifier votre propre fichier d'index personnalisé lorsque vous construisez le paquet (voir les options de jeu HTML5). Vous pouvez en savoir plus en consultant le centre d'aide de YoYo Games.
  • Android - Pour les appareils Android, vous pouvez choisir de créer un fichier *.apk ou *.aab (Android App Bundle ) à partir de la fenêtre qui s'affiche pour enregistrer le jeu :

    Le type de fichier que vous choisirez dépendra du magasin que vous souhaitez cibler, le fichier *.aab étant nécessaire pour Google Play, tandis que le fichier *.apk peut être utilisé sur d'autres magasins. Vous pouvez en savoir plus en consultant le centre d'aide de YoYo Games.
  • iOS - La compilation pour iOS crée un fichier xarchive qui est ensuite utilisé dans Xtools pour créer le paquet final iOS. Notez que pour compiler pour iOS, vous aurez besoin d'un ordinateur Apple Mac fonctionnant sous OSX ou plus, ainsi que des certificats et autorisations appropriés. Vous pouvez en savoir plus en consultant le centre d'aide de YoYo Games.
  • macOS - La compilation pour la cible macOS vous demandera d'abord de choisir entre la création d'un DMG ou d'un fichier Zip :
    L'option "Package as DMG" crée un paquet d'installation .dmg qui peut être utilisé pour installer le jeu sur un ordinateur Mac. Notez que pendant ce processus de construction, vous verrez l'installateur apparaître sur le Mac, qui disparaîtra une fois le processus de construction terminé.

    L'option "Package as Zip" crée soit un fichier *.app, soit un fichier *.pkg, selon que vous souhaitez ou non le télécharger ultérieurement sur le Mac App Store. Comme pour iOS, vous aurez besoin d'un ordinateur Apple Mac équipé d'OSX ou d'une version supérieure, ainsi que des certificats et autorisations nécessaires. Vous pouvez en savoir plus en consultant le centre d'aide de YoYo Games.
  • Windows UWP - Pour Windows UWP, GameMaker créera un package *.appxbundle qui pourra ensuite être téléchargé sur la boutique Windows ou distribué ailleurs. Lorsque vous cliquez sur le bouton Créer un exécutable, vous êtes invité à indiquer à GameMaker le type de package que vous souhaitez créer (pour les architectures ARM, x64 et/ou x86), et vous devez choisir les cibles sur lesquelles le jeu sera pris en charge (vous devez en sélectionner au moins une, mais vous pouvez sélectionner les trois ou toute combinaison) :

    Si vous cochez la case Package For Store Upload, le paquet final créé sera un fichier *.appxupload, qui est ce que Microsoft spécifie comme devant être utilisé pour soumettre des applications à leur magasin, comme expliqué dans cet article ici (toutes les architectures choisies seront incluses, tout comme avec le *.appxbundle). Vous pouvez trouver plus d'informations sur la mise en place et la compilation vers la plateforme UWP dans le centre d'aide de YoYo Games.

Une fois que vous avez créé votre package d'actifs exécutables, vous pouvez ensuite donner le fichier à d'autres personnes ou le placer sur votre site Web pour qu'il soit téléchargé, ou encore télécharger ces fichiers vers les différents services d'hébergement pour une distribution individuelle ou même vers des magasins en ligne (comme Google Play, Apple App Store ou le Microsoft Store) pour une distribution générale et la vente au détail.

Notez que vous êtes libre de distribuer les jeux que vous créez avec GameMaker de la manière que vous souhaitez, y compris de les vendre. Bien entendu, cela suppose que le site sprites, les images et les sons que vous avez utilisés pour créer le jeu peuvent également être distribués ou vendus et que vous disposez des droits légaux sur tous les actifs. Cela suppose également que le jeu est conforme au CLUF de YoYo Games pour GameMaker.
