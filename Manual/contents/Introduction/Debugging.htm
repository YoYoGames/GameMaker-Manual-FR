Débogage

Lors de la programmation de votre jeu, que vous utilisiez GML Code ou GML Visual, il peut être très facile de faire des erreurs - utiliser les mauvaises variables, passer les mauvais arguments ou utiliser les mauvaises fonctions ne sont que quelques-unes des erreurs les plus courantes que nous faisons tous - et ces erreurs ne sont pas toujours détectées par le vérificateur de syntaxe qui est inclus dans l'éditeur de script/action. Cela signifie que ces erreurs n'apparaîtront que lorsque vous exécuterez ou compilerez votre jeu, et même dans ce cas, il peut être difficile de savoir exactement ce qui ne va pas. Il peut donc être d'une importance vitale de savoir comment votre jeu se comporte sur l'appareil choisi, ainsi que de voir quelles instances sont dans la pièce, quelles variables globales ont été créées et toute une série d'autres détails. Vérifier ces éléments s'appelle "déboguer votre code", et les erreurs ou problèmes que vous pouvez trouver sont appelés "bugs".

Vous pouvez vérifier beaucoup de ces choses vous-même en ajoutant ce que l'on appelle du "code de débogage" à vos jeux pour afficher à l'écran les informations dont vous avez besoin, ou vous pouvez faire en sorte qu'il affiche des messages lorsqu'il y a une erreur, mais c'est souvent beaucoup de travail et si vous n'êtes pas sûr de l'origine du problème, cela peut prendre beaucoup de temps de procéder par essais et erreurs pour retrouver votre bogue. Pour cela, GameMaker dispose d'un mode de test spécial appelé Debug Mode qui démarre le module de débogage. Il existe également un système complet de rapport d'erreurs qui affiche des messages lors de la compilation pour vous aider à localiser les problèmes de votre jeu.

Les sections ci-dessous couvrent les différentes façons dont vous pouvez déboguer votre jeu :

Rapport d'erreur

Lorsqu'une erreur se produit au cours de la compilation et de l'exécution d'un projet, elle est signalée par un message dans la fenêtre de sortie des erreurs de compilation, comme celui illustré ci-dessous :

La plupart des messages d'erreur du compilateur suivent le même schéma, avec un message indiquant l'objet dans lequel l'erreur s'est produite, puis l'événement avec le numéro de ligne, et enfin une courte description de l'erreur elle-même. Ce type d'information fourni par le rapport d'erreur est très important car il détecte et signale les bogues liés à la syntaxe et au langage qui vont faire planter votre jeu fini, mais il ne signalera pas les bogues plus subtils et ne donnera pas de retour sur les performances. Pour ce genre de choses, vous devez utiliser le mode Debug (expliqué ci-dessous).

En dehors de ces erreurs de compilation, vous pouvez également obtenir des erreurs de syntaxe. Celles-ci sont détectées lorsque vous ajoutez vos actions ou votre code sur GML Visual et sont affichées dans la fenêtre de sortie des erreurs de syntaxe, comme dans l'image ci-dessous :

Comme pour les erreurs de compilation, la fenêtre des erreurs de syntaxe vous dirigera vers l'objet, l'événement et les lignes spécifiques qui posent problème, et sera souvent affichée avec une brève explication de l'erreur.

Mode débogage

Si vous avez besoin de vérifier les choses plus attentivement, vous pouvez exécuter le jeu en mode débogage en cliquant sur le bouton debug run . Le module de débogage s'ouvre alors en même temps que votre jeu. Ce module vous permet de contrôler le fonctionnement de votre jeu, depuis les performances globales jusqu'aux valeurs des variables individuelles dans les différentes instances. Vous trouverez de plus amples informations sur le module de débogage, ses possibilités et la manière de l'utiliser à la page suivante :

  • Le débogueur

Notez que lorsque vous utilisez le module de débogage, vous pouvez également ajouter des points d'arrêt à votre code ou à GML Visual. Un point d'arrêt est un point dans la boucle du jeu où vous voulez que le module de débogage fasse une pause et vous permette de parcourir le code du jeu ligne par ligne. Vous pouvez ajouter des points d'arrêt en utilisant la touche "F9", et ils seront affichés dans la fenêtre de sortie où vous pouvez les activer/désactiver selon vos besoins.

Outils supplémentaires

En tant qu'outil supplémentaire pour les jeux compilés sur debugging, il existe certains paramètres de ligne de commande qui peuvent être utilisés lorsque vous exécutez le paquetage final du jeu sur Windows. Pour plus d'informations sur la façon dont cela fonctionne, voir ici.

Vous pouvez également générer une superposition de débogage lorsque vous testez votre jeu, qui affichera des informations de débogage supplémentaires sur les performances de votre jeu pendant son exécution, et vous pouvez également envoyer des messages de débogage à la console de compilation où vous pouvez fournir votre propre texte pour aider à déboguer le comportement de vos jeux. Pour plus d'informations, consultez les pages suivantes :
