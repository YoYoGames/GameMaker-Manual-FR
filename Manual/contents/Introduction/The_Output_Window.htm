La fenêtre de sortie

Lorsque vous ouvrez un projet dans GameMaker pour la première fois, la fenêtre de sortie est affichée en bas de l'écran. Cette fenêtre contient plusieurs onglets qui affichent les différentes informations de sortie de votre projet, en fonction de certaines circonstances. La station d'accueil peut être fermée en cliquant sur le bouton situé en bas de l'IDE. Vous pouvez cliquer sur et faire glisser n'importe quel onglet de la fenêtre ancrée vers une autre station d'accueil pour modifier sa position, ou vous pouvez cliquer sur dans la barre de contenu de la station d'accueil et la faire glisser dans l'espace de travail pour créer une fenêtre autonome :

Vous pouvez également cliquer sur et faire glisser une sortie ancrée dans une autre pour créer une fenêtre de sortie à vue partagée, comme le montre l'exemple ci-dessous où les deux sorties d'erreur ont été placées dans le même onglet (vous pouvez cliquer lentement sur sur un onglet pour changer son nom également) :

Vous pouvez retrouver l'état par défaut du site IDE à tout moment en sélectionnant Réinitialiser la mise en page dans la fenêtre Mise en page et vous pouvez rouvrir tout onglet fermé à partir du menu Fenêtre. Vous pouvez également modifier le mode d'affichage du contenu de chacune des fenêtres de sortie en modifiant les préférences des fenêtres de sortie.

Sortie

La fenêtre de sortie générale affiche toutes les données de sortie du compilateur, ainsi que les messages que vous avez ajoutés à votre jeu à l'aide de la commande la fonction show_debug_message() de votre jeu. La plupart des informations initiales affichées sont des informations de débogage sur la façon dont le jeu est construit et peuvent être ignorées. en tant que telles peuvent généralement être ignorées. Cependant, si vous rencontrez un problème lors de la construction de votre projet pour une plate-forme cible, elles peuvent s'avérer utiles pour trouver la cause et fournir des informations au support technique si vous le contactez. Notez que vous pouvez modifier la quantité d'informations affichées ici dans la section Compilation des Préférences.

Résultats de la recherche

Vous pouvez ouvrir la fenêtre de recherche et de remplacement en utilisant le raccourci clavier / + + " F", ou en allant dans le menu Edition. Une fois que vous avez saisi vos termes de recherche, les résultats s'affichent dans cette fenêtre au format :

[object] - [event] - [Line Number]: [search string]

Si le terme recherché se trouve dans un script, il sera tout simplement :

[script] - [Line Number]: [search string]

Vous pouvez ensuite double-cliquer sur sur l'une des entrées retournées pour ouvrir l'actif donné à la position correcte pour l'édition.

Cette fenêtre affichera également les résultats de la fonctionnalité "Rechercher toutes les références" de Plume.

Contrôle des sources

Cette fenêtre affichera tous les résultats de votre plugin SCM. Vous pouvez trouver plus d'informations sur la configuration de Source Control ici.

Points d'arrêt

Les points d'arrêt sont des endroits de votre code de jeu ou de GML Visual où vous avez demandé à GameMaker de mettre en pause l'exécution du projet pendant qu'il est en mode débogage. Vous pouvez ajouter un point d'arrêt n'importe où dans la boucle du jeu en utilisant la touche " F9", et lorsque vous le faites, il apparaîtra dans cet onglet de sortie. Vous pouvez les activer et les désactiver (sans les supprimer) en cliquant sur la case à cocher à gauche et si vous utilisez le bouton droit de la souris sur l'un d'entre eux, vous obtenez un petit menu qui vous permet de supprimer le point d'arrêt ou d'ouvrir la fenêtre du code/GML Visual où se trouve le point d'arrêt.

Erreurs de syntaxe

Les erreurs de syntaxe sont les erreurs dans votre code que le vérificateur de syntaxe de GameMaker a détecté dans votre code. Il existe de nombreuses raisons pour lesquelles vous pouvez obtenir une erreur de syntaxe - comme avoir un " ;" dans votre code alors qu'un " :" est attendu - et celles-ci doivent être corrigées avant d'essayer de tester ou de compiler votre jeu. Chaque entrée d'erreur de syntaxe dans la fenêtre de sortie peut être double-cliquée à l'aide du bouton gauche de la souris pour ouvrir une fenêtre à la position indiquée afin de la résoudre.

Les erreurs de syntaxe seront mises à jour au fur et à mesure que vous écrirez votre code (il y aura une courte pause entre le moment où vous tapez quelque chose et celui où les erreurs apparaissent dans cette fenêtre, afin d'éviter que des erreurs soient signalées pour un code inachevé), et suivront le format suivant :

[object] - [event] - [Line Number] - [Position In Line]: [error string]

Ou si l'erreur est dans un script, elle le sera :

[script] - [Line Number] - [Position in line]: [error string]

Notez que les erreurs de syntaxe empêchent généralement la compilation du jeu, mais il existe des erreurs qui seront signalées et qui n'empêcheront pas la compilation, mais qui doivent néanmoins être résolues. Les erreurs en question sont :

  • pour les variables déclarées qui ne sont utilisées nulle part
  • pour les variables utilisées quelque part qui n'ont pas encore été déclarées

En général, l'un ou l'autre de ces cas suggère une faute de frappe dans le nom d'une variable, ou un autre type d'erreur facile à commettre - et ils sont donc signalés pour que vous les révisiez. Mais cela peut aussi être intentionnel, car vous avez peut-être déclaré la variable pour un usage futur, mais vous n'avez pas encore eu le temps de l'utiliser. C'est pourquoi ces erreurs permettront quand même à votre jeu de compiler. Pour une liste complète des messages d'erreur de syntaxe possibles, veuillez consulter cette page.

Erreurs de compilation

Une erreur de compilation se produit lorsque votre jeu rencontre un type d'erreur que le vérificateur de syntaxe n'a pas pu détecter, ou lorsqu'une erreur est liée à la façon dont vous avez configuré les options de compilation. Dans ce cas, le jeu affiche une erreur de compilation et cesse de fonctionner. Toutes les erreurs de compilation seront également affichées dans la fenêtre General Output, mais elles seront également listées ici séparément (car elles peuvent être "perdues" dans le reste de la sortie générale).

Les messages d'erreur du compilateur suivront tous le même format :

[object] - [event] - [Line Number]: [error string]

Si l'erreur se trouve dans un script, ce sera simplement :

[script] - [Line Number]: [error string]

Vous pouvez ensuite double-cliquer sur sur l'une des entrées d'erreur du compilateur pour ouvrir le poste donné à la position signalée comme donnant lieu à l'erreur. Pour une liste complète des messages d'erreur possibles, veuillez consulter ici.

Messages de plumes

La fenêtre Messages de Feather vous montre toutes les erreurs, avertissements et suggestions signalés par Feather. Vous pouvez double-cliquer sur n'importe quel élément de la liste pour naviguer vers sa fenêtre de code correspondante.

Voir Messages de la Plume (Plume activée) pour plus d'informations.
