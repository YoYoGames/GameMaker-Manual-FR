Langue de GameMaker

Vous pouvez utiliser GameMaker Language pour programmer vos jeux de deux manières différentes : en plaçant des blocs pour "écrire" le code visuellement, ou en écrivant le code manuellement.

La première option, GML Visual, est la plus adaptée aux débutants et aux amateurs qui souhaitent se lancer rapidement dans la création de leurs propres jeux.

La deuxième option, GML Code, vous permet d'exploiter toute la puissance de GameMaker Language en écrivant votre code manuellement.

Si GML Visual est plus facile à utiliser au départ, GML Code reste un langage de programmation facile à utiliser et doté de fonctionnalités puissantes (fonctions, méthodes, Structs & Constructors, etc.).

Création de scripts visuels/codes

Il vous est demandé de choisir entre GML Code et GML Visual lors de la création d'un nouvel événement d'objet, d'une nouvelle ressource de script ou de toute autre chose qui entraîne la création d'un script :

En plus du choix entre GML Code et GML Visual, cette fenêtre contient deux cases à cocher :

  • Ne pas demander à nouveau pour ce projet: Lorsque cette option est activée, elle définit l'option de jeu "Langage de script par défaut" à votre nouvelle sélection, de sorte qu'il ne vous est pas demandé de choisir à nouveau pour le projet actuel.
  • Ne plus jamais demander: Lorsque cette option est activée, elle définit la préférence "Langage de script par défaut" à votre nouvelle sélection, de sorte que vous n'êtes jamais demandé de choisir à nouveau, pour tous les projets.
